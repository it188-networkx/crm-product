**上下文**

- 上游 Epic 定义（A0303）：`requirements/executive-dashboard/opportunity-health/README.md`，T-03/E-03 业务定位：构建商机健康度感知与预警中心，系统自动基于漏斗停滞时长、AI 评分与销售跟进行为三个维度持续评估每个在途商机的健康状态，高风险商机主动推送给 CEO 实时干预与追踪
- Feature 清单（共 5 个，本 Epic 全部 Feature 均为 P1，纳入本冲刺规划范围）：
  - F-01 商机健康度多维评分：综合漏斗停滞时长、AI 评分与销售跟进行为三个维度生成健康度等级，等级阈值按客户类型（终端/外单）分别配置（BR-01 ~ BR-04 / 场景一步骤 1）
  - F-02 高风险商机预警列表：聚合展示健康度达到高/中风险等级的在途商机，支持按负责人、阶段、风险类型和客户类型筛选与排序（BR-07 / 场景一步骤 2~3）
  - F-03 预警处置与关注标记：CEO 对预警商机执行关注标记、忽略或转派三类处置操作，操作结果同步通知销售负责人，所有操作写入审计日志（BR-05, BR-06 / 场景一步骤 3~5）
  - F-04 预警主动推送通知：商机健康度新进入高风险等级或竞品威胁等级升高时，系统主动向 CEO 推送预警通知，含商机摘要与处置入口（场景二步骤 2~3）
  - F-05 竞争威胁商机追踪（可独立交付）：消费 T-02/E-03 竞品威胁信号，按商机与责任人维度展示威胁等级及趋势走势，支持 CEO 判断战略干预时机（BR-08, DEP-03 / 场景二）
- 跨 Epic 约束：AI 评分维度由 T-02/E-01 提供（BR-01 / DEP-01）；商机阶段数据来源于 T-01/E-03（DEP-02）；竞品威胁信号来源于 T-02/E-03（DEP-03 / BR-08 只读消费）；漏斗停留基准须与 T-03/E-01 对齐（DEP-04 / BR-04）
- 设计变体：lowcode（见 `requirements/AGENTS.md`）
