**上下文**

- 上游 Epic 定义（A0303）：`requirements/knowledge-repository/knowledge-ingestion/README.md`，T-04/E-01 业务定位：构建受控的知识资产录入与发布通道，确保所有进入知识库的内容均经结构化分类、人工脱敏审核和版本化管理，成为 E-02 检索与 E-03 向量化引用的唯一可信来源
- 交付阶段说明：本 Epic 标注为 Phase 2 交付（会议纪要 20260304 决议：第一阶段不交付知识库），但本冲刺目标为完成 MVP 全部产品设计（需求特性定义），不涉及交付排期
- P0 Feature 清单（共 3 个）：
  - F-01 结构化内容录入：知识单元录入通道，支持分类标签（行业/客户类型/竞品/销售阶段）选择、正文与附件提交、元数据填写，可携带商机上下文预填；录入后保存为草稿状态等待审核（BR-04 / 场景一步骤 1~2）
  - F-02 审核与脱敏发布：审核员对草稿内容进行脱敏标记与发布决策，标记客户/金额/人员等敏感信息并替换为泛化描述，决定发布或退回；发布后触发 E-02/E-03 订阅事件（BR-01, BR-02, BR-03 / 场景一步骤 3~4）
  - F-03 知识单元版本管理：已发布知识单元支持版本迭代，修订后重新进入 F-02 审核流，新版发布后旧版自动归档（不可检索引用），历史版本快照保留供审计查阅（BR-05 / 场景二）
- P1 Feature（不含在本次范围）：F-04 内容质量反馈
- 跨 Epic 约束：GC-03 信息脱敏合规要求（BR-02）；GC-04 审计日志不可篡改（BR-03）；E-02 检索与 E-03 向量化仅消费「已发布」状态知识单元（BR-01）；事业部数据隔离（BR-06 / GC-02）
- 设计变体：lowcode（见 `requirements/AGENTS.md`）
