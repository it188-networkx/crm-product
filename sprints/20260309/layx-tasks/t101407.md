**上下文**

- 上游 Epic 定义（A0303）：`requirements/structured-sales-cycle/data-integration/README.md`，E-07 业务定位：将主要外部渠道数据源统一接入线索池，实现客户交互行为的自动记录与映射，为 E-01 提供完整线索来源与行为意向分，为 E-02 提供自动触达行为记录，为 E-06 提供行为特征数据
- P0 Feature 清单（共 3 个）：
  - F-01 渠道接入配置：支持管理员配置外部渠道连接参数与数据映射规则，提供连接测试、渠道激活与停用管理（BR-02, BR-06 / 场景一）
  - F-02 增量数据同步：按配置的同步频率拉取已激活渠道的增量互动数据，按映射规则转换为线索来源记录或 E-02 跟进记录写入；失败时按 BR-05 自动重试（BR-01, BR-03, BR-05 / 场景二）
  - F-03 手动导入：支持上传标准格式文件批量导入线索数据，含字段映射预览、去重提示与导入摘要（BR-01 / 场景三）
- P1 Feature（不含在本次范围）：F-04 同步状态监控、F-05 EDM 外发行为回写
- 跨 Epic 约束：接入数据须服从 E-01 去重校验规则（DEP-01）；行为数据须按 E-02 跟进记录格式写入（DEP-02）；行为特征须能被 E-06 标签体系引用（DEP-03）；各渠道接入须获得外部平台授权（DEP-04）；渠道接入优先级分三阶段：M1 手动导入 → M3 邮件+官网 → M6 企微+社媒
- 设计变体：lowcode（见 `requirements/AGENTS.md`）
