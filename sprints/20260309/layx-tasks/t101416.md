**上下文**

- 上游 Epic 定义（A0303）：`requirements/knowledge-repository/knowledge-retrieval/README.md`，T-04/E-02 业务定位：构建统一的知识检索与场景推送能力，使售前在商机跟进场景中可按客户类型、行业、销售阶段等多维度快速检索 E-01 已发布知识单元，并在商机详情页自动推送相关案例与话术
- 交付阶段说明：T-04 知识库标注为 Phase 2 交付，但本冲刺目标为完成 MVP 全部产品设计（需求特性定义），不涉及交付排期
- P0 Feature 清单（共 3 个）：
  - F-01 多维度知识检索：提供关键词与多维筛选器（客户类型/行业/竞品/销售阶段）的知识检索能力，结果按热度与匹配度排序，仅展示「已发布」知识单元；展示版本号与发布时间（BR-01, BR-02 / 场景一）
  - F-02 商机场景智能推送：在 T-01/E-03 商机详情页扩展区域，基于商机属性自动推送 Top-N 相关知识单元，支持展开预览；推送上限 N=5（可配置），不影响核心商机信息展示（BR-01, BR-03, BR-04 / 场景二）
  - F-03 检索反馈与热度排序：售前可对检索与推送的知识单元提交反馈（有用/不相关/已过时），标签统一采用 Theme 级反馈分类法；反馈权重影响热度分；反馈数据汇总至内容运营看板（BR-05, BR-06, BR-07 / 场景三）
- 跨 Epic 约束：检索与推送仅来自 E-01 状态为「已发布」的知识单元（BR-01）；F-02 集成于 T-01/E-03 商机详情页（BR-03）；反馈数据供 E-01 内容运营看板消费；智能推送相关性算法优先级需与 E-03 协调确认
- 设计变体：lowcode（见 `requirements/AGENTS.md`）
