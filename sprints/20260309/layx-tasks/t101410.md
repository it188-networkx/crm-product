**上下文**

- 上游 Epic 定义（A0303）：`requirements/executive-dashboard/funnel-analytics/README.md`，T-03/E-01 业务定位：为 CEO 构建以线索到成单全链路转化为核心的自助数据视图，通过漏斗阶段分布与停留时长分析实现转化瓶颈识别与团队执行健康度自主研判
- P0 Feature 清单（共 3 个）：
  - F-01 全链路漏斗可视化：展示从线索进入到成单全链路各阶段的数量、转化率与平均停留时长；支持时间窗口切换与历史基准对比高亮（BR-01, BR-03, BR-04, BR-06 / 场景一）
  - F-02 按负责人筛选视图：支持按销售人员筛选漏斗视图，展示各阶段转化率与跟进及时率排行及周环比变化（BR-02, BR-04, BR-05 / 场景一、场景二）
  - F-03 阶段商机明细下钻：从漏斗阶段进入该阶段商机明细列表，展示商机名称、负责人、停留时长与最近跟进时间；支持按停留时长排序（BR-01, BR-03, BR-05 / 场景一、场景二）
- 跨 Epic 约束：漏斗数据须与 T-01/E-03 商机推进模块同源（BR-01 / DEP-01）；负责人映射须与 T-01 线索分配一致（BR-02 / DEP-02）；停留时长基准须与 T-03/E-03 商机健康度预警对齐（BR-03 / DEP-03）；漏斗指标为 T-03/E-04 Go/No-Go 看板提供 KPI 基线（DEP-04）；F-03 须以 F-01 为前提（INT-01）
- 设计变体：lowcode（见 `requirements/AGENTS.md`）
