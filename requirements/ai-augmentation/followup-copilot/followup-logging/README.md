# Feature Definition - [T-02/E-02/F-03] 跟进记录沉淀 (Followup Record Persistence)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-02/E-02 AI 跟进辅助 (AI Followup Copilot)](../README.md)

**业务定义**：
该特性用于在销售完成一次跟进触达（即确认发送草稿）后，将本次跟进的核心结果（触达时间、行动类型、草稿摘要、发送状态与确认人）结构化写入跟进记录，并允许销售在记录上标注客户反馈结果。跟进记录须写入 T-01/E-02 统一记录体系，保证客户互动上下文的完整性和可被后续策略建议引用。

该特性关注的是"如何将每次已确认的跟进触达结构化记录下来，并使之可被引用"，不定义策略建议的生成方式（由 F-01 承接），也不定义内容草稿的起草与发送确认（由 F-02 承接）。

**范围边界**：

1. 包含：将已确认发送的跟进触达结构化写入跟进记录；在跟进记录上标注客户反馈；跟进记录写入 T-01/E-02 统一记录体系；为后续策略建议提供可引用的历史互动上下文。
2. 不包含：策略建议生成（F-01）；内容草稿生成与发送确认（F-02）；跟进记录的查询与展示视图（属于 T-01/E-02 或单独的记录管理 Feature）；客户反馈的自动分析与评分影响（属于上游 E-01 或独立分析能力）。
3. 参见 Epic [DEP-03]：跟进记录须写入 T-01/E-02 统一记录体系；若 T-01/E-02 记录体系未就绪，本 Feature 功能不可用，须向销售提示降级状态。

## 用户故事 (User Stories)

| ID | 角色 (As a...) | 想要 (I want to...) | 从而 (So that...) | 验收标准索引 (AC Ref) |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | 销售执行层 | 在确认发送触达内容后，本次跟进结果自动写入跟进记录，无需手动填报 | 确保每次有效跟进的关键信息被完整留存，后续跟进可引用历史上下文 | AC-01, AC-02, AC-05 |
| US-02 | 销售执行层 | 在完成跟进后，对本次跟进标注客户反馈结果（如客户有回应、客户无回应、客户明确拒绝等） | 将客户实际反应纳入互动上下文，支撑后续策略建议的准确性 | AC-03, AC-04 |

## 业务规则 (Business Rules)

- **[BR-01]记录完整性**：参见 Epic [BR-04]。每次人工确认发送的跟进必须写入跟进记录，记录内容须包含触达时间、行动类型、草稿摘要及发送确认人，缺少其中任意字段则记录不允许写入成功，须向系统触发方标注写入失败原因。
- **[BR-02]记录体系一致性**：参见 Epic [BR-04]。跟进记录须写入 T-01/E-02 统一记录体系，不允许孤立存储；两者共享客户互动数据结构，本 Feature 创建的跟进记录须与 T-01/E-02 中的记录格式保持兼容。
- **[BR-03]记录触发条件**：跟进记录只能由 F-02 的"已确认发送"事件触发写入，不允许在草稿未确认（待确认或已放弃状态）时创建跟进记录，也不允许手动独立创建无关联草稿的空白记录。`[推导-待确认]`
- **[BR-04]反馈标注非强制**：客户反馈标注为可选操作，不阻塞跟进记录的写入；反馈可在记录写入后的任意时刻补充标注，但一条跟进记录只允许存在一个当前有效的反馈标注，后续更新须覆盖前值并保留变更时间。`[推导-待确认：反馈标注的可选枚举值定义]`
- **[BR-05]记录不可删除**：已写入的跟进记录不允许被删除，确保跟进历史的完整可追溯性；若存在错误记录，须通过追加更正说明的方式处理，不允许覆盖或删除原始记录。`[推导-待确认]`

## 业务对象定义 (Business Object Definition)

### 跟进记录

跟进记录是一次已确认完成的客户跟进触达的结构化业务凭证，承载触达结果的核心事实，供后续跟进策略建议引用，同时写入 T-01/E-02 统一记录体系以确保互动上下文的完整性。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联客户 | 标识本次跟进针对哪位客户 |
| 关联草稿 | 标识本记录由哪份已确认草稿触发生成 |
| 触达时间 | 销售确认发送的时间点，作为跟进时序的基准 |
| 行动类型 | 本次跟进采用的行动分类（从已确认策略继承） |
| 草稿摘要 | 触达内容草稿的简要摘要，不含客户 PII 明文 |
| 发送状态 | 标识本次跟进触达的最终状态（已确认发送） |
| 确认人 | 执行确认发送操作的销售人员标识 |
| 客户反馈标记 | 销售对本次跟进客户反应的标注结果（可选），如"有回应""无回应""明确拒绝"等 `[推导-待确认：枚举值定义]` |
| 反馈标注时间 | 客户反馈标记的最近更新时间 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | F-02 中销售完成草稿确认发送；T-01/E-02 记录体系可用 | 草稿确认发送事件触发 | 跟进记录自动写入，包含触达时间、行动类型、草稿摘要与确认人（参见 BR-01），记录写入 T-01/E-02 统一记录体系（参见 BR-02） | US-01 |
| AC-02 | 草稿为待确认或已放弃状态 | 系统尝试创建跟进记录 | 跟进记录不被创建（参见 BR-03）；销售无感知，无误操作风险 | US-01 |
| AC-03 | 存在已写入的跟进记录 | 销售对该记录标注客户反馈 | 跟进记录的客户反馈标记更新为销售选择的反馈值，记录反馈标注时间（参见 BR-04） | US-02 |
| AC-04 | 存在已标注客户反馈的跟进记录 | 销售更新对同一记录的反馈标注 | 原反馈标记被新值覆盖，反馈标注时间更新为当前时间（参见 BR-04）；原始跟进记录其余字段不受影响 | US-02 |
| AC-05 | T-01/E-02 记录体系不可用（参见 Epic DEP-03） | 草稿确认发送事件触发 | 系统向销售提示跟进记录写入失败及降级状态（参见 BR-01/BR-02）；销售的确认发送操作本身不受阻断，提示中建议待记录体系恢复后补录 | US-01 |
