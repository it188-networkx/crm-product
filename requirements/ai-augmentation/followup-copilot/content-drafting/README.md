# Feature Definition - [T-02/E-02/F-02] 个性化内容草稿生成 (Personalized Content Drafting)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-02/E-02 AI 跟进辅助 (AI Followup Copilot)](../README.md)

**业务定义**：
该特性用于在销售确认跟进策略建议后，基于已确认的策略自动生成面向该客户的个性化触达内容草稿。草稿生成时不包含客户姓名、联系方式等个人可识别信息明文，以满足信息脱敏要求。销售可在草稿基础上进行编辑，完成最终确认后将触达内容标记为已发送，并触发跟进记录写入（由 F-03 承接）。当关联客户存在竞品预警时，草稿内容突出我方优势与竞品对比价值点。

该特性关注的是"基于已确认策略生成什么样的内容草稿"以及"草稿如何经人工确认进入发送状态"，不定义策略建议的生成方式（由 F-01 承接），也不定义跟进记录的结构化写入（由 F-03 承接）。

**范围边界**：

1. 包含：基于已确认跟进策略生成个性化内容草稿；草稿脱敏（不含 PII 明文）；人工编辑草稿；人工确认并标记为已发送；竞品上下文时的差异化草稿生成。
2. 不包含：策略建议生成（F-01）；跟进结果的结构化记录（F-03）；实际发送渠道的技术对接；内容合规审查（非本 Feature 业务范围）。
3. 参见 Epic [DEP-04]：内容草稿依赖外部 AI 推理服务；服务不可用时草稿不可生成，系统须向销售明确提示并建议人工准备触达内容。
4. 参见 Epic [DEP-05]（可选增强）：T-04/E-03 知识向量化就绪后，可为草稿生成提供行业贴合知识内容增强；该依赖为可选，不影响基础 MVP 交付。`[推导-待确认：T-04 依赖接口尚未明确]`

## 业务规则 (Business Rules)

- **[BR-01]草稿确认发送**：参见 Epic [BR-02]。AI 生成的触达内容草稿须经销售人工确认操作后方可标记为已发送，系统不允许 AI 直接触达客户，无强制绕过入口。
- **[BR-02]草稿脱敏**：参见 Epic [BR-06]。AI 内容草稿中不得包含客户姓名、联系方式、组织机构代码等个人可识别信息明文；所有客户敏感信息在传入外部 AI 推理服务前必须完成脱敏处理。
- **[BR-03]策略前置依赖**：内容草稿生成必须以存在已确认的跟进策略建议为前置条件；无已确认策略时，草稿生成不可触发，系统须提示销售先完成策略确认（参见 F-01）。
- **[BR-04]草稿与策略关联**：每份草稿必须关联至生成它的已确认策略建议，以确保内容与策略的一致性可追溯；策略被撤销时，基于该策略生成的草稿须标记为策略已失效，提示销售重新确认策略后再生成草稿。`[推导-待确认]`
- **[BR-05]竞品草稿差异化**：当上游已确认策略包含竞品应对上下文时，生成的草稿内容必须包含竞品对比价值点，不允许忽略竞品上下文生成通用草稿。`[推导-待确认：竞品价值点的具体来源与表达边界]`

## 业务对象定义 (Business Object Definition)

### 触达内容草稿

触达内容草稿是基于已确认跟进策略由 AI 生成的个性化外发内容，承载待发送的触达文字及其确认状态，作为触发跟进记录写入的上游凭证。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联策略建议 | 标识本草稿基于哪条已确认的跟进策略建议生成 |
| 关联客户 | 标识本草稿面向哪位客户 |
| 草稿内容 | AI 生成并经销售编辑的最终触达文字，不含 PII 明文 |
| 生成时间 | 草稿的 AI 生成时间点 |
| 编辑状态 | 标识草稿是否经过销售人工修改（未编辑 / 已编辑） |
| 确认状态 | 标识销售对草稿的处理结果（待确认 / 已确认发送 / 已放弃） |
| 确认人 | 执行确认发送操作的销售人员标识 |
| 确认时间 | 执行确认发送操作的时间点 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 存在已确认的跟进策略建议；外部 AI 推理服务可用 | 销售触发内容草稿生成 | 生成与策略一致的个性化触达内容草稿，草稿不含客户 PII 明文（参见 BR-02）；草稿关联至对应策略（参见 BR-04） | US-01 |
| AC-02 | 存在已生成的内容草稿 | 销售对草稿内容进行编辑 | 草稿内容更新为编辑后版本，编辑状态标记为"已编辑"；确认状态保持"待确认" | US-02 |
| AC-03 | 存在已生成（或已编辑）的内容草稿 | 销售执行确认发送操作 | 草稿确认状态标记为"已确认发送"，记录确认人与确认时间（参见 BR-01）；触发 F-03 跟进记录写入流程 | US-02 |
| AC-04 | 上游已确认策略包含竞品应对上下文 | 销售触发内容草稿生成 | 生成的草稿内容包含竞品对比价值点，体现我方优势（参见 BR-05）；草稿不含客户 PII 明文（参见 BR-02） | US-03 |
| AC-05 | 无已确认的跟进策略建议（策略尚未生成或尚未确认） | 销售尝试触发内容草稿生成 | 系统提示需先完成策略确认方可生成草稿（参见 BR-03），不触发 AI 调用 | US-01 |
| AC-06 | 外部 AI 推理服务不可用（参见 Epic DEP-04） | 销售触发内容草稿生成 | 系统明确提示服务不可用，建议销售人工准备触达内容；不生成任何草稿内容 | US-01 |
| AC-07 | 存在已生成的草稿；上游关联策略被替换（F-01 BR-06 触发） | 策略替换事件 | 草稿自动标记为"策略已失效"，提示销售重新确认策略后重新生成草稿（参见 BR-04）；不允许基于已失效草稿执行确认发送 | US-01 |
