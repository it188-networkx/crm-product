# Feature Definition - [T-02/E-02/F-01] 跟进策略建议 (Followup Strategy Recommendation)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-02/E-02 AI 跟进辅助 (AI Followup Copilot)](../README.md)

**业务定义**：
该特性用于在销售准备跟进客户时，基于客户当前 AI 评分与分级、历史互动记录及跟进阶段，自动生成下一步行动建议，包含行动类型、紧迫度与建议时间节点。当关联客户存在有效竞品预警时，策略建议自动注入竞品应对上下文。销售可在查看建议后选择接受，或补充背景信息后要求重新生成，确认后方作为执行依据。

该特性关注的是"基于客户上下文生成什么样的建议"以及"建议如何经人工确认进入执行"，不定义触达内容的具体起草方式（由 F-02 承接），也不定义跟进结果的记录方式（由 F-03 承接）。

**范围边界**：

1. 包含：基于客户评分、历史互动记录与跟进阶段生成策略建议；注入竞品应对上下文；人工确认/修改/忽略建议；补充背景信息后重新生成。
2. 不包含：触达内容草稿生成（F-02）；跟进结果记录（F-03）；客户评分计算（E-01 上游）；竞品预警数据生成（E-03 上游）。
3. 参见 Epic [DEP-01]：策略建议依赖 E-01 提供的客户评分；E-01 不可用时降级为仅基于历史跟进记录生成建议，并标注"评分数据不可用"。
4. 参见 Epic [DEP-02]：差异化策略依赖 E-03 提供竞品预警上下文；E-03 无预警数据时跳过竞品应对视角，不影响基础跟进辅助功能。
5. 参见 Epic [DEP-04]：策略建议依赖外部 AI 推理服务；服务不可用时建议不可生成，系统须向销售明确提示并建议人工准备。

## 用户故事 (User Stories)

| ID | 角色 (As a...) | 想要 (I want to...) | 从而 (So that...) | 验收标准索引 (AC Ref) |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | 销售执行层 | 进入待跟进客户详情后查看 AI 生成的下一步跟进策略建议（含行动类型、紧迫度与建议时间节点） | 基于客户数据有依据地决策本次跟进行动，不再依赖人工回顾 | AC-01, AC-02, AC-06 |
| US-02 | 销售执行层 | 在策略建议不完全符合实际情境时，补充背景信息后要求重新生成建议 | 让 AI 建议更贴合当前客户实际情况，提升建议可操作性 | AC-03, AC-07 |
| US-03 | 销售执行层 | 当关联客户存在有效竞品预警时，查看含竞品应对上下文的差异化跟进策略建议 | 在有竞争威胁的跟进中制定包含竞品应对视角的行动方案 | AC-04, AC-05 |

## 业务规则 (Business Rules)

- **[BR-01]建议确认生效**：参见 Epic [BR-01]。所有 AI 生成的跟进策略建议须经销售人工确认（接受/修改/忽略）后方可作为执行依据，系统不得自动触发跟进动作。
- **[BR-02]评分上下文依赖**：参见 Epic [BR-03]。策略建议必须关联客户当前 AI 评分结果与分级；缺少评分数据时，策略建议必须明确标注"评分数据不可用，建议人工判断优先级"，不允许以空白或隐式方式处理评分缺失情形。
- **[BR-03]竞品上下文注入**：参见 Epic [BR-05]。当关联客户存在有效竞品预警时，策略建议必须包含竞品应对上下文（竞品威胁描述与差异化应对要点）；竞品预警有效期过期时，须向销售提示刷新预警数据，建议仍可生成但不含竞品上下文。`[推导-待确认：竞品预警有效期定义]`
- **[BR-04]建议可操作性**：参见 Epic [BR-07]。AI 跟进建议必须包含明确的行动指引（下一步动作类型 + 时间节点建议），不允许输出仅包含泛化描述（如"继续跟进"）的无效建议；若无法生成可操作建议，系统须向销售提示原因并建议补充客户信息。`[推导-待确认]`
- **[BR-05]脱敏传输**：参见 Epic [BR-06]。客户个人可识别信息在传入外部 AI 推理服务前必须完成脱敏处理，策略建议生成输入不得包含客户姓名、联系方式、组织机构代码等 PII 明文。
- **[BR-06]建议唯一激活**：同一客户同一时刻仅允许存在一条待确认的策略建议；补充信息重新生成时，旧建议自动标记为已替换，不得同时保留多条并列的未确认建议。`[推导-待确认]`

## 业务对象定义 (Business Object Definition)

### 跟进策略建议

跟进策略建议是针对特定客户、基于其当前上下文由 AI 生成的下一步行动方案，承载行动指引信息并记录销售的确认决策，作为后续内容草稿生成的上游输入。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联客户 | 标识本建议针对哪位客户生成 |
| 生成时间 | 建议的生成时间点，作为时效性判断基准 |
| 行动类型 | 建议的下一步行动分类（如电话跟进、邮件触达、上门拜访等）`[推导-待确认：行动类型枚举值]` |
| 紧迫度 | 基于客户评分与跟进阶段判定的建议优先程度 |
| 建议时间节点 | AI 推荐的执行时间范围 |
| 竞品上下文标识 | 标识本建议是否注入了竞品预警上下文，及其有效期状态 |
| 生成依据摘要 | 支撑本建议的核心输入信息摘要（客户评分等级、历史跟进轮次等），不含 PII |
| 确认状态 | 标识销售对本建议的处理结果（待确认 / 已接受 / 已修改 / 已忽略 / 已替换） |
| 确认人 | 执行确认操作的销售人员标识 |
| 确认时间 | 执行确认操作的时间点 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 客户存在有效 AI 评分；历史互动记录可访问；外部 AI 推理服务可用 | 销售进入待跟进客户详情，触发策略建议生成 | 生成包含行动类型、紧迫度与建议时间节点的可操作策略建议（参见 BR-04），不含客户 PII（参见 BR-05）；销售可对建议执行接受/修改/忽略操作（参见 BR-01） | US-01 |
| AC-02 | 客户 AI 评分不可用（E-01 服务异常或未评分）；外部 AI 推理服务可用 | 销售触发策略建议生成 | 建议生成并标注"评分数据不可用，建议人工判断优先级"（参见 BR-02），建议内容仅基于历史跟进记录生成 | US-01 |
| AC-03 | 存在已生成的策略建议；销售判断需补充背景信息 | 销售补充背景信息后提交重新生成请求 | 旧建议标记为"已替换"（参见 BR-06）；系统基于补充信息重新生成新建议（参见 BR-04） | US-02 |
| AC-04 | 关联客户存在有效竞品预警（未过期）；外部 AI 推理服务可用 | 销售触发策略建议生成 | 建议中包含竞品应对上下文（竞品威胁描述与差异化应对要点）（参见 BR-03），竞品上下文标识标记为已注入 | US-03 |
| AC-05 | 关联客户竞品预警已过期 | 销售触发策略建议生成 | 系统提示销售竞品预警数据已过期，建议其刷新（参见 BR-03）；建议仍可生成，但不含竞品应对上下文，竞品上下文标识标记为已过期 | US-03 |
| AC-06 | 外部 AI 推理服务不可用（参见 Epic DEP-04） | 销售触发策略建议生成 | 系统明确提示服务不可用，建议销售人工准备跟进策略；不生成任何建议内容 | US-01 |
| AC-07 | 策略建议已生成；销售未补充任何背景信息 | 销售提交空白背景信息的重新生成请求 | 系统提示需提供至少一项补充信息方可重新生成，不触发 AI 调用 | US-02 |
