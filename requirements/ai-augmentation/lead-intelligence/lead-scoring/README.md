# Feature Definition - F-01 线索动态评分与分级 (Lead Dynamic Scoring & Classification)

## 特性概述 (Feature Overview)

**所属 Epic**：[[T-02/E-01] AI 线索智能研判 (AI Lead Intelligence)](../README.md)

**业务定义**：
该特性用于对新进及存量线索持续实施动态评分，输出分级建议（高/中/低）及可解释的推荐理由，帮助 MKT Leader 与销售执行层统一线索优先级判断口径。当销售认为评分与实际不符时，可补充线索背景信息后提交重评，系统更新评分并维护完整的评分变更历史，为后续异常检测与分配推荐提供评分基准。

**范围边界**：

1. 包含：线索批量评分触发、评分结果与分级建议呈现、推荐理由可解释说明、人工信息补充触发重评、评分变更历史记录。
2. 不包含：异常线索识别与处置（由 F-02 承接）、分配推荐排序（由 F-03 承接）、线索录入与基础信息管理。
3. 外部依赖约束：动态评分依赖外部 AI 推理服务，服务不可用时降级为规则引擎静态评分（参见 Epic [DEP-01]）；评分准确性依赖 T-01 线索与跟进记录历史上下文，数据缺失时标记提示（参见 Epic [DEP-02]）。
4. 下游消费约束：本特性输出的转商机合理性评分被 T-01/E-03 商机推进消费，评分模型变更须提前协同（参见 Epic [DEP-03]）；健康度评分被 T-03/E-03 预警下游消费，变更须通知（参见 Epic [DEP-04]）。

| US-01 | MKT Leader | 触发线索池中全部新进线索的批量评分 | 快速获得统一口径的优先级分级建议，识别高价值线索并提升分配效率 | AC-01, AC-05 |
| US-02 | 销售执行层 | 查看自己负责线索的评分结果与可解释推荐理由 | 理解评分依据，判断是否需要补充信息发起重评 | AC-02 |

## 业务规则 (Business Rules)

- **[BR-01]评分覆盖**：参见 Epic [BR-01]。所有进入系统的线索必须在录入后纳入评分队列；评分结果、分级建议（高/中/低）及推荐理由必须对授权用户可见且可查。
- **[BR-02]分级一致性**：参见 Epic [BR-02]。线索分级必须基于同一评分规则集输出；不同授权用户对同一线索看到的分级结果必须一致，不得因查看角色不同而异。
- **[BR-03]评分版本追踪**：参见 Epic [BR-03]。每次评分或重评触发后，系统必须记录评分时间、触发来源（批量自动/销售手动重评）及因子变化摘要，以支持审计与争议复核。
- **[BR-04]AI 降级处理**：外部 AI 推理服务不可用时，系统必须降级为规则引擎静态评分，并在评分结果中标记"AI 能力不可用"；降级期间评分结果仍须满足 BR-01 的可见性要求。`[推导-待确认]`
- **[BR-05]历史数据不足处理**：当线索的历史跟进上下文（T-01 数据）不足以支撑准确评分时，评分结果必须标记"历史上下文不足"，并建议用户手动补充背景信息后重评，不得静默输出无依据评分。`[推导-待确认]`
- **[BR-06]重评资格限制**：重评请求仅允许由该线索的责任销售或 MKT Leader 发起；其他角色不得触发重评。`[推导-待确认]`
- **[BR-07]重评信息要求**：提交重评请求时，必须附带至少一项新增或更新的线索背景信息；未包含增量信息的重评请求不得触发评分计算。`[推导-待确认]`
- **[BR-08]重评后高优先级升级通知**：重评完成后，若线索分级由中/低升级为高优先级，系统必须将该线索推送至 MKT Leader 待确认队列，由 MKT Leader 决定是否调整分配；未经确认前原有分配关系不变。`[推导-待确认]`

## 业务对象定义 (Business Object Definition)

### 线索评分记录

线索评分记录是系统对某条线索在特定时刻实施评分的业务凭证，承载评分结论、分级建议与可解释推荐理由，并记录触发来源与因子变化，以支持争议复核和历史追溯。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识该评分记录归属于哪一条线索 |
| 评分时间 | 本次评分结果产出的业务时间基准 |
| 触发来源 | 标识本次评分是系统批量自动触发还是销售手动重评触发 |
| 评分结果 | 线索在评分规则集下获得的量化分值或等级指示 |
| 分级建议 | 基于评分结果对线索优先级的分级输出（高/中/低） |
| 推荐理由说明 | 对分级建议的可解释性业务说明，供用户理解评分依据 |
| 因子变化摘要 | 与上一版本评分相比，本次影响评分的关键因子变化说明 |
| 数据状态标记 | 标识评分结果是否因 AI 不可用（降级为规则引擎）或历史上下文不足而受限 |

### 线索信息补充申请

线索信息补充申请是销售执行层在对当前评分结果存疑时提交的业务请求，关联补充的背景信息并触发重评流程，其结果作为重评历史的触发依据可供追溯。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识补充申请归属的线索 |
| 申请人 | 发起重评请求的销售执行层或 MKT Leader |
| 申请时间 | 重评请求提交的业务时间 |
| 补充信息内容 | 申请人提供的新增或更新的线索背景信息，作为重评的输入依据 |
| 关联评分记录 | 重评完成后产出的新评分记录引用，用于追溯重评结果 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 线索池中存在新进线索；外部 AI 推理服务可用 | MKT Leader 触发批量评分 | 全部新进线索获得评分结果、分级建议（高/中/低）及推荐理由；评分时间与触发来源记录在评分版本中（参见 BR-01、BR-02、BR-03） | US-01 |
| AC-02 | 销售已获授权查看其负责线索；线索已存在有效评分记录 | 销售查看线索评分详情 | 销售可见该线索的当前分级建议及推荐理由说明，可识别评分依据（参见 BR-01） | US-02 |
| AC-03 | 线索已存在有效评分记录；申请人为该线索责任销售或 MKT Leader | 申请人提交含增量背景信息的重评请求 | 系统触发重评，产出新评分记录（含更新的分级、推荐理由与因子变化摘要），原评分记录保留（参见 BR-03、BR-06、BR-07） | US-03 |
| AC-04 | 重评已完成；重评前分级为中或低；重评后分级升级为高 | 重评结果产出 | 该线索被推送至 MKT Leader 待确认队列；MKT Leader 可查看升级详情并确认是否调整分配；原分配关系在确认前保持不变（参见 BR-08） | US-03, US-04 |
| AC-05 | 外部 AI 推理服务不可用 | MKT Leader 触发批量评分 | 系统使用规则引擎完成静态评分，评分结果中标记"AI 能力不可用"；评分结果仍对授权用户可见（参见 BR-04） | US-01 |
| AC-06 | 线索历史跟进上下文（T-01 数据）不足 | 评分触发（批量或重评） | 评分结果标记"历史上下文不足"；系统建议用户手动补充背景信息后重评（参见 BR-05） | US-01, US-03 |
