# Feature Definition - F-03 智能分配推荐 (Intelligent Assignment Recommendation)

## 特性概述 (Feature Overview)

**所属 Epic**：[[T-02/E-01] AI 线索智能研判 (AI Lead Intelligence)](../README.md)

**业务定义**：
该特性基于当前线索池负载、各销售的历史转化率及公平性检测结果，为 MKT Leader 提供线索分配推荐排序，并附带公平性检测告警。所有分配推荐须经 MKT Leader 显式确认后方可生效，系统不自动执行分配，且每次确认操作均留存完整操作审计日志。当销售重评使线索升至高优先级时，F-01 会将其推送至待确认队列，由本特性承接后续的分配确认流程。

**范围边界**：

1. 包含：分配推荐排序生成、推荐依据（线索池负载/历史转化率）呈现、公平性检测与告警、MKT Leader 确认分配操作、分配操作审计日志记录。
2. 不包含：线索评分与分级（由 F-01 承接）、异常线索识别与处置（由 F-02 承接）、销售人员绩效管理与考核、分配确认后的线索跟进追踪。
3. 输入前置约束：未完成处置的异常线索不进入分配推荐队列（参见 Epic [BR-05] 及 F-02）。
4. 外部依赖约束：分配推荐依赖外部 AI 推理服务，服务不可用时降级为基于规则的静态推荐（参见 Epic [DEP-01]）；历史转化率依赖 T-01 历史跟进数据，数据不足时标记提示（参见 Epic [DEP-02]）。
5. 人工决策边界：所有分配推荐须经 MKT Leader 确认后方可更新线索归属，系统不自动执行分配（参见 Epic [BR-06]）。
6. 公平性阈值：公平性检测中"优质线索过度集中"的判定阈值待业务定版（参见 Epic [BR-07]，`[推导-待确认：公平性阈值定义]`）。

## 业务规则 (Business Rules)

- **[BR-01]异常线索排除**：参见 Epic [BR-05]。被标记为异常且尚未完成处置的线索不得进入分配推荐队列。
- **[BR-02]人工确认生效**：参见 Epic [BR-06]。所有分配推荐须经 MKT Leader 或责任人显式确认后方可更新线索归属；系统不得自动执行分配。
- **[BR-03]公平性检测**：参见 Epic [BR-07]。分配推荐算法必须检测优质线索（高优先级分级）是否过度集中于特定销售；发现集中分布时须生成告警，并在推荐排序中体现均衡分配意图。集中度判定阈值待业务定版，阈值缺失时默认输出告警但不阻断推荐生成。`[推导-待确认：公平性阈值定义，责任归属：MKT Leader 确认]`
- **[BR-04]分配操作审计**：参见 Epic [BR-08]。每次分配确认操作必须留存操作人、操作时间、分配前责任人及分配后责任人信息，供审计查询。
- **[BR-05]AI 降级处理**：外部 AI 推理服务不可用时，系统必须降级为基于规则的静态分配推荐，并在推荐结果中标记"AI 能力不可用"；BR-02 的人工确认要求在降级状态下仍须满足。`[推导-待确认]`
- **[BR-06]历史数据不足处理**：当目标销售的历史转化率数据不足时，该销售在推荐排序中的依据项须标记"历史数据不足"，不得以缺省高分代入推荐排序。`[推导-待确认]`

## 业务对象定义 (Business Object Definition)

### 分配推荐记录

分配推荐记录是系统为某批次线索生成的分配建议凭证，承载推荐排序、推荐依据与公平性检测结论，供 MKT Leader 审查并发起确认操作。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识该推荐记录归属于哪一条待分配线索 |
| 推荐排序列表 | 按优先级排列的推荐销售列表，每项包含推荐依据摘要 |
| 推荐依据摘要 | 影响当前排序的关键因子说明（线索池负载比例、历史转化率） |
| 公平性检测结论 | 标识是否检测到优质线索过度集中，以及检测是否触发告警 |
| 数据状态标记 | 标识推荐结果是否因 AI 不可用降级或存在历史数据不足情况 |
| 生成时间 | 推荐记录产出的业务时间 |

### 分配确认操作记录

分配确认操作记录是 MKT Leader 对某条分配推荐进行确认时产出的审计凭证，完整记录操作人、时间及分配前后的责任人变更，以满足分配操作的可审计性要求。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识本次确认操作归属于哪一条线索 |
| 操作人 | 执行分配确认的 MKT Leader |
| 操作时间 | 分配确认完成的业务时间 |
| 分配前责任人 | 确认操作执行前线索的原归属销售 |
| 分配后责任人 | 确认操作执行后线索的新归属销售 |
| 关联推荐记录 | 本次确认所依据的分配推荐记录引用 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 待分配线索已完成评分且无未处置异常标记；外部 AI 推理服务可用 | MKT Leader 查看分配推荐 | 系统展示基于线索池负载与历史转化率的推荐排序，每项附带推荐依据摘要（参见 BR-01、BR-05） | US-01 |
| AC-02 | 推荐排序中检测到优质线索（高优先级）过度集中于特定销售 | 系统完成公平性检测 | 系统生成公平性告警并在推荐排序中体现均衡分配意图；MKT Leader 可见告警详情（参见 BR-03） | US-02 |
| AC-03 | MKT Leader 已审查分配推荐；选定目标销售 | MKT Leader 执行确认分配操作 | 线索责任归属更新至所选销售；分配确认操作记录留存（含操作人、时间、分配前后责任人）（参见 BR-02、BR-04） | US-03 |
| AC-04 | 异常线索尚未完成处置（处置状态为待处置） | 分配推荐排序触发 | 该线索不出现在分配推荐队列中（参见 BR-01） | US-03 |
| AC-05 | 外部 AI 推理服务不可用 | MKT Leader 查看分配推荐 | 系统使用规则引擎生成静态分配推荐，推荐结果中标记"AI 能力不可用"；MKT Leader 的确认要求仍须满足（参见 BR-05） | US-01 |
