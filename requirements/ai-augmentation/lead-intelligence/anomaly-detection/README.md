# Feature Definition - F-02 异常线索识别与复核 (Anomaly Lead Identification & Review)

## 特性概述 (Feature Overview)

**所属 Epic**：[[T-02/E-01] AI 线索智能研判 (AI Lead Intelligence)](../README.md)

**业务定义**：
该特性用于在线索进入分配推荐前，自动识别并标记垃圾、重复和高风险线索，生成识别依据与人工复核建议，并通过 MKT Leader 的人工处置（保留/删除/合并）确保进入分配环节的线索具备基本质量。所有异常标记均附带可解释的识别依据，处置结果完整记录以支持后续追溯。

**范围边界**：

1. 包含：异常线索自动识别（垃圾/重复/高风险三类）、识别依据呈现、人工复核建议生成、MKT Leader 处置操作（保留/删除/合并）、处置结果记录。
2. 不包含：线索评分与分级（由 F-01 承接）、分配推荐排序（由 F-03 承接）、线索录入与基础信息管理、被删除线索的数据清理策略。
3. 外部依赖约束：异常识别依赖外部 AI 推理服务，服务不可用时降级为规则引擎静态识别（参见 Epic [DEP-01]）。

## 业务规则 (Business Rules)

- **[BR-01]异常标记透明**：参见 Epic [BR-04]。异常线索标记（垃圾/重复/高风险）必须附带可解释的识别依据；不得以隐式方式静默丢弃被识别为异常的线索。
- **[BR-02]异常优先处置**：参见 Epic [BR-05]。被标记为异常的线索在 MKT Leader 完成处置（保留/删除/合并）前，不得进入分配推荐队列。
- **[BR-03]异常类型定义**：系统识别的异常类型限定为以下三类，且识别结论必须唯一归属某一类型：（1）垃圾线索：信息残缺率超过阈值或明显无效联系方式；（2）重复线索：与现有线索存在关键字段（如联系方式或公司名称）高度重合；（3）高风险线索：携带异常行为特征或与已知风险标签匹配。`[推导-待确认：各类型具体识别阈值待业务定版]`
- **[BR-04]AI 降级处理**：外部 AI 推理服务不可用时，系统必须降级为规则引擎执行静态异常识别，并在识别结果中标记"AI 能力不可用"；降级期间 BR-01 的透明度要求仍须满足。`[推导-待确认]`
- **[BR-05]处置结果可追溯**：每次处置操作（保留/删除/合并）完成后，系统必须记录操作人、操作时间及处置结论；合并操作还须记录被合并的目标线索引用。
- **[BR-06]合并操作约束**：合并操作仅当两条线索均处于待处置状态时方可执行，且合并后应保留主线索并标记被合并线索的处置去向。`[推导-待确认]`

## 业务对象定义 (Business Object Definition)

### 异常线索标记记录

异常线索标记记录是系统对某条线索实施异常识别后产出的业务凭证，承载异常类型结论、识别依据与处置状态，供 MKT Leader 进行人工复核并记录最终处置结果。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识该异常记录归属于哪一条线索 |
| 异常类型 | 识别结论所属类别（垃圾/重复/高风险） |
| 识别依据 | 触发当前异常类型标记的可解释业务说明 |
| 识别时间 | 系统完成异常识别的业务时间 |
| 数据状态标记 | 标识识别结果是否因 AI 不可用而降级为规则引擎输出 |
| 处置状态 | 当前记录是否已经过 MKT Leader 人工处置（待处置/已处置） |
| 处置结论 | 人工处置的最终结果（保留/删除/合并） |
| 处置人 | 执行处置操作的 MKT Leader |
| 处置时间 | 完成处置操作的业务时间 |
| 合并目标线索引用 | 仅当处置结论为合并时，标识被合并至的目标线索 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 线索池中存在符合异常识别条件的线索（垃圾/重复/高风险）；外部 AI 推理服务可用 | 系统完成批量异常识别 | 每条符合条件的线索被标记对应异常类型，并附带识别依据说明；标记结果对 MKT Leader 可见（参见 BR-01、BR-03） | US-01 |
| AC-02 | 异常线索已被标记且处于待处置状态 | MKT Leader 查看异常线索列表及识别依据 | MKT Leader 可见异常类型、识别依据与系统复核建议，可逐项发起处置操作（参见 BR-01） | US-01, US-02 |
| AC-03 | 异常线索处于待处置状态；MKT Leader 选择处置结论 | MKT Leader 执行保留、删除或合并操作 | 处置结论被记录（含操作人、操作时间、处置结论）；合并操作同时记录目标线索引用（参见 BR-05、BR-06） | US-02 |
| AC-04 | 异常线索尚未完成处置（处置状态为待处置） | 分配推荐排序触发 | 未处置的异常线索不出现在分配推荐队列中（参见 BR-02） | US-02 |
| AC-05 | 外部 AI 推理服务不可用 | 系统完成批量异常识别 | 系统使用规则引擎完成静态异常识别，识别结果中标记"AI 能力不可用"；识别依据仍对 MKT Leader 可见（参见 BR-04） | US-01 |
