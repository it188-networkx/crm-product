# Feature Definition - 竞品预警（情报提交、售前定版与风险等级管理）(Competitive Alert – Submission, Presales Finalization & Risk Level Management)

## 特性概述 (Feature Overview)

**所属 Epic**：[[T-02/E-03] 竞品预警与监控 (Competitive Alerts & Monitoring)](../README.md)

**业务定义**：

该特性支持 MKT、售前与责任销售三方通过语音输入、文字输入或文档输入提交竞品情报，系统基于关键词检测自动提取结构化竞品信号（竞品名称、威胁要点、涉及客户等），提交人确认或修正后正式录入；由售前审核、裁决冲突并选定最准确的录入信息作为最终竞品预警记录（可选补充备注），系统基于定版内容自动判定风险等级并标注，售前确认或修正后定版生效，系统同时通知 MKT 与责任销售。
此外，当已定版预警关联到新的情报信息时，AI 基于新信息重新评估风险等级并向售前推荐调整方案，售前确认或修正后生效；AI 还会对高风险预警持续 2 周无新信息补充或任意预警持续 4 周未降级的情况主动提醒售前复核，并同时提示责任销售是否有新的竞品信息可补充，确保风险等级始终反映最新竞品态势。
它关注的是"竞品情报从多通道输入、系统提取、人工确认到售前定版、风险等级动态管理再到三方周知的完整信息流"，同时支持 MKT 查看各竞品名称被提及的累计次数以辅助竞品活跃度判断。不定义预警处置状态流转或应对结果的闭环记录。

**范围边界**：

1. 包含：MKT / 售前 / 责任销售通过语音、文字或文档通道提交竞品情报、系统关键词提取与提交人确认、售前审核裁决与定版预警记录（可选备注）、AI 自动判定风险等级与售前确认/修正、定版后通知 MKT 与责任销售、已定版预警新增关联信息时 AI 推荐风险等级调整、AI 主动提醒售前复核并提示责任销售补充信息（高风险 2 周无新信息 / 任意等级 4 周未降级）、MKT 可查看竞品名称累计提及次数。
2. 不包含：预警处置状态变更与闭环记录、应对结果追踪。
3. 外部依赖：竞品信息亦可来自外部数据订阅或内部知识库，与人工提交并行；外部来源不可用时不影响人工提交通道（参见 Epic [DEP-01]）。
4. 责任销售确定依赖 E-01 客户分配推荐结果，分配结果不可用时通知降级为仅通知 MKT Leader 人工分发（参见 Epic [DEP-02]）。
5. 预警推送优先级受当前商机阶段影响，阶段数据不可用时不按阶段区分优先级（参见 Epic [DEP-03]）。
6. 预警记录的数据结构须与 E-02 AI 跟进辅助共享，变更须协同确认（参见 Epic [DEP-04]）。

## 业务规则 (Business Rules)

- **[BR-01]预警必须关联客户**：每条竞品预警必须关联至少一名受影响客户；不得产生无客户关联的空泛预警。（参见 Epic [BR-01]）
- **[BR-02]情报输入通道与提取**：MKT、售前、责任销售三类角色均可通过语音输入、文字输入或文档输入提交竞品情报；系统对输入内容进行关键词检测，自动提取竞品名称、威胁要点与受影响客户，生成情报草稿；提交人须对草稿进行确认或修正后方可正式录入，不得跳过人工确认直接录入；其他角色不得提交竞品情报。
- **[BR-03]预警定版权限**：预警记录的定版（含确认/修正 AI 标注的风险等级与可选备注）仅限售前角色执行；MKT 和责任销售不得直接定版或修改已定版预警内容。
- **[BR-04]冲突裁决**：当同一竞品威胁下存在多条情报提交且内容存在冲突时，售前须查阅全部已提交情报后方可定版；定版结果须体现裁决结论，未被采纳的提交须标注未采纳原因，不得静默忽略。
- **[BR-05]风险等级标注**：竞品预警的风险等级（高/中/低）由系统基于定版内容自动判定，售前确认或修正后生效，依据竞品威胁接近度与受影响商机价值两个维度交叉判定。竞品威胁接近度分三档：（1）直接接触——竞品已接触客户决策层、客户主动提及竞品方案或竞品已提交报价/POC；（2）间接信号——竞品在客户周边活跃（如同行业客户中标）或客户技术层有接触但未上升决策层；（3）市场传闻——竞品发布新产品或新策略但无客户侧直接接触证据。受影响商机价值参考商机金额、客户战略等级与当前漏斗阶段综合评估，分为高/中/低三档。交叉判定规则：直接接触×高价值=高风险、直接接触×中价值=高风险、直接接触×低价值=中风险；间接信号×高价值=高风险、间接信号×中价值=中风险、间接信号×低价值=低风险；市场传闻×高价值=中风险、市场传闻×中/低价值=低风险。未完成风险等级标注的预警不得进入定版状态，不得触发通知。（参见 Epic [BR-02]）
- **[BR-06]定版通知规则**：售前完成预警定版后，系统须同时通知 MKT Leader 与受影响客户的责任销售；责任销售的确定依据 E-01 分配推荐的最终确认结果；E-01 不可用时，降级为仅通知 MKT Leader 由其人工分发，并标记为降级通知状态。`[推导-待确认：降级触发阈值与恢复时机]`（参见 Epic [BR-03]、[DEP-02]）
- **[BR-07]去重规则**：同一竞品针对同一客户在同一自然日内，多条情报提交须合并至同一预警上下文供售前统一裁决，不允许产生多条内容实质相同的定版预警。`[推导-待确认：去重判定口径]`
- **[BR-08]定版备注**：售前在定版时可选择性补充备注信息（如补充说明、风险判断依据等）；备注为可选字段，不强制填写；备注内容随预警记录一并存储，不独立为单独业务对象。
- **[BR-09]提取确认必经**：系统关键词提取生成的情报草稿在提交人确认前不具有正式效力；若提取结果未识别出竞品名称或受影响客户，须提示提交人手动补全，确认后的情报记录须至少包含竞品名称、威胁描述与受影响客户（≥1 名），否则不可录入。
- **[BR-10]关联信息触发风险等级调整推荐**：已定版预警新增关联情报信息后，AI 须基于新、旧信息综合重新评估风险等级（沿用 BR-05 交叉判定规则），并向售前推荐调整方案（维持 / 升级 / 降级），售前确认或修正后生效；AI 推荐仅供参考，最终调整决定权归售前；每次调整须留存变更前后等级、触发信息与确认人记录。（参见 Epic [BR-04]）
- **[BR-12]竞品提及次数统计**：系统须按竞品名称维度统计其在全部已定版预警中被提及的累计次数；统计范围仅包含已定版预警记录中的竞品名称字段，未定版的情报提交不计入统计；MKT Leader 可按时间段筛选统计范围（如近 30 天、近 90 天、自定义起止日期），默认展示全量累计；MKT Leader 可查看该统计数据，用于辅助判断竞品活跃度与威胁趋势。（参见 Epic [BR-06]）
- **[BR-11]AI 主动提醒规则**：AI 须对以下两种情形发起提醒——（1）高风险预警自上次信息更新起超过 2 周（14 自然日）无新情报信息补充；（2）任意风险等级的预警自定版或上次等级调整起超过 4 周（28 自然日）未降级。提醒分两路发送：向定版售前发送复核提醒（售前可选择维持当前等级并附理由，或执行等级调整）；向受影响客户的责任销售发送信息补充提示（询问是否有新竞品动态可提交，销售可响应也可忽略，不强制要求回复）。已执行操作的提醒不重复触发，直至再次满足触发条件。`[推导-待确认：提醒时间窗口 2 周/4 周需与业务方确认]`（参见 Epic [BR-05]）

## 业务对象定义 (Business Object Definition)

### 竞品情报提交记录

竞品情报提交记录是 MKT、售前或责任销售通过语音、文字或文档通道输入竞品动态后，经系统提取与人工确认形成的正式情报凭证，承载提交人视角的威胁描述与受影响客户，供售前审核裁决时参照。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 提交人 | 标识本条情报由哪位人员提交 |
| 提交人角色 | MKT / 售前 / 责任销售 |
| 输入通道 | 标识本条情报的输入方式：语音输入 / 文字输入 / 文档输入 |
| 竞品名称 | 本条情报所涉及的竞品主体（系统提取 + 人工确认） |
| 威胁描述 | 提交人对竞品动态的描述（系统提取 + 人工确认） |
| 受影响客户 | 提交人认为受本次竞品动态影响的客户（≥1 名）（系统提取 + 人工确认） |
| 提交时间 | 情报被提交的时间点 |
| 采纳状态 | 售前定版后标记：已采纳 / 未采纳（含未采纳原因） |

### 竞品预警记录

竞品预警记录是售前对一条或多条竞品情报提交进行审核裁决后形成的定版业务凭证，承载权威的竞品威胁判定、受影响范围、风险等级与通知结论，并作为后续处置闭环和 AI 辅助的上下文基准。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 竞品名称 | 标识本条预警所针对的竞品主体 |
| 威胁描述 | 售前定版后的竞品威胁判定要点，作为 MKT 与责任销售理解竞品动向的核心信息 |
| 风险等级 | 售前在定版时标注的威胁严重程度（高/中/低），作为通知优先级与处置优先级的判定依据 |
| 受影响客户列表 | 本次竞品威胁所关联的受影响客户集合，每条预警至少关联一名客户 |
| 关联情报提交 | 标识本条定版预警所裁决的全部原始情报提交记录，含采纳/未采纳标记 |
| 通知对象 | 标识本次定版后应通知的 MKT Leader 与责任销售，或在降级场景下仅通知 MKT Leader |
| 通知状态 | 标识本条预警的通知是否已完成发送，以及是否处于降级通知状态 |
| 定版售前 | 执行定版操作的售前人员 |
| 定版时间 | 预警完成定版的时间点 |
| 定版备注 | 售前在定版时可选补充的说明信息（如风险判断依据、额外背景等），可选字段 |
| 信息来源标识 | 关联触发本次预警的竞品信息来源（若有外部数据源），供追溯核查 |

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 操作人为 MKT / 售前 / 责任销售之一；通过语音、文字或文档通道输入竞品相关内容 | 系统完成关键词提取并展示情报草稿，提交人确认或修正后提交 | 情报提交记录创建成功，包含竞品名称、威胁描述与受影响客户（≥1 名），进入待定版状态；非上述三类角色提交时被拒绝（参见 BR-01、BR-02、BR-09） | US-01 |
| AC-02 | 同一竞品威胁下存在 ≥2 条情报提交且内容存在冲突；操作人为售前 | 售前查阅全部提交并执行定版操作 | 定版预警记录创建，未采纳提交标注未采纳原因；非售前角色尝试定版时被拒绝（参见 BR-03、BR-04） | US-02 |
| AC-03 | 售前执行定版操作 | 售前标注风险等级，可选补充备注 | 风险等级写入预警记录；备注（如有）随预警记录存储；未标注风险等级时定版操作不可完成（参见 BR-05、BR-08） | US-02 |
| AC-04 | 预警已定版；E-01 分配结果可用 | 定版完成触发通知 | MKT Leader 与受影响客户责任销售同时收到定版预警通知，含受影响客户名单、风险等级（参见 BR-06） | US-03, US-04 |
| AC-05 | 预警已定版；E-01 分配结果不可用 | 定版完成触发通知 | 降级为仅通知 MKT Leader，预警记录标记降级通知状态（参见 BR-06） | US-04 |
| AC-06 | 同一竞品同一客户当日已存在情报提交；同日再次收到同源情报 | 新情报提交 | 新提交合并至同一预警上下文，不产生独立预警记录（参见 BR-07） | US-01 |
| AC-07 | 仅存在 1 条情报提交且无冲突；操作人为售前 | 售前执行定版操作 | 定版预警记录创建成功，流程与多条提交一致（参见 BR-03、BR-05） | US-02 |
| AC-08 | 已定版预警新增关联情报信息 | AI 完成重新评估并推荐风险等级调整方案 | 售前收到调整推荐（维持/升级/降级），确认或修正后新等级生效；变更前后等级、触发信息与确认人记录留存（参见 BR-10） | US-05 |
| AC-09 | 高风险预警 ≥2 周无新信息，或任意预警 ≥4 周未降级 | AI 触发提醒 | 定版售前收到复核提醒，可选择维持（须附理由）或调整等级；责任销售收到信息补充提示；已处理提醒不重复触发（参见 BR-11） | US-06, US-07 |
| AC-10 | MKT Leader 查看竞品提及统计 | 查看竞品名称累计提及次数，可按时间段筛选 | 系统按竞品名称展示已定版预警中的累计提及次数，支持按时间段筛选（近 30 天、近 90 天、自定义起止日期），默认全量累计；未定版情报不计入（参见 BR-12） | US-08 |
