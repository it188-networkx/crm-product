# Feature Definition - 竞品动态推送与客户关联 (Competitive Alert Push & Customer Association)

## 特性概述 (Feature Overview)

**所属 Epic**：[[T-02/E-03] 竞品预警与监控 (Competitive Alerts & Monitoring)](../README.md)

**业务定义**：

该特性用于将竞品信息来源中检测到的竞品动态转化为结构化竞品预警，并将预警与受影响客户名单自动关联，推送给对应责任销售；同时允许售前人员针对具体预警补充技术应对建议，使预警具备可操作的业务参考价值。
它关注的是"竞品动态如何被识别并变成可追溯、可推送、可引用的结构化预警记录"以及"应对建议如何被完整绑定至预警上下文"，不定义预警在看板中的展示形式、处置状态流转，也不定义应对结果的闭环记录。

**范围边界**：

1. 包含：竞品预警生成与风险等级标注、受影响客户关联、向责任销售推送预警、售前补充技术应对建议并关联预警记录。
2. 不包含：竞品抢单监控看板展示（F-02）、处置状态变更与闭环记录（F-03）、日报竞品挑刺高亮呈现（F-02）。
3. 外部依赖：竞品信息来源须由外部数据订阅或内部知识库提供，来源不可用时预警生成能力降级（参见 Epic [DEP-01]）。
4. 责任销售确定依赖 E-01 客户分配推荐结果，分配结果不可用时推送降级为通知 MKT Leader 人工分发（参见 Epic [DEP-02]）。
5. 预警推送优先级受当前商机阶段影响，阶段数据不可用时不按阶段区分优先级（参见 Epic [DEP-03]）。
6. 预警记录的数据结构须与 E-02 AI 跟进辅助共享，变更须协同确认（参见 Epic [DEP-04]）。

## 业务规则 (Business Rules)

- **[BR-01]预警必须关联客户**：每条竞品预警必须关联至少一名受影响客户；不得产生无客户关联的空泛预警。（参见 Epic [BR-01]）
- **[BR-02]预警分级**：竞品预警必须附带风险等级标注（高/中/低），等级依据竞品威胁程度与受影响商机价值综合评估；未完成风险等级标注的预警不得进入推送队列。`[推导-待确认：风险等级评估规则]`（参见 Epic [BR-02]）
- **[BR-03]预警推送责任人**：预警必须推送给受影响客户的责任销售，不得推送给与该客户无负责关系的其他销售人员；责任销售的确定依据 E-01 分配推荐的最终确认结果。（参见 Epic [BR-03]）
- **[BR-04]应对建议关联**：售前补充的技术应对建议必须关联至对应竞品预警记录，不允许应对建议与预警分离存储。（参见 Epic [BR-07]）
- **[BR-05]推送降级规则**：当 E-01 分配结果不可用时，预警无法精准推送至责任销售，须降级为推送给 MKT Leader 由其人工分发，并明确标记当前为降级推送状态。`[推导-待确认：降级触发阈值与恢复时机]`
- **[BR-06]去重规则**：同一竞品针对同一客户在同一自然日内不得重复生成独立预警记录；若同日出现多条同源动态，须合并为单条预警并更新威胁描述，不允许产生多条内容实质相同的预警。`[推导-待确认：去重判定口径]`
- **[BR-07]应对建议提交资格**：技术应对建议仅允许售前人员提交；责任销售和 MKT Leader 不得直接编辑应对建议内容，但可查看。`[推导-待确认：角色权限边界]`

## 业务对象定义 (Business Object Definition)

### 竞品预警记录

竞品预警记录是竞品动态被系统识别后形成的结构化业务凭证，承载竞品威胁来源、受影响范围、风险等级与推送结论，并作为后续处置闭环和 AI 辅助的上下文基准。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 竞品名称 | 标识本条预警所针对的竞品主体 |
| 威胁描述 | 描述触发本次预警的竞品动态要点，作为责任销售与售前理解竞品动向的核心信息 |
| 风险等级 | 标识本条预警的威胁严重程度（高/中/低），作为推送优先级与处置优先级的判定依据 |
| 受影响客户列表 | 记录与本次竞品动态相关的受影响客户集合，每条预警至少关联一名客户 |
| 责任推送对象 | 标识本次预警应推送的责任销售或在降级场景下推送的 MKT Leader |
| 推送状态 | 标识本次预警是否已完成推送，以及是否处于降级推送状态 |
| 信息来源标识 | 关联触发本次预警的竞品信息来源，供追溯核查 |
| 预警生成时间 | 记录系统识别竞品动态并完成预警生成的时间点，作为时效管理和审计依据 |

### 技术应对建议

技术应对建议是售前针对某条竞品预警提供的结构化应对参考，与竞品预警记录绑定存储，供责任销售在商机跟进中直接引用，同时作为 E-02 AI 跟进辅助的上下文输入。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属预警 | 标识本条建议归属的竞品预警记录，确保建议与预警不分离 |
| 建议内容摘要 | 售前针对该竞品威胁撰写的技术层面差异化应对要点 |
| 提交售前 | 标识由哪位售前人员提交了该条建议，作为责任归属与追溯依据 |
| 提交时间 | 记录建议被提交的时间点 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 竞品信息来源正常可用；存在可关联的受影响客户 | 系统检测到新竞品动态 | 生成竞品预警记录，包含威胁描述、关联受影响客户列表（≥1 名）、风险等级标注；未附带风险等级的预警不进入推送队列（参见 BR-01、BR-02） | US-01 |
| AC-02 | 竞品预警记录已生成 | MKT Leader 查看预警摘要 | 可查看竞品预警摘要，包含受影响客户名单、威胁描述和风险等级，不同风险等级的预警在摘要中可区分 | US-01 |
| AC-03 | 竞品预警已完成风险等级标注；E-01 分配结果可用 | 触发预警推送 | 预警推送至受影响客户对应的责任销售，不推送给无客户负责关系的人员（参见 BR-03） | US-02, US-03 |
| AC-04 | 责任销售已接收到推送预警 | 责任销售查看预警通知 | 可查看预警详情，包括竞品威胁描述、风险等级和受影响客户信息 | US-03 |
| AC-05 | E-01 分配结果不可用 | 触发预警推送 | 预警降级推送至 MKT Leader，并在预警记录中明确标记当前为降级推送状态（参见 BR-05） | US-02 |
| AC-06 | 已存在目标竞品预警记录；操作人为售前角色 | 售前提交技术应对建议 | 建议被记录并与竞品预警关联存储，非售前角色不得提交建议（参见 BR-04、BR-07） | US-04 |
| AC-07 | 同一竞品同一客户当日已存在预警记录；同日再次出现同源动态 | 系统检测到重复竞品动态 | 不产生新的独立预警记录，现有预警的威胁描述被更新合并，最终为单条记录（参见 BR-06） | US-01 |
