# Epic - [T-02/E-03] 竞品预警与监控 (Competitive Alerts & Monitoring)

## 概述 (Overview)

**业务背景：**

当前团队缺乏系统性竞品动态监控机制，销售人员只能通过客户临时反馈或偶发信息了解竞品动向，无法提前识别受竞品威胁的商机并主动应对。竞品相关风险在商机层面缺乏系统记录和处理闭环，日报中竞品挑刺信息也未被结构化沉淀供后续决策引用。`[缺少需求输入-待补充：无 A0106 需求报告，As-Is 数据来源于 A0302 推导]`

**核心目标：**

本 Epic 旨在建立竞品动态监控与预警推送能力：通过持续监控竞品动态，将预警关联到具体受影响客户并推送给对应责任销售；同时提供竞品抢单监控看板，按商机和责任人追踪竞品风险等级，确保竞品威胁从识别到处置形成完整闭环。

**价值预估：**

- **预警覆盖**：受竞品威胁的商机预警覆盖率从 0 提升至可量化水平。`[推导-待确认：目标值待业务定义]`（支撑 Theme OC-05 竞品预警可关联）
- **处置闭环**：收到竞品预警的商机完成处理闭环的比例 ≥ 80%（M6）。`[推导-待确认]`（支撑 Theme OC-05）
- **建议质量**：竞品应对建议满意度 ≥ 3.5/5（M3）。（承接 Theme KPI：AI 建议满意度 ≥ 3.5/5 M3）

## 业务场景 (Business Scenarios)

### 场景一：MKT Leader 接收竞品预警并调度差异化应对

**用户**：MKT Leader（市场管理）

**情境**：系统识别到竞品主动攻入某批客户池，MKT Leader 需快速评估影响范围并协调销售与售前制定应对策略。

**业务流程**：

1. 系统检测到新竞品动态（信息来源变化或关键词触发），生成竞品预警并关联到受影响客户名单。（F-01）
2. MKT Leader 查看竞品预警摘要：受影响客户列表、竞品威胁描述与风险等级评估。（F-01）
3. MKT Leader 将高风险预警推送给对应责任销售，并触发 E-02 AI 跟进辅助生成差异化应对跟进建议。（F-01）
4. MKT Leader 在竞品抢单监控看板中追踪各商机的竞品风险状态与处置进度。（F-02）
5. 受影响商机完成应对后，责任销售在看板中标记处置闭环状态，系统结束该预警的追踪周期。（F-03）

### 场景二：日报竞品挑刺高亮与责任人跟进

**用户**：销售执行层（Sales Rep）/ 售前执行层

**情境**：每日例行中，销售在日报中看到针对自己负责商机的竞品挑刺高亮，需要评估并完成处理。

**业务流程**：

1. 系统在日报中高亮展示与销售负责商机相关的竞品挑刺要点（新预警、升级风险）。（F-01、F-02）
2. 销售查看竞品挑刺详情，评估风险等级并决定应对方式（自行应对 / 协同售前）。（F-02）
3. 售前（如协同）补充技术应对建议，关联至该商机的竞品预警记录。（F-01）
4. 销售完成差异化应对后，在竞品监控看板中更新处理状态，系统将本次应对结果沉淀至竞品风险记录。（F-02、F-03）

## 业务规则 (Business Rules)

### 预警推送规则

- **[BR-01]预警必须关联客户**：每条竞品预警必须关联至少一名受影响客户；不得产生无客户关联的空泛预警。（适用 F-01；对应 Theme OC-05 竞品预警可关联）
- **[BR-02]预警分级**：竞品预警必须附带风险等级标注（高/中/低），等级依据竞品威胁程度与受影响商机价值综合评估，不允许未分级的预警进入推送队列。`[推导-待确认：风险等级评估规则]`（适用 F-01、F-02）
- **[BR-03]预警推送责任人**：预警必须推送给受影响客户的责任销售，不得发送给无负责关系的销售人员；责任归属依据 E-01 分配推荐的最终确认结果。（适用 F-01）

### 监控看板规则

- **[BR-04]日报高亮逻辑**：日报中的竞品挑刺高亮须仅包含当天或最近 N 天内新产生的预警，已闭环的预警不再高亮。`[推导-待确认：N 天定义]`（适用 F-02）
- **[BR-05]看板筛选能力**：竞品抢单监控看板必须支持按商机、责任人、风险等级三个维度独立或组合筛选；不得仅提供固定的全量视图。（适用 F-02）

### 处置闭环规则

- **[BR-06]处置结果记录**：每次竞品应对完成后，责任销售必须填写处置结果（应对摘要 + 处置状态），系统方可将该预警标记为已闭环，不允许静默自动关闭预警。（适用 F-02、F-03）
- **[BR-07]应对建议关联**：售前补充的技术应对建议必须关联至对应竞品预警记录，不允许应对建议与预警分离存储，以保障 E-02 跟进辅助在引用应对上下文时的一致性。（适用 F-01、F-03）

## 非功能需求 (Non-Functional Requirements)

- **[NFR-01]预警推送时效**：竞品动态被系统识别后到预警推送至责任销售，端到端时延不超过 24 小时。`[推导-待确认：时效要求视竞品信息来源刷新频率而定]`
- **[NFR-02]看板加载时效**：竞品抢单监控看板全量加载时间不超过 3 秒（基于当前数据规模）。`[推导-待确认]`
- **[NFR-03]操作可审计**：竞品预警推送、处置状态变更等关键操作均须留存操作人、时间及变更内容记录，支持按商机、责任人检索。

## 验收标准 (Acceptance Criteria)

- **[AC-01]预警关联客户**：每条竞品预警关联至少一名受影响客户，不产生无关联空泛预警（承接 Theme OC-05，落地 BR-01、BR-03）。
- **[AC-02]预警分级可见**：预警附带风险等级标注（高/中/低），未分级预警不进入推送队列（落地 BR-02）。
- **[AC-03]看板可筛选**：竞品抢单监控看板支持按商机、责任人、风险等级独立或组合筛选（承接 Theme OC-05，落地 BR-04、BR-05）。
- **[AC-04]处置闭环完整**：责任销售填写处置结果后系统方可标记预警已闭环，不允许静默自动关闭（承接 Theme OC-05，落地 BR-06、BR-07）。
- **[AC-05]操作可审计**：预警推送与处置变更留存完整审计日志（落地 NFR-03）。
- **[AC-06]降级可用**：外部竞品数据源或 E-01 分配结果不可用时，预警降级为推送 MKT Leader 人工分发（落地 DEP-01、DEP-02）。

## 约束与依赖 (Constraints & Dependencies)

- **[DEP-01]外部竞品信息来源**：竞品动态监控依赖外部数据订阅或内部知识库提供稳定的竞品信息输入；无稳定竞品数据源时，竞品预警功能不可用，需向用户明确提示。（影响 F-01）
- **[DEP-02]E-01 客户分类与分配结果**：预警推送对象（责任销售）依赖 E-01 分配推荐的最终分配结果；E-01 分配结果不可用时，预警无法精准推送，降级为推送给 MKT Leader 人工分发。（影响 F-01）
- **[DEP-03]T-01 E-03 商机阶段联动**：竞品预警推送时机须参考 T-01/E-03（商机推进与阶段门控）的商机当前阶段，优先对处于关键决策阶段的商机推送高优先级预警；T-01/E-03 阶段数据不可用时，预警推送不按商机阶段区分优先级。`[推导-待确认：协同界面尚未明确]`（影响 F-01、F-02）
- **[DEP-04]E-02 跟进辅助上下文注入**：本 Epic 的竞品预警上下文须能被 E-02（AI 跟进辅助）消费，以生成差异化跟进建议；两者共享竞品预警记录的数据结构，变更须协同确认。（影响 F-01、F-03）
- **[ASM-01]竞品信息时效假设**：假设外部竞品信息来源提供的数据具备足够时效性（如日更新），若更新频率不足，则预警时效性 NFR-01 可能无法满足，需要业务方降低预警时效要求。（影响 F-01）
