# Feature - [F-02] 规则自动触发执行 (Rule Auto-Trigger Execution)

> 需求特性定义（A0304）｜所属 Epic：E-05 营销自动化规则管理
> **Feature 编号**：F-02 ｜ **优先级**：P0 ｜ **状态**：Draft

## 特性概述 (Feature Overview)

**所属 Epic**：[E-05 营销自动化规则管理](../README.md)

**业务定义**：
本特性由系统自动执行，无需人工逐条触发。系统定期扫描所有处于"激活"状态的培育触发规则，逐一判断客户是否满足触发条件；对于条件满足且当前不在冷却期内的客户，系统在配置的触发延迟时长到期后执行规则动作——向负责销售推送待跟进提醒，并在 E-02 中写入"自动触达"类型跟进记录，同时关联触发规则 ID；客户进入冷却期以防止短期内重复触达；满足退出条件的客户立即退出规则触达范围。

本特性聚焦"扫描—条件匹配—冷却判定—延迟等待—执行动作—E-02 记录写入—退出判定"的自动化执行链路，不涵盖规则的配置与管理（见 F-01）及运行数据监控（见 F-03）。

**范围边界**：

1. 包含：定期规则触发扫描、触发条件匹配（基于 E-06 标签）、冷却期判定、触发延迟等待与取消、执行动作（提醒推送 + E-02 跟进记录写入）、客户退出规则范围的自动处理。
2. 不包含：规则配置与发布管理（F-01）、规则运行监控报告（F-03）、自动外发邮件/微信/短信等外部消息（参见 Epic [BR-05]）、销售人工完成跟进后的记录更新（销售自主操作，属于 E-02 范围）。
3. 跨 Epic 约束承接：触发条件基于 E-06 标签扫描（参见 Epic [DEP-01]）；执行动作须向 E-02 写入自动触达记录（参见 Epic [DEP-02]）；触发条件可使用 E-01 线索分级结果（参见 Epic [DEP-03]）；扫描周期须在 1 小时以内（参见 Epic [NFR-01]）。

## 业务规则 (Business Rules)

- **[BR-01]仅激活规则参与扫描**：系统扫描范围严格限于处于"激活"状态的规则；草稿、待审批、暂停、归档状态的规则不参与任何扫描与触发执行（参见 Epic [BR-04]）。
- **[BR-02]触发条件基于标签**：扫描时触发条件的匹配判断须基于 E-06 标签体系的当前标签值；若触发条件引用的标签在 E-06 中已废弃，该规则的本轮扫描结果跳过该条件匹配，不触发任何动作，并生成异常记录（参见 Epic [BR-01]）。
- **[BR-03]冷却期优先于重复触发**：对同一客户，同一规则在冷却期内不得重新触发执行动作，即使触发条件在冷却期内再次满足；冷却期从上一次动作执行完成时开始计算，以冷却时长（天）为单位（参见 Epic [BR-02]）。
- **[BR-04]退出条件优先于冷却**：客户满足退出条件时，系统立即将其从该规则的触达范围中移除，不论其当前是否处于冷却期，后续扫描不再匹配该客户（参见 Epic [BR-06]）。
- **[BR-05]触发延迟处理**：触发条件满足后，若规则配置的触发延迟为 0，系统在当前扫描周期内立即执行动作；延迟 > 0 时，系统记录待执行状态并在延迟时长到期后方可执行；延迟等待期间若退出条件率先满足，该次待执行动作取消，不再执行（参见 Epic [BR-07]）。
- **[BR-06]执行动作范围限定**：系统执行的动作严格限于：向负责销售推送待跟进提醒通知（含规则名称与触发原因摘要），以及在 E-02 中写入"自动触达"类型跟进记录并关联触发规则 ID；不允许自动发送任何外部消息（参见 Epic [BR-05]）。
- **[BR-07]E-02 记录写入原子性**：动作执行的提醒推送与 E-02 跟进记录写入须视为一个业务操作单元；若 E-02 不可用，提醒推送降级为仅发送提醒（不写记录），并在系统层面标记本次执行为"降级执行"（参见 Epic [DEP-02]）。
- **[BR-08]扫描周期时效**：规则触发扫描的执行周期不超过 1 小时，确保时间敏感触发条件（如"7 日无互动到期"）在下一个扫描窗口内被识别并触发（参见 Epic [NFR-01]）。

## 业务对象定义 (Business Object Definition)

### 自动触达记录

自动触达记录是系统在规则执行动作时向 E-02 写入的跟进记录类型，用于在销售跟进时间线上标识一次由规则触发的系统自动培育动作，并保留触发来源可追溯性。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联客户 | 标识本次自动触达针对的客户 |
| 关联规则 | 标识触发本次动作的培育触发规则 ID 与版本号 |
| 触发时间 | 系统执行动作的时间戳（延迟到期后实际执行时间） |
| 记录类型 | 固定为"自动触达"，与人工跟进记录区分 |
| 提醒接收人 | 标识本次提醒通知发送给哪位负责销售 |

### 规则触发执行状态

规则触发执行状态描述某条规则对某个客户在当前周期内的执行进程，用于支撑冷却期判定、延迟等待管理与退出条件检测。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属规则 | 标识该执行状态归属哪条培育触发规则 |
| 关联客户 | 标识该执行状态针对哪个客户 |
| 执行状态 | 标识当前进程阶段（延迟等待中 / 已执行 / 已取消 / 已退出） |
| 冷却期截止时间 | 本次执行完成后不得重复触发的期限，基于冷却时长计算 |
| 执行时间 | 动作实际执行完成的时间（延迟等待期动作取消时不填写） |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 存在至少一条激活规则；存在符合其触发条件且不在冷却期内的客户 | 系统执行周期性扫描 | 系统识别出符合条件的客户，触发后续执行流程（参见 BR-01、BR-03） | US-01 |
| AC-02 | 存在激活规则；客户处于冷却期内 | 系统执行周期性扫描 | 该客户在本次扫描中不触发执行动作（参见 BR-03） | US-01, US-04 |
| AC-03 | 存在草稿、暂停或归档状态规则 | 系统执行周期性扫描 | 上述状态的规则不参与本次扫描，不产生任何触发记录（参见 BR-01） | US-01 |
| AC-04 | 触发条件满足；触发延迟为 0；客户不在冷却期内 | 扫描周期到达执行时点 | 系统立即推送提醒给负责销售，并在 E-02 写入"自动触达"类型跟进记录（含规则 ID）；客户进入冷却期（参见 BR-05、BR-06） | US-02, US-04 |
| AC-05 | 触发条件满足；触发延迟 > 0；客户不在冷却期内；延迟期内退出条件未满足 | 延迟时长到期 | 系统推送提醒给负责销售，并在 E-02 写入"自动触达"类型跟进记录；提醒信息包含规则名称与触发原因摘要；客户进入冷却期 | US-02, US-06 |
| AC-06 | 触发条件已满足，处于延迟等待期；客户在等待期间满足退出条件 | 退出条件检测生效 | 本次待执行动作取消，不推送提醒，不写 E-02 记录；执行状态标记为"已取消"；客户从该规则触达范围移除（参见 BR-04、BR-05） | US-03, US-05 |
| AC-07 | 客户已被规则触发执行动作 | 系统下一次扫描到来 | 冷却期内客户不被再次匹配为待触发对象；冷却期截止时间由上次执行时间加冷却时长计算（参见 BR-03） | US-04 |
| AC-08 | 客户当前在规则触达范围内（非冷却期） | 客户的标签变更导致满足退出条件 | 系统在下一次扫描中识别退出条件成立，立即将该客户从规则触达范围中移除；后续扫描不再匹配该客户（参见 BR-04） | US-05 |
