# Feature - [F-01] 规则配置与管理 (Rule Configuration & Management)

> 需求特性定义（A0304）｜所属 Epic：E-05 营销自动化规则管理
> **Feature 编号**：F-01 ｜ **优先级**：P0 ｜ **状态**：Draft

## 特性概述 (Feature Overview)

**所属 Epic**：[E-05 营销自动化规则管理](../README.md)

**业务定义**：
本特性面向 MKT Leader，提供培育触发规则的全生命周期配置与管理能力。MKT Leader 可通过该特性新建规则并定义触发条件（基于 E-06 标签体系）、触发延迟时长、执行动作（向销售推送提醒并写入 E-02 跟进记录）、冷却策略（冷却时长）与退出条件；规则完成配置后须经发布审批方可生效，每次变更均留存版本记录，支持历史版本查阅与对比，实现规则策略的可配置、可审计、可迭代闭环。

本特性聚焦规则的"定义—审批—发布—变更—版本管理"全链路，不覆盖规则的自动触发执行（见 F-02）及运行数据监控（见 F-03）。

**范围边界**：

1. 包含：规则新建、触发条件配置（基于 E-06 标签）、触发延迟时长配置、执行动作配置（提醒通知 + E-02 记录写入）、冷却策略配置、退出条件配置、规则合规性校验、发布审批流、版本记录与历史查阅、规则暂停与归档。
2. 不包含：规则触发扫描与执行（F-02）、规则运行监控报告（F-03）、对外发送邮件/微信/短信（参见 Epic [BR-05]）、标签体系维护（E-06 范围）、执行动作超出提醒+E-02记录的高级动作类型（V1 范围外，参见 Epic [BR-05]）。
3. 跨 Epic 约束承接：触发条件须基于 E-06 标签体系（参见 Epic [DEP-01]）；执行动作须向 E-02 写入记录（参见 Epic [DEP-02]）；触发条件可使用 E-01 线索分级结果（参见 Epic [DEP-03]）。

## 业务规则 (Business Rules)

- **[BR-01]触发条件须基于标签**：自动化规则的触发条件必须基于 E-06 客户标签体系的标签值定义，不允许绕过标签体系直接引用原始线索字段；配置时须从标签字典中选取有效标签；若触发条件引用的标签在 E-06 标签字典中已被废弃或删除，系统须阻止该规则发布并提示条件不合规（参见 Epic [BR-01]）。
- **[BR-02]冷却时长必须大于零**：每条规则的冷却时长必须设置为大于 0 的正整数（单位：天），冷却时长缺失或为零时系统阻止规则进入发布审批流程（参见 Epic [BR-02]）。
- **[BR-03]发布须经审批**：规则的首次发布与每次变更发布均须提交审批，记录操作人与提交时间；审批通过后方可生效，审批拒绝时规则退回草稿状态并保留拒绝原因（参见 Epic [BR-03]）。
- **[BR-04]版本号自动递增**：规则每次审批通过发布后，系统自动为该规则分配新版本号，并记录发布时间戳、变更摘要与审批人；历史版本须留存且不可删除，支持查阅但不可重新激活（参见 Epic [BR-03]）。
- **[BR-05]规则状态流转约束**：规则的合法状态为：草稿 → 待审批 → 激活；激活状态可转为暂停或归档；草稿与待审批状态不参与扫描执行（参见 Epic [BR-04]）；已归档规则不可恢复为激活或草稿。
- **[BR-06]执行动作范围限定**：执行动作配置范围仅限于：向负责销售推送待跟进提醒，以及指定在 E-02 中写入"自动触达"类型跟进记录；不允许配置自动外发邮件、微信或短信等外部消息（参见 Epic [BR-05]）。
- **[BR-07]退出条件可选但须逻辑一致**：退出条件为可选配置；若配置了退出条件，其标签引用须与 E-06 标签字典一致，且退出条件不得与触发条件在逻辑上相互包含（即触发条件成立时退出条件同时成立视为配置冲突，系统须阻止发布并提示冲突原因）。
- **[BR-08]规则唯一名称**：同一系统内规则名称须唯一，不允许存在两条同名规则（含已归档规则）；重命名时须同样校验唯一性。

## 业务对象定义 (Business Object Definition)

### 培育触发规则

培育触发规则是 MKT Leader 定义的一套培育执行策略单元，描述"在什么条件下、等待多久、对哪类客户执行什么动作、多久不重复、什么情况下退出"。规则是 F-02 自动触发执行的直接配置依据。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 规则名称 | 唯一标识该规则的业务名称，便于 MKT 识别与引用 |
| 触发条件 | 基于 E-06 标签字典定义的一组标签匹配逻辑，用于判断客户是否应进入本规则的触发范围 |
| 触发延迟时长 | 触发条件满足后至实际执行动作的等待时间（单位：天），可为 0（立即执行） |
| 执行动作 | 条件满足后系统执行的业务行为：向负责销售推送提醒，并在 E-02 写入"自动触达"类型跟进记录 |
| 冷却时长 | 同一客户被本规则触发执行动作后，在该时长内不再重复触发（单位：天，必须 > 0） |
| 退出条件 | 基于 E-06 标签字典定义的一组标签匹配逻辑，满足时客户从本规则的触达范围中移除；可选 |
| 规则状态 | 标识规则当前所处的生命周期阶段（草稿 / 待审批 / 激活 / 暂停 / 归档） |
| 当前版本号 | 标识当前生效版本的版本号，每次审批通过后自动递增 |

### 规则版本记录

规则版本记录是培育触发规则每次审批通过发布后的快照存档，用于回溯配置演进历史与支撑合规审计。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属规则 | 标识该版本记录归属哪条培育触发规则 |
| 版本号 | 唯一标识该次发布的版本序号 |
| 发布时间 | 该版本审批通过并生效的时间戳 |
| 审批人 | 完成本次发布审批的责任人标识 |
| 变更摘要 | 本版本相对上一版本的关键变更说明（由操作人填写） |
| 配置快照 | 该版本下规则全量配置内容（触发条件、延迟、执行动作、冷却、退出条件） |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 用户已进入规则新建流程 | 填写规则名称、触发条件（至少一个基于 E-06 标签的条件）、执行动作（提醒+E-02记录）、冷却时长（>0） | 规则以草稿状态保存，可在规则列表中查看 | US-01 |
| AC-02 | 规则草稿已存在 | 配置触发延迟时长（含 0 值） | 触发延迟时长成功保存，0 表示条件满足后立即执行 | US-01 |
| AC-03 | 规则草稿已存在 | 配置退出条件（基于 E-06 标签） | 退出条件成功保存；若未配置退出条件，规则仍可正常进入校验 | US-01 |
| AC-04 | 规则草稿已完成配置 | 触发合规性校验 | 校验通过时允许进入发布审批流程；若冷却时长为 0，阻止并提示须 >0；若触发条件引用的标签在 E-06 中不存在或已废弃，阻止并提示条件不合规；若退出条件与触发条件逻辑冲突，阻止并提示冲突原因（参见 BR-02、BR-01、BR-07） | US-02 |
| AC-05 | 规则通过合规性校验 | 提交发布审批 | 规则状态变为"待审批"，审批请求记录审批提交人与时间 | US-03 |
| AC-06 | 规则处于"待审批"状态 | 审批人审批通过 | 规则状态变为"激活"，系统分配新版本号并记录发布时间、审批人与变更摘要；规则立即纳入 F-02 触发扫描范围（参见 BR-03、BR-04） | US-03 |
| AC-07 | 规则已处于"激活"状态 | MKT Leader 编辑规则配置（如修改冷却时长） | 规则产生新草稿版本，编辑期间原版本继续有效，直至新版本审批通过后替换 | US-04 |
| AC-08 | 规则变更草稿已通过校验并审批通过 | 新版本发布生效 | 系统分配新版本号，新配置立即对 F-02 扫描生效，旧版本归档可查（参见 BR-04） | US-04 |
| AC-09 | 规则存在多个历史版本 | MKT Leader 查阅规则版本历史 | 展示各历史版本的版本号、发布时间、审批人与变更摘要；可查看任一版本的配置快照 | US-05 |
| AC-10 | 规则处于"激活"状态 | MKT Leader 暂停规则 | 规则状态变为"暂停"，F-02 触发扫描立即停止处理该规则；暂停操作留存审计日志 | US-06 |
| AC-11 | 规则处于"激活"或"暂停"状态 | MKT Leader 归档规则 | 规则状态变为"归档"，不可再激活；历史版本与配置记录保留可查；归档操作留存审计日志（参见 BR-05） | US-06 |
