# Feature Definition - [E-07/F-01] 渠道接入配置 (Channel Integration Configuration)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-07] 数据接入与同步 (Data Integration & Sync)]

**业务定义**：
本特性用于支持管理员将外部数据渠道（如邮件平台、官网表单、企业微信等）配置为线索行为数据来源，包括填写连接凭据、定义字段映射规则、执行连接测试，以及对渠道进行激活与停用管理。渠道配置是整个数据接入流程的入口控制点，只有经过管理员显式激活的渠道才能进入增量同步流程（F-02）。

本特性关注"如何建立受控的渠道接入通道"，不涉及激活后的数据拉取与同步执行逻辑。

**范围边界**：

1. 包含：渠道连接参数录入与修改、连接测试、字段映射规则配置、渠道激活与停用、配置变更操作日志留存。
2. 不包含：增量数据拉取与写入（F-02）、手动导入（F-03）、同步状态监控（F-04）、EDM 外发行为回写（F-05）。
3. 连接凭据的访问与修改权限限制为管理员角色，普通用户不可见（参见 Epic [BR-06]）。
4. 映射规则变更须留存版本记录（参见 Epic [BR-02]）。
5. 社媒渠道（企业微信、LinkedIn、微博等）自动接入技术可行性尚未确认，上述渠道不应纳入可激活范围（参见 Epic [ASM-03]）；各渠道须获得对应平台开放授权后方可激活（参见 Epic [DEP-04]）。
6. 官网渠道接入范围限于可关联至具体线索的表单填写提交事件，浏览行为追踪未纳入本特性（参见 Epic [ASM-04]）。

## 业务规则 (Business Rules)

- **[BR-01]连接测试前置**：渠道首次激活前必须完成连接测试且结果为成功；未通过连接测试的渠道不允许激活。
- **[BR-02]激活状态唯一性**：同一渠道在同一时刻只能处于"已激活"或"已停用"两种状态之一；停用操作不删除配置，保留历史参数和映射规则以便重新激活。
- **[BR-03]凭据访问控制**：连接凭据（服务地址、认证信息等）的查看与修改权限仅限管理员角色，其他角色不可见；凭据不得以明文形式出现于操作日志或任何展示界面中（参见 Epic [BR-06]）。
- **[BR-04]映射规则版本化**：每次映射规则变更（新增、修改、删除映射项）须留存变更版本记录，包含变更时间与操作人（参见 Epic [BR-02]）；变更仅影响当前渠道，不影响其他渠道的映射规则（参见 Epic [BR-02]）。
- **[BR-05]授权前置**：渠道须获得对应平台的开放授权后方可配置激活；缺少平台授权的渠道不得进入激活流程（参见 Epic [DEP-04]）。
- **[BR-06]操作留存**：渠道配置的新增、修改、激活、停用操作均须写入操作日志，包含操作时间与操作人，支持后续审计检索（参见 Epic [NFR-03]）。
- **[BR-07]连接失败诊断**：连接测试失败时，结果须包含可定位失败原因的诊断信息（如连接超时、认证失败、地址不可达），不得仅返回通过/失败标记。

## 业务对象定义 (Business Object Definition)

### 渠道配置

渠道配置是管理员建立与外部数据源接入关系的业务记录，承载渠道类型标识、连接参数、映射规则以及当前接入状态。每条渠道配置对应一个外部数据渠道，并在激活后被 F-02 增量同步引用。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 渠道名称 | 由管理员自定义命名，用于在管理界面中唯一标识本条配置 |
| 渠道类型 | 标识外部数据来源类别（如邮件平台、官网表单等），约束可用的连接参数与默认映射结构 |
| 连接凭据 | 建立与外部平台数据通道所需的认证信息（如服务地址、令牌等），访问受 BR-03 约束 |
| 接入状态 | 标识渠道当前是否处于已激活状态，决定 F-02 是否将其纳入同步执行范围 |
| 上次连接测试结果 | 记录最近一次连接测试的通过/失败结论及测试时间，作为激活前置依据 |

### 字段映射规则

字段映射规则定义外部渠道数据字段与线索池业务字段之间的对应关系，是数据写入前的转换契约，确保不同渠道数据按统一业务语义写入线索池。每条规则归属于特定渠道配置，与其他渠道相互独立。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属渠道 | 标识本映射规则归属于哪条渠道配置 |
| 外部字段标识 | 标识来源渠道中的原始字段名称或代码 |
| 目标业务字段 | 标识线索池中对应的业务属性，包含线索来源字段与 E-02 跟进记录动作类型等 |
| 映射版本记录 | 标识该规则的变更历史，包含变更时间与操作人，用于规则变更追溯 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 管理员已具备目标渠道平台授权 | 新增渠道配置，填写渠道名称、类型与连接凭据并保存 | 渠道配置以"待激活"状态留存，操作写入日志 | US-01 |
| AC-02 | 渠道配置已存在且连接参数已填写 | 触发连接测试 | 测试成功时返回连接通过确认；测试失败时返回包含失败原因的诊断信息（参见 BR-07），渠道不得自动激活 | US-02 |
| AC-03 | 渠道配置已存在 | 配置或修改字段映射规则 | 映射规则保存成功，变更留存版本记录（参见 BR-04），不影响其他渠道映射规则 | US-03, US-06 |
| AC-04 | 渠道配置存在且上次连接测试结果为成功 | 激活渠道 | 渠道状态变更为"已激活"，后续被 F-02 纳入同步执行范围，操作写入日志；连接测试未通过时激活操作不生效（参见 BR-01）；已激活渠道执行停用操作后状态变更为"已停用"，F-02 不再拉取其数据，配置保留 | US-04, US-05 |
| AC-05 | 渠道配置已存在 | 修改连接凭据或映射规则后保存 | 变更生效，操作日志记录变更时间与操作人（参见 BR-06）；凭据不以明文形式出现于日志中（参见 BR-03） | US-01, US-06 |
