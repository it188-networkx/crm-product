# Feature Definition - [E-07/F-02] 增量数据同步 (Incremental Data Sync)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-07] 数据接入与同步 (Data Integration & Sync)]

**业务定义**：
本特性用于按配置的同步频率自动从已激活渠道拉取客户交互行为增量数据，并按映射规则将其转换为线索来源记录或 E-02 跟进记录写入线索池。当同步遭遇连接失败时，系统按有限次数自动重试；全部重试失败后将渠道标记为同步异常并向管理员推送告警通知。

本特性是自动化数据接入的执行层，依赖 F-01 完成渠道配置激活后方可运行，对已接入数据须先执行 E-01 去重校验再写入，不直接维护去重规则。

**范围边界**：

1. 包含：已激活渠道的周期性增量数据拉取、按映射规则转换数据、写入线索来源记录或 E-02 跟进记录、连接失败时的自动重试、全量重试失败后的异常告警。
2. 不包含：渠道配置管理（F-01）、手动导入（F-03）、同步状态监控与补抓入口（F-04）、EDM 外发行为回写（F-05）。
3. 接入数据须服从 E-01 定义的去重校验规则，E-07 不独立维护去重逻辑（参见 Epic [DEP-01]）；E-01 未就绪时，接入数据需整体暂存，待 E-01 就绪后批量补处理（参见 Epic [DEP-01]）。
4. 行为数据须按 E-02 定义的跟进记录格式写入；E-02 未就绪时，行为数据仅作为原始事件暂存，不触发跟进记录创建（参见 Epic [DEP-02]）。
5. 行为特征须能被 E-06 标签体系引用；E-06 未就绪时，行为特征仅在线索字段中存储（参见 Epic [DEP-03]）。
6. 自动接入渠道发生同步异常时，系统须保留手动导入通道作为容灾路径（参见 Epic [BR-03]）。
7. 同一外部渠道事件无论同步触发多少次，系统均不得产生重复记录；去重依据为外部渠道中该事件的来源唯一标识（参见 Epic [NFR-04]）。

## 业务规则 (Business Rules)

- **[BR-01]仅同步已激活渠道**：增量同步仅对处于"已激活"状态的渠道执行；已停用渠道不参与同步，直至重新激活。
- **[BR-02]去重写入**：拉取的外部数据在写入线索池前须先执行 E-01 定义的去重校验，重复数据按 E-01 BR-01 处理，不得绕过去重直接写入（参见 Epic [BR-01]）。
- **[BR-03]行为数据写入格式**：自动接入的行为数据（如邮件互动事件）须按 E-02 定义的跟进记录格式写入；邮件渠道中普通往来邮件事件与主动 EDM 推送反馈事件须分别映射为不同动作类型的跟进记录。
- **[BR-04]同步幂等性**：同一外部渠道事件，无论同步触发多少次，系统均不得产生重复的线索来源记录或跟进记录；去重依据为该事件在外部渠道中的来源唯一标识（参见 Epic [NFR-04]）。
- **[BR-05]有限重试**：增量同步遭遇连接失败时，系统须按由短至长的间隔自动重试，且重试次数须有上限；全部重试失败后方可将渠道标记为"同步异常"并触发告警，不得无限重试（参见 Epic [BR-05]）。
- **[BR-06]同步时效**：增量同步的默认执行频率不超过 1 小时，确保外部互动事件在 1 小时内反映至线索池（参见 Epic [NFR-01]）。
- **[BR-07]容灾路径保留**：自动接入渠道发生同步异常期间，手动导入通道（F-03）必须保持可用，不得因自动同步异常而被关闭（参见 Epic [BR-03]）。

## 业务对象定义 (Business Object Definition)

### 同步执行记录

同步执行记录是每次增量同步尝试的业务凭证，记录本次同步的渠道来源、执行结果及数据写入情况，供故障追溯与数据完整性核查使用。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属渠道 | 标识本次同步归属的渠道配置 |
| 执行时间 | 标识本次同步任务的触发时间 |
| 执行结果 | 标识本次同步是成功完成还是遭遇失败（含失败阶段：连接失败、数据解析失败、写入失败） |
| 写入条数 | 本次同步成功写入的记录数量 |
| 跳过条数 | 因去重规则被跳过的记录数量 |
| 重试次数 | 本次同步尝试中的重试轮数 |

### 渠道异常状态

渠道异常状态是全部重试失败后系统对该渠道置入的业务标记，触发向管理员推送告警通知，并留存故障诊断信息供管理员定位根因。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属渠道 | 标识进入异常状态的渠道配置 |
| 异常进入时间 | 标识渠道首次被标记为同步异常的时间 |
| 诊断信息 | 描述导致同步失败的原因（如连接超时、认证过期、数据格式变更等） |
| 告警状态 | 标识告警是否已推送给管理员及管理员是否已处理 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 存在已激活渠道且配置同步频率 | 系统到达同步周期 | 从已激活渠道拉取增量互动数据，同步间隔不超过配置值（最长 1 小时；参见 BR-06）；已停用渠道不被触发（参见 BR-01） | US-01 |
| AC-02 | 增量数据拉取成功，映射规则已配置 | 系统对拉取数据执行转换与写入 | 数据按映射规则转换为线索来源记录或 E-02 跟进记录；写入前执行 E-01 去重校验，重复数据不写入（参见 BR-02）；同一事件不产生重复记录（参见 BR-04）；邮件渠道 EDM 反馈事件与普通往来邮件事件映射为不同动作类型的跟进记录（参见 BR-03） | US-01, US-02 |
| AC-03 | 渠道同步遭遇连接失败 | 系统按重试策略尝试重连 | 按由短至长间隔自动重试，重试次数不超过上限（参见 BR-05）；全部重试失败后渠道标记为"同步异常"，并向管理员推送告警通知（含诊断信息） | US-03 |
| AC-04 | 渠道处于"同步异常"状态，管理员已处理故障 | 管理员手动触发补抓同步 | 系统执行中断期间数据的补抓追赶（补抓逻辑由 F-04 提供入口，本特性保障补抓数据的写入去重与幂等性，参见 BR-04）；补抓完成后渠道恢复正常同步状态 | US-04 |
