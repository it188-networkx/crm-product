# Feature Definition - [E-07/F-03] 手动导入 (Manual Import)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-07] 数据接入与同步 (Data Integration & Sync)]

**业务定义**：
本特性支持 MKT Leader 和销售执行层通过上传标准格式数据文件，将来自社媒渠道、线下活动或其他暂无法自动接入的数据源的线索批量录入线索池。文件上传后，系统解析文件内容并以字段映射预览方式呈现拟导入结果，标注可能存在的字段缺失或格式冲突；用户确认后系统执行导入，对重复记录进行去重提示，导入完成后提供导入摘要并留存操作日志。

本特性既是自动接入不可用时的容灾降级路径，也是社媒等渠道的首选录入方式，与自动同步路径平行存在而非互斥。

**范围边界**：

1. 包含：标准格式文件上传（CSV/Excel 及主流三方 CRM 导出格式）、字段映射预览与冲突提示、去重标注与提示、导入确认执行、导入摘要生成、导入操作日志留存。
2. 不包含：渠道配置管理（F-01）、自动增量同步（F-02）、同步状态监控（F-04）、EDM 外发行为回写（F-05）。
3. 手动导入在自动接入渠道发生同步异常期间必须保持可用，不因自动接入就绪而被关闭（参见 Epic [BR-03]）。
4. 导入数据须服从 E-01 定义的去重校验规则，E-07 不独立维护去重逻辑（参见 Epic [DEP-01]）；E-01 未就绪时，导入数据需整体暂存，待 E-01 就绪后批量补处理（参见 Epic [DEP-01]）。
5. 单次导入须支持 ≥ 1000 条记录，导入完成时延不超过 30 秒（参见 Epic [NFR-02]）。
6. 所有导入操作须留存完整操作日志，支持后续审计检索（参见 Epic [NFR-03]、GC-04）。

## 业务规则 (Business Rules)

- **[BR-01]去重写入**：导入数据在写入线索池前须先执行 E-01 定义的去重校验（手机号或企业名称匹配），重复数据按 E-01 BR-01 处理，不得绕过去重直接写入（参见 Epic [BR-01]）；在导入预览阶段须标注检测到的重复记录并提示合并。
- **[BR-02]支持格式**：手动导入须支持 CSV、Excel 及主流三方 CRM 导出格式；不支持的文件格式须在上传阶段给出明确的格式不支持说明，不得静默失败。
- **[BR-03]容灾可用性**：手动导入能力在自动接入渠道发生同步异常期间必须保持可用，不得因自动接入就绪或运行而被关闭（参见 Epic [BR-03]）。
- **[BR-04]导入操作留痕**：每次导入操作须写入操作日志，包含操作人、操作时间、文件标识、导入结果（成功条数、跳过条数、异常条数），支持后续审计检索（参见 Epic [NFR-03]）。
- **[BR-05]批量导入承载量**：单次导入须支持 ≥ 1000 条记录，导入完成时延不超过 30 秒（参见 Epic [NFR-02]）；超出承载量上限时须明确拒绝并提示分批上传，不得静默截断。
- **[BR-06]预览确认前置**：导入执行须以用户明确确认预览结果为前置条件；未经确认不得直接写入线索池。
- **[BR-07]异常记录说明**：导入过程中遭遇格式异常、必填字段缺失或无法匹配映射规则的记录须单独计入异常条数，并在导入摘要中提供可定位原因的说明；异常记录不阻断其余有效记录的正常写入。

## 业务对象定义 (Business Object Definition)

### 导入批次

导入批次是一次手动导入操作的业务凭证，记录本次导入的来源文件、操作人、执行结果及摘要，用于后续数据溯源与操作审计。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 操作人 | 标识发起本次导入操作的用户 |
| 导入时间 | 标识导入操作的发起时间 |
| 来源文件标识 | 标识本次导入所用的文件（文件名及版本信息），用于溯源 |
| 成功条数 | 本次导入成功写入线索池的记录数量 |
| 跳过条数 | 因去重规则识别为重复而被跳过的记录数量 |
| 异常条数 | 因格式异常或字段映射失败而未能写入的记录数量 |

### 字段映射预览

字段映射预览是在导入执行前向用户呈现的拟导入结果快照，展示外部文件字段与线索池业务字段的对应关系，并标注检测到的冲突、缺失字段及潜在重复记录，供用户确认后决定是否继续。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 文件字段列表 | 来源文件中识别到的所有字段名称 |
| 目标业务字段映射 | 每个文件字段对应的线索池业务属性（包含映射成功、未匹配、冲突状态） |
| 冲突与缺失标注 | 标识存在格式冲突或必填字段缺失的字段项 |
| 重复记录标注 | 标识与线索池已有记录存在去重命中的行数及匹配依据 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 用户持有 MKT Leader 或销售执行层角色权限 | 上传支持格式（CSV/Excel/主流三方 CRM 导出格式）的数据文件 | 文件被成功接收并解析，进入字段映射预览阶段（参见 BR-02）；文件格式不支持时给出明确说明，不静默失败（参见 BR-02）；单次 ≥ 1000 条时在规定时延内完成（参见 BR-05） | US-01 |
| AC-02 | 文件已上传并解析完成 | 查看字段映射预览 | 预览展示文件字段与线索池业务字段的映射关系，标注冲突与缺失字段（参见 BR-06）；存在潜在重复记录时标注命中行数及匹配依据（参见 BR-01） | US-02, US-03 |
| AC-03 | 用户确认预览结果，发起导入 | 系统执行导入写入 | 有效记录按去重校验写入线索池（参见 BR-01）；重复记录按 E-01 BR-01 处理后跳过（不直接写入）；异常记录不阻断其余有效记录写入（参见 BR-07） | US-03 |
| AC-04 | 导入执行完成 | 系统生成导入摘要并留存操作日志 | 用户可查看本次导入的成功条数、跳过条数与异常条数；异常条数 > 0 时提供可定位原因的说明（参见 BR-07）；操作日志包含操作人、时间、文件标识与导入结果（参见 BR-04） | US-04 |
