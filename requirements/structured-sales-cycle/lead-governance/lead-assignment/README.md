# Feature Definition - [F-03] 线索分配与流转 (Lead Assignment & Transfer)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-01] 线索治理与分配 (Lead Governance & Assignment)](../README.md)

**业务定义**：
本特性用于将已完成分级确认的线索，按分级结果生成分配推荐，经管理者确认后路由至指定销售，并支持对已分配线索的再分配与回池操作，全程留存可检索的审计日志。它关注的是"合格线索如何从线索池流转至责任销售、流转权限边界与操作轨迹如何保障"，不定义分级结果如何产生（F-02）、销售跟进行为（E-03）或培育策略触发（E-02）。

**范围边界**：

1. 包含：基于分级的分配推荐生成（A/B 级进入分配池）、管理者确认分配、销售接收线索通知、管理者触发的再分配与回池操作、分配操作审计日志。
2. 不包含：线索分级确认（F-02）、线索池全局视图与筛选（F-04）、销售跟进过程管理（E-03）、E-02 培育策略触发（由分级结果直接驱动，不经本 Feature）。

## 用户故事 (User Stories)

| ID | 角色 (As a...) | 想要 (I want to...) | 从而 (So that...) | 验收标准索引 (AC Ref) |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | MKT Leader | 查看系统基于分级结果生成的分配推荐，并确认将线索分配给指定销售 | 确保高意向线索及时触达合适销售，分配决策留有人工判断空间 | AC-01, AC-02 |
| US-02 | 销售 | 收到分配通知并确认接收线索 | 明确自身负责线索范围，开始跟进准备 | AC-03 |
| US-03 | MKT Leader / 销售 Leader | 将长期未跟进或原负责销售离职的线索再分配给其他销售 | 避免高价值线索因人员变动而长期搁置 | AC-04, AC-05 |

## 业务规则 (Business Rules)

- **[BR-01] 分配前置条件**：仅分级已经 MKT Leader 确认生效（状态为已确认）的线索方可进入分配推荐流程（参见 Epic [BR-04] 及 F-02）；未确认分级的线索不得被系统生成分配推荐，亦不得接受手工指派。
- **[BR-02] 分级路由规则**：已确认分级为 A 或 B 的线索进入分配池，系统须生成分配推荐（参见 Epic [BR-06]）；已确认分级为 C 或 D 的线索进入市场培育（E-02），不进入分配推荐池；推荐仅作参考，最终分配须由 MKT Leader 或销售 Leader 人工确认执行，系统不得自动分配（参见 Epic [BR-04]）。
- **[BR-03] 唯一归属**：同一线索在任意时刻仅能归属一名销售；新分配执行前须终止原归属关系，并在审计日志中记录原归属终止（参见 Epic [BR-02]）；分配执行前须校验目标销售处于有效在职状态（阈值 TBD：账号状态有效，责任方：系统权限模块；若无法校验则以管理者确认为准，标记 [待确认-GC-06 就绪后验证]）。
- **[BR-04] 再分配权限**：再分配操作须由 MKT Leader 或销售 Leader 触发，普通销售不得主动发起再分配；再分配须经操作人显式确认后执行，不得静默变更归属关系。
- **[BR-05] 审计日志完整性**：所有线索分配与再分配操作须在执行时同步写入审计日志（参见 Epic [BR-07]），日志须包含：操作人标识、操作时间戳、操作类型（初次分配 / 再分配 / 回池）、旧归属销售标识（再分配时）、新归属销售标识；日志须支持按线索 ID、操作人、时间范围检索（参见 Epic [NFR-03]）；日志保留周期不低于 12 个月。

## 业务对象定义 (Business Object Definition)

### 分配推荐

分配推荐是系统基于线索分级与分配规则为可分配线索生成的建议对象，为管理者提供分配决策参考。推荐不产生实际归属关系，须经确认后方产生分配记录。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识本次推荐针对的线索 |
| 推荐销售 | 系统建议分配的目标销售标识 |
| 推荐依据摘要 | 推荐的分级等级与推荐理由概要，供管理者参考决策 |
| 推荐状态 | 标识推荐是否已被确认执行或管理者已覆盖指定（待确认 / 已确认 / 已覆盖） |

### 线索归属记录

线索归属记录是线索分配或再分配执行后产生的正式业务凭证，确立线索与销售之间的责任关系，并作为审计与追溯的基础。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识归属记录对应的线索 |
| 负责销售 | 当前承担该线索跟进责任的销售标识 |
| 归属生效时间 | 本次分配或再分配确认执行的时间戳 |
| 归属终止时间 | 本条归属关系终止的时间戳（再分配执行时自动填写） |
| 归属终止原因 | 标识终止类型（再分配 / 回池 / 销售离职），供审计参考 |

### 分配审计日志

分配审计日志是对线索分配类操作的完整操作轨迹记录，独立于归属记录存在，用于合规审计与责任追溯，不可删除、不可篡改。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识本条日志对应的线索 |
| 操作类型 | 区分初次分配、再分配或回池操作 |
| 操作人 | 执行或确认本次操作的管理者标识 |
| 操作时间 | 操作完成的时间戳 |
| 旧归属销售 | 再分配前的原负责销售标识（初次分配时为空） |
| 新归属销售 | 本次操作指定的新负责销售标识（回池时为空） |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 线索分级已确认为 A 或 B 级 | 系统生成分配推荐 | 系统在分级确认后生成包含推荐销售与推荐依据摘要的分配推荐，推荐处于待确认状态，不自动执行归属关系变更 | US-01 |
| AC-02 | 线索存在待确认的分配推荐 | MKT Leader 确认推荐（或覆盖指定其他销售）并执行分配 | 线索归属记录生效，对应销售收到分配通知，分配操作写入审计日志（参见 BR-05） | US-01 |
| AC-03 | 线索已分配至某销售 | 被分配销售查看并确认接收线索 | 系统记录销售接收确认时间，线索进入该销售的跟进队列 | US-02 |
| AC-04 | 线索已分配至某销售，管理者发起再分配 | MKT Leader 或销售 Leader 指定新负责销售并确认再分配 | 原归属关系终止，新归属关系生效；原负责销售归属终止时间更新；再分配操作写入审计日志，包含新旧负责人信息（参见 BR-05） | US-03 |
| AC-05 | 线索分级为 C 或 D 级 | 系统处理分级确认结果 | 线索不进入分配推荐池，不产生分配推荐记录；线索流向 E-02 培育（参见 BR-02） | US-01 |
| AC-06 | 线索存在多条历史分配操作记录 | MKT Leader 或销售 Leader 查询该线索的分配历史 | 系统展示该线索所有分配与再分配操作的审计日志，支持按操作时间排序检索（参见 BR-05） | US-04 |
