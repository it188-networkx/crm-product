# Feature Definition - [F-05] 评分规则管理 (Scoring Rule Management)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-01] 线索治理与分配 (Lead Governance & Assignment)](../README.md)

**业务定义**：
本特性用于支持 MKT Leader 对线索评分所依赖的维度与权重参数进行创建、编辑、版本管理、启用停用及发布审批，同时承接来自 E-04 成交复盘模块的评分规则优化建议，由 MKT Leader 人工审核后选择性采纳并发布生效。它关注的是"评分规则如何被业务可信地管理与迭代"，不定义具体的评分计算过程（F-02）、分级结果的使用（F-03、E-02）或 E-04 复盘分析本身。

**范围边界**：

1. 包含：评分维度与权重参数的创建与编辑、规则版本记录与历史对比、发布前审批确认、规则启用与停用状态管理、来自 E-04 的回流建议查看与逐条审核。
2. 不包含：评分的实际计算执行（F-02）、分级阈值（A/B/C/D）的调整（阈值由 Epic 业务规则固定，参见 Epic [BR-04]）、E-04 复盘分析的生成过程（E-04）、分配规则的配置（F-03）。

## 用户故事 (User Stories)

| ID | 角色 (As a...) | 想要 (I want to...) | 从而 (So that...) | 验收标准索引 (AC Ref) |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | MKT Leader | 创建或编辑评分规则的维度与权重参数配置 | 确保评分标准反映当前业务对线索质量的判断维度 | AC-01, AC-02 |
| US-02 | MKT Leader | 对编辑后的规则执行发布审批，使新规则对后续线索生效 | 控制规则变更节奏，避免未经确认的参数影响线索评分结果 | AC-03, AC-04 |
| US-03 | MKT Leader | 启用或停用某条评分规则 | 灵活管理多套规则的适用范围，支持规则试运行与下线 | AC-05 |
| US-04 | MKT Leader | 查看当前规则与历史版本的参数对比 | 追溯规则迭代历史，为下一次优化提供参照基准 | AC-06 |

## 业务规则 (Business Rules)

- **[BR-01] 版本控制必执行**：每次评分规则维度或权重参数发生变更，系统须自动产生新版本记录（参见 Epic [BR-03]）；版本记录须包含变更人、变更时间、修改前后的维度与权重参数对比；历史版本记录须可查询，不允许删除任何已存在的版本记录。
- **[BR-02] 发布审批前置**：变更后的评分规则须经 MKT Leader 显式确认发布后方可生效（参见 Epic [BR-03]）；发布操作须产生发布审批记录，包含发布人与发布时间；已发布规则仅对发布时间之后新入池的线索生效，历史已分级线索不受影响。
- **[BR-03] 启停状态约束**：评分规则须支持启用与停用状态切换；停用状态的规则不参与任何线索的评分计算；任意时刻须至少有一条规则处于启用状态，不允许将当前唯一启用中的规则停用；若停用操作导致无启用规则，系统须阻止并提示操作人。
- **[BR-04] 回流建议处理**：来自 E-04 的评分规则优化建议须以待审核状态呈现给 MKT Leader（参见 Epic [BR-05]）；建议内容须标注调整维度与推荐权重变更方向；MKT Leader 须逐条做出采纳或拒绝决定，系统不得自动更新任何未经确认的建议；已采纳建议转化为规则草稿，须经 BR-02 发布审批流程后方可生效。
- **[BR-05] 初始规则基线**：系统上线时须至少有一条评分规则处于启用并已发布状态（参见 Epic [ASM-02]）；若初始规则参数未就绪，F-02 AI 评分功能不可用，系统须明确提示当前不可用原因，降级为人工分级模式。

## 业务对象定义 (Business Object Definition)

### 评分规则

评分规则是线索评分计算的配置参数载体，定义了参与评分的维度集合与各维度权重，是 F-02 评分计算的基础输入。每条规则存在独立的版本序列与启用状态。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 规则名称 | 业务可识别的规则标识名称，用于管理界面区分多条规则 |
| 评分维度列表 | 参与评分计算的维度集合，每个维度包含维度名称与对应权重比例 |
| 启用状态 | 标识规则当前是否参与评分计算（启用 / 停用） |
| 当前版本号 | 标识规则当前所处版本，随每次参数变更递增 |
| 发布状态 | 标识规则是否已经审批发布（草稿 / 已发布） |

### 规则版本记录

规则版本记录是对每次规则参数变更的完整快照，构成规则的可追溯变更历史，不可删除或修改。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属规则 | 标识版本记录归属的评分规则 |
| 版本号 | 本次版本的唯一序列号 |
| 变更前参数快照 | 变更前的维度与权重配置完整记录 |
| 变更后参数快照 | 变更后的维度与权重配置完整记录 |
| 变更人 | 执行本次参数变更的操作人标识 |
| 变更时间 | 本次变更操作完成的时间戳 |
| 发布人 | 确认执行发布操作的 MKT Leader 标识（草稿状态为空） |
| 发布时间 | 本版本生效的时间戳（草稿状态为空） |

### E-04 回流建议

E-04 回流建议是来自成交复盘模块的评分规则优化推荐，以待审核状态进入本特性处理流程，经 MKT Leader 逐条审核后产生采纳或拒绝结论，不自动影响评分规则配置。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 建议来源 | 标识建议来自 E-04 的哪次复盘分析 |
| 调整维度 | 建议调整的评分维度名称 |
| 推荐权重变更方向 | 建议增加或减少的权重幅度或比例 |
| 建议理由摘要 | E-04 输出的调整依据概要，供 MKT Leader 参考判断 |
| 审核状态 | 标识 MKT Leader 的处理结论（待审核 / 已采纳 / 已拒绝） |
| 审核时间 | MKT Leader 完成审核操作的时间戳 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 系统中无同名启用规则 | MKT Leader 创建新评分规则并填写维度与权重参数 | 规则以草稿状态保存，版本记录产生（版本 1），发布前不参与任何线索评分 | US-01 |
| AC-02 | 已存在一条规则，MKT Leader 修改其维度或权重参数 | 提交规则参数变更 | 系统产生新版本记录，包含变更前后参数对比、变更人与变更时间；规则更新为新版本草稿状态（参见 BR-01） | US-01 |
| AC-03 | 规则处于草稿状态，参数已完整填写 | MKT Leader 执行发布确认操作 | 规则状态变更为已发布，发布记录（发布人、发布时间）写入版本记录；规则对发布后新入池线索生效（参见 BR-02） | US-02 |
| AC-04 | 规则处于草稿状态，尚未发布 | 线索入池触发评分计算 | 草稿状态规则不参与评分计算；若无已发布启用规则则 F-02 不可用，系统提示（参见 BR-03、BR-05） | US-02 |
| AC-05 | 系统中存在至少两条启用状态的规则 | MKT Leader 将其中一条规则切换为停用 | 目标规则状态变更为停用，不再参与后续线索评分；剩余启用规则正常运作（参见 BR-03） | US-03 |
| AC-06 | 某条规则已存在多个历史版本记录 | MKT Leader 查看该规则的版本历史 | 系统展示所有版本的参数快照、变更人、变更时间及发布状态，支持版本间参数对比（参见 BR-01） | US-04 |
| AC-07 | E-04 输出了评分规则优化建议，系统收到待审核建议 | MKT Leader 查看建议列表并对某条建议选择采纳 | 该建议状态更新为已采纳，系统生成对应规则草稿（维度与权重按建议内容预填），须经 BR-02 发布审批后方可生效（参见 BR-04） | US-05 |
| AC-08 | E-04 建议处于待审核状态 | MKT Leader 对某条建议选择拒绝 | 建议状态更新为已拒绝，不产生任何规则草稿，评分规则参数不变（参见 BR-04） | US-05 |
