# Feature Definition - [F-02] 线索评分与分级 (Lead Scoring & Grading)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-01] 线索治理与分配 (Lead Governance & Assignment)](../README.md)

**业务定义**：
本特性用于对进入线索池的每条线索按可配置评分规则计算多维评分与总分，生成 A/B/C/D 四级分级建议，并经 MKT Leader 人工确认后驱动后续分配与培育决策。它同时支持线索入池后根据跟进行为与客户反馈进行动态评分更新，确保分级结果持续反映线索当前意向状态。本特性关注的是"如何量化评估线索意向、分级结论如何产生与确认"，不定义评分规则的创建与配置（F-05）、分配路由规则（F-03）或培育策略（E-02）。

**范围边界**：

1. 包含：基于评分规则的自动初始评分计算、AI 分级建议生成与人工确认、分级阈值适用、动态评分更新与分级变动提示、历史评分记录留存。
2. 不包含：评分维度与权重参数的配置管理（F-05）、分级结果触发的分配路由（F-03）、分级结果触发的培育策略（E-02）、评分规则回流审核（F-05）。

## 业务规则 (Business Rules)

- **[BR-01] 初始评分自动触发**：线索完成录入并进入线索池后，系统须自动触发初始评分计算；50 条以内的批次评分须在 10 秒内完成，不得影响其他用户正常操作（参见 Epic [NFR-02]）；评分规则基准未就绪时 AI 评分功能不可用，系统须明确提示并降级为人工分级（参见 Epic [ASM-02]）。
- **[BR-02] 评分维度来源**：评分计算须使用当前生效版本评分规则中定义的维度与权重参数（参见 Epic [BR-03] 及 F-05 评分规则管理）；E-06 标签体系未就绪时降级为仅使用手工标注的行业与规模字段，AI 评分准确度受限须在系统中明确说明（参见 Epic [DEP-02]）。
- **[BR-03] 分级阈值与 AI 建议边界**：AI 分级建议按总分映射为四个等级：A（80–100 分）、B（70–79 分）、C（50–69 分）、D（50 分以下）（参见 Epic [BR-04]）；系统生成的分级建议须经 MKT Leader 人工确认后方可生效；未经确认的建议不得驱动分配路由或培育策略触发（参见 Epic [BR-04]）。
- **[BR-04] 动态评分更新**：线索入池后，系统须根据跟进频率与客户反馈频率持续更新行为意向分与总分（参见 Epic [BR-06a]）；行为意向分可累计但最高不超过 20 分；线索进入商机后行为意向分停止累计，若回退到线索阶段，行为意向分从 0 重新计算。当总分变化导致分级等级变动时，系统须向 MKT Leader 推送确认提示，变动后的新分级须再次经确认方可生效；历史各版本评分快照须留存，支撑趋势查询（参见 Epic [NFR-03]）。
- **[BR-05] 评分可见性**：线索的总分、各维度得分、AI 分级建议及当前已确认分级须对 MKT Leader 可见；历史评分快照须支持按线索 ID 检索，以支撑分析与复核（参见 Epic [NFR-03]）。

## 业务对象定义 (Business Object Definition)

### 线索评分记录

线索评分记录是每次评分计算行为的业务凭证，承载某一时间点线索的量化评估结果，是分级判断与历史趋势分析的基础。每条线索可存在多条历史评分记录，当前生效记录由最后一次人工确认决定。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识评分记录归属的线索 |
| 评分快照时间 | 本次评分计算完成并触发确认流程的时间戳 |
| 各维度得分 | 按评分规则各维度分别计算的分项得分，用于可视化评分结构 |
| 总分 | 各维度加权计算后的总分，作为分级映射的输入 |
| AI 分级建议 | 系统基于总分映射输出的 A/B/C/D 分级建议，须经人工确认方可生效 |
| 使用的规则版本 | 本次评分采用的评分规则版本标识，用于追溯评分依据 |
| 确认状态 | 标识 AI 建议是否已经 MKT Leader 确认（待确认 / 已确认） |
| 触发类型 | 标识本次评分是初始评分还是动态更新评分 |

### 已确认分级

已确认分级是线索当前有效的分级结论，是驱动分配路由与培育策略的业务依据。它由最近一次经 MKT Leader 确认的 AI 分级建议产生，变更须再次经确认。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属线索 | 标识分级结论归属的线索 |
| 分级等级 | 当前有效分级（A / B / C / D） |
| 确认人 | 执行确认操作的 MKT Leader 标识 |
| 确认时间 | 分级结论生效的时间戳 |
| 对应评分记录 | 与本次确认操作关联的评分快照，支撑追溯 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 线索已入池，当前存在一条生效中的评分规则 | 线索完成录入进入线索池 | 系统在 10 秒内完成初始评分计算，生成各维度得分、总分与 AI 分级建议，处于待确认状态；该建议不驱动任何分配或培育动作 | US-01 |
| AC-02 | 线索存在待确认的 AI 分级建议 | MKT Leader 查看分级建议并执行确认 | 分级建议状态更新为已确认，已确认分级结论生效，系统可据此驱动后续分配路由与培育策略（参见 BR-03） | US-02 |
| AC-03 | 线索存在待确认的 AI 分级建议 | MKT Leader 未执行确认操作 | 线索分级保持待确认状态，分配路由与培育策略不触发（参见 Epic [BR-04]） | US-02 |
| AC-04 | 线索已有已确认分级，且后续跟进行为导致行为意向分变化触发分级等级变动 | 系统检测到动态评分更新后等级变化 | 系统向 MKT Leader 推送分级变动确认提示；变动后的新分级须再次经确认方可生效（参见 BR-04） | US-03 |
| AC-05 | 线索存在多条历史评分快照 | MKT Leader 查询该线索的评分历史 | 系统展示该线索各时间点的总分、分级建议与确认状态，支持按时间排序查看趋势（参见 BR-05） | US-04 |
