# Feature Definition - [F-02] 阶段门控规则配置 (Stage Gating Rule Configuration)

## 特性概述 (Feature Overview)

**所属 Epic**：[E-03] 商机推进与阶段门控 (Opportunity Pipeline)

**业务定义**：
本特性用于支持管理员对商机推进各阶段的准入条件进行集中管理，将阶段推进的准入标准转化为可配置、可版本追踪、可审计的业务规则集合。它关注的是"管理员如何定义和维护各阶段的晋级门槛、如何对规则变更进行版本管控、规则如何在商机推进时被消费"，不定义商机如何推进阶段（由 F-01 承接）及商机健康度评估（由 F-03 承接）。

**范围边界**：

1. 包含：阶段准入条件的创建与编辑（必填字段、最低停留时长、决策人接触状态等条件类型）、规则启用/停用、规则版本记录、特殊跳转补录项定义、商机推进时门控条件的实时校验与结果反馈。
2. 不包含：商机阶段流转与结案操作（F-01）、商机健康度评估与预警（F-03）、权限体系定义（承接 GC-06，不在本 Feature 内定义）。
3. 参见 Epic [DEP-02]：门控规则中的"最新跟进记录意向等级"条件依赖 E-02 提供跟进数据；E-02 未就绪时该条件降级为仅校验必填字段，意向等级驱动阶段的能力缺失。
4. 参见 Epic [ASM-01]：假设业务方在系统上线前能提供至少一套初始阶段定义与准入条件参数；初始规则未确认时阶段推进降级为无门控模式。

## 业务规则 (Business Rules)

- **[BR-01]规则不得硬编码**：各阶段准入条件须以可配置方式管理，不得以固定逻辑嵌入系统；管理员须能通过规则配置界面创建、编辑和启停规则，无需修改代码（承接 Epic [BR-02]）。

- **[BR-02]条件类型约束**：准入条件支持以下标准类型：必填字段（指定字段必须有值）、最低停留时长（商机在当前阶段停留的最短时长，单位为天，最小值为 0）、决策人接触状态（接触状态须达到指定等级）；条件类型扩展须经发布审批（承接 GC-07）。

- **[BR-03]关键门控字段强校验**：从"需求确认"阶段起，以下条件为系统内置强制项，不受管理员配置影响：决策人接触状态须为"已接触"以上；当前阶段的下一步行动日期为必填项；上述条件缺失时系统统一返回提示：「阶段推进失败：缺少有效结构化跟进记录」（承接 Epic [BR-07]）。

- **[BR-04]规则变更版本管理**：每次规则发布（新建或编辑后保存）须自动生成版本快照，版本快照包含：版本号（自增整数）、变更内容、发布时间、发布人；版本快照不可删除或修改（承接 GC-07，承接 Epic [BR-02]）。规则发布须经发布审批，审批通过后方可生效，审批记录留存于版本快照中。

- **[BR-05]规则生效范围**：规则变更发布后仅对新发起的阶段推进操作生效；已完成的历史阶段变更按发生时的规则版本判定，不受新版本影响。

- **[BR-06]停用规则的影响**：阶段门控规则停用后，对应阶段的推进操作降级为无门控模式（仅校验 BR-03 的内置强制项）；停用操作须留存操作人、停用时间与停用原因（承接 GC-04）。

- **[BR-07]校验结果反馈**：商机推进时门控校验结果须以统一格式返回：通过时允许推进；阻止时返回所有未满足条件的列表，每条条件说明具体未满足的判断依据（如"决策人接触状态当前为未接触，要求为已接触以上"）；校验结果不持久化，以推进时的实时计算结果为准。

- **[BR-08]特殊跳转补录项定义**：管理员须能为各阶段配置特殊跳转时的补录项；补录项定义包含：项目名称、是否必填、输入类型（布尔型/文本型）；系统内置四项不可删除的跳转补录项：跳转原因说明（文本型，必填）、是否已口头确认预算（布尔型，必填）、是否有历史合作基础（布尔型，必填）、是否为特殊项目（布尔型，必填）（承接 Epic [BR-08]，承接 F-01 [BR-10]）。

- **[BR-09]配置权限约束**：阶段门控规则的创建、编辑、启停与版本发布操作须限定为管理员角色；销售执行层与销售 Leader 仅可读取规则配置结果，无法修改（承接 GC-06）。

## 业务对象定义 (Business Object Definition)

### 阶段门控规则

阶段门控规则是管理员为某一商机推进阶段定义的准入条件集合，是门控校验的业务基准文件。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 目标阶段 | 标识本规则所约束的商机推进阶段 |
| 条件列表 | 本阶段准入所需满足的条件项集合（含条件类型与阈值） |
| 特殊跳转补录项列表 | 销售发起特殊跳转至本阶段时须补录的信息项集合 |
| 启用状态 | 标识本规则当前是否处于有效执行状态 |
| 当前版本号 | 标识本规则当前生效的版本快照编号 |

### 门控规则版本快照

门控规则版本快照是每次规则发布时自动生成的不可变历史记录，用于支撑版本追踪与合规审计。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属规则 | 标识本快照归属的阶段门控规则 |
| 版本号 | 自增整数，唯一标识本次发布版本 |
| 快照内容 | 本版本生效时的完整规则内容（条件列表与补录项定义） |
| 发布人 | 发起本次发布的管理员身份标识 |
| 发布时间 | 本版本正式生效的时间戳 |
| 审批结果 | 记录本次发布审批的结论与审批人 |

### 门控准入条件项

门控准入条件项是构成阶段门控规则的最小业务约束单元，描述单一可判定的准入要求。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 条件类型 | 标识本条件属于必填字段、最低停留时长还是决策人接触状态 |
| 条件参数 | 承载判定所需的阈值或目标值（如停留天数下限、接触状态等级） |
| 是否必填 | 标识本条件是否为强制执行项；非强制项满足时作为推荐提示 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 管理员已登录；目标阶段不存在有效规则 | 管理员为指定阶段创建新的门控规则，配置至少一条准入条件并提交发布审批 | 审批通过后规则生效，版本快照（版本 1）自动创建，规则启用状态为"启用"（参见 BR-04） | US-01 |
| AC-02 | 管理员已登录；准入条件类型为"最低停留时长"，输入天数为负数 | 管理员尝试保存条件配置 | 系统阻止保存并提示参数无效，最低停留时长最小值为 0（参见 BR-02） | US-01 |
| AC-03 | 管理员已登录；目标阶段已存在生效规则 | 管理员修改已有规则的一项准入条件并提交发布审批 | 审批通过后新版本生效，版本快照（版本号自增）自动创建，旧版本快照保留且内容不变（参见 BR-04, BR-05） | US-02 |
| AC-04 | 管理员已登录；目标规则当前处于启用状态 | 管理员填写停用原因并确认停用操作 | 规则状态变更为停用，对应阶段推进降级为无门控模式（保留 BR-03 内置强制项），停用操作留存记录（参见 BR-06） | US-03 |
| AC-05 | 管理员已登录；目标阶段规则已有至少 2 个历史版本 | 管理员查阅该规则的版本列表 | 系统展示所有历史版本列表（含版本号、发布时间、发布人），管理员可查看任意版本的完整快照内容（参见 BR-04） | US-04 |
| AC-06 | 商机处于活跃状态；目标阶段门控规则处于启用状态；所有准入条件均已满足 | 销售尝试推进商机至下一阶段 | 门控校验通过，允许推进（参见 BR-07） | US-06 |
| AC-07 | 商机处于活跃状态；目标阶段门控规则处于启用状态；存在未满足的准入条件 | 销售尝试推进商机至下一阶段 | 门控校验阻止推进，返回所有未满足条件的列表，每条条件注明具体未满足的判断依据（参见 BR-07）；商机阶段保持不变 | US-06 |
| AC-08 | 管理员已登录；目标阶段已存在门控规则 | 管理员为该阶段新增一项特殊跳转补录项（文本型、必填）并发布 | 审批通过后补录项定义生效，四项内置跳转补录项始终存在且不可删除（参见 BR-08） | US-07 |
| AC-09 | 管理员已提交规则发布审批请求（新建或编辑后） | 审批人通过审批 | 规则变更正式生效，版本快照自动创建并关联审批通过记录（含审批人与审批时间）（参见 BR-04） | US-05 |
| AC-10 | 管理员已提交规则发布审批请求（新建或编辑后） | 审批人驳回审批并填写驳回原因 | 规则变更不生效，已有规则内容保持不变；驳回记录（含驳回原因、审批人、时间）留存于版本快照草稿中（参见 BR-04） | US-05 |
