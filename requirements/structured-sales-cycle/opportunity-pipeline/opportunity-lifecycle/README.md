# Feature Definition - [F-01] 商机生命周期管理 (Opportunity Lifecycle Management)

## 特性概述 (Feature Overview)

**所属 Epic**：[E-03] 商机推进与阶段门控 (Opportunity Pipeline)

**业务定义**：
本特性用于管理商机从创建到最终结案（成交或失单）的完整生命周期，将商机推进过程中的阶段变更、里程碑确认与结案动作转化为可追踪、可审计的正式业务记录。它关注的是"商机如何创建、阶段如何流转、结案如何规范、操作历史如何留存"，不定义各阶段的准入门控规则（由 F-02 承接）及健康度预警（由 F-03 承接）。

**范围边界**：

1. 包含：商机创建（从线索转化或手动创建）、阶段推进（含子里程碑字段：预算确认、报价确认、条款确认）、成交结案（含成交金额、签约日期、归因渠道与关键影响因素）、失单结案（含失单原因一级分类）、商机退回、每次阶段变更的操作记录留存。
2. 不包含：阶段准入条件的配置与强制校验（F-02）、健康度评估与风险预警（F-03）、商机详情时间线展示（F-04）、失单复盘流程（E-04）、漏斗数据消费（T-03）。
3. 参见 Epic [DEP-01]：商机从线索转化时的意向等级门槛依赖 E-01 线索分级；E-01 未就绪时商机须手动创建，线索上下文自动关联能力缺失。
4. 参见 Epic [DEP-04]：失单结案后触发 E-04 复盘流程依赖 E-04 就绪；E-04 未就绪时仅归档失单记录，不触发复盘。

## 业务规则 (Business Rules)

- **[BR-01]门控不可绕过**：销售在未满足当前阶段准入条件前无法推进至下一阶段；管理员可在确认风险后强制覆盖，强制覆盖须留存操作人与覆盖原因（承接 GC-04）。阶段准入条件的具体参数由 F-02 管理，本 Feature 仅消费门控结果（通过/阻止）。

- **[BR-02]阶段规则可配置**：各阶段准入条件须通过 F-02 的配置方式管理，不得硬编码；本 Feature 不直接维护规则内容，仅在推进时调用门控校验结果。

- **[BR-03]终态不可回退**：商机结案（成交/失单）后状态为终态，不允许回退至活跃阶段。如确认结案有误，须通过新建商机重新推进；原商机档案完整保留，不得删除或覆盖。

- **[BR-04]跟进记录耦合**：阶段推进建议基于最新跟进记录状态（承接 E-02）；若当前阶段跟进记录缺失，系统须提示补充后方可触发门控校验；最新跟进记录的意向等级须作为门控校验的参考输入之一。

- **[BR-05]漏斗数据输出**：商机各阶段的进入数量、停留时长与结案结果数据须以可消费格式实时输出供 T-03 消费；数据输出须经断链检测，避免商机状态与漏斗数据不一致（承接 Epic [BR-05]）。

- **[BR-06]停滞判定基准**：商机停滞预警的判定由 F-03 承接；本 Feature 的职责是在商机状态变更时同步更新阶段进入时间戳，为 F-03 的停滞计算提供时间基准（承接 Epic [BR-06]）。

- **[BR-07]成交结案必填**：成交结案时，成交金额、签约日期、归因渠道、关键影响因素均为必填项；缺少任一项时系统阻止结案并返回提示：「成交结案失败：缺少必要的成交信息」。归因渠道从预设分类中单选；关键影响因素支持多选（选项由管理员在系统参数配置中维护，初始选项随系统上线时提供）。

- **[BR-08]失单结案必填**：失单结案时，失单原因一级分类为必填项；缺失时系统阻止结案并返回提示：「失单结案失败：缺少失单原因分类」。失单原因一级分类从预设分类中单选，分类选项随阶段规则同批次上线，不在本 Feature 中自定义。

- **[BR-09]商机退回机制**：活跃商机若因客户交互频率持续下降达到停滞阈值且销售确认暂停推进但未达成交/失单，可由销售或管理层发起退回；退回须记录退回原因；商机状态置为"退回待重启"；原商机历史记录完整保留。条件具备时可由管理层或销售重新激活；激活视为同一商机继续推进，不触发失单复盘流程（承接 Epic [BR-09]）。

- **[BR-10]特殊阶段跳转**：销售可发起特殊阶段跳转（如"需求确认"直接跳转至"商务谈判"），跳转时须补录以下信息：跳转原因说明、是否已口头确认预算、是否有历史合作基础、是否为特殊项目；补录完成后方可执行跳转；操作留存完整审计记录（承接 GC-04）；特殊跳转不取消被跳过阶段的子里程碑记录需求（承接 Epic [BR-08]）。

- **[BR-11]阶段变更操作记录**：每次阶段变更（正常推进、特殊跳转、管理员强制覆盖）须自动留存操作记录，记录内容包括：变更前阶段、变更后阶段、操作人、操作时间戳、变更类型（正常/跳转/强制）；操作记录不可删除或修改（承接 GC-04，承接 Epic [NFR-02]）。

- **[BR-12]线索转商机门槛**：从线索触发转商机时，系统须校验线索意向等级是否满足转商机门槛（门槛值依赖 E-01 定义，初始阈值随系统上线时提供）；门槛不满足时系统提示意向等级不足并阻止转化；E-01 未就绪或门槛值未完成配置时，降级为手动创建模式，不校验意向等级（承接 Epic [DEP-01]）。

## 业务对象定义 (Business Object Definition)

### 商机记录

商机记录是从商机创建至最终结案的核心业务实体，承载商机的全生命周期状态与阶段推进上下文。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 商机名称 | 标识本商机的唯一业务标题，通常包含客户名称与需求摘要 |
| 关联客户 | 标识商机归属的客户实体，来源于客户档案 |
| 来源线索 | 标识触发本商机创建的线索记录；手动创建时为空 |
| 当前阶段 | 标识商机当前所处的推进阶段，取值范围由阶段规则配置定义 |
| 商机状态 | 标识商机的整体状态：活跃、退回待重启、成交、失单 |
| 负责销售 | 标识商机的主要责任销售人员 |
| 阶段进入时间 | 标识商机进入当前阶段的时间戳，为停滞计算提供基准 |
| 预算确认状态 | 标识当前阶段内预算确认子里程碑是否已完成 |
| 报价确认状态 | 标识当前阶段内报价确认子里程碑是否已完成 |
| 条款确认状态 | 标识当前阶段内条款确认子里程碑是否已完成 |

### 成交结案记录

成交结案记录是商机以成交终态关闭时的正式业务凭证，承载成交事实与归因信息，供漏斗数据与业绩分析消费。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识该记录归属的商机实体 |
| 成交金额 | 最终确认的合同金额，作为业绩统计的业务基准 |
| 签约日期 | 合同签署完成的日期，作为成交归期的业务基准 |
| 归因渠道 | 标识本次成交的主要来源渠道，从预设分类中单选 |
| 关键影响因素 | 标识促成本次成交的关键因素，支持多选 |

### 失单结案记录

失单结案记录是商机以失单终态关闭时的正式业务凭证，承载失单原因分类，供赢单率分析与复盘流程消费。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识该记录归属的商机实体 |
| 失单原因分类 | 标识失单的一级原因分类，从预设分类中单选 |

### 阶段变更记录

阶段变更记录是每次商机阶段推进操作的不可变审计凭证，确保推进历史完整可追溯。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识该记录归属的商机实体 |
| 变更前阶段 | 本次操作前商机所处阶段 |
| 变更后阶段 | 本次操作后商机所处阶段 |
| 变更类型 | 标识本次变更属于正常推进、特殊跳转还是管理员强制覆盖 |
| 操作人 | 发起本次阶段变更的用户身份标识 |
| 操作时间戳 | 阶段变更发生的精确时间，用于停滞计算和审计检索 |
| 跳转补录信息 | 特殊跳转时补录的原因与背景信息；非跳转操作时为空 |
| 强制覆盖原因 | 管理员强制覆盖时记录的原因说明；非强制覆盖时为空 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 线索意向等级满足转商机门槛；E-01 已就绪 | 在线索/跟进记录中触发转商机操作 | 系统创建商机记录并自动关联线索上下文（客户信息、跟进记录），初始阶段设为"初步接触"，留存阶段变更记录 | US-01 |
| AC-02 | 线索意向等级不满足门槛，或无线索来源 | 触发转商机（门槛不满足时）或手动创建商机 | 门槛不满足时系统阻止转化并提示意向等级不足；手动创建时允许操作，商机来源线索字段为空 | US-01, US-02 |
| AC-03 | 商机处于活跃状态；当前阶段准入条件已满足（由 F-02 校验通过） | 销售推进商机至下一阶段并填写进展摘要 | 系统变更商机阶段，记录新阶段进入时间戳，留存阶段变更记录（类型：正常推进），进展摘要关联至商机记录 | US-03 |
| AC-04 | 商机处于活跃状态；当前阶段准入条件未满足（由 F-02 校验阻止） | 销售尝试推进商机至下一阶段 | 系统阻止推进并返回统一提示，说明未满足的具体条件；商机阶段保持不变（参见 BR-01） | US-03 |
| AC-05 | 商机处于支持子里程碑确认的阶段 | 销售确认预算确认、报价确认或条款确认子里程碑 | 对应子里程碑状态更新为已确认，记录确认时间与操作人，确认结果作为 F-02 门控校验的可用输入 | US-04 |
| AC-06 | 商机处于活跃状态；成交结案所需必填项（成交金额、签约日期、归因渠道、关键影响因素）均已填写 | 销售提交成交结案 | 商机状态变更为成交（终态），留存成交结案记录与阶段变更记录，漏斗数据更新，不可回退（参见 BR-03）；若任一必填项缺失则阻止结案并返回提示（参见 BR-07） | US-05 |
| AC-07 | 商机处于活跃状态；失单原因一级分类已选择 | 销售提交失单结案 | 商机状态变更为失单（终态），留存失单结案记录与阶段变更记录，漏斗数据更新，不可回退（参见 BR-03）；若失单原因分类缺失则阻止结案并返回提示（参见 BR-08）；E-04 就绪时触发复盘流程 | US-06 |
| AC-08 | 商机处于活跃状态；退回原因已填写 | 销售或销售 Leader 发起商机退回 | 商机状态变更为"退回待重启"，留存退回原因与操作记录，历史推进记录完整保留 | US-07 |
| AC-09 | 商机处于"退回待重启"状态 | 管理层或销售重新激活商机 | 商机状态恢复为活跃，激活操作留存记录，不触发失单复盘流程，历史推进记录完整延续（参见 BR-09） | US-07 |
