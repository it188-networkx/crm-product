# Feature Definition - [F-03] 商机健康度视图 (Opportunity Health View)

## 特性概述 (Feature Overview)

**所属 Epic**：[E-03] 商机推进与阶段门控 (Opportunity Pipeline)

**业务定义**：
本特性用于对在途商机进行健康状态评估，将分散在商机记录、阶段变更历史和跟进记录中的推进状态数据转化为可视化的风险等级分组，使销售 Leader 与管理层能基于客观数据自助识别停滞和高风险商机，替代依赖人工汇报的介入模式。它关注的是"在途商机按什么维度分级、哪些商机需要预警、管理层如何以此发起介入决策"，不定义单个商机的阶段推进操作（F-01）、门控规则的内容配置（F-02）或商机时间线的事件展示（F-04）。

**范围边界**：

1. 包含：在途商机按健康/预警/停滞三级风险等级的聚合展示、基于可配置阶段预期时长的停滞判定、按阶段/负责销售/客户类型的筛选与过滤、预警商机的停滞时长与上次推进记录摘要展示、管理介入决策的记录（阶段策略备注与下步行动）。
2. 不包含：商机阶段推进操作（F-01）、门控规则配置（F-02）、单个商机的完整时间线（F-04）、漏斗转化率统计（T-03）。
3. 参见 Epic [DEP-02]：停滞判定中"近期跟进记录缺失"的判断依赖 E-02 跟进记录数据；E-02 未就绪时停滞判定仅依赖阶段停留时长，缺失跟进维度的健康度评估会降低预警准确性。
4. 参见 Epic [ASM-01]：阶段预期时长的初始参数值须在系统上线前由管理员配置完成；初始值未配置时停滞判定无法生效，健康度视图中所有商机均显示为"健康"状态。

## 业务规则 (Business Rules)

- **[BR-01]健康度视图范围**：健康度视图仅展示当前处于活跃状态的在途商机（商机状态为"活跃"）；已结案（成交/失单）、退回待重启的商机不纳入视图范围。

- **[BR-02]风险等级判定**：在途商机的风险等级按以下规则判定：
  - **健康**：当前阶段停留时长未超出该阶段预期停留时长阈值，且近 N 天内存在有效跟进记录（N 的初始值为 7 天，由管理员可配置；E-02 未就绪时忽略跟进记录维度）。
  - **预警**：当前阶段停留时长超出该阶段预期停留时长阈值的 50%，但未超出 100%；或近 N 天内无有效跟进记录（E-02 就绪时参与判定）。
  - **停滞**：当前阶段停留时长超出该阶段预期停留时长阈值的 100%。
  - 各判定维度同时满足时，取最高风险等级（停滞 > 预警 > 健康）。
  - 阶段预期停留时长阈值由管理员在本 Feature 中配置，初始值随阶段规则同批次上线（承接 Epic [BR-06]，承接 Epic [ASM-01]）。

- **[BR-03]停滞计算基准**：停滞时长的计算起点为商机进入当前阶段的阶段进入时间戳（由 F-01 维护）；当前时间与阶段进入时间之差即为当前阶段停留时长，单位为天（不足一天按一天计）。

- **[BR-04]视图数据时效**：健康度视图中的商机状态与风险等级数据须基于实时计算结果；视图加载时须重新计算各商机的当前阶段停留时长与风险等级，不依赖缓存的历史评估结果（承接 Epic [NFR-01]）。

- **[BR-05]视图响应性能**：在途商机数量不超过 500 条时，健康度视图的加载时间须不超过 3 秒；超过 500 条时性能目标为尽力而为，须明确提示数据量超限（承接 Epic [NFR-01]）。

- **[BR-06]筛选维度**：视图支持以下三个独立筛选维度，可组合使用：阶段（取当前阶段值，单选或多选）、负责销售（单选或多选，列表来源于系统用户目录）、客户类型（取商机关联客户的类型属性，单选或多选）；筛选条件须在同一视图内同时生效。

- **[BR-07]管理介入记录**：管理介入决策（阶段策略备注与下步行动）须关联至对应商机记录；每次介入须记录操作人与操作时间；介入记录不可删除，多次介入按时间顺序追加（承接 GC-04，承接 Epic [NFR-02]）。

- **[BR-08]上次推进记录摘要**：预警/停滞商机须展示上次阶段推进操作的摘要，包含：推进时间、推进后阶段、操作人；若无历史推进记录（商机从未推进过阶段），则显示商机创建时间作为基准并标注"尚未推进"。

- **[BR-09]阶段预期时长配置权限**：阶段预期停留时长阈值的配置须限定为管理员角色；销售执行层与销售 Leader 仅可读取，无法修改（承接 GC-06）。

## 业务对象定义 (Business Object Definition)

### 商机健康度快照

商机健康度快照是健康度视图中每条在途商机的实时评估结果，用于支撑风险分组展示与管理介入决策，是视图加载时动态计算产生的只读对象。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识本快照对应的商机记录 |
| 当前风险等级 | 标识本商机当前评估的风险等级（健康/预警/停滞） |
| 当前阶段停留时长 | 商机在当前阶段已停留的时长（天），由阶段进入时间戳实时计算 |
| 阶段预期时长阈值 | 当前阶段对应的管理员配置预期停留时长，作为停滞判定基准 |
| 上次推进摘要 | 最近一次阶段变更的时间、推进后阶段与操作人，或"尚未推进"标注 |

### 阶段预期时长配置

阶段预期时长配置是管理员为各商机推进阶段设定的停滞判定参数，是健康度评估的业务基准文件。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 目标阶段 | 标识本配置约束的商机推进阶段 |
| 预期停留时长 | 该阶段商机的预期停留天数上限，超出后触发预警或停滞判定 |
| 跟进记录有效窗口 | 近 N 天内须存在有效跟进记录的时间窗口，单位为天 |
| 最后更新人 | 最近一次修改本配置的管理员身份标识 |
| 最后更新时间 | 最近一次配置变更的时间戳 |

### 管理介入记录

管理介入记录是销售 Leader 或管理层针对特定商机发起介入决策时的正式业务凭证，用于追踪管理层的主动干预行为。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识本记录归属的商机实体 |
| 阶段策略备注 | 管理层对当前商机推进策略的判断与调整建议 |
| 下步行动 | 管理层指定的下一步具体行动要求 |
| 操作人 | 发起本次介入的管理层用户身份标识 |
| 操作时间 | 本次介入记录创建的时间戳 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 系统中存在至少 1 条在途（活跃）商机；阶段预期时长已配置 | 销售 Leader 打开商机健康度视图 | 视图按健康/预警/停滞三个风险等级分组展示所有在途商机，每组显示商机数量；视图在 500 条以内数据时 3 秒内加载完成（参见 BR-01, BR-04, BR-05） | US-01 |
| AC-02 | 健康度视图已加载；存在可筛选的阶段、销售与客户类型数据 | 销售 Leader 组合使用阶段、负责销售、客户类型三个筛选条件 | 视图实时更新仅展示同时满足所有已选筛选条件的商机，风险等级分组随筛选结果联动更新（参见 BR-06） | US-02 |
| AC-03 | 健康度视图中存在预警或停滞商机 | 销售 Leader 查看某条预警/停滞商机的详情 | 系统展示该商机的当前阶段停留时长、阶段预期时长阈值、上次推进摘要（含推进时间、推进后阶段、操作人）；若无推进历史则显示"尚未推进"（参见 BR-03, BR-08） | US-03 |
| AC-04 | 销售 Leader 已查看某条预警/停滞商机的详情 | 销售 Leader 填写阶段策略备注与下步行动并确认提交 | 管理介入记录创建并关联至该商机（含操作人与操作时间），记录不可删除；该商机详情中可查看介入记录历史（参见 BR-07） | US-04 |
| AC-05 | 管理员已登录；目标阶段存在预期时长配置 | 管理员修改目标阶段的预期停留时长阈值并保存 | 配置更新成功，健康度视图在下次加载时使用新阈值计算各商机风险等级；变更记录（操作人、变更前后值、时间）留存（参见 BR-09） | US-05 |
| AC-06 | 管理员已登录；输入预期停留时长为 0 天 | 管理员尝试将某阶段预期停留时长配置为 0 天并保存 | 系统允许保存（0 天表示无最短停留要求），视图中该阶段商机的停滞判定仅依赖跟进记录维度（E-02 就绪时）或显示为健康（E-02 未就绪时） | US-05 |
