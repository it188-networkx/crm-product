# Epic - [E-03] 商机推进与阶段门控 (Opportunity Pipeline)

## 概述 (Overview)

**业务背景：**

当前商机推进标准不统一，销售与管理层对同一商机所处阶段存在认知偏差；没有统一的阶段门槛定义，销售可以随意跳跃推进阶段而不留止损依据；管理层只能通过人工汇报了解商机进展，无法及时识别停滞或高风险商机，导致关键介入时机滞后，失单后也缺乏复盘数据。

**核心目标**

本 Epic 旨在构建统一的商机生命周期管理能力，以可配置的阶段规则与准入门槛控制商机推进节奏，并对在途商机提供健康度可视化，让管理层能基于客观数据识别停滞与高风险商机，及时决策介入。

**价值预估：**

- **推进一致性**：统一阶段定义与门控规则后，销售、管理层对商机进展形成共识认知，消除汇报信息失真（支撑 T-01 OC-03）。
- **决策效率**：管理层通过健康度视图自助识别问题商机，替代依赖人工汇报的介入模式，为 T-03 提供商机漏斗底层数据（支撑 T-03 价值主张）。
- **覆盖规模**：结构化商机推进消除人工记录盲区，为月均有效跟进客户数从 ≤ 30 家提升至 ≥ 100 家提供结构化支撑（支撑 T-01 KPI M6）。

## 业务场景 (Business Scenarios)

### 场景一：线索转商机与阶段推进

**用户**：销售执行层
**情境**：高意向线索经 E-02 多轮跟进后达到转商机条件，销售创建商机并逐步推进至报价阶段。
**业务流程**：

1. 销售在线索/跟进记录中触发转商机操作，系统校验意向等级是否满足转商机门槛。
2. 满足门槛时系统创建商机记录，以当前线索上下文（客户信息、跟进记录）作为商机初始状态。
3. 销售确认商机阶段（初步接触），并记录当前阶段进展摘要。
4. 销售推进至下一阶段（需求确认）时，系统校验当前阶段准入条件是否满足。
5. 条件满足后允许阶段推进，系统记录阶段变更时间戳与操作人。

### 场景二：商机健康度监控与介入

**用户**：销售 Leader / 管理层
**情境**：每周例会前，销售 Leader 需快速识别当前在途商机中停滞超期或高风险的商机，主动介入。
**业务流程**：

1. 销售 Leader 查看商机健康度视图，系统按风险等级（健康/预警/停滞）对在途商机分组。
2. 系统对超出当前阶段预期停留时长的商机自动标记预警，显示停滞时长与上次推进记录。
3. 销售 Leader 选定预警商机，查看推进历史与当前阻塞原因。
4. 销售 Leader 决策介入，更新阶段策略备注并指定下步行动。
5. 系统保留介入记录，关联至商机时间线。

### 场景三：商机结案（成交/失单）

**用户**：销售执行层
**情境**：商机谈判完成，销售需结案并触发后续复盘流程。
**业务流程**：

1. 销售将商机阶段推进至终态（成交 / 失单），系统要求填写结案摘要（结案原因、关键影响因素）。
2. 成交结案时，系统记录成交金额与时间，更新漏斗数据供 T-03 消费。
3. 失单结案时，系统记录失单原因分类，触发 E-04 成交复盘流程（若启用）。
4. 结案后商机状态不可回退，记录归档至客户档案供历史查询。

## 业务规则 (Business Rules)

### 门控与推进

- **[BR-01]门控不可绕过**：销售在满足当前阶段准入条件前无法推进至下一阶段；管理员可在确认风险后强制覆盖，强制覆盖须留存操作人与覆盖原因（承接 GC-04）。
- **[BR-02]阶段规则可配置**：各阶段准入条件参数须通过配置方式管理，不得硬编码；变更须经发布审批并留存版本记录（承接 GC-07）。
- **[BR-03]终态不可回退**：商机结案（成交/失单）后状态为终态，不允许回退至活跃阶段；如确认结案有误，须通过新建商机重新推进，原商机档案保留。

### 数据联动

- **[BR-04]跟进记录耦合**：E-02 的意向等级更新须作为 E-03 阶段判断的参考依据之一，阶段推进建议基于最新跟进记录状态；若跟进记录缺失，系统提示补充后方可推进。
- **[BR-05]漏斗数据输出**：商机各阶段的进入数量、停留时长与结案结果数据须以可消费格式实时输出，供 T-03 决策驾驶舱漏斗视图消费；数据输出须经断链检测，避免商机状态与漏斗数据不一致。

### 健康度预警

- **[BR-06]停滞判定基准**：商机停滞预警须基于各阶段可配置的预期停留时长（承接 GC-07）触发，超出阈值后系统自动标记预警状态；阈值由管理员配置，初始值随阶段规则一并上线。

## 非功能需求 (Non-Functional Requirements)

- **[NFR-01]健康度视图响应**：在途商机 ≤ 500 条时，健康度视图加载时间不超过 3 秒，支持多人同时查看不产生数据竞争问题（承接 GC-06 权限分级）。
- **[NFR-02]操作可审计**：阶段推进、门控强制覆盖、管理介入、结案等关键操作须留存完整审计日志，按商机 ID、操作人、时间检索（承接 GC-04）。
- **[NFR-03]漏斗数据一致性**：商机状态变更后漏斗数据更新时延不超过 1 分钟，确保 T-03 决策视图数据时效性。

## 约束与依赖 (Constraints & Dependencies)

- **[DEP-01]线索分级依赖**：商机创建的转化来源依赖 E-01 线索分级结果；E-01 未就绪时，商机须手动创建，失去线索上下文自动关联能力，推进记录连续性下降。
- **[DEP-02]跟进记录依赖**：阶段门控判断中的"最新跟进记录"条件依赖 E-02 提供；E-02 未就绪时，门控检查降级为仅校验必填字段，意向等级驱动阶段的能力缺失。
- **[DEP-03]T-03 数据消费**：T-03 决策驾驶舱的漏斗视图依赖本 Epic 输出的商机阶段数据；本 Epic 数据格式变更须与 T-03 协同确认兼容性。
- **[DEP-04]复盘流程依赖**：失单结案后触发 E-04 复盘流程依赖 E-04 就绪；E-04 未就绪时，失单结案记录仅归档，不触发复盘。
- **[ASM-01]阶段规则基准可用**：假设业务方在系统上线前能提供至少一套初始阶段定义与准入条件参数；阶段规则未确认时阶段推进降级为无门控模式，健康度预警无法生效。
