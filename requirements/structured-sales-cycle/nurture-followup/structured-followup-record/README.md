# Feature Definition - [T-01/E-02/F-02] 结构化跟进记录 (Structured Follow-up Record)

> 需求特性定义（A0304）｜所属主题：T-01 结构化销售过程
> 所属 Epic：E-02 培育与跟进沉淀 ｜ **Feature 编号**：F-02 ｜ **优先级**：P0 ｜ **状态**：规划中

---

## 特性概述 (Feature Overview)

**所属 Epic**：[E-02 培育与跟进沉淀 (Nurture & Follow-up)](../README.md)

**业务定义**：

结构化跟进记录用于将每次销售触达客户的过程固化为统一格式的不可删除档案，解决当前跟进记录分散于微信、口头备忘和个人笔记中、缺乏统一结构化归档的核心问题。本特性关注"单次跟进记录的创建、追加修正与意向等级驱动"，确保每次触达可追溯、可分析，并将意向变化实时反馈至商机阶段判断。它不定义跟进节奏的建立与管理（归属 F-01），也不定义跟进记录在时间线上的聚合展示（归属 F-03）。

**范围边界**：

1. 包含：标准化跟进记录的创建（触达方式、沟通要点、意向等级、内容互动结果、下次跟进计划）；可选竞品信息字段的填写；已提交记录的追加修正说明；意向等级变化向 E-03 商机阶段的实时同步；E-05 自动化触达动作的跟进记录写入。
2. 不包含：跟进节奏的建立与调整（F-01）；跟进记录的时间线聚合视图（F-03）；竞品监控与预警看板（归属 T-02 E-03）；完整的商机阶段流转逻辑（归属 E-03）。
3. 外部依赖：意向等级更新须实时驱动 E-03 商机阶段判断（参见 Epic [DEP-02]），E-03 未就绪时意向等级仅作为记录内部字段；E-05 自动化规则执行后须在本特性写入跟进记录（参见 Epic [DEP-03]），E-05 未就绪时所有记录均为人工创建；销售须在每次触达后 24 小时内完成录入（参见 Epic [ASM-01]）。

## 业务规则 (Business Rules)

- **[BR-01] 记录不可删除**：参见 Epic [BR-03]。已提交的跟进记录为不可删除的追加式档案；仅允许在原记录下追加修正说明，修正说明须注明修正人与修正时间，原记录内容必须完整保留。
- **[BR-02] 意向驱动阶段**：参见 Epic [BR-04]。跟进记录中的意向等级一经提交，必须实时同步至 E-03 商机阶段判断逻辑；E-03 未就绪时意向等级仅作为跟进记录内部字段保留，待 E-03 就绪后补充同步逻辑。
- **[BR-03] 竞品字段可选**：参见 Epic [BR-05]。竞品信息字段（竞品名称、风险等级、预警状态）为可选非必填；填写后数据归属 E-02 跟进档案，竞品监控与预警分析能力不在本特性范围内。
- **[BR-04] 自动化记录写入**：参见 Epic [BR-06]。E-05 自动化规则触发的触达动作必须在本特性生成对应跟进记录，动作类型标记为"自动触达"，并关联触发规则 ID；内容推送类动作产生的互动结果（如内容是否被打开）须在结果可获取后回写至对应记录。
- **[BR-05] 标准字段完整性**：触达方式和意向等级为必填字段；沟通要点、内容互动结果、下次跟进计划为建议填写项；缺失必填字段时不得提交记录，须向销售明确提示待填字段。
- **[BR-06] 记录提交时效**：参见 Epic [NFR-01]。从销售提交跟进记录到系统完成归档并更新意向状态，业务感知延迟不超过 2 秒。

## 业务对象定义 (Business Object Definition)

### 跟进记录

跟进记录是销售或自动化规则每次触达客户后生成的正式业务档案，承载触达事实与意向状态变化，作为客户跟进历史的不可变业务基准。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属客户 | 标识该记录归属于哪位客户 |
| 记录人 | 标识由哪位销售人员创建（或标注为系统自动创建） |
| 创建类型 | 标识本次记录来源是"人工跟进"还是"自动触达" |
| 关联规则 ID | 仅自动触达记录需填写，关联触发本次记录的 E-05 规则 ID |
| 触达方式 | 本次触达客户的渠道或方式（必填） |
| 意向等级 | 本次跟进后销售对客户意向的判断结果（必填） |
| 沟通要点 | 本次触达的核心沟通内容摘要 |
| 内容互动结果 | 自动触达时记录客户对推送内容的互动情况（如未读/已读/回复） |
| 下次跟进计划 | 销售记录的下次触达时间与意图 |
| 竞品信息 | 可选的竞品名称、风险等级与预警状态（参见 BR-03） |
| 提交时间 | 记录被正式提交归档的业务时间戳 |

### 跟进记录修正说明

跟进记录修正说明是对已提交跟进记录的追加补充，保障原始档案完整性的前提下修正信息误差，作为独立的审计凭证附挂于原记录之下。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属跟进记录 | 标识该修正说明归属的原始跟进记录 |
| 修正人 | 提交本条修正说明的销售人员 |
| 修正说明内容 | 对原记录中错误或缺漏信息的补充说明 |
| 修正时间 | 本次修正说明提交的业务时间戳 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 客户存在激活状态的跟进节奏计划或销售主动发起记录 | 销售填写触达方式与意向等级后提交跟进记录 | 记录以追加式档案形式归档，含触达方式、意向等级与提交时间；归档延迟不超过 2 秒（参见 BR-06） | US-01 |
| AC-02 | 销售在提交时未填写触达方式或意向等级任一必填字段 | 销售点击提交 | 记录不得提交，系统向销售明确提示待填必填字段（参见 BR-05） | US-01 |
| AC-03 | 销售已提交含意向等级的跟进记录；E-03 已就绪 | 跟进记录归档完成 | 意向等级实时同步至 E-03 商机阶段判断逻辑；若等级达到转商机建议阈值，系统提示销售确认转商机操作（参见 BR-02） | US-02 |
| AC-04 | 销售在跟进记录中填写竞品名称、风险等级或预警状态 | 销售提交含竞品信息的跟进记录 | 竞品字段内容随记录一并归档，不影响记录提交流程；竞品数据不触发本特性内的分析或告警逻辑（参见 BR-03） | US-03 |
| AC-05 | 已存在一条已提交的跟进记录 | 销售填写修正说明内容后提交修正 | 修正说明以追加形式附挂于原记录之下，含修正人与修正时间；原记录内容保持完整不变（参见 BR-01） | US-04 |
| AC-06 | 销售尝试删除已提交的跟进记录 | 销售发起删除操作 | 操作被拒绝，不得删除已提交记录（参见 BR-01） | US-04 |
| AC-07 | E-05 自动化规则触发一次触达动作；E-05 已就绪 | E-05 触发触达动作完成 | 本特性自动生成一条创建类型为"自动触达"的跟进记录，关联触发规则 ID；记录归档延迟不超过 2 秒（参见 BR-04、BR-06） | US-05 |
| AC-08 | 已存在一条"自动触达"类型的跟进记录；内容互动结果可获取 | 互动结果数据回传 | 对应记录的内容互动结果字段更新为实际互动情况，更新时间可追溯（参见 BR-04） | US-05 |
