# Feature Definition - [T-01/E-02/F-01] 跟进节奏计划 (Follow-up Rhythm Plan)

> 需求特性定义（A0304）｜所属主题：T-01 结构化销售过程
> 所属 Epic：E-02 培育与跟进沉淀 ｜ **Feature 编号**：F-01 ｜ **优先级**：P0 ｜ **状态**：规划中

---

## 特性概述 (Feature Overview)

**所属 Epic**：[E-02 培育与跟进沉淀 (Nurture & Follow-up)](../README.md)

**业务定义**：

跟进节奏计划用于为已入池的客户建立结构化、可持续的跟进提醒体系。它将"按节奏主动触达客户"这一销售基本行为转化为可配置、可移交、可审计的正式业务记录，解决当前人工凭记忆维系跟进节奏导致高频遗忘与商机流失的核心问题。本特性关注"节奏的建立、调整和移交"，不定义节奏执行后的记录内容（归属 F-02）与提醒的主动推送机制（归属 F-04）。

**范围边界**：

1. 包含：按客户类型（终端/二次激活/外单转售）差异化生成节奏计划；节奏间隔的手动调整；节奏变更的审计记录留存；节奏计划的责任人移交。
2. 不包含：每次跟进的记录内容（F-02）；到期提醒的主动推送与逾期标记（F-04）；节奏的全自动 AI 优化（参见 Epic [DEP-05]，F-04 P1 范围）。
3. 外部依赖：节奏计划的自动启动依赖 E-01 线索分级分配结果（参见 Epic [DEP-01]）；E-01 未就绪时须由销售手动创建计划；节奏计划的自动移交依赖 E-01 责任人变更操作（参见 Epic [DEP-04]）；客户类型标注须在入池时完成（参见 Epic [ASM-02]），缺失类型时按终端客户默认节奏处理。

## 业务规则 (Business Rules)

- **[BR-01] 类型差异化节奏**：参见 Epic [BR-01]。终端客户、二次激活客户、外单转售客户的默认跟进节奏间隔必须按各自决策周期差异化设定；节奏间隔参数须通过配置方式维护，不得硬编码为固定值。
- **[BR-02] 节奏不可静默跳过**：参见 Epic [BR-02]。跟进节奏到期后，销售必须通过提交一次跟进记录或主动执行延期操作来响应；系统不得自动将逾期节奏标记为已完成或静默推进至下一周期。
- **[BR-03] 节奏变更审计留存**：每次节奏间隔调整须完整留存审计记录，包含变更人身份、变更前间隔值、变更后间隔值和变更时间；审计记录不可删除（参见 Epic NFR-03）。
- **[BR-04] 节奏计划唯一性**：同一客户在同一责任人下仅允许存在一条激活状态的节奏计划；创建前须检查是否存在激活计划，已存在时不得重复创建，须转入更新或移交流程。
- **[BR-05] 类型缺省处理**：参见 Epic [ASM-02]。当客户类型标注缺失时，系统须按终端客户默认节奏生成计划，并向当前责任人发出类型补充提醒；补充类型后须提示责任人确认是否调整节奏间隔。

## 业务对象定义 (Business Object Definition)

### 跟进节奏计划

跟进节奏计划是为特定客户建立的持续性跟进提醒约定，承载客户类型、节奏间隔配置与当前执行状态，作为驱动销售按周期主动触达客户的业务基准。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属客户 | 标识该计划归属于哪位客户 |
| 当前责任人 | 标识当前承担节奏执行义务的销售人员 |
| 客户类型 | 驱动节奏间隔差异化配置的分类依据（终端/二次激活/外单转售） |
| 节奏间隔 | 当前有效的两次跟进之间的建议时间间隔 |
| 计划状态 | 标识节奏计划当前是否处于激活、暂停或已完成状态 |
| 下次跟进建议时间 | 基于上次跟进完成时间与当前节奏间隔推算的下一次建议触达时间 |
| 创建来源 | 标识计划由 E-01 自动触发创建还是由销售手动创建 |

### 节奏变更记录

节奏变更记录是节奏计划每次间隔调整的不可变审计凭证，用于支撑管理审计与决策追溯。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属节奏计划 | 标识该记录归属于哪条节奏计划 |
| 变更人 | 执行本次间隔调整的销售人员或系统动作 |
| 变更前间隔 | 调整前的节奏间隔值 |
| 变更后间隔 | 调整后的节奏间隔值 |
| 变更时间 | 本次调整发生的业务时间戳 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 客户已入池且客户类型已标注；E-01 分级分配结果已就绪 | E-01 触发节奏计划自动创建 | 按客户类型生成对应默认间隔的激活状态节奏计划，创建来源标记为"自动"，责任人为分配结果中的销售 | US-01 |
| AC-02 | 客户已入池但 E-01 未就绪；销售手动发起计划创建 | 销售选择客户类型并确认节奏间隔 | 生成激活状态节奏计划，创建来源标记为"手动"，计划内容与手动输入一致 | US-01 |
| AC-03 | 客户节奏计划处于激活状态 | 销售修改节奏间隔并确认 | 节奏计划间隔更新为新值；自动生成一条节奏变更记录，包含变更人、变更前后间隔值与变更时间（参见 BR-03） | US-02, US-04 |
| AC-04 | 节奏计划到期；销售未提交跟进记录且未主动延期 | 系统检测到节奏逾期状态 | 节奏计划保持逾期状态，不自动推进至下一周期（参见 BR-02）；F-04 负责提醒推送（不在本特性范围内） | US-03 |
| AC-05 | 某客户节奏计划已发生过至少一次间隔调整 | 销售 Leader 查看该计划的变更历史 | 按变更时间倒序展示全部节奏变更记录，每条含变更人、变更前后间隔值与变更时间 | US-04 |
| AC-06 | E-01 完成责任人变更；原责任人存在激活状态节奏计划 | 责任人变更操作完成 | 节奏计划的当前责任人自动更新为新责任人，原节奏间隔配置保持不变；生成责任人变更审计记录（参见 Epic NFR-03） | US-05 |
| AC-07 | 客户类型标注缺失；触发节奏计划创建 | 系统按终端客户默认节奏生成计划 | 计划以终端客户默认间隔创建并激活；同步向当前责任人发出类型补充提醒（参见 BR-05） | US-01 |
