# Feature Definition - [T-01/E-02/F-03] 客户跟进时间线 (Customer Follow-up Timeline)

> 需求特性定义（A0304）｜所属主题：T-01 结构化销售过程
> 所属 Epic：E-02 培育与跟进沉淀 ｜ **Feature 编号**：F-03 ｜ **优先级**：P0 ｜ **状态**：规划中

---

## 特性概述 (Feature Overview)

**所属 Epic**：[E-02 培育与跟进沉淀 (Nurture & Follow-up)](../README.md)

**业务定义**：

客户跟进时间线将单客户在全生命周期内积累的所有跟进事件按时间顺序聚合呈现，形成可视化的互动轨迹视图。它解决人员流动时客户关系上下文随之丢失、新成员无法基于历史记录连续推进的核心问题，同时为销售 Leader 提供客户意向变化的全程可观测性。本特性关注"时间线的聚合展示与筛选"，不定义单条跟进记录的创建与修正（归属 F-02），也不定义节奏计划的管理（归属 F-01）。

**范围边界**：

1. 包含：人工跟进记录与自动触达记录的统一时间线展示；按时间范围筛选跟进事件；意向等级变化轨迹的可视化呈现；责任人变更后时间线的跨责任人连续展示。
2. 不包含：跟进记录的创建与修正（F-02）；节奏计划的管理与变更（F-01）；跨客户的汇总分析视图（归属 T-03 决策驾驶舱）；商机阶段流转记录（归属 E-03）。
3. 外部依赖：时间线内容来源于 F-02 生成的跟进记录（本特性依赖 F-02 已就绪）；责任人变更后时间线的连续显示依赖 E-01 责任人变更完成（参见 Epic [DEP-04]）；E-05 自动触达记录在时间线中统一展示（参见 Epic [BR-06]），E-05 未就绪时仅展示人工跟进记录；销售须持续录入跟进记录，否则时间线连续性将显著下降（参见 Epic [ASM-01]）。

## 业务规则 (Business Rules)

- **[BR-01] 人工与自动记录统一展示**：参见 Epic [BR-06]。人工跟进记录与 E-05 自动触达记录必须在同一时间线中按时间顺序统一展示，不得因来源不同而分离呈现；自动触达记录须在展示中标注其来源类型。
- **[BR-02] 时间线不可过滤删除**：时间线须全量呈现 F-02 归档的所有跟进记录，不得因来源类型、记录人或意向等级而隐藏任何已归档记录；筛选操作仅作用于当前视图，不影响底层记录完整性。
- **[BR-03] 跨责任人连续展示**：责任人变更后，时间线须连续展示原责任人与新负责销售的全部历史跟进记录，不得因责任人变更而截断历史视图（参见 Epic [DEP-04]）。
- **[BR-04] 承载量下限**：参见 Epic [NFR-02]。单客户跟进时间线须支持展示不少于 200 条历史记录，且加载完成时间不超过 3 秒，满足老客户多年份跟进沉淀的展示场景。

## 业务对象定义 (Business Object Definition)

### 跟进时间线

跟进时间线是单客户所有跟进事件的聚合视图，以时间顺序将 F-02 归档的全部跟进记录与意向变化呈现为可检索的历史轨迹，是客户关系上下文的唯一可视化入口。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属客户 | 标识该时间线聚合呈现的是哪位客户的跟进历史 |
| 事件总数 | 当前客户时间线上已归档的跟进事件总量 |
| 时间范围筛选条件 | 当前视图所应用的起止时间范围过滤参数（默认全量） |

### 时间线事件

时间线事件是时间线上的单个跟进记录展示单元，来源于 F-02 归档的跟进记录，承载该次跟进的关键摘要信息供快速浏览。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联跟进记录 | 标识该展示单元对应的 F-02 原始跟进记录 |
| 事件时间 | 该次跟进记录的提交时间，作为时间线排序依据 |
| 来源类型 | 标识该事件是"人工跟进"还是"自动触达" |
| 记录人 | 标识由哪位销售人员创建（或标注为系统自动创建） |
| 意向等级 | 该次跟进记录的意向等级结论，用于展示意向变化轨迹 |
| 触达方式摘要 | 该次跟进的触达渠道摘要，供时间线快速识别 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 某客户已存在至少一条归档的跟进记录（含人工跟进或自动触达） | 销售执行层或销售 Leader 打开该客户的跟进时间线 | 全部已归档跟进事件按提交时间倒序排列展示；每条事件含来源类型、记录人、意向等级与触达方式摘要（参见 BR-01） | US-01, US-02 |
| AC-02 | 某客户的时间线同时包含人工跟进记录与 E-05 自动触达记录 | 销售执行层打开该客户的跟进时间线 | 两类记录在同一时间线中混合排列，自动触达记录在展示中标注"自动触达"来源（参见 BR-01） | US-01 |
| AC-03 | 某客户在历史跟进中有意向等级变化 | 销售 Leader 查看该客户的跟进时间线 | 时间线中各事件的意向等级按时间顺序可见，意向等级变化轨迹可从时间线中完整追溯 | US-02 |
| AC-04 | 某客户跟进时间线已加载 | 销售或销售 Leader 设置时间范围筛选条件（如过去 3 个月） | 时间线仅展示所选时间范围内的跟进事件；筛选不影响底层记录完整性，清除筛选后全量记录恢复展示（参见 BR-02） | US-03 |
| AC-05 | E-01 已完成责任人变更；原责任人存在历史跟进记录 | 新负责销售打开该客户的跟进时间线 | 时间线连续展示原责任人与新负责销售的全部历史跟进记录，无历史截断（参见 BR-03） | US-04 |
| AC-06 | 某客户已累积不少于 200 条历史跟进记录 | 销售打开该客户的跟进时间线 | 全部记录成功加载展示，加载完成时间不超过 3 秒（参见 BR-04） | US-01, US-04 |
| AC-07 | 销售尝试在时间线视图中删除某条历史跟进记录 | 销售发起删除操作 | 操作被拒绝；时间线不提供删除历史记录的入口（参见 BR-02，原始记录不可删除由 F-02 BR-01 保障） | US-01 |
