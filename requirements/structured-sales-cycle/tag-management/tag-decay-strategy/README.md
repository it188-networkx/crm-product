# Feature Definition - [F-03] 标签衰减策略配置 (Tag Decay Strategy Configuration)

> 需求特性定义（A0304）｜所属 Epic：[E-06] 客户标签体系管理 ｜ 所属主题：T-01 结构化销售周期
> **Feature 编号**：F-03 ｜ **优先级**：P0 ｜ **状态**：Draft

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-06] 客户标签体系管理 (Customer Tag Management)](../README.md)

**业务定义**：
该特性支持 MKT Leader 与管理员对指定标签维度配置时间衰减规则，定义"当客户满足特定触发条件（如 N 天无跟进记录）时，系统自动将该维度标签值按配置的幅度降级，但不低于配置的最低边界"。衰减策略以配置记录的形式管理，参数变更须复用 F-01 的发布审批机制；衰减执行过程由系统自动完成，每次衰减均留存操作日志。它关注的是"衰减规则如何被定义、修改、发布和追溯"，不包含衰减结果的统计报告。

**范围边界**：

1. 包含：衰减策略的创建与参数配置、发布审批（复用 F-01 机制）、衰减规则的启用与停用、系统自动执行衰减并留存日志。
2. 不包含：衰减统计报告与异常客户群识别（见 F-04）、标签字典维护（见 F-01）、客户打标操作（见 F-02）。
3. 参见 Epic [DEP-02]：基于跟进活跃度的衰减触发条件依赖 E-02 跟进记录的时间戳；E-02 未就绪时，基于跟进天数的衰减规则不可用，仅支持以固定时间窗触发。
4. 参见 Epic [DEP-01]：衰减策略仅作用于 F-01 已发布字典中定义的标签维度。

## 业务规则 (Business Rules)

- **[BR-01] 适用维度约束**：衰减策略只能针对 F-01 已发布字典中存在的标签维度配置；维度被停用或从字典中删除时，关联的衰减策略须同步进入停用状态，不得继续执行。
- **[BR-02] 参数必须可配**：每条衰减规则的触发条件（如无跟进天数，阈值 TBD——由 MKT Leader 在创建或修改策略时显式指定；若未填写触发阈值，则该规则不允许激活）、衰减幅度与最低边界须通过配置管理，不得硬编码（参见 Epic [BR-05]）。
- **[BR-03] 最低边界保护**：衰减不得将标签值降至配置的最低边界以下；客户标签已处于最低边界时，系统不再执行衰减（参见 Epic [BR-06]）。
- **[BR-04] 参数变更须审批**：衰减策略的新增与参数修改须复用 F-01 发布审批机制；未通过审批的参数变更不得生效；审批期间原已发布的参数持续生效（参见 Epic [BR-05]）。
- **[BR-05] 配置权限**：衰减策略的创建与参数修改须限制为 MKT Leader 及管理员角色；其他角色不得发起策略创建或参数变更（参见 Epic [BR-08]）。
- **[BR-06] 衰减日志留存**：每次系统自动衰减须留存受影响客户标识、标签维度、触发条件快照、原标签值、新标签值与执行时间，支持按客户 ID、标签维度、时间范围检索（参见 Epic NFR-03）。
- **[BR-07] E-02 降级处理**：当 E-02 跟进记录时间戳不可用时，基于跟进活跃度的触发条件不可激活；已激活但跟进数据中断的规则须暂停执行并将规则状态标记为"数据源不可用"，不得静默跳过或以上次可用值继续执行（参见 Epic [DEP-02]）。

## 业务对象定义 (Business Object Definition)

### 衰减策略 (Decay Strategy)

衰减策略是针对某一标签维度的时间衰减配置记录，它定义了系统何时触发降级、每次降级幅度为多少、以及降级的最低边界，是衰减自动执行的业务依据。配置记录须经发布审批后方可生效，并以版本方式留存历史变更。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 适用标签维度 | 标识该策略作用于哪一个已发布字典中的标签维度 |
| 触发条件描述 | 人可读的触发条件说明（如"无 E-02 跟进记录满 N 天"） |
| 触发阈值 | 触发条件的量化参数，由 MKT Leader 在配置时显式指定；缺失时规则不允许激活（TBD：具体数值由业务方在配置时决定） |
| 衰减幅度 | 每次系统执行衰减时标签值的降级步长 |
| 最低边界值 | 衰减不得突破的底线标签值 |
| 当前状态 | 草稿 / 待审批 / 已发布 / 已停用 / 数据源不可用 |
| 版本号 | 标识该配置记录当前所处的版本，随每次审批发布递增 |

### 衰减执行记录 (Decay Execution Record)

衰减执行记录是单次系统自动衰减操作的日志凭证，用于承载本次衰减的业务上下文，支持对特定客户或标签维度的衰减历史进行追溯与审计。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属客户 | 标识此次衰减操作作用于哪位客户 |
| 标签维度 | 标识本次衰减操作针对的标签维度 |
| 触发条件快照 | 记录本次触发时所依据的规则条件（包含策略版本号与阈值） |
| 原标签值 | 衰减执行前客户在该维度的标签值 |
| 新标签值 | 衰减执行后客户在该维度的标签值 |
| 执行时间 | 系统完成本次衰减操作的时间戳 |
| 关联策略版本 | 本次衰减执行所依据的衰减策略版本号 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | MKT Leader 已登录；目标标签维度在 F-01 已发布字典中存在；触发阈值已显式指定 | MKT Leader 创建衰减策略并提交审批，审批通过后发布 | 衰减策略以"已发布"状态生效，版本号记录；策略仅对 F-01 已发布字典维度生效（参见 BR-01）；阈值未填写时不允许激活（参见 BR-02） | US-01 |
| AC-02 | 存在已发布衰减策略；系统到达定期检查时点；存在满足触发条件的客户 | 系统执行衰减检查并对满足条件的客户自动降级标签值 | 满足条件的客户标签值按配置幅度降级；每次操作生成衰减执行记录，包含客户标识、标签维度、原值、新值、执行时间与策略版本（参见 BR-06） | US-03 |
| AC-03 | 客户标签值已处于配置的最低边界 | 系统执行衰减检查，该客户满足触发条件 | 系统不对该客户执行降级操作；衰减执行记录标注"已达最低边界，跳过执行"（参见 BR-03） | US-01, US-03 |
| AC-04 | 已有"已发布"状态的衰减策略 | MKT Leader 修改策略参数并提交审批 | 修改进入待审批状态；审批通过前原已发布参数持续生效；审批通过后新参数以新版本号生效（参见 BR-04） | US-02 |
| AC-05 | 已激活的衰减规则其触发条件依赖 E-02 跟进记录；E-02 数据源不可用 | 系统到达定期检查时点 | 相关衰减规则暂停执行；规则状态变更为"数据源不可用"，不得静默跳过（参见 BR-07） | US-03 |
| AC-06 | 存在已执行的衰减记录 | MKT Leader 按客户 ID 或标签维度发起日志查询 | 返回符合条件的衰减执行记录列表，包含客户标识、标签维度、原值、新值、执行时间、关联策略版本；支持按时间范围过滤（参见 BR-06） | US-04 |
