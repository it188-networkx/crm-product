# Feature Definition - [F-01] 标签字典管理 (Tag Dictionary Management)

> 需求特性定义（A0304）｜所属 Epic：[E-06] 客户标签体系管理 ｜ 所属主题：T-01 结构化销售周期
> **Feature 编号**：F-01 ｜ **优先级**：P0 ｜ **状态**：Draft

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-06] 客户标签体系管理 (Customer Tag Management)](../README.md)

**业务定义**：
该特性支持管理员与 MKT Leader 按业务维度（如行业、规模、意向等级、客户类型）创建、编辑和发布标签字典条目，建立全团队统一可复用的客户标签资产。它关注的是"标签条目如何被定义、维护和发布"，包含对标签生命周期（草稿 → 待审批 → 已发布 → 已停用）的管控与历史版本追溯，不涉及标签如何被赋值给客户、冲突解决逻辑或衰减行为。

**范围边界**：

1. 包含：标签维度分组管理、标签条目创建与编辑、发布审批、版本历史查询、引用保护下的停用与删除控制。
2. 不包含：客户打标操作（见 F-02）、标签衰减策略配置（见 F-03）、标签使用统计报告（见 F-04）。
3. 参见 Epic [DEP-01]：标签字典是 E-01 线索评分特征和 E-05 自动化规则触发条件的基础，字典未就绪时下游能力须降级或延迟。
4. 参见 Epic [DEP-03]：标签维度设计须与 T-02 AI 特征工程需求协商对齐。

## 业务规则 (Business Rules)

- **[BR-01] 唯一性约束**：同一维度内标签可用值不得重名；标签名称在全库唯一，防止跨维度引用歧义。（参见 Epic [BR-01]）
- **[BR-02] 引用保护**：被 E-01 评分规则、E-05 自动化规则、E-02 跟进记录或客户档案引用的标签不允许直接删除；须先解除全部引用或将引用迁移至替代标签后方可删除。（参见 Epic [BR-02]）
- **[BR-03] 状态流转**：标签条目须经历草稿 → 待审批 → 已发布的生命周期；处于草稿或待审批状态的标签不可被外部引用；已发布标签可进入已停用状态，但停用前须满足 BR-02 的引用保护约束。
- **[BR-04] 发布审批**：标签字典的新增与修改须提交发布审批后方可生效；审批须由具有审批权限的管理员或 MKT Leader 执行，提交人不得自行审批同一变更。
- **[BR-05] 版本记录**：每次发布审批通过须留存版本快照（版本号、变更摘要、发布人、发布时间）；历史版本须可被检索查看；版本回滚须经重新发布审批，不支持静默覆盖。（参见 Epic [BR-09]）
- **[BR-06] 维护权限**：标签字典的创建、编辑、发布与删除操作须限制为管理员及 MKT Leader 角色；销售执行层不得访问字典的编辑与发布入口。（参见 Epic [BR-07]）
- **[BR-07] 删除路径**：被引用的标签须先展示当前全量引用清单，告知操作人后续须解除全部引用或迁移至替代标签，方可执行删除；不得在未展示引用清单的情况下直接阻断删除操作而不给出处置路径。（参见 Epic [BR-02]）

## 业务对象定义 (Business Object Definition)

### 标签维度 (Tag Dimension)

标签维度是将标签条目按业务属性分组的组织单元（如行业、规模、意向等级、客户类型），用于确保同一维度内标签可用值的唯一性管控，并支撑下游评分与规则引用时的维度定向查询。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 维度名称 | 标识该分组的业务含义，全库唯一 |
| 维度描述 | 说明该维度的适用场景与业务范围，供维护人员理解 |
| 适用角色范围 | 标识哪些角色可在打标时引用本维度下的标签条目 |

### 标签条目 (Tag Entry)

标签条目是标签字典中单个可被下游引用的标签值，承载标签的业务语义、生命周期状态与版本归属，是标签体系一切引用关系的原子单元。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属维度 | 标识该条目归属于哪个标签维度 |
| 标签名称 | 该标签条目的可读业务名称，全库唯一 |
| 可用值列表 | 该标签条目允许被赋予的有效取值范围 |
| 当前状态 | 标签条目所处生命周期阶段（草稿 / 待审批 / 已发布 / 已停用） |
| 创建人 | 标识由哪位管理员或 MKT Leader 创建，用于责任追溯 |
| 当前版本号 | 标识该条目最近一次发布审批通过后的版本归属 |

### 标签版本记录 (Tag Version Record)

标签版本记录是某次发布审批通过后的字典快照，用于支撑历史版本追溯、审计与回滚决策，是标签字典变更管控的正式凭证。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联标签条目 | 标识本次版本快照归属于哪条标签条目 |
| 版本号 | 标识该次发布在该条目历史中的顺序位置 |
| 变更摘要 | 描述本次发布相较上一版本的业务变更内容 |
| 发布人 | 标识执行审批通过操作的管理员或 MKT Leader |
| 发布时间 | 记录审批通过并生效的业务时间点 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 管理员或 MKT Leader 已登录；目标维度已存在或正在新建 | 填写标签名称与可用值后提交发布审批，由具有审批权限的不同账号审核通过 | 标签条目进入已发布状态，对全员可见并可被下游引用；版本记录留存（含版本号、变更摘要、发布人、发布时间）（参见 BR-03、BR-04、BR-05） | US-01, US-02 |
| AC-02 | 已有标签条目处于已发布状态 | 管理员或 MKT Leader 修改该条目的标签名称或可用值后提交 | 该条目回退至待审批状态，原已发布版本继续对外有效，直至新版本审批通过后方可替换；提交人不得自行审批（参见 BR-03、BR-04） | US-01, US-02 |
| AC-03 | 目标标签条目已被至少一个外部来源（E-01 评分规则、E-05 自动化规则、E-02 跟进记录或客户档案）引用 | 管理员发起删除该标签条目操作 | 系统展示当前全量引用清单（含引用来源与数量），阻止直接删除，并提示须先解除全部引用或迁移至替代标签后再执行删除（参见 BR-02、BR-07） | US-04 |
| AC-04 | 目标标签条目已存在至少两个历史已发布版本 | 管理员或 MKT Leader 进入版本历史查询 | 可检索到按发布时间排序的所有历史版本列表，每条记录包含版本号、变更摘要、发布人与发布时间；版本内容可查看但不可静默覆盖（参见 BR-05） | US-03 |
| AC-05 | 销售执行层账号已登录 | 尝试访问标签字典的编辑或发布入口 | 编辑与发布入口对销售执行层不可见或不可操作（参见 BR-06） | US-01, US-02 |
