# Feature Definition - [F-01] 结构化复盘录入 (Structured Review Entry)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-04] 成交复盘与策略回流 (Win/Loss Retrospective)](../README.md)

**业务定义**：
本特性为销售团队提供标准化复盘录入能力，将商机成交或失单的结果转化为结构化、可追溯的复盘档案。失单场景下系统强制触发复盘录入，要求销售在规定时限内完成含原因分类、竞争分析和内部评估的结构化表单；成交场景下系统以建议性提示触发录入，鼓励销售归档关键成功因素与竞争优势点。本特性聚焦于"录入哪些内容、在什么条件下录入、如何保证记录不可篡改"，不涉及 AI 比对、策略回流或复盘结论的下游消费逻辑。

**范围边界**：

1. 包含失单结构化复盘表单录入（强制触发）、成交复盘录入（建议性触发）、逾期提醒、修正说明追加。
2. 不包含 AI 一致性比对（由 F-02 承接）、策略回流建议归纳（由 F-03 承接）、归因渠道字段的初始采集（由 E-03 承接）。
3. 参见 Epic [DEP-01]：E-04 复盘录入依赖 E-03 的成交/失单结案状态作为触发上游；E-03 未就绪时无法自动触发复盘流程，须手动创建复盘记录。
4. 参见 Epic [ASM-01]：假设销售在商机结案后 48 小时内完成复盘录入；长期延迟录入将导致记忆偏差，影响复盘质量。

## 用户故事 (User Stories)

| ID | 角色 (As a...) | 想要 (I want to...) | 从而 (So that...) | 验收标准索引 (AC Ref) |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | 销售执行层 | 在商机失单结案后完成结构化复盘录入 | 记录失单原因分类、竞争对手与内部评估，形成可追溯的失单诊断档案 | AC-01, AC-03, AC-06 |
| US-02 | 销售执行层 | 在商机成交结案后自愿填写成交复盘 | 归档关键成功因素、决策触发要素与竞争优势点，为同类商机推进提供参考 | AC-02, AC-04 |
| US-03 | 销售执行层 | 对已提交的复盘记录追加修正说明 | 在发现填写偏差时补充说明，保留原始记录的同时标注修正意见 | AC-05 |
| US-04 | 销售 Leader | 查看本团队已归档的复盘记录 | 掌握团队失单原因分布，识别共性规律并采取管理行动 | AC-01, AC-02 |

## 业务规则 (Business Rules)

- **[BR-01]失单强制触发**：商机失单结案后系统必须自动触发复盘录入流程，销售须在失单结案后配置时限内（默认 48 小时）完成复盘提交；超时未提交时系统发送逾期提醒，不自动关闭或跳过复盘流程（继承 Epic BR-01）。
- **[BR-02]失单复盘必填字段**：失单复盘表单中，失单原因一级分类（由 E-03 预填）、主要竞争对手和关键决策障碍为必填字段，内部评估为选填；缺失任一必填字段则不得提交，系统须提示补充缺失字段。
- **[BR-03]归因渠道不重复采集**：归因渠道字段由 E-03 成交结案时录入，本特性复用 E-03 已采集的字段内容，不得重复采集或覆盖；若 E-03 未提供预填内容，该字段在复盘表单中显示为空，不阻断复盘提交。
- **[BR-04]成交复盘建议性触发**：商机成交结案后系统提示销售填写成交复盘，录入为建议性非强制，不阻断成交结案流程；管理员可通过系统配置将成交复盘设置为必填（继承 Epic BR-02）。
- **[BR-05]不可删除仅可追加修正**：复盘记录提交后不得删除，亦不得修改或覆盖原始内容；如发现填写偏差，只允许追加修正说明，修正说明须关联至原始复盘记录并标注追加人与追加时间。

## 业务对象定义 (Business Object Definition)

### 复盘记录

复盘记录是商机结案后对成交或失单过程的正式业务归档，用于承载结构化诊断内容、触发 AI 比对及策略回流等下游消费。失单复盘记录以失单原因与竞争分析为核心；成交复盘记录以成功因素与经验归纳为核心。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识该复盘记录归属的商机，保证每条复盘可唯一追溯至具体商机 |
| 复盘类型 | 区分失单复盘或成交复盘，决定适用的字段范围与触发规则 |
| 触发方式 | 标识复盘由系统自动触发还是人工手动创建，用于追溯触发异常 |
| 失单原因一级分类 | 失单复盘专属，由 E-03 预填，标识失单的主要分类原因 |
| 主要竞争对手 | 失单复盘必填，标识本次商机竞争中的核心竞品或竞争方 |
| 关键决策障碍 | 失单复盘必填，记录导致失单的关键业务或决策阻碍 |
| 内部评估 | 失单复盘选填，记录销售对本次推进过程的内部反思与评估 |
| 关键成功因素 | 成交复盘专属，记录本次成交的关键驱动要素 |
| 决策触发要素 | 成交复盘专属，记录促成客户最终决策的关键触发因素 |
| 竞争优势点 | 成交复盘专属，记录在竞争中发挥作用的差异化优势 |
| 提交人 | 标识由哪位销售人员提交复盘记录，作为责任归属依据 |
| 提交时间 | 记录复盘正式提交的时间，用于计算是否在规定时限内完成 |
| 逾期状态 | 标识失单复盘是否在规定时限内完成，用于逾期统计与提醒 |

### 修正说明

修正说明是对已归档复盘记录的补充性业务凭证，不覆盖原始内容，仅以追加方式记录后续发现的偏差说明。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属复盘记录 | 标识修正说明关联的原始复盘记录 |
| 修正内容说明 | 描述原始记录中需修正或补充的具体内容 |
| 追加人 | 标识提交修正说明的责任人 |
| 追加时间 | 记录修正说明追加的时间，形成完整的修改时间线 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 商机处于失单结案状态 | 系统自动触发复盘录入流程 | 生成含 E-03 预填摘要的失单复盘表单，标注规定完成时限，销售可查看并开始录入 | US-01, US-04 |
| AC-02 | 商机处于成交结案状态 | 系统提示销售填写成交复盘 | 成交结案流程正常完成，系统提示可选填写成交复盘，不阻断成交流程（参见 BR-04） | US-02 |
| AC-03 | 失单复盘表单缺少必填字段（主要竞争对手或关键决策障碍未填写） | 销售提交复盘 | 系统拒绝提交，明确提示缺失的必填字段名称（参见 BR-02） | US-01 |
| AC-04 | 成交复盘字段已填写（关键成功因素、决策触发要素、竞争优势点） | 销售提交成交复盘 | 成交复盘记录归档至对应商机，包含全部填写内容，对销售 Leader 可见 | US-02, US-04 |
| AC-05 | 复盘记录已归档提交 | 销售追加修正说明并提交 | 修正说明追加至原始复盘记录，原始内容不被修改，追加人与时间均有记录（参见 BR-05） | US-03 |
| AC-06 | 失单复盘超过规定时限仍未提交 | 时限到期 | 系统向销售发送逾期提醒，复盘录入入口保持开放，不自动关闭或跳过（参见 BR-01） | US-01 |
