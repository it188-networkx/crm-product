# Feature Definition - [F-03] 策略回流审核 (Strategy Feedback Review)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-04] 成交复盘与策略回流 (Win/Loss Retrospective)](../README.md)

**业务定义**：
本特性将累积的复盘结论转化为可操作的策略改进建议，并经由人工审核确认后同步至 E-01 评分规则或 E-02 培育策略，形成从复盘结论到销售策略优化的闭环反馈链路。系统依据复盘数据归纳共性偏差，生成评分规则回流建议（面向 MKT Leader）和培育策略回流建议（面向销售 Leader）；所有建议须经负责人逐条确认方可触发下游变更；全部审批记录留存可查。本特性聚焦于"建议如何生成、如何审核、如何同步、如何留存审批轨迹"，不涉及复盘录入（由 F-01 承接）或 AI 一致性比对（由 F-02 承接）。

**范围边界**：

1. 包含评分规则回流建议归纳、培育策略回流建议归纳、MKT Leader / 销售 Leader 审核（采纳/修改/拒绝）、采纳建议触发下游同步、审批记录留存与检索。
2. 不包含 E-01 评分规则的直接编辑（由 E-01 承接）、E-02 培育节奏参数的直接修改（由 E-02 承接）、下游变更发布审批流的执行（由 GC-07 承接）。
3. 参见 Epic [DEP-04]：评分规则回流须写入 E-01 配置，培育策略回流须写入 E-02 节奏参数；若目标 Epic 未提供配置入口，建议仅停留在已采纳状态，不能自动同步，标注"同步目标不可用"。

## 业务规则 (Business Rules)

- **[BR-01]建议生成触发条件**：系统在同类失单原因达到归纳阈值时生成回流建议（阈值待定 TBD；责任归属：产品负责人与 MKT Leader / 销售 Leader 确认；默认分流策略：默认阈值为近 30 天内同一失单原因分类出现 ≥ 3 次；未达到阈值则不生成建议，数据不足时建议列表为空）；建议以"待审核"状态呈现给对应负责人。
- **[BR-02]回流须人工确认方可生效**：系统归纳的所有回流建议，须经 MKT Leader（评分规则类）或销售 Leader（培育策略类）逐条人工确认后方可触发下游参数变更请求；未经确认的建议不得自动生效（继承 Epic BR-05）。
- **[BR-03]采纳须经下游审批流**：采纳的评分规则回流建议触发 E-01 参数变更时须经 GC-07 发布审批流；培育策略回流建议触发 E-02 参数变更时同样须经相应审批流；下游审批流未通过时，建议状态回退至"待审核"，并标注审批未通过原因。
- **[BR-04]所有审批记录须留存**：全部回流建议及其审批动作（采纳/修改/拒绝）须留存完整审批记录，记录须包含审批人、审批时间、审批意见（拒绝原因或修改说明）；留存记录须支持按日期和建议类型检索（继承 Epic BR-06）。
- **[BR-05]回流目标不可用时降级**：若 E-01 或 E-02 未提供可接受回流建议的配置入口，已采纳建议保持"已采纳"状态但不触发自动同步，在建议详情中标注"同步目标不可用"（参见 Epic DEP-04）。

## 业务对象定义 (Business Object Definition)

### 策略回流建议

策略回流建议是系统依据一批复盘结论自动归纳生成的改进建议凭证，用于驱动 MKT Leader / 销售 Leader 的审核决策，并在采纳后触发下游策略参数变更。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 建议类型 | 区分评分规则回流建议（面向 E-01）或培育策略回流建议（面向 E-02） |
| 归纳来源摘要 | 标识支撑本条建议的复盘结论条目引用，作为建议可信度的依据 |
| 识别的偏差描述 | 系统归纳的当前规则与复盘结论之间的偏差说明 |
| 建议调整内容 | 系统建议的具体规则或策略改进方向 |
| 生成时间 | 记录建议生成的时间 |
| 当前状态 | 标识建议所处的审核阶段（待审核/已采纳/已修改/已拒绝） |
| 审核责任人 | 标识负责审核本条建议的 MKT Leader 或销售 Leader |
| 下游同步状态 | 标识建议采纳后的下游同步结果（待同步/已同步/同步目标不可用/审批未通过） |

### 回流审批记录

回流审批记录是对每条回流建议审核动作的业务凭证，确保决策过程可追溯、可审计，支持策略回顾与合规核查。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属回流建议 | 标识该审批记录归属的策略回流建议 |
| 审批动作 | 记录审核人对建议的具体操作（采纳/修改/拒绝） |
| 审批人 | 标识执行审批的 MKT Leader 或销售 Leader |
| 审批时间 | 记录审批操作完成的时间 |
| 审批意见 | 拒绝时填写拒绝原因，修改时填写修改说明；采纳时可选填确认依据 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 近期复盘结论中同类失单原因达到归纳阈值 | 系统归纳生成回流建议 | 评分规则回流建议以"待审核"状态呈现给 MKT Leader，包含归纳来源摘要与建议内容（参见 BR-01） | US-01 |
| AC-02 | MKT Leader 查看评分规则回流建议；决策采纳 | MKT Leader 确认采纳并提交 | 建议状态变为"已采纳"，触发 E-01 参数变更的 GC-07 审批流；审批记录含审批人与时间（参见 BR-02、BR-03） | US-01, US-02 |
| AC-03 | MKT Leader 查看评分规则回流建议；决策拒绝 | MKT Leader 填写拒绝原因并确认拒绝 | 建议状态变为"已拒绝"，拒绝记录留存，包含拒绝原因、审批人与时间；不触发任何下游变更（参见 BR-04） | US-02 |
| AC-04 | 近期复盘结论中培育节奏相关偏差达到归纳阈值 | 系统归纳生成回流建议 | 培育策略回流建议以"待审核"状态呈现给销售 Leader，包含归纳来源摘要与建议内容（参见 BR-01） | US-03 |
| AC-05 | 销售 Leader 查看培育策略回流建议；决策采纳 | 销售 Leader 确认采纳并提交 | 建议状态变为"已采纳"，触发 E-02 节奏参数变更的审批流；审批记录含审批人与时间（参见 BR-02、BR-03） | US-03, US-04 |
| AC-06 | 审核人查询历史回流建议审批记录 | 按日期或建议类型筛选检索 | 返回符合条件的全部历史建议及其审批结果（状态、审批人、审批时间、审批意见），支持按日期和类型过滤（参见 BR-04） | US-05 |
| AC-07 | 建议已被采纳；但 E-01 或 E-02 同步目标不可用 | 采纳操作完成 | 建议保持"已采纳"状态，不触发下游自动同步，建议详情标注"同步目标不可用"（参见 BR-05） | US-02, US-04 |
| AC-08 | 建议已被采纳；下游 GC-07 审批流未通过 | 下游审批流返回未通过结果 | 建议状态回退至"待审核"，并在审批记录中标注审批未通过原因（参见 BR-03） | US-02, US-04 |
