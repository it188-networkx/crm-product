# Feature Definition - [F-02] AI 一致性比对 (AI Consistency Check)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-04] 成交复盘与策略回流 (Win/Loss Retrospective)](../README.md)

**业务定义**：
本特性在失单复盘提交后，利用 AI 能力自动比对销售填写的失单原因与 E-02 跟进记录中的沟通证据之间的一致性，对偏差超过阈值的复盘记录标记为异常单并触发二次确认。AI 比对结果仅作辅助建议，不自动修改复盘内容；最终复盘结论以销售人工裁定为准。本特性聚焦于"AI 比对在什么条件下触发、偏差如何判定与分流、人工裁定如何收尾"，不涉及复盘录入字段定义（由 F-01 承接）或策略回流（由 F-03 承接）。

**范围边界**：

1. 包含失单复盘的 AI 一致性比对触发、偏差检测与异常标记、二次确认请求、人工裁定记录。
2. 不包含成交复盘 AI 比对（成交场景不触发）、复盘录入字段定义（由 F-01 承接）、跟进证据的归档与管理（由 E-02 承接）。
3. 参见 Epic [DEP-02]：AI 比对所需证据来源于 E-02 跟进记录；E-02 未就绪或跟进记录缺失时，AI 比对降级不可用，但不阻断复盘录入。
4. 参见 Epic [DEP-03]：AI 一致性比对功能依赖 T-02 提供的文本分析能力；T-02 未就绪时，本特性整体降级，复盘以人工归纳替代。

## 业务规则 (Business Rules)

- **[BR-01]仅限失单场景触发**：AI 一致性比对仅在失单复盘提交后自动触发；成交复盘不触发 AI 比对流程。
- **[BR-02]AI 比对结果仅辅助**：AI 比对结果仅作为建议呈现给销售，不得自动修改销售已填写的复盘内容；最终复盘结论以销售人工裁定为准（继承 Epic BR-03）。
- **[BR-03]比对基于可用跟进证据**：AI 比对基于 E-02 中已归档的结构化跟进证据（含文本记录及可结构化提取的音频转写内容）；比对前系统检查证据可用性：若相关跟进证据缺失，判定为"证据不足"并跳过比对，不阻断复盘录入（继承 Epic BR-04）。
- **[BR-04]偏差阈值与异常分流**：AI 比对输出一致性评分，评分低于偏差阈值时判定为偏差超阈值并触发异常标记（阈值待定 TBD；责任归属：产品负责人与业务负责人确认；默认分流策略：采用默认阈值 60%，低于 60% → 标记异常单并发起二次确认；高于或等于 60% → 比对结果附加至复盘记录，不触发异常流程；证据不足 → 跳过比对，标记"证据不足"）。
- **[BR-05]二次确认后须有裁定结论**：异常标记复盘须由对应销售人员在规定时限内（与 BR-01 中复盘完成时限保持一致）完成二次确认；销售可选择修正复盘内容后重新提交，或确认维持原填写内容并附偏差说明；两种路径均须记录裁定结论、裁定人与裁定时间。
- **[BR-06]AI 服务不可用时降级**：AI 比对服务不可用时（参见 Epic NFR-01 降级策略），系统跳过本次比对，复盘记录正常归档，同时在记录中标注"AI 比对未执行-服务降级"，不阻断复盘提交流程。

## 业务对象定义 (Business Object Definition)

### AI 比对结果

AI 比对结果是对失单复盘记录中填写原因与跟进记录证据一致性评估的业务凭证，用于驱动异常标记与二次确认流程，并在最终裁定后作为复盘质量的可追溯依据。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属复盘记录 | 标识该比对结果归属的失单复盘记录 |
| 比对执行时间 | 记录 AI 比对触发与完成的时间，用于时效核查（参见 Epic NFR-01） |
| 证据来源引用 | 标识参与比对的 E-02 跟进记录条目，作为比对依据的溯源 |
| 一致性评分 | AI 输出的原因与证据相似度分值，是异常判定的量化基准 |
| 偏差描述 | AI 归纳的原因与证据之间的主要偏差内容，供销售参考 |
| 异常标记 | 标识本次比对结果是否触发异常（超阈值/正常/证据不足/服务降级） |

### 二次确认裁定记录

二次确认裁定记录是销售对异常标记复盘进行人工裁定的业务凭证，确保 AI 比对建议不自动生效，最终复盘结论以人工意志为准。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属复盘记录 | 标识该裁定记录归属的失单复盘记录 |
| 关联 AI 比对结果 | 引用触发本次二次确认的 AI 比对结果 |
| 裁定结论 | 标识销售最终选择修正复盘内容还是维持原填写 |
| 偏差说明 | 选择维持原填写时，销售对原因与证据偏差的主观解释说明 |
| 裁定人 | 标识执行裁定的销售人员 |
| 裁定时间 | 记录裁定操作完成的时间 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 失单复盘已提交；E-02 跟进证据可用；AI 服务正常 | 系统触发 AI 一致性比对 | 系统在规定时延内（≤5 秒，参见 Epic NFR-01）完成比对，并将结果呈现给销售 | US-01 |
| AC-02 | AI 比对完成；一致性评分高于或等于偏差阈值 | AI 比对结果生成 | 比对结果以建议附加至复盘记录，不触发异常标记，不修改复盘内容（参见 BR-02、BR-04） | US-01 |
| AC-03 | AI 比对完成；一致性评分低于偏差阈值 | AI 比对结果生成 | 系统标记该复盘为异常单，向销售发送二次确认请求，异常复盘对销售 Leader 可见（参见 BR-04） | US-01, US-02, US-03 |
| AC-04 | 复盘被标记为异常单；销售执行二次确认裁定 | 销售选择维持原填写或修正复盘内容后提交裁定 | 裁定结论、裁定人与裁定时间记录至复盘档案，最终复盘结论以销售裁定为准，AI 比对建议不自动覆盖（参见 BR-02、BR-05） | US-02 |
| AC-05 | 失单复盘已提交；E-02 相关跟进证据缺失 | 系统尝试触发 AI 比对 | 系统标记"证据不足"并跳过 AI 比对，复盘录入流程正常完成（参见 BR-03） | US-01 |
| AC-06 | 失单复盘已提交；AI 比对服务不可用 | 系统尝试触发 AI 比对 | 系统跳过比对步骤，复盘正常归档，记录中标注"AI 比对未执行-服务降级"，不阻断复盘流程（参见 BR-06） | US-01 |
