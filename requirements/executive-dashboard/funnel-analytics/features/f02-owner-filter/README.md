# Feature Definition - F-02 按负责人筛选视图 (Owner Filter View)

> 需求特性定义（A0304）｜所属 Epic：E-01 销售漏斗全链路视图 ｜所属主题：T-03 CEO 决策驾驶舱

## 特性概述 (Feature Overview)

**所属 Epic**：[E-01 销售漏斗全链路视图](../../README.md)

**业务定义**：
本特性在全链路漏斗基础上增加按销售负责人维度的筛选能力，使 CEO 能够查看特定销售人员或团队在各漏斗阶段的转化率与跟进及时率，并通过排行与周环比变化识别执行健康度下滑的个人。本特性服务于两类场景：月初例会前的瓶颈定位（按负责人确认某阶段执行问题的来源），以及每周例行健康度巡检（持续跟踪跟进及时率下滑趋势）。

负责人与商机的归属映射须与 T-01 线索分配记录中的负责人映射保持一致（参见 Epic [BR-02] / [DEP-02]）；跟进及时率的计算口径须与 F-01 全链路视图中使用的口径相同（参见 Epic [BR-04]）；数据访问须遵循 GC-06 权限分级约束（参见 Epic [BR-05]）。

**范围边界**：

1. 包含：按单个或多个销售负责人筛选漏斗视图、团队汇总维度与个人维度切换、各负责人各阶段转化率展示、各负责人跟进及时率展示、跟进及时率排行与周环比变化展示、跟进及时率低于阈值的负责人标注。
2. 不包含：全链路漏斗的无负责人汇总视图（由 F-01 承接）、从漏斗阶段进入商机明细列表（由 F-03 承接）、对负责人发起介入或通知动作（超出本 Epic 范围）。

## 业务规则 (Business Rules)

- **[BR-01] 负责人数据同源**：按负责人筛选所使用的人员归属关系必须与 T-01 线索分配记录中的负责人映射保持一致（参见 Epic [BR-02]），本特性不得建立独立的人员结构。
- **[BR-02] 跟进及时率定义**：跟进及时率 = 在选定时间窗口内，商机进入某阶段后，在规定时间窗口阈值内产生首次跟进记录的商机数量 ÷ 该负责人在该阶段的总商机数量；时间窗口阈值与 F-01 中使用的口径完全一致（参见 Epic [BR-04]）。[推导-待确认：跟进及时率时间窗口阈值待产品与业务确认，责任归属：产品负责人 layx，默认分流策略：阈值未确认前跟进及时率字段标记"[待定义]"，不展示计算结果]
- **[BR-03] 漏斗数据一致性**：按负责人筛选后某负责人各阶段的商机数量之和，须与 T-01/E-03 商机推进模块中该负责人名下对应阶段的商机总数一致（参见 Epic [BR-01]）。
- **[BR-04] 多人选择逻辑**：当 CEO 选择多个负责人时，视图分别展示各负责人的独立数据，支持横向对比；同时提供"合并为团队汇总"切换，合并后各阶段数量为所选负责人的并集（同一商机不重复计入）。[推导-待确认]
- **[BR-05] 跟进及时率低阈值标注**：当某负责人的跟进及时率低于标注阈值时，该负责人须被标注为执行风险；标注阈值业务值待定（TBD，责任归属：产品负责人 layx），默认分流策略：阈值未确认前不触发标注，排行数据仍正常展示。[推导-待确认]
- **[BR-06] 周环比计算**：周环比变化 = （本周跟进及时率 − 上周跟进及时率）÷ 上周跟进及时率；上周数据缺失时，环比标记为"—"，不得显示为 0 或产生计算错误。[推导-待确认]
- **[BR-07] 环比下滑标注阈值**：当某负责人的周环比变化为负且绝对值超过阈值时，须触发下滑标注；阈值业务值待定（TBD，责任归属：产品负责人 layx），默认分流策略：阈值未确认前不触发标注。[推导-待确认]
- **[BR-08] 权限边界**：CEO 可查看全团队所有销售负责人的数据；销售人员仅可查看本人及其下属的数据；无权查看的负责人数据不得出现在筛选列表中（参见 Epic [BR-05]）。

## 业务对象定义 (Business Object Definition)

### 负责人漏斗快照

负责人漏斗快照是特定时间窗口下某一销售负责人在各漏斗阶段的转化与执行汇总，是 CEO 判断该负责人执行健康度的业务依据。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属时间窗口 | 标识本快照所对应的统计周期，与 F-01 漏斗视图时间窗口对齐 |
| 负责人标识 | 标识本快照归属哪一销售负责人，与 T-01 线索分配记录中的人员映射对齐 |
| 各阶段商机数量 | 该负责人在选定时间窗口内各漏斗阶段的商机数量 |
| 各阶段流入转化率 | 该负责人名下相邻阶段的转化率 |
| 跟进及时率 | 该负责人在选定时间窗口内商机进入阶段后及时跟进的比率 |
| 团队排行位次 | 该负责人跟进及时率在可见团队范围内的排行位次 |
| 周环比变化 | 本周跟进及时率相对上周的变化幅度 |
| 执行风险标注 | 标识该负责人是否因跟进及时率低于阈值而被标注为执行风险 |
| 环比下滑标注 | 标识该负责人是否因周环比下滑幅度超过阈值而被触发下滑标注 |
