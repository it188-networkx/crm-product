# Feature Definition - F-03 阶段商机明细下钻 (Stage Opportunity Drilldown)

> 需求特性定义（A0304）｜所属 Epic：E-01 销售漏斗全链路视图 ｜所属主题：T-03 CEO 决策驾驶舱

## 特性概述 (Feature Overview)

**所属 Epic**：[E-01 销售漏斗全链路视图](../../README.md)

**业务定义**：
本特性允许 CEO 从全链路漏斗视图（F-01）的特定阶段，进入该阶段的商机明细列表，查看每条商机的名称、负责人、当前阶段停留时长与最近跟进时间；并支持按停留时长排序，使 CEO 快速定位在该阶段停留时间最长、最可能出现停滞风险的商机，为介入决策提供具体依据。

本特性以 F-01 全链路漏斗可视化为前提，F-01 未交付时本特性无法独立提供完整业务价值（参见 Epic [INT-01]）。商机明细数据须与 T-01/E-03 商机推进模块保持同源（参见 Epic [BR-01] / [DEP-01]）；停留时长计算口径须与 E-03 商机健康度预警的停滞判断基准完全一致（参见 Epic [BR-03] / [DEP-03]）；数据访问须遵循 GC-06 权限分级约束（参见 Epic [BR-05]）。

**范围边界**：

1. 包含：从漏斗阶段进入该阶段的商机明细列表、商机名称展示、商机负责人展示、当前阶段停留时长展示、最近跟进时间展示、按停留时长排序（升序/降序）。
2. 不包含：漏斗各阶段整体统计视图（由 F-01 承接）、按负责人筛选的漏斗视图（由 F-02 承接）、商机详情查看与编辑（超出本 Epic 范围）、介入与干预动作（由 E-03 商机健康度预警承接）。

## 用户故事 (User Stories)

| ID | 角色 (As a...) | 想要 (I want to...) | 从而 (So that...) | 验收标准索引 (AC Ref) |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | CEO | 从漏斗视图的特定阶段进入该阶段的商机明细列表 | 从汇总数字下钻到具体商机层级，为后续逐条分析提供入口 | AC-01, AC-02 |
| US-02 | CEO | 在商机明细列表中查看每条商机的负责人与当前阶段停留时长 | 判断哪位负责人名下的商机停留时间最长，快速定位执行层面的具体问题来源 | AC-01, AC-03 |
| US-03 | CEO | 在商机明细列表中查看每条商机的最近跟进时间 | 区分"停滞但有跟进"与"停滞且无跟进"两类商机，判断风险严重程度 | AC-01, AC-04 |
| US-04 | CEO | 在商机明细列表中按停留时长排序（从长到短或从短到长） | 快速将停留时间最长、停滞风险最高的商机置于列表顶部，优先关注 | AC-05 |

## 业务规则 (Business Rules)

- **[BR-01] 数据同源**：商机明细列表中展示的商机必须与 T-01/E-03 商机推进模块中该阶段的商机记录完全一致，不得出现遗漏或额外商机（参见 Epic [BR-01]）。
- **[BR-02] 下钻范围**：从漏斗某阶段进入明细列表时，列表仅展示在当前漏斗视图所选时间窗口内，于该阶段状态下且尚未推进至下一阶段或成单的商机；已完成该阶段推进的商机不纳入列表。[推导-待确认：如需展示历史已推进商机，须在产品评审中明确，默认分流策略：仅展示当前仍在该阶段的商机]
- **[BR-03] 停留时长计算**：单条商机的当前阶段停留时长 = 查询时间点 − 商机进入该阶段的时间点；起止边界定义与 E-03 商机健康度预警中"停滞超期"的判断基准完全一致（参见 Epic [BR-03]）。
- **[BR-04] 最近跟进时间定义**：最近跟进时间取该商机在当前阶段内最后一次跟进记录的时间戳；若该商机在当前阶段尚无任何跟进记录，最近跟进时间标记为"暂无跟进"，不得显示为空值或其他阶段的跟进时间。[推导-待确认]
- **[BR-05] 排序规则**：按停留时长排序时，降序为停留时长最长者排在最前；升序为停留时长最短者排在最前；停留时长相同时，以商机进入该阶段的时间点早者优先（先进先排）。[推导-待确认]
- **[BR-06] 权限边界**：CEO 可查看全团队所有商机的阶段明细；销售人员仅可查看本人及其下属负责的商机明细；无权查看的商机不得出现在列表中（参见 Epic [BR-05]）。

## 业务对象定义 (Business Object Definition)

### 阶段商机明细记录

阶段商机明细记录是 CEO 在从漏斗阶段下钻时所查看的单条商机的执行状态快照，是 CEO 判断具体商机停滞风险与介入优先级的业务依据。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 商机名称 | 商机的业务标识，与 T-01/E-03 商机推进模块中的名称保持一致 |
| 负责人 | 该商机在 T-01 线索分配记录中的当前负责人 |
| 当前阶段 | 该商机当前所处的漏斗阶段，与 T-01 商机推进阶段定义对齐 |
| 当前阶段停留时长 | 从商机进入当前阶段至查询时间点的持续时长 |
| 最近跟进时间 | 该商机在当前阶段内最后一次跟进记录的时间戳；无跟进时标记为"暂无跟进" |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | CEO 处于全链路漏斗视图（F-01）；T-01/E-03 商机数据已就绪 | CEO 从某漏斗阶段进入商机明细列表 | 列表展示当前仍在该阶段的所有商机，每条商机显示商机名称、负责人、当前阶段停留时长与最近跟进时间，列表中商机数量与漏斗阶段显示的商机总数一致（参见 BR-01、BR-02） | US-01, US-02, US-03 |
| AC-02 | 某阶段当前无商机（商机数量为零） | CEO 从该阶段进入商机明细列表 | 列表展示为空，并提供明确的"该阶段当前无商机"提示，不报错（参见 BR-02） | US-01 |
| AC-03 | CEO 处于商机明细列表 | CEO 查看某条商机的停留时长 | 停留时长为查询时间点减去该商机进入该阶段的时间点（参见 BR-03） | US-02 |
| AC-04 | 某商机在当前阶段无任何跟进记录 | CEO 查看该商机的最近跟进时间 | 最近跟进时间显示为"暂无跟进"（参见 BR-04） | US-03 |
| AC-05 | CEO 处于商机明细列表 | CEO 选择按停留时长降序排序 | 列表按停留时长由长到短重新排列，停留时长相同时按进入阶段时间早者优先（参见 BR-05） | US-04 |
