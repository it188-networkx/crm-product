# Feature Definition - F-01 全链路漏斗可视化 (Full-Chain Funnel Visualization)

> 需求特性定义（A0304）｜所属 Epic：E-01 销售漏斗全链路视图 ｜所属主题：T-03 CEO 决策驾驶舱

## 特性概述 (Feature Overview)

**所属 Epic**：[E-01 销售漏斗全链路视图](../../README.md)

**业务定义**：
本特性为 CEO 提供从线索进入到成单全链路各阶段的数量、转化率与平均停留时长的统一视图，使 CEO 能够独立识别转化瓶颈阶段，不再依赖销售人员口头汇报。视图支持时间窗口切换（7 天 / 30 天 / 90 天及自然月），并对转化率显著低于历史基准的阶段进行高亮标注，辅助 CEO 快速锁定需要介入的问题节点。

漏斗数据须与 T-01/E-03 商机推进模块保持同源（参见 Epic [BR-01] / [DEP-01]）；漏斗阶段划分与阶段名称须与 T-01 商机推进中的阶段定义一致（参见 Epic [BR-06]）；停留时长的起止边界定义须与 E-03 商机健康度预警对齐（参见 Epic [BR-03] / [DEP-03]）；本特性产出的漏斗指标为 E-04 Go/No-Go 看板提供 KPI 基线（参见 Epic [DEP-04]）。

**范围边界**：

1. 包含：全链路各阶段商机数量展示、相邻阶段转化率计算与展示、各阶段平均停留时长展示、时间窗口切换（7 天 / 30 天 / 90 天及自然月）、与历史基准对比的低转化率阶段高亮。
2. 不包含：按负责人维度筛选漏斗（由 F-02 承接）、从漏斗阶段进入商机明细列表（由 F-03 承接）、商机健康度预警与介入动作（由 E-03 承接）、Go/No-Go 指标看板（由 E-04 承接）。

| US-01 | CEO | 查看从线索进入到成单全链路各阶段的商机数量与相邻阶段转化率 | 快速掌握整体销售转化状况，识别数量骤降的瓶颈阶段 | AC-01, AC-02 |
| US-02 | CEO | 查看各阶段商机的平均停留时长 | 判断哪些阶段存在明显停滞，区分"数量少"与"推进慢"两类问题 | AC-01, AC-03 |

## 业务规则 (Business Rules)

- **[BR-01] 漏斗阶段完整性**：漏斗视图必须展示从线索进入到成单的全部阶段，不得遗漏中间阶段；各阶段顺序与阶段名称须与 T-01 商机推进中的阶段定义完全一致（参见 Epic [BR-06]）。
- **[BR-02] 数据同源**：各阶段商机数量与转化率数字必须与 T-01/E-03 商机推进模块中的商机状态记录保持同源（参见 Epic [BR-01]），本特性不得维护独立的统计口径。
- **[BR-03] 转化率计算**：相邻阶段转化率 = 下一阶段商机数量 ÷ 当前阶段商机数量；当分母为零时，转化率标记为"—"，不得显示为 0% 或产生计算错误。
- **[BR-04] 停留时长定义**：阶段平均停留时长 = 该阶段内所有商机（在选定时间窗口内已进入该阶段的商机）停留时长的算术平均值；单条商机停留时长的起止边界（进入阶段时间点 / 推进至下一阶段或成单时间点）须与 E-03 商机健康度预警中"停滞超期"的判断基准完全一致（参见 Epic [BR-03]）。
- **[BR-05] 仍在途商机处理**：对于在选定时间窗口内已进入某阶段但尚未推进到下一阶段的商机，其停留时长以查询时间点作为截止计算，不得排除在外，同时须在视图中标注该阶段存在仍在途商机。[推导-待确认]
- **[BR-06] 时间窗口边界**：时间窗口以"商机进入该阶段的时间戳"为纳入依据；凡在选定时间窗口范围内进入该阶段的商机均须纳入统计，不得以"离开阶段时间"或"成单时间"作为纳入条件。
- **[BR-07] 历史基准定义**：用于高亮比对的历史转化率基准，取前三个自然月对应阶段转化率的算术平均值；历史数据不足三个月时，可用数据期间的平均值作为基准，并在视图中标注基准周期。[推导-待确认：阈值待产品与数据团队确认，默认分流策略：数据不足时仍展示漏斗但不触发高亮]
- **[BR-08] 高亮触发条件**：当某阶段的当前周期转化率低于历史基准转化率超过阈值时，该阶段须触发高亮标注；阈值业务值待定（TBD，责任归属：产品负责人 layx），默认分流策略：阈值未确认前不触发高亮，历史基准数据仍正常展示。[推导-待确认]
- **[BR-09] 权限边界**：漏斗视图的数据访问须遵循 GC-06 权限分级约束（参见 Epic [BR-05]）；CEO 角色可查看全团队全阶段数据，无需按负责人筛选。

## 业务对象定义 (Business Object Definition)

### 漏斗阶段快照

漏斗阶段快照是特定时间窗口下某一漏斗阶段的统计汇总，是 CEO 判断该阶段转化健康度的业务基准。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属时间窗口 | 标识本快照所对应的统计周期（7 天 / 30 天 / 90 天 / 自然月） |
| 阶段标识 | 标识本快照归属哪一销售漏斗阶段，与 T-01 商机推进阶段定义对齐 |
| 阶段商机数量 | 在选定时间窗口内进入该阶段的商机总数 |
| 流入转化率 | 从上一阶段流入本阶段的转化率（本阶段数量 ÷ 上一阶段数量） |
| 平均停留时长 | 在选定时间窗口内进入该阶段的所有商机的平均阶段停留时长 |
| 历史基准转化率 | 用于对比高亮的历史参照转化率，取前三个自然月均值 |
| 基准覆盖周期 | 历史基准所依据的实际数据起止期间，供 CEO 验证基准可信度 |
| 是否包含仍在途商机 | 标识该阶段快照中是否存在尚未推进至下一阶段的商机，影响停留时长解读 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | T-01/E-03 商机状态数据已就绪；CEO 已通过权限验证 | CEO 打开全链路漏斗视图（默认时间窗口 30 天） | 视图展示全部漏斗阶段（与 T-01 阶段定义一致，无遗漏），每个阶段显示商机数量、流入转化率与平均停留时长（参见 BR-01、BR-02、BR-04） | US-01, US-02 |
| AC-02 | 相邻两阶段中某阶段商机数量为零 | CEO 查看该阶段转化率 | 该阶段转化率标记为"—"，不显示 0% 或报错（参见 BR-03） | US-01 |
| AC-03 | 选定时间窗口内某阶段存在尚未推进的仍在途商机 | CEO 查看该阶段停留时长 | 该阶段平均停留时长已纳入仍在途商机，且视图标注该阶段包含仍在途商机（参见 BR-05） | US-02 |
| AC-04 | CEO 当前处于漏斗视图 | CEO 切换时间窗口（如从 30 天切换为自然月） | 视图各阶段数量、转化率与平均停留时长全部按新时间窗口重新计算展示，时间窗口边界以商机进入阶段时间为准（参见 BR-06） | US-03 |
| AC-05 | 历史基准数据已就绪（≥ 1 个自然月可用数据） | CEO 查看漏斗视图 | 某阶段当前转化率低于历史基准超过高亮阈值时，该阶段具有视觉上可区分的高亮标注（参见 BR-07、BR-08） | US-04 |
| AC-06 | 历史基准数据不足三个自然月 | CEO 查看漏斗视图 | 视图基于可用数据期间计算基准并展示，视图标注基准覆盖周期，高亮逻辑不受影响（参见 BR-07） | US-04 |
