# Epic - [E-01] 销售漏斗全链路视图 (Funnel Analytics)

## 概述 (Overview)

**业务背景：**

CEO 当前对销售转化全链路的判断完全依赖销售人员的口头汇报，缺乏从线索进入到成单的统一数据视图。各阶段转化率、停留时长与跟进及时率均无可信的量化依据，当汇报口径不一致时，CEO 无法独立辨别"真实高质量商机"与"停滞但仍占用注意力的低质商机"，导致高价值商机的介入时机常被延误，资源分配决策只能凭经验拍板。

**核心目标：**

本 Epic 旨在为 CEO 构建一个以线索到成单全链路转化为核心的自助数据视图，通过漏斗阶段分布与停留时长分析，让 CEO 自主识别转化瓶颈及团队执行健康度，替代依赖口头汇报的判断模式。

**价值预估：**

- **决策自主**：老板通过系统做投入决策的事项占比从 0% 提升至 ≥ 60%（M6）。（支撑 Theme T-03 核心 KPI）
- **感知及时**：高风险商机从停滞到管理层感知并决策介入的平均时延，从依赖汇报（≥ 1 周）降低至系统可见（≤ 24 小时）。[推导-待确认]
- **口径统一**：CEO 与销售层对同一商机的状态数字完全一致，消除因口径差异引发的数据可信度质疑。

## 业务场景 (Business Scenarios)

### 场景一：全链路漏斗浏览与转化瓶颈定位

**用户**：CEO（王总）

**情境**：月初例会前，CEO 需要快速了解上月销售转化全链路的整体状况，识别转化率最低的阶段，并确认该阶段的主要负责人与停滞分布。

**业务流程**：
1. CEO 查看销售漏斗全链路转化视图，获取从线索到成单各阶段的数量与转化率分布。
2. 系统高亮转化率显著低于历史基准的阶段，并展示该阶段商机的平均停留时长。
3. CEO 按负责人维度筛选漏斗视图，查看各销售人员在该瓶颈阶段的跟进及时率。
4. 系统对跟进及时率低于阈值的负责人进行标注，辅助 CEO 识别执行层面的具体问题。
5. CEO 完成定位后选择具体商机进入详情，为后续介入决策提供依据。

**跨特性协同**：步骤 1-2 依赖 F-01（全链路漏斗可视化），步骤 3-4 依赖 F-02（按负责人筛选视图），步骤 5 依赖 F-03（阶段商机明细下钻）。

### 场景二：销售团队例行健康度巡检

**用户**：CEO（王总）

**情境**：每周固定时段，CEO 对销售团队执行健康度进行例行巡检，关注哪些团队成员的跟进及时率出现明显下滑趋势。

**业务流程**：
1. CEO 查看按负责人维度的跟进及时率排行与周环比变化。
2. 系统将跟进及时率环比下滑超过阈值的负责人及其关联商机列表展示给 CEO。[推导-待确认]
3. CEO 下钻查看特定负责人所负责商机的阶段分布与停留时长明细。
4. 系统展示该负责人商机在各阶段的停留时长分布与行业基准对比。
5. CEO 基于数据做出是否需要直接介入的判断，并通过 E-03 预警模块触发后续跟进闭环。

**跨特性协同**：步骤 1-2 依赖 F-02（按负责人筛选视图），步骤 3-4 依赖 F-03（阶段商机明细下钻）；步骤 5 的介入动作由 E-03 商机健康度预警 Epic 承接，不在本 Epic 范围内。

## 功能特性 (Functional Features)

| Feature ID | 名称 | 优先级 | 范围说明 |
| :--- | :--- | :--- | :--- |
| F-01 | 全链路漏斗可视化 | P0 | 展示从线索进入到成单全链路各阶段的数量、转化率与平均停留时长；支持时间窗口切换（7/30/90 天及自然月）；历史基准对比高亮低于均值的阶段 |
| F-02 | 按负责人筛选视图 | P0 | 支持按销售人员筛选漏斗视图，展示该负责人各阶段转化率与跟进及时率；支持多人对比与团队汇总维度切换 |
| F-03 | 阶段商机明细下钻 | P0 | 从漏斗阶段点击进入该阶段商机明细列表，展示商机名称、负责人、当前阶段停留时长与最近跟进时间；支持按停留时长排序 |

## 业务规则 (Business Rules)

| 编号 | 规则描述 | 适用 Feature |
| :--- | :--- | :--- |
| BR-01 | 漏斗数据必须与 T-01 E-03 商机推进模块的商机状态记录保持同源，两者展示的各阶段商机数量与转化率数字必须完全一致，不得独立维护统计口径。 | F-01, F-02, F-03 |
| BR-02 | 按负责人筛选所使用的人员归属关系必须与 T-01 线索分配记录中的负责人映射保持一致，不得出现两套人员结构。 | F-02 |
| BR-03 | 阶段停留时长的起止边界定义（进入阶段时间点 / 推进至下一阶段或成单时间点）必须与 E-03 商机健康度预警中"停滞超期"的判断基准完全一致，不得有歧义。 | F-01, F-02, F-03 |
| BR-04 | 跟进及时率的计算口径（从商机进入阶段到首次跟进记录的时间窗口阈值）在 F-01 全链路视图与 F-02 负责人视图中必须使用相同口径。[推导-待确认] | F-01, F-02 |
| BR-05 | 数据访问须遵循 GC-06 权限分级约束：CEO 视图与销售人员视图在权限边界内独立，CEO 可查看全团队数据，销售人员仅可查看本人及下属数据。 | F-01, F-02, F-03 |
| BR-06 | 漏斗各阶段名称与阶段划分标准必须与 T-01 商机推进中的阶段定义保持一致，不得在本 Epic 中自定义独立的阶段体系。 | F-01, F-02, F-03 |

## 非功能需求 (Non-Functional Requirements)

| 编号 | 类别 | 描述 |
| :--- | :--- | :--- |
| NFR-01 | 数据时效性 | 漏斗数据刷新延迟 ≤ 1 小时，确保 CEO 在查阅时看到的是当日可信快照，而非过期数据。[推导-待确认] |
| NFR-02 | 数据主权 | 漏斗 ROI 计算与指标口径支持管理员配置，关键指标提供下钻路径，CEO 可自验证数据来源，满足 GC-02 数据主权约束。 |
| NFR-03 | 操作审计 | CEO 对漏斗数据的查看与筛选操作须形成可审计的操作日志，满足 GC-04 操作审计约束。 |

## 约束与依赖 (Constraints & Dependencies)

| 编号 | 类别 | 描述 |
| :--- | :--- | :--- |
| DEP-01 | 外部依赖 | T-01 商机推进模块（E-03 商机推进与阶段门控）提供从线索到成单各阶段的状态记录与时间戳；此依赖未就绪时，漏斗数据不完整，本 Epic 无法交付完整价值。 |
| DEP-02 | 外部依赖 | T-01 线索分配模块提供负责人与商机的映射关系；此依赖未就绪时，F-02 按负责人筛选视图无法正常工作。 |
| DEP-03 | 跨 Epic 数据约束 | E-03 商机健康度预警（本 T-03 内部）依赖本 Epic 的漏斗阶段定义与停留时长数据作为判断基线；阶段定义一旦确认，不得在 E-03 上线后单独调整，须协同变更。 |
| DEP-04 | 跨 Epic 数据约束 | E-04 Go/No-Go 验收看板依赖本 Epic 的漏斗指标作为 KPI 达成率计算基线；指标口径须在 E-01 与 E-04 设计阶段同步对齐。 |
| ASM-01 | 前提假设 | 假设 T-01 商机推进模块已上线且各阶段状态记录数据完整，足以支撑全链路漏斗计算；若 T-01 数据不完整，漏斗视图降级为仅展示可用阶段的局部转化链路。[推导-待确认] |
