# T-03/E-03 商机健康度预警

> 需求史诗定义（A0303）｜所属主题：T-03 CEO 决策驾驶舱
> **Epic 编号**：E-03 ｜ **优先级**：P1 ｜ **状态**：规划中
> 关联 Issue：[#40](https://github.com/it188-networkx/crm-product/issues/40)

---

## 1. 业务背景

### As-Is（当前状态）

- CEO/管理层无法主动感知哪些商机出现停滞、竞争威胁或跟进异常，只能被动等待销售汇报
- 健康度判断完全依赖销售自评，数据主观性强，预警触发滞后
- 竞品威胁信号分散在竞品监控系统中，未与商机维度关联，CEO 无法直接感知影响范围

### To-Be（目标状态）

- 系统自动基于漏斗停留时长、AI 健康度评分、竞品预警信号，生成商机健康度评级并实时推送预警
- CEO 可在驾驶舱一键查看高风险商机列表，支持"关注标记"和"转派跟进"操作，推进动作留存审计日志
- 跨渠道竞争威胁商机被优先标出，辅助 CEO 判断是否需要战略干预

> [缺少需求输入-待补充]：健康度阈值的业务基准（客单价大小、行业差异）需与销售总监访谈确认

## 2. 核心 Feature 拆分

| Feature ID | Feature 名称 | 优先级 | 简述 |
|---|---|---|---|
| F-01 | 商机健康度评分与预警列表 | P1 | 基于漏斗停留时长 + AI 评分模型生成健康度等级，列表展示高/中/低风险商机，支持按负责人/阶段/风险类型筛选 |
| F-02 | 预警处置与关注标记 | P1 | CEO 可对预警商机标记"关注"、"转派"或"忽略"，操作记录写入审计日志（GC-04），处置状态实时同步销售负责人 |
| F-03 | 跨渠道竞争威胁预警 | P1 | 联动 T-02 E-03（竞品预警）信号，识别受竞争威胁商机并在驾驶舱高亮展示，支持威胁等级展示与趋势走势 |

## 3. 业务规则约束（Business Rules）

| 规则 ID | 描述 |
|---|---|
| BR-01 | 健康度评分模型由 T-02 E-01（AI 线索研判）提供，T-03 E-03 负责呈现与预警触发逻辑，不得重复建模 |
| BR-02 | 商机健康度分级阈值（高/中/低风险）由产品管理员在设置页配置，须与 E-01 漏斗阶段定义一致 |
| BR-03 | CEO"关注标记"和"转派"操作必须写入不可篡改审计日志（GC-04），日志保留 ≥ 12 个月 |
| BR-04 | 竞争威胁信号来自 T-02 E-03，T-03 E-03 仅消费信号，不自行采集竞品数据 |
| BR-05 | 预警列表仅展示 CEO 权限范围内的商机数据（GC-02 数据权限隔离），跨事业部数据须授权 |

## 4. 非功能需求（NFR）

| 编号 | 类型 | 要求 |
|---|---|---|
| NFR-01 | 性能 | 健康度预警列表加载 ≤ 2s（1000 条数据基准） |
| NFR-02 | 实时性 | 竞品威胁信号同步延迟 ≤ 15 分钟（来自 T-02 E-03 事件推送） |
| NFR-03 | 审计 | 所有 CEO 操作（关注/转派/忽略）审计日志写入成功率 100%，失败须回滚并告警 |

## 5. 跨 Epic 依赖

| 依赖标识 | 说明 |
|---|---|
| DEP-01 | **T-03/E-01（漏斗分析）**：漏斗阶段定义与停滞时长计算作为健康度判断基线，E-03 不得自行定义阶段 |
| DEP-02 | **T-02/E-01（AI 线索研判）**：提供 AI 商机健康度评分模型，API 须在 GA 前完成集成测试 |
| DEP-03 | **T-02/E-03（竞品预警）**：提供竞争威胁信号事件流，E-03 通过订阅消费，要求信号延迟 ≤ 15 分钟 |
| DEP-04 | **T-01/E-01（机会管理）**：商机基础数据（所有者、阶段、金额）读自 T-01 E-01 数据层 |
