# Feature - [F-01] 商机健康度多维评分 (Opportunity Health Scoring)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-03/E-03 商机健康度预警 (Opportunity Health)](../README.md)

**业务定义**：
本特性用于对每个在途商机进行持续、自动化的健康状态评估，将漏斗停滞时长、AI 评分与销售跟进行为三个独立维度的信号综合计算为健康度等级（高风险 / 中风险 / 健康），使 CEO 与管理层不再依赖销售主动上报，即可从系统层面感知哪些商机处于停滞或跟进异常状态。
本特性关注的是"商机当前健康状态如何评定"，不定义预警列表的呈现方式、处置操作或推送机制。

**范围边界**：

1. 包含：三维度数据采集（停滞时长 / AI 评分 / 跟进行为）、维度得分合并为健康度等级、按客户类型（终端/外单）分别应用阈值配置、评分结果存储与等级变更触发。
2. 不包含：AI 评分模型自身建模（参见 Epic [DEP-02]，由 T-02/E-01 提供）；漏斗阶段定义与停滞时长计算逻辑（参见 Epic [DEP-01]，由 T-03/E-01 提供）；商机基础字段（参见 Epic [DEP-04]，由 T-01/E-01 提供）；预警列表呈现（F-02）；处置操作（F-03）；主动推送（F-04）。
3. 阈值配置的管理界面不在本特性范围，由系统设置模块承接；本特性仅消费已配置的阈值进行等级判定（参见 Epic [BR-04]）。
4. 客户类型分类阈值配置（终端/外单）依赖销售总监阈值访谈前置（参见 Epic [ASM-01]）；若访谈延迟，初始版本降级为统一默认阈值，后续版本切换为分类配置。

## 业务规则 (Business Rules)

- **[BR-01]AI 评分来源约束**：商机健康度中的 AI 评分维度须直接使用 T-02/E-01 提供的评分结论，本特性不得自行建模或推导替代评分；若 T-02/E-01 评分能力尚未覆盖商机域，须在评分结果中标记"AI 评分待补充"，健康度等级降级为仅依赖另外两个维度计算，并在等级结果中注明评分来源不完整。（参见 Epic [BR-01]、[DEP-02]）
- **[BR-02]跟进缺失降分**：当商机连续 14 天无销售跟进记录时，"跟进行为"维度得分必须降至该维度最低分值，并触发健康度综合等级重新评估；14 天窗口以自然日计算，跟进记录缺失数据或数据异常时不得静默忽略，须降级为最低分并标注数据异常。（参见 Epic [BR-02]）
- **[BR-03]客户类型分权阈值**：高/中/低风险等级判定阈值必须按客户类型（终端客户 / 外单客户）分别维护，终端客户阈值集与外单客户阈值集相互独立，不得跨类型共用同一组阈值；若商机的客户类型字段缺失，系统须标记"类型未知"并暂停等级判定，不得降级套用任意一组阈值。（参见 Epic [BR-03]）
- **[BR-04]阈值版本追踪**：每次阈值配置变更须记录变更版本（含变更时间与变更人），历史版本须可查询；阈值变更后所有在途商机须在下一个评分周期内按新阈值重新评定等级。（参见 Epic [BR-04]）
- **[BR-05]评分触发时机**：健康度等级评估须在以下任一事件发生后触发重新计算：销售跟进记录新增、AI 评分结论更新、漏斗阶段变更、评分周期到期（周期长度由系统设置，TBD，责任归属：产品管理员；周期到期前若上述事件均未发生，维持当前等级不变）。
- **[BR-06]评分结果完整性**：每次评估结论须包含健康度等级、三个维度得分及其对应权重、评估时间与触发原因；缺少任一必要维度数据时，等级输出必须携带"数据不完整"标记，不得输出无标记的等级结论。

## 业务对象定义 (Business Object Definition)

### 商机健康度评分记录

商机健康度评分记录是对某一在途商机在特定评估时点的健康状态的正式业务快照，用于承载三个评分维度的当前状态、综合等级结论及评估过程的可追溯信息。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识该评分记录归属哪一个在途商机 |
| 漏斗停滞得分 | 基于当前漏斗阶段停留时长计算的维度分值 |
| AI 评分得分 | 来自 T-02/E-01 的商机健康度评分结论（含数据不完整标记） |
| 跟进行为得分 | 基于最近跟进记录时间计算的维度分值（含跟进缺失降分标记） |
| 健康度等级 | 综合三个维度后判定的等级结论：高风险 / 中风险 / 健康 |
| 客户类型 | 用于确定适用阈值集合的客户分类（终端/外单/未知） |
| 阈值版本引用 | 本次判定所使用的阈值配置版本，用于结果追溯 |
| 评估时间 | 本次等级评估完成的时间基准 |
| 触发原因 | 本次重新评估的触发事件类型 |
| 数据完整性标记 | 标识本次评分是否因数据缺失而处于降级状态 |

### 健康度阈值配置

健康度阈值配置定义了在特定客户类型下，各维度得分组合如何映射到健康度等级，是等级判定的业务规则基线。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 客户类型 | 该阈值集合适用的客户分类（终端/外单） |
| 高风险阈值边界 | 触发高风险等级的综合得分上限（含各维度权重） |
| 中风险阈值边界 | 触发中风险等级的综合得分范围 |
| 配置版本 | 阈值的唯一版本标识，用于历史追溯 |
| 变更时间 | 该版本配置生效的时间 |
| 变更责任人 | 执行该次配置变更的产品管理员 |
