# Feature - [F-04] 预警主动推送通知 (Proactive Alert Push Notification)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-03/E-03 商机健康度预警 (Opportunity Health)](../README.md)

**业务定义**：
本特性使系统能够在商机健康度等级首次进入高风险或竞品威胁等级升高时，主动向 CEO 推送预警通知，而无需 CEO 定期主动巡检。通知内容包含商机摘要与处置操作入口，使 CEO 能够直接从通知跳转至处置场景，压缩从风险产生到管理层感知的时延。
本特性关注的是"系统主动触达 CEO 的预警通知触发机制与通知内容"，不定义健康度等级的计算方式（F-01），不定义预警列表的呈现（F-02），也不定义处置操作的执行逻辑（F-03）。

**范围边界**：

1. 包含：高风险首次进入触发推送、竞品威胁等级升高触发推送、通知内容定义（商机摘要 + 处置入口）、通知频率控制（防重推机制）、通知可见性范围控制。
2. 不包含：健康度等级计算逻辑（参见 F-01）；竞品威胁信号的采集与关联逻辑（参见 F-05，上游信号来自 T-02/E-03，参见 Epic [DEP-03]）；处置操作功能实现（参见 F-03）；预警列表（参见 F-02）；通知的具体送达渠道技术实现（由系统架构确定）。
3. 推送通知仅面向 CEO 数据权限范围内的商机，跨事业部商机须经独立授权（参见 Epic [BR-07]）。

## 用户故事 (User Stories)

| ID | 角色 (As a...) | 想要 (I want to...) | 从而 (So that...) | 验收标准索引 (AC Ref) |
| :--- | :--- | :--- | :--- | :--- |
| US-01 | CEO | 当商机首次进入高风险等级时主动收到系统推送通知 | 第一时间感知风险恶化，不依赖定期巡检即可触发干预 | AC-01, AC-04 |
| US-02 | CEO | 当竞品威胁等级升高时主动收到系统推送通知 | 及时了解受竞品威胁的在途商机，快速判断是否需要战略干预 | AC-02, AC-04 |

| US-04 | CEO | 同一商机在短期内不被重复推送相同类型的预警通知 | 避免因通知泛滥导致重要预警被忽视 | AC-05 |

## 业务规则 (Business Rules)

- **[BR-01]高风险首次进入触发**：仅当商机健康度等级从非高风险状态变更为高风险时，才触发高风险预警通知；商机在高风险状态内的持续评分刷新不重复触发；商机从高风险降级后再次升级至高风险视为新一次首次进入，允许重新触发推送。
- **[BR-02]竞品威胁升高触发**：仅当 T-02/E-03 提供的竞品威胁等级在本 Epic 关联的商机维度发生等级上升（即由低威胁升至中威胁，或由中威胁升至高威胁）时触发推送；威胁等级维持不变或下降时不触发；竞品威胁信号延迟超过 15 分钟须在通知中标注信号时延状态（参见 Epic [DEP-03]、[NFR-02]）。
- **[BR-03]通知内容完整性**：每条推送通知须包含：商机名称、客户名称、客户类型、当前负责人、健康度等级（或竞品威胁等级）、主要风险类型标记、直达处置操作的入口；缺少必要字段时通知须标注"数据不完整"，不得静默发出不完整通知。
- **[BR-04]防重推机制**：同一商机同一风险类型的预警通知，在 24 小时内不得重复推送（24 小时窗口以通知发出时间计算）；24 小时窗口结束后，若商机仍处于同一风险状态且符合触发条件，可重新触发推送；防重推规则须独立记录推送历史，不得依赖 CEO 是否已处置来判断是否推送。
- **[BR-05]数据权限边界**：推送通知仅针对 CEO 当前数据权限范围内的商机；跨事业部商机须独立授权后方可触发推送，未授权的跨事业部预警不得向 CEO 推送。（参见 Epic [BR-07]）
- **[BR-06]推送触发可靠性**：满足触发条件的推送通知，系统须在触发事件发生后 5 分钟内完成推送（TBD 具体时限，责任归属：产品负责人；超时需触发告警，不得静默丢失）；推送失败须记录失败原因并触发告警，不允许静默丢失。

## 业务对象定义 (Business Object Definition)

### 预警推送通知记录

预警推送通知记录是系统向 CEO 主动发出预警通知的正式业务凭证，用于追溯通知触发原因、内容完整性与送达状态，并支撑防重推机制判断。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识本次通知对应的预警商机 |
| 推送类型 | 触发本次通知的风险类型：高风险首次进入 / 竞品威胁等级升高 |
| 接收人 | 通知目标接收人（CEO） |
| 商机摘要 | 通知内含的商机关键信息（名称、客户、负责人、等级、风险类型） |
| 处置入口标记 | 通知中是否包含直达处置操作的入口，及其对应的处置操作类型 |
| 推送时间 | 通知成功发出的时间 |
| 推送状态 | 通知的发送结果（成功/失败） |
| 失败原因 | 发送失败时的原因说明，用于告警追溯 |
| 数据完整性标记 | 标识本次通知是否因数据缺失而以不完整状态发出 |
| 信号时延标记 | 竞品威胁通知中，标识上游信号是否存在超时延迟 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 商机健康度等级从中风险变更为高风险（首次进入高风险） | F-01 评分变更事件发生 | 系统在 5 分钟内向 CEO 推送高风险预警通知，推送记录生成（参见 BR-01、BR-06） | US-01 |
| AC-02 | 竞品威胁等级从中威胁升至高威胁，涉及 CEO 权限范围内的在途商机 | T-02/E-03 竞品威胁信号到达 | 系统在 5 分钟内向 CEO 推送竞品威胁预警通知，推送记录生成；若信号时延 > 15 分钟，通知中标注信号时延状态（参见 BR-02、BR-06） | US-02 |
| AC-03 | 预警推送通知已生成 | CEO 收到推送通知 | 通知内容包含：商机名称、客户名称、客户类型、当前负责人、健康度/竞品威胁等级、主要风险类型、直达处置操作入口（参见 BR-03） | US-03 |
| AC-04 | 商机已推送高风险预警通知且在高风险状态内持续评分刷新 | F-01 再次评分但等级未发生降级后升级 | 不触发重复推送（参见 BR-01） | US-01 |
| AC-05 | 某商机在 20 小时前已推送高风险通知且仍处于高风险状态 | 24 小时窗口未到期，再次满足高风险触发条件 | 不触发新通知；24 小时窗口到期后若仍满足触发条件则重新触发（参见 BR-04） | US-04 |
| AC-06 | CEO 不具备某事业部商机数据权限 | 该商机进入高风险状态 | 不向 CEO 推送预警通知（参见 BR-05、Epic [BR-07]） | US-01 |
| AC-07 | 推送通知所需商机字段存在缺失 | 触发推送 | 通知以"数据不完整"标记发出，缺失字段须明确标注，不得静默发出无标记的不完整通知（参见 BR-03） | US-03 |
