# Feature - [F-03] 预警处置与关注标记 (Alert Action & Attention Marking)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-03/E-03 商机健康度预警 (Opportunity Health)](../README.md)

**业务定义**：
本特性使 CEO 能够对预警商机执行关注标记、忽略或转派三类处置操作，将其决策意图记录为可追溯的正式业务事件，并通过通知机制将结果同步给受影响的销售负责人。所有操作须写入不可篡改的审计日志，确保管理行为的合规性与可追溯性。
本特性关注的是"CEO 对已识别预警商机的处置行为"，不定义预警列表的呈现逻辑（F-02），也不定义商机健康度等级的计算方式（F-01）。

**范围边界**：

1. 包含：关注标记操作（含介入意图说明）、忽略操作、转派操作（含转派说明与新受让人指定）、操作结果通知（通知销售负责人）、操作审计日志写入。
2. 不包含：预警商机的筛选与聚合展示（参见 F-02）；处置入口的主动推送触达方式（参见 F-04）；处置操作后的健康度等级重新评估（参见 F-01）；竞品威胁追踪（F-05）。
3. 所有操作须写入不可篡改审计日志，日志保留期 ≥ 36 个月，审计日志写入成功率须达到 100%（参见 Epic [BR-05]、[NFR-03]）。
4. 转派操作完成后须同步通知新受让销售负责人，通知内容须包含商机摘要、CEO 转派说明与当前健康度状态（参见 Epic [BR-06]）。

## 业务规则 (Business Rules)

- **[BR-01]处置操作范围**：CEO 可对预警商机执行且仅执行以下三类处置操作：（1）关注标记——必须填写介入意图说明方可提交；（2）忽略——可选填忽略原因；（3）转派——必须同时指定新受让销售负责人且必须填写转派说明方可提交。三类操作之外的商机修改不属于本特性范围。
- **[BR-02]操作前置状态**：仅允许对当前健康度等级为高风险或中风险的在途商机发起处置操作；已关闭、已赢单、已失单或健康度等级为"健康"的商机不得接受处置操作。
- **[BR-03]审计日志写入约束**：三类处置操作全部执行成功后，系统必须同步将操作记录写入不可篡改审计日志；日志写入失败时操作必须回滚，禁止静默丢失；日志保留期 ≥ 36 个月。（参见 Epic [BR-05]、[NFR-03]）
- **[BR-04]转派通知强制**：转派操作完成且审计日志写入成功后，系统必须同步发出通知至新受让销售负责人；通知内容须包含：商机名称与摘要、CEO 的转派说明、当前健康度等级与主要风险类型；通知发送失败须触发告警，不得静默忽略。（参见 Epic [BR-06]）
- **[BR-05]关注标记唯一性**：同一商机在同一时间只能处于一个关注标记状态；已被标记"关注"的商机可被 CEO 更新介入意图说明，但不得重复创建新的关注标记记录，须在原记录上追加更新历史。
- **[BR-06]忽略可撤销**：忽略操作在当日内可由 CEO 主动撤销；撤销操作须记录至审计日志；忽略状态被撤销后商机恢复至未处置状态；忽略撤销是否通知原负责人不在本特性范围内（TBD，责任归属：产品负责人）。
- **[BR-07]操作可见性**：CEO 可查看其权限范围内任一预警商机的全部历史处置操作记录；销售负责人仅可查看自己所负责商机上的处置操作结果（不含其他商机），不可查看其他人的处置历史。

## 业务对象定义 (Business Object Definition)

### 处置操作记录

处置操作记录是 CEO 对预警商机执行关注标记、忽略或转派操作的正式业务凭证，用于追溯处置历史、支撑审计合规与后续状态管理。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识该操作记录归属哪一个预警商机 |
| 操作类型 | 三类操作之一：关注标记 / 忽略 / 转派 |
| 操作人 | 执行本次处置操作的 CEO |
| 操作时间 | 操作成功完成并写入审计日志的时间 |
| 介入说明 | 关注标记时填写的介入意图说明（关注标记操作必填） |
| 忽略原因 | 忽略操作时填写的原因（可选） |
| 转派说明 | 转派操作时填写的说明（转派操作必填） |
| 新受让负责人 | 转派操作时指定的新销售负责人（转派操作必填） |
| 撤销状态 | 标识忽略操作是否已被撤销及撤销时间 |
| 审计日志写入状态 | 标识本条记录是否已成功写入不可篡改审计日志 |

### 处置通知记录

处置通知记录是转派操作完成后系统向新受让销售负责人发出通知的业务凭证，用于确认通知是否送达及支撑告警追溯。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联处置操作 | 标识该通知由哪条转派处置操作触发 |
| 接收人 | 通知的目标接收人（新受让销售负责人） |
| 通知内容摘要 | 商机名称、CEO 转派说明、当前健康度等级与主要风险类型 |
| 发送状态 | 通知是否成功发送（成功/失败） |
| 失败原因 | 发送失败时的原因说明，用于告警追溯 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 商机当前健康度等级为高风险或中风险，处于在途状态 | CEO 填写介入意图说明并提交关注标记操作 | 关注标记状态写入，处置操作记录创建，审计日志同步写入；同一商机不重复创建新关注标记记录（参见 BR-01、BR-03、BR-05） | US-01 |
| AC-02 | 商机当前健康度等级为高风险或中风险，处于在途状态 | CEO 提交忽略操作 | 忽略状态写入，处置操作记录创建，审计日志同步写入（参见 BR-01、BR-03） | US-02 |
| AC-03 | 商机当前健康度等级为高风险或中风险，处于在途状态；CEO 已指定新受让人并填写转派说明 | CEO 提交转派操作 | 转派操作写入，处置操作记录创建，审计日志同步写入，通知同步发出至新受让销售负责人（参见 BR-01、BR-03、BR-04） | US-03, US-04 |
| AC-04 | 转派操作已成功提交且审计日志已写入 | 系统发出转派通知 | 通知内容包含商机名称与摘要、CEO 转派说明、当前健康度等级与主要风险类型；通知发送失败须触发告警（参见 BR-04） | US-04 |
| AC-05 | 某商机上存在多条历史处置操作记录 | CEO 查看该商机的处置历史 | 历史操作记录按时间倒序展示，每条记录含操作类型、操作时间、操作人与说明内容（参见 BR-07） | US-05 |
| AC-06 | 审计日志写入失败 | CEO 提交任意处置操作 | 操作回滚，CEO 收到操作失败提示，不产生任何静默丢失的操作记录（参见 BR-03、Epic [BR-05]） | US-01, US-02, US-03 |
| AC-07 | 商机健康度等级为"健康"或商机已关闭/赢单/失单 | CEO 尝试提交处置操作 | 操作被拒绝，不产生任何处置记录（参见 BR-02） | US-01, US-02, US-03 |
