# Feature - [F-02] 高风险商机预警列表 (Risk Alert List)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-03/E-03 商机健康度预警 (Opportunity Health)](../README.md)

**业务定义**：
本特性为 CEO 提供一个聚合视图，将健康度等级达到高风险或中风险的在途商机汇集展示，使 CEO 在开展周度巡检时无需在多个入口手动筛查，即可在单一预警列表中快速定位需重点关注的商机范围并进行多维度过滤排序。
本特性关注的是"高风险/中风险商机的聚合呈现与筛查导航"，不定义健康度等级的计算方式（F-01），也不定义 CEO 在预警列表上执行的处置操作（F-03）。

**范围边界**：

1. 包含：高/中风险商机聚合展示、按负责人/阶段/风险类型/客户类型的筛选与排序、列表内单条商机摘要的必要信息展示。
2. 不包含：健康度等级计算逻辑（参见 F-01）；处置操作（关注标记、忽略、转派）入口的功能实现（参见 F-03）；主动推送通知（F-04）；竞争威胁追踪列表（F-05）。
3. 列表仅展示 CEO 数据权限范围内的商机数据，跨事业部数据须经独立授权（参见 Epic [BR-07]）。
4. 列表数据来源依赖 F-01 提供的健康度评分结果；若 F-01 评分数据不可用，列表应展示空状态而非展示未评分商机（参见 Epic [DEP-01]、[DEP-04]）。

## 业务规则 (Business Rules)

- **[BR-01]预警门控**：列表仅聚合展示健康度等级为"高风险"或"中风险"的在途商机；健康状态为"健康"的商机不得出现在预警列表中；已关闭、已赢单、已失单的商机不属于"在途"范围，不得纳入列表。
- **[BR-02]数据权限边界**：列表展示的商机范围须严格限定在 CEO 当前数据权限范围内；跨事业部商机须独立授权后方可展示，未经授权的跨事业部数据禁止越权呈现。（参见 Epic [BR-07]）
- **[BR-03]筛选条件独立可组合**：负责人、漏斗阶段、风险类型、客户类型四类筛选条件须支持独立使用及任意组合叠加，任一筛选条件清空后列表须恢复至仅受数据权限过滤的完整结果集。
- **[BR-04]风险类型标记**：每条预警商机须标注导致当前风险等级的主要风险类型（漏斗停滞 / 跟进缺失 / AI 评分低 / 竞品威胁）；单条商机可同时具有多个风险类型标记；风险类型标记须与 F-01 评分记录保持一致，不得自行推断。
- **[BR-05]排序基准**：列表默认排序规则为风险等级降序（高风险优先）；风险等级相同时，以漏斗停滞时长降序作为次级排序条件；CEO 可手动切换排序维度，切换后本次会话内保持所选排序，会话结束后恢复默认排序。
- **[BR-06]列表响应时效**：预警列表基于 1000 条数据量级，业务感知加载时间不超过 2 秒；筛选条件变更后结果刷新时间同样不超过 2 秒。（参见 Epic [NFR-01]）
- **[BR-07]摘要信息完整性**：列表中每条商机摘要须至少包含：商机名称、客户名称、客户类型、负责人、当前漏斗阶段、健康度等级、主要风险类型标记、停滞时长；缺少任一字段时，对应字段以"数据不完整"标记展示，不得静默留空。

## 业务对象定义 (Business Object Definition)

### 预警商机列表条目

预警商机列表条目是高/中风险在途商机在预警列表中的业务摘要快照，供 CEO 进行风险识别和筛选决策，不承载完整的商机详情。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 商机名称 | 标识商机的业务名称 |
| 客户名称 | 归属客户的名称，用于快速识别客户背景 |
| 客户类型 | 终端客户或外单客户，用于筛选与阈值关联 |
| 负责人 | 当前负责跟进该商机的销售人员 |
| 当前漏斗阶段 | 商机在销售漏斗中的当前所处阶段 |
| 健康度等级 | 由 F-01 评定的当前风险等级（高风险/中风险） |
| 主要风险类型 | 导致当前风险等级的风险原因标记（可多选） |
| 漏斗停滞时长 | 商机在当前阶段停留的自然日数，作为排序与筛选依据 |
| 数据完整性标记 | 标识该条目中是否存在数据不完整的字段 |

### 预警列表筛选状态

预警列表筛选状态描述 CEO 在当前会话中对预警列表应用的筛选与排序条件组合。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 负责人筛选 | 当前选中的一个或多个销售负责人 |
| 漏斗阶段筛选 | 当前选中的一个或多个漏斗阶段 |
| 风险类型筛选 | 当前选中的一个或多个风险类型标记 |
| 客户类型筛选 | 当前选中的客户类型（终端/外单/全部） |
| 排序维度 | 当前选择的排序字段（风险等级/停滞时长） |
| 排序方向 | 升序或降序 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | CEO 数据权限内存在高风险和中风险在途商机；F-01 评分数据可用 | CEO 打开预警列表 | 列表展示全部高风险和中风险在途商机，每条条目包含规定摘要字段，默认按风险等级降序排列（参见 BR-01、BR-05） | US-01 |
| AC-02 | F-01 评分数据不可用 | CEO 打开预警列表 | 列表展示空状态，明确标注数据来源不可用，不展示任何未评分商机 | US-01 |
| AC-03 | 预警列表已加载；CEO 设置一个或多个筛选条件（负责人、阶段、风险类型、客户类型中的一个或组合） | 提交筛选条件 | 列表结果仅展示满足全部筛选条件的商机，响应时间 ≤ 2 秒；清空筛选条件后恢复完整结果集（参见 BR-03、BR-06） | US-02, US-03, US-04, US-05 |
| AC-04 | 预警列表已加载 | CEO 切换排序维度为"停滞时长"降序 | 列表按停滞时长降序重排，本次会话内保持所选排序，会话结束后恢复默认排序（参见 BR-05） | US-06 |
| AC-05 | CEO 数据权限不覆盖某事业部商机 | CEO 打开预警列表 | 该事业部商机不出现在列表中（参见 BR-02、Epic [BR-07]） | US-01 |
| AC-06 | 某条预警商机存在数据不完整字段（如负责人字段缺失） | 列表加载 | 该条目正常展示，缺失字段以"数据不完整"标记呈现，不影响其他字段展示（参见 BR-07） | US-01 |
