# Feature - [F-05] 竞争威胁商机追踪 (Competitive Threat Opportunity Tracking)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-03/E-03 商机健康度预警 (Opportunity Health)](../README.md)

**业务定义**：
本特性消费 T-02/E-03 提供的竞品威胁信号，将威胁信号与在途商机进行关联，按商机与责任人两个维度展示威胁等级及历史趋势走势，使 CEO 能够在单一视图中全面评估竞品对在途商机群体的实际威胁范围，判断是否需要战略级干预。本特性仅只读消费竞品威胁信号，不自行采集或修改竞品数据。
本特性关注的是"竞品威胁对在途商机的关联呈现与趋势追踪"，不定义竞品信号的采集与评分（上游 T-02/E-03），不定义通知推送机制（F-04），也不定义 CEO 转派操作的执行逻辑（F-03）。

**范围边界**：

1. 包含：竞品威胁信号与在途商机的关联展示、按商机维度和责任人维度的威胁等级分布、单条商机的竞品威胁等级历史趋势走势、竞品威胁追踪视图的筛选与排序。
2. 不包含：竞品信号采集与威胁等级计算（参见 Epic [DEP-03]，由 T-02/E-03 提供，参见 Epic [BR-08]）；主动推送通知（参见 F-04）；处置操作执行（参见 F-03）；商机健康度多维评分（参见 F-01）；预警列表（参见 F-02）。
3. 本特性可独立交付：数据来源（T-02/E-03 竞品信号）、依赖链路（参见 Epic [DEP-03]）与用户场景（场景二）均独立于 F-01~F-04，可作为独立单元单独排期上线。
4. 本特性展示范围须限定在 CEO 数据权限范围内的商机（参见 Epic [BR-07]）。
5. 竞品信号端到端延迟须 ≤ 15 分钟（参见 Epic [NFR-02]、[DEP-03]）；若信号延迟超出，须在视图中标注数据时效状态。

## 业务规则 (Business Rules)

- **[BR-01]信号只读消费**：本特性对 T-02/E-03 提供的竞品威胁信号只读消费，不得修改信号源数据、不得自行建模评估竞品威胁，也不得向 T-02/E-03 写回任何数据。（参见 Epic [BR-08]）
- **[BR-02]关联范围**：仅将竞品威胁信号关联至当前在途商机；已关闭、已赢单、已失单的商机不纳入追踪视图；商机关联准确率依赖上游 T-02/E-03 信号质量，须经业务团队确认 ≥ 70%（参见 Epic [DEP-03]）。
- **[BR-03]威胁等级分级展示**：竞品威胁等级采用 T-02/E-03 定义的等级体系（具体等级名称与边界定义以 T-02/E-03 文档为准），本特性不得自行定义或修改等级边界；等级展示须与上游信号保持严格一致，不得重新诠释等级含义。
- **[BR-04]趋势走势周期**：单条商机的竞品威胁趋势走势须展示最近 90 天的等级变化历史（90 天窗口以自然日计算）；90 天内无威胁信号的商机不展示趋势图，以"无竞品威胁记录"标注；数据不足 90 天时按实际可用周期展示，须注明起始日期。
- **[BR-05]数据时效标注**：竞品威胁信号端到端延迟超过 15 分钟时，视图须标注信号时延状态，注明最后一次信号刷新时间；延迟状态下的数据仍可展示，但 CEO 须能识别数据非最新。（参见 Epic [NFR-02]）
- **[BR-06]数据权限边界**：追踪视图展示范围须严格限定在 CEO 当前数据权限范围内的在途商机；跨事业部商机须独立授权，未授权数据禁止越权展示。（参见 Epic [BR-07]）
- **[BR-07]排序基准**：商机维度列表默认按竞品威胁等级降序排列（高威胁优先）；等级相同时按商机金额降序作为次级排序；责任人维度默认按该负责人名下高威胁商机数量降序排列；CEO 可手动切换排序维度，会话内保持所选排序，会话结束后恢复默认。

## 业务对象定义 (Business Object Definition)

### 竞品威胁商机关联记录

竞品威胁商机关联记录是将 T-02/E-03 竞品威胁信号与具体在途商机进行关联后的业务快照，用于支撑威胁等级展示与趋势追踪，不包含竞品信号原始数据的完整内容。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识本次关联的在途商机 |
| 竞品名称 | 本次威胁信号关联的竞品主体 |
| 威胁等级 | 来自 T-02/E-03 的当前威胁等级结论（等级体系以上游定义为准） |
| 信号来源版本 | 标识本次关联记录使用的竞品信号版本，用于时效追溯 |
| 信号时间 | 上游 T-02/E-03 产生本次威胁信号的时间 |
| 关联时间 | 本特性完成信号与商机关联的时间 |
| 时延标记 | 标识本次关联时端到端信号延迟是否超出 15 分钟 |

### 竞品威胁趋势记录

竞品威胁趋势记录是单条商机的竞品威胁等级历史变化序列，用于支撑 CEO 判断威胁走势与干预时机。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属商机 | 标识该趋势记录归属的在途商机 |
| 记录时间点 | 历史等级快照的时间基准 |
| 威胁等级快照 | 该时间点的竞品威胁等级 |
| 趋势区间说明 | 展示区间的起止日期与数据覆盖状态（完整90天/部分/无记录） |

### 责任人威胁分布摘要

责任人威胁分布摘要是按销售负责人维度聚合的竞品威胁商机概览，用于 CEO 在战略层面快速识别团队级竞品压力分布。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 销售负责人 | 汇总对象的销售人员 |
| 高威胁商机数 | 该负责人名下当前威胁等级最高级别的在途商机数量 |
| 受威胁商机总数 | 该负责人名下存在任意竞品威胁信号的在途商机总数 |
| 最近信号时间 | 该负责人名下最新竞品威胁信号的产生时间 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | CEO 数据权限内存在受竞品威胁的在途商机；T-02/E-03 信号可用 | CEO 打开竞品威胁追踪视图 | 视图按商机维度展示全部受威胁在途商机及其威胁等级，默认按威胁等级降序排列（参见 BR-02、BR-07） | US-01 |
| AC-02 | T-02/E-03 竞品信号不可用 | CEO 打开竞品威胁追踪视图 | 视图展示空状态，明确标注上游数据不可用，不展示任何无信号支撑的商机 | US-01 |
| AC-03 | 竞品威胁追踪视图已加载 | CEO 切换至责任人维度视图 | 视图按销售负责人聚合展示各人名下受威胁商机数，默认按高威胁商机数降序排列（参见 BR-07） | US-02 |
| AC-04 | 某商机存在最近 90 天内的竞品威胁历史记录 | CEO 查看该商机的威胁趋势走势 | 趋势图展示最近 90 天等级变化，起始日期明确标注；数据不足 90 天时展示实际可用周期并注明起始日期；无威胁记录时标注"无竞品威胁记录"（参见 BR-04） | US-03 |
| AC-05 | 竞品信号端到端延迟超过 15 分钟 | CEO 查看竞品威胁追踪视图 | 视图标注信号时延状态及最后信号刷新时间，数据仍可展示（参见 BR-05、Epic [NFR-02]） | US-04 |
| AC-06 | CEO 不具备某事业部商机数据权限 | CEO 查看竞品威胁追踪视图 | 该事业部商机不出现在追踪视图中（参见 BR-06、Epic [BR-07]） | US-01 |
| AC-07 | 竞品威胁等级维持不变（无等级升高事件） | T-02/E-03 信号刷新但等级未变化 | 视图数据同步更新，不触发 F-04 推送通知；趋势走势记录新增一条等级不变的快照 | US-03 |
