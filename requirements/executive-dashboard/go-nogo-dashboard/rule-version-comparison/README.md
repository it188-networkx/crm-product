# Feature Definition - F-02 规则版本效果对比

## 特性概述 (Feature Overview)

**所属 Epic**：T-03/E-04 规则效果验收看板

**业务定义**：

规则版本效果对比特性为 MKT Leader 提供将当前活跃规则版本与星标（历史最优）版本进行 KPI 并排对比的能力，量化规则变更对线索质量分布、跟进成功率、MQL→SQL 转化率等核心指标的实际影响。同时，本特性承担星标版本管理职责：MKT Leader 可将任意已发布的规则版本标记为星标（同一规则类型仅一个星标），建立后续版本对比与 No-Go 回退的基准。CEO 可只读查看对比结果，不具备对比操作与星标管理权限。KPI 数据复用 F-01 所引用的数据源，不独立计算。

**范围边界**：

1. 包含：将当前活跃规则版本与星标版本进行 KPI 并排对比（绝对差值 + 相对差值）；展示星标版本与活跃版本的元数据（版本号、生效日期、变更摘要）；星标版本管理（标记 / 取消）；星标变更操作日志
2. 不包含：
   - 创建、编辑或发布任何规则版本（职责归属 T-01/E-01 F-05 及 T-01/E-05 F-01）
   - 独立获取或重新计算原始 KPI 数据（数据来源职责归属 F-01）
   - 与星标以外的历史版本任意对比（对比基准固定为星标版本）
   - 跨两个以上版本的多版本同时对比
   - `[推导-待确认]` 版本对比结果的导出或归档功能

## 业务规则 (Business Rules)

- **[BR-01]对比基准固定**：KPI 对比的基准方固定为当前星标版本，比较方为当前活跃版本；MKT Leader 不可自由选择任意历史版本进行对比。若尚未设定星标版本，则对比功能不可用，须提示先标记星标。
- **[BR-02]星标唯一性**：参见 Epic [BR-03]。同一规则类型（线索评分规则 / 培育触发规则）在同一时刻仅允许一个版本被标记为星标；新标记时旧星标自动取消。
- **[BR-03]星标可变更**：参见 Epic [BR-04]。MKT Leader 可随时将任意已发布的历史版本或当前活跃版本标记为星标；活跃版本 = 星标版本时属合法状态。
- **[BR-04]版本只读引用**：参见 Epic [BR-01]。对比视图对所引用的规则版本仅具备只读查阅权限，不提供修改规则内容的入口。
- **[BR-05]数据不重算**：参见 Epic [BR-02]。KPI 对比数据直接引用 T-03/E-01 与 T-03/E-02 的计算结果，不独立发起新的数据获取请求。
- **[BR-06]权限区分**：参见 Epic [BR-06]。MKT Leader 可执行对比操作与星标管理；CEO 仅可查看对比结果，无操作权限。
- **[BR-07]星标变更审计**：参见 Epic [BR-09]。每次星标标记或取消操作须写入审计日志，包含操作人、操作时间、规则类型、涉及版本号。

## 业务对象定义 (Business Object Definition)

### 星标版本记录 (Star Version Record)

星标版本记录标识某一规则类型下被 MKT Leader 认定为"历史最优"的版本，是版本对比与 No-Go 回退的基准。同一规则类型同一时刻仅存在一条有效星标记录。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 规则类型 | 星标所属的规则分类：线索评分规则 / 培育触发规则 |
| 星标版本号 | 被标记为星标的规则版本标识，来源于 T-01/E-01 F-05 或 T-01/E-05 F-01 |
| 标记人 | 执行本次星标操作的 MKT Leader 账号 |
| 标记时间 | 本次星标操作的精确时间戳 |
| 前一星标版本号 | 本次标记前该规则类型的旧星标版本标识；首次标记时为空 |

### 版本对比快照 (Version Comparison Snapshot)

版本对比快照是 MKT Leader 在单次对比操作中，系统呈现当前活跃版本与星标版本 KPI 差异的逻辑集合。它并非持久化存储对象，仅在当次查看会话中存在。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 规则类型 | 本次对比涉及的规则分类 |
| 活跃版本号 | 当前生效的规则版本标识（比较方） |
| 星标版本号 | 当前星标版本标识（基准方） |
| KPI 差异列表 | 各核心 KPI 在活跃版本与星标版本下的实际值、绝对差值及相对差值（百分比）的集合 |
| 数据引用期次 | 对比数据所对应的统计期次标识，确保数据来源可溯 |
| 对比触发时间 | 本次对比操作发生的时间戳 |

| :--- | :--- | :--- | :--- | :--- |
| AC-01 | MKT Leader 已登录；当前存在活跃线索评分规则版本；该规则类型已有星标版本；T-03/E-01 和 T-03/E-02 已输出数据 | MKT Leader 进入版本对比视图 | 看板以并排形式展示活跃版本与星标版本的 KPI（线索质量分布、跟进成功率、MQL→SQL 转化率），每项指标同时展示绝对差值与相对差值；加载 ≤ 3 秒（参见 Epic NFR-01） | US-01 |

| AC-04 | MKT Leader 已登录；当前活跃版本 = 当前星标版本 | MKT Leader 进入版本对比视图 | 对比正常展示，所有 KPI 差值为零；`[推导-待确认]` 系统可提示"当前版本与星标版本效果表现一致" | US-02 |
| AC-05 | MKT Leader 已登录；版本列表中存在多个已发布版本 | MKT Leader 浏览版本列表 | 每个版本条目展示版本号、生效日期、变更摘要及星标状态（是否为当前星标），信息来源与 T-01 版本库一致 | US-03 |
| AC-06 | CEO 以只读角色登录；当前存在可用的版本对比结果 | CEO 进入版本对比视图 | CEO 可查看版本对比的全部 KPI 差异数据，但不可执行星标管理操作；页面无星标标记/取消入口 | US-04 |
