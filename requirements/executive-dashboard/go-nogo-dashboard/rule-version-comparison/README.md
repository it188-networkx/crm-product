# Feature Definition - F-02 规则版本切换与对比分析

## 特性概述 (Feature Overview)

**所属 Epic**：T-03/E-04 Go/No-Go 验收看板

**业务定义**：

规则版本切换与对比分析特性允许管理层（以 CEO 为代表）从 T-01 策略版本库中选取不同的历史规则版本，并在 Go/No-Go 验收看板内以并排视图呈现当期版本与所选历史版本之间的 KPI 达成差异。差值计算基于 F-01（当期 KPI 达成总览）所提供的当期数据集，不独立重新获取原始数据。本特性的核心业务价值在于量化规则变更对 KPI 达成率的实际影响，为管理层的策略决策提供可溯源的数字依据。

**范围边界**：

1. 包含：从 T-01/E-01（机会管理策略版本库）与 T-01/E-05（市场策略配置版本库）中读取可用的历史规则版本列表；展示版本元数据（版本号、生效日期、变更摘要）；按所选版本对比当期 KPI 达成数据，以并排视图呈现绝对差值与相对差值；版本列表随 T-01 发布新版本后的自动刷新
2. 不包含：
   - 创建、编辑或发布任何新的规则版本（版本管理职责归属 T-01/E-01 及 T-01/E-05）
   - 独立获取或重新计算原始 KPI 数据（数据来源职责归属 F-01）
   - 跨两个以上版本的多版本同时对比
   - [推导-待确认] 版本对比结果的导出或归档功能

## 业务规则 (Business Rules)

- **[BR-F02-01]版本来源约束**：可供对比选择的规则版本须且仅来自 T-01/E-01（机会管理策略）或 T-01/E-05（市场渠道策略）所产出的已发布版本号；E-04 看板不得自行创建或衍生新版本；参见 Epic 层级规则 [BR-03]

- **[BR-F02-02]差值计算数据源唯一性**：KPI 差值计算须复用 F-01 当期数据集作为唯一数据源，在对比视图内不独立发起新的原始数据获取请求；参见 Epic 层级规则 [BR-02]

- **[BR-F02-03]版本只读引用**：对比视图对所引用的规则版本仅具备只读查阅权限，不提供任何修改规则内容的入口；参见 Epic 层级规则 [BR-03]

- **[BR-F02-04]KPI 目标基准统一**：当期版本与历史版本并排对比时，须使用同一期次的 KPI 目标值作为共同基准计算达成率，不得因版本切换而替换目标基准；参见 Epic 层级规则 [BR-01]

- **[BR-F02-05]权限维度过滤**：版本对比视图中呈现的渠道维度数据须严格依据当前登录角色的权限范围进行过滤，无权限的维度数据不得出现在对比结果中；参见 Epic 层级规则 [BR-05]

- **[BR-F02-06]版本列表实时性**：[推导-待确认] 当 T-01 发布新规则版本后，E-04 看板的可选版本列表须在当次页面加载或主动刷新时同步更新，确保可选版本与上游版本库保持一致

- **[BR-F02-07]维度口径一致性校验**：[推导-待确认] 若当期版本与所选历史版本涉及的 KPI 维度口径不一致（如维度定义发生结构性变更），系统须向用户显示明确的告警提示，说明差值数据的可比性存在局限

## 业务对象定义 (Business Object Definition)

### 规则版本引用 (Rule Version Reference)

规则版本引用是 E-04 看板对 T-01 策略版本库中已发布版本的只读描述载体，记录供管理层在对比分析中识别和选择版本所需的元数据。其内容由 T-01/E-01 或 T-01/E-05 的版本发布动作产生，E-04 不对其进行写操作。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 版本号 | 由上游版本库（T-01/E-01 或 T-01/E-05）分配的唯一标识符，在 E-04 看板中作为版本选择的主键 |
| 版本来源 | 标识该版本归属于机会管理策略（T-01/E-01）还是市场渠道策略（T-01/E-05） |
| 生效日期 | 该版本规则在业务上正式生效的日期，用于帮助管理层对应业务时间轴定位版本 |
| 变更摘要 | 由版本发布方填写的本次变更的简要描述，显示在版本选择器中供选版前快速阅读 |
| 版本状态 | [推导-待确认] 标识该版本当前是否为当期生效版本（当期版本 / 历史版本），用于区分并排视图左右两侧的角色归属 |

### 版本对比报告 (Version Comparison Report)

版本对比报告是管理层在选定历史版本后，系统在单次对比操作范围内生成的 KPI 差值分析结果的逻辑集合。它并非持久化存档对象，而是在当次会话中呈现并排对比所需的结构化业务数据。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 当期版本引用 | 本次对比中作为基准的当期生效规则版本号 |
| 对比版本引用 | 本次对比中由管理层主动选取的历史规则版本号 |
| 数据集引用期次 | 差值计算所依赖的 F-01 当期数据集的所属期次标识，确保数据来源可溯 |
| KPI 指标差值列表 | 各 KPI 指标在当期版本与对比版本下的达成值及绝对差值、相对差值（百分比）的集合 |
| 告警标记 | [推导-待确认] 当维度口径不一致时附加的可比性告警信息，说明特定指标差值的局限性 |
| 对比触发时间戳 | [推导-待确认] 本次对比操作发生的时间，用于会话内的操作追溯 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | CEO 已登录，F-01 当期数据集已就绪，T-01 版本库中存在至少一个已发布的历史版本 | CEO 在版本选择器中选取一个历史规则版本并确认 | 看板切换至并排视图，左侧显示当期版本的 KPI 达成数据，右侧显示所选历史版本对应的 KPI 达成数据；两侧版本号均清晰标注 | US-01 |
| AC-02 | 并排视图已加载当期版本与历史版本数据 | 查看并排视图中任意 KPI 指标行 | 该行同时展示当期版本达成值、历史版本达成值、两者绝对差值（±数值）及相对差值（±百分比）；差值为正时以正向视觉标识区分，差值为负时以负向视觉标识区分 | US-01、US-03 |
| AC-03 | T-01 版本库中不存在任何已发布的历史版本，或当前角色权限下无可见版本 | CEO 打开版本选择器 | 版本选择器显示"暂无可用历史版本"的业务提示，不呈现空列表或异常状态；并排对比功能入口不可触发 | US-01 |
| AC-04 | CEO 已打开版本选择器 | CEO 浏览版本列表中的某一条目 | 该条目显示版本号、版本来源（机会管理策略 / 市场渠道策略）、生效日期及变更摘要，信息完整且与 T-01 版本库记录一致 | US-02 |
| AC-05 | T-01/E-01 或 T-01/E-05 在当次会话内发布了新版本 | CEO 刷新当前 Go/No-Go 看板页面 | 版本选择器中的可选版本列表包含新发布的版本条目，与 T-01 版本库当前状态保持一致 | US-02 |
| AC-06 | 并排视图已加载，[推导-待确认] 当期版本与所选历史版本存在维度口径不一致的 KPI 指标 | 查看并排视图 | 受影响的 KPI 指标行显示可比性告警标记，并附有说明文字提示管理层该指标差值存在口径局限，其他指标行正常显示差值，整体对比视图不被阻断 | US-03 |
