# Feature Definition - F-01 KPI 达成率多维追踪看板

## 特性概述 (Feature Overview)

**所属 Epic**：T-03/E-04 Go/No-Go 验收看板

**业务定义**：
KPI 达成率多维追踪看板为管理层（以 CEO 为核心用户）提供当期经营 KPI 的全景监控能力。看板以"实际值 vs 目标值"为核心呈现结构，按市场、渠道、漏斗阶段三个维度交叉展示各 KPI 的达成率，并依据达成率区间对每个 KPI 项进行分级标注（达成 / 预警 / 未达成），使管理层无需人工汇总即可即时识别偏差最大的维度，为 Go/No-Go 验收决策提供可信的数据基础。该特性聚焦于当期状态监控，不涵盖历史趋势分析。

**范围边界**：
1. 包含：按市场、渠道、漏斗阶段维度展示当期 KPI 实际值与目标值的对比；支持按周 / 月 / 季切换时间粒度；按达成率阈值对 KPI 状态进行分级标注（达成 / 预警 / 未达成）；维度筛选交互（市场、渠道、漏斗阶段的单选或多选过滤）；基于角色权限对可见维度数据进行过滤
2. 不包含：KPI 目标值的录入或修改（目标值配置归属 T-03/E-01 与 T-03/E-02 F-04）；历史期间趋势对比与走势图；原始明细数据的下钻穿透；KPI 数据的自主计算或数据源另立（直接引用 E-01 / E-02 输出结果）；决策结论的标注与归档（属于 E-04 其他特性）

## 业务规则 (Business Rules)

- **[BR-01]KPI目标引用**：参见 Epic [BR-01]。KPI 目标值由管理员预设，须与 T-03/E-01 漏斗指标口径保持一致；各维度不允许在本看板内自定义目标值。

- **[BR-02]数据不重算**：参见 Epic [BR-02]。渠道维度 KPI 数据直接引用 T-03/E-02（渠道 ROI）的计算结果；漏斗各阶段基础数据引用 T-03/E-01；本特性不得重复计算或另立数据源。

- **[BR-03]达成率计算口径**：达成率 = 当期实际值 ÷ 当期目标值 × 100%。当目标值为零或未配置时，该维度 KPI 的达成率显示为"—"（不可用），不参与分级标注。[推导-待确认]

- **[BR-04]KPI达成分级阈值**：依据达成率数值对 KPI 状态进行三级标注：达成率 ≥ 100% 标注为【达成】；达成率在 80%（含）至 100%（不含）之间标注为【预警】；达成率 < 80% 标注为【未达成】。[推导-待确认]

- **[BR-05]权限隔离**：参见 Epic [BR-05]。看板严格按角色权限（GC-06）过滤维度数据，禁止越权查看；受权限限制而无法展示的维度须显示明确提示信息，不得以空白代替。

- **[BR-06]时间粒度与当期定义**：周粒度以自然周（周一至周日）为一期，月粒度以自然月为一期，季粒度以自然季度（Q1–Q4）为一期；"当期"默认为系统当日所在的最近一个完整统计周期（当周 / 当月 / 当季）。[推导-待确认]

- **[BR-07]维度组合完整性**：当某维度（如某渠道）当期无实际数据时，该维度仍须在看板中保留条目展示，实际值显示为 0，达成率正常计算并参与分级标注。[推导-待确认]

## 业务对象定义 (Business Object Definition)

### KPI 达成率条目

KPI 达成率条目是本特性的核心业务对象，代表某一维度（市场 / 渠道 / 漏斗阶段）在特定时间粒度下某项 KPI 指标的当期监控快照。每个条目唯一对应一个"维度 × KPI 指标 × 时间粒度 × 统计周期"的组合，其目标值来源于上游配置，实际值来源于 T-03/E-01 或 T-03/E-02 的计算输出，达成率与分级状态由本特性依据业务规则派生。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 维度类型 | 该条目所属的观测维度，取值为：市场 / 渠道 / 漏斗阶段 |
| 维度值 | 维度的具体取值，例如：华东市场、SEM 渠道、商机阶段 |
| KPI 指标名称 | 被监控的业务指标名称，例如：线索量、ROI、机会转化率 |
| 时间粒度 | 当前统计口径，取值为：周 / 月 / 季 |
| 统计周期 | 当期具体时间范围，例如：2026-W10、2026-03、2026-Q1 |
| 目标值 | 管理员预设的当期 KPI 目标数值，来源于 [DEP-05] |
| 实际值 | 当期 KPI 的实际完成数值，来源于 [DEP-01] 或 [DEP-02] |
| 达成率 | 实际值 ÷ 目标值 × 100%；目标值为零或未配置时显示"—" |
| 达成状态 | 依据 [BR-04] 阈值派生的三级分类：达成 / 预警 / 未达成；目标值缺失时显示"—" |
| 数据刷新时间 | 本条目实际值最近一次从上游同步的时间戳，用于判断数据新鲜度 |

### KPI 维度筛选条件

KPI 维度筛选条件是用户在当次查看会话中所选定的观测范围与时间粒度的组合，决定看板当前所呈现的 KPI 达成率条目集合。筛选条件不持久化保存，刷新后恢复默认状态。[推导-待确认]

| 业务属性 | 业务说明 |
| :--- | :--- |
| 选定时间粒度 | 用户当前选择的统计口径，取值为：周 / 月 / 季，默认为月 [推导-待确认] |
| 选定市场范围 | 用户选定的一个或多个市场维度值；未选定时展示用户权限内全部市场 |
| 选定渠道范围 | 用户选定的一个或多个渠道维度值；未选定时展示用户权限内全部渠道 |
| 选定漏斗阶段范围 | 用户选定的一个或多个漏斗阶段值；未选定时展示用户权限内全部阶段 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 管理员已完成当期 KPI 目标值配置；T-03/E-01 和 T-03/E-02 已输出当期实际值；用户以 CEO 角色登录，拥有全维度查看权限 | 用户进入 KPI 达成率多维追踪看板，保持默认筛选条件（月粒度，全维度） | 看板在 3 秒内完成加载（满足 NFR-01）；展示所有权限内维度的 KPI 条目，每条目包含：维度值、KPI 指标名称、目标值、实际值、达成率（百分比）、达成状态标注（达成 / 预警 / 未达成）；达成率 ≥ 100% 的条目标注【达成】，80%–100% 的标注【预警】，< 80% 的标注【未达成】 | US-01 |
| AC-02 | 同 AC-01 前置条件；其中某漏斗阶段的 KPI 目标值在 [DEP-05] 中未配置 | 用户查看该漏斗阶段的 KPI 条目 | 该条目的目标值字段显示"—"，达成率字段显示"—"，达成状态字段显示"—"，不进行任何分级标注；其他已配置目标值的条目不受影响，正常展示 | US-01 |
| AC-03 | 同 AC-01 前置条件；T-03/E-02 在 E-04 看板上次加载后完成了数据刷新 | 用户在 E-02 数据刷新后的 5 分钟内（满足 NFR-03）重新加载或刷新看板 | 看板渠道维度的 KPI 实际值已更新为 E-02 最新输出值；页面展示的"数据刷新时间"时间戳与 E-02 本次刷新时间一致（或在允许延迟范围内） | US-01 |
| AC-04 | 看板已正常加载（同 AC-01） | 用户将时间粒度从"月"切换为"季" | 看板在 3 秒内完成重新加载；所有 KPI 条目的目标值、实际值、达成率均切换为当季统计口径下的对应数值；分级状态依据新数值重新判定；筛选条件区域显示"季"为当前选中粒度 | US-02 |
| AC-05 | 看板已正常加载（同 AC-01） | 用户在市场维度筛选器中选择"华东市场"，其余维度筛选条件不变 | 看板仅展示华东市场维度下的 KPI 达成率条目；其他市场维度条目从视图中隐藏；渠道与漏斗阶段维度条目不受市场筛选影响，仍按全量展示（或根据市场与渠道/阶段的业务关联关系过滤，以实际业务定义为准）[推导-待确认]；筛选器保持"华东市场"选中状态 | US-02 |
| AC-06 | 当前登录用户的角色权限（GC-06）限定其仅可查看"华东市场"数据，不可查看"华南市场"数据 | 用户进入看板，查看市场维度下的 KPI 条目列表 | "华南市场"相关 KPI 条目不出现在列表中；若用户尝试通过手动修改筛选参数的方式请求华南市场数据，看板显示明确的权限受限提示信息（例如："您无权查看该维度数据"），不展示任何华南市场数据内容 | US-03 |
