# Feature Definition - F-01 规则效果 KPI 追踪看板

## 特性概述 (Feature Overview)

**所属 Epic**：T-03/E-04 规则效果验收看板

**业务定义**：
规则效果 KPI 追踪看板为 MKT Leader 提供当前活跃规则版本下核心 KPI 的实时监控能力。看板按规则类型（线索评分规则 / 培育触发规则）展示三项核心指标：线索质量分布（各分级占比）、跟进成功率、MQL→SQL 转化率，并支持按周/月/季切换时间粒度。MKT Leader 通过看板直观了解当前规则版本的运行效果，为后续版本对比与 Go/No-Go 决策提供数据基础。CEO 以只读角色查看看板，不具备操作权限。

**范围边界**：
1. 包含：按规则类型（评分规则/培育触发规则）展示当前活跃版本下的核心 KPI；支持周/月/季时间粒度切换；展示当前活跃规则版本号与生效日期；基于角色权限区分 MKT Leader 操作权限与 CEO 只读权限
2. 不包含：KPI 目标值的录入或修改；历史期间趋势对比与走势图（属 F-03）；规则版本间的并排对比（属 F-02）；Go/No-Go 决策操作（属 F-04）；KPI 数据的自主计算（直接引用 T-03/E-01 与 T-03/E-02 输出）

| :--- | :--- | :--- | :--- | :--- |
| US-01 | MKT Leader | 查看当前活跃线索评分规则版本下的核心 KPI（线索质量分布、跟进成功率、MQL→SQL 转化率） | 直观了解当前评分规则的运行效果，判断是否需要与历史版本对比或执行 Go/No-Go 决策 | AC-01、AC-02 |

## 业务规则 (Business Rules)

- **[BR-01]数据不重算**：参见 Epic [BR-02]。看板 KPI 数据直接引用 T-03/E-01 与 T-03/E-02 的计算结果，不得重复计算或另立数据源。
- **[BR-02]规则版本标注**：看板须在显著位置展示当前活跃规则版本号与生效日期，使 MKT Leader 明确知道正在观测哪个版本的效果。
- **[BR-03]权限区分**：参见 Epic [BR-06]。MKT Leader 可操作看板（切换视图、切换粒度）；CEO 仅可查看，无操作权限；看板按角色权限过滤数据（参见 Epic [BR-11]）。
- **[BR-04]时间粒度定义**：周粒度以自然周（周一至周日）为一期，月粒度以自然月为一期，季粒度以自然季度（Q1–Q4）为一期；默认展示月粒度。`[推导-待确认]`
- **[BR-05]无数据处理**：当某规则类型在当期无 KPI 数据时，看板须展示"暂无数据"提示，不以零值替代。`[推导-待确认]`

## 业务对象定义 (Business Object Definition)

### 规则效果 KPI 条目

规则效果 KPI 条目是本特性的核心业务对象，代表某一规则类型在特定时间粒度下某项核心 KPI 的当期表现快照。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 规则类型 | 当前条目所属的规则分类：线索评分规则 / 培育触发规则 |
| 活跃版本号 | 当前生效的规则版本标识，来源于 T-01/E-01 F-05 或 T-01/E-05 F-01 |
| 版本生效日期 | 当前活跃版本开始生效的日期 |
| KPI 指标名称 | 被监控的业务指标：线索质量分布 / 跟进成功率 / MQL→SQL 转化率 |
| 时间粒度 | 当前统计口径：周 / 月 / 季 |
| 统计周期 | 当期具体时间范围 |
| 实际值 | 当期 KPI 的实际完成数值，来源于 T-03/E-01 或 T-03/E-02 |
| 数据刷新时间 | 本条目实际值最近一次从上游同步的时间戳 |
