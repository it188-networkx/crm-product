# Feature Definition - [F-04] Go/No-Go 决策与版本回退

## 特性概述 (Feature Overview)

**所属 Epic**：T-03/E-04 规则效果验收看板

**业务定义**：

本特性为 MKT Leader 提供对当前活跃规则版本的正式 Go/No-Go 决策标注能力。MKT Leader 在观察期（14 个自然日）满后，基于 F-01 KPI 数据和 F-02 版本对比结果，对当前活跃的线索评分规则或培育触发规则版本标注"Go（继续采用）"或"No-Go（回退至星标版本）"。Go 决策确认后，MKT Leader 可选择将当前版本标记为新星标；No-Go 决策经二次确认后，系统自动向上游（T-01/E-01 F-05 或 T-01/E-05 F-01）发起版本切换请求，将规则切回至当前星标版本。所有决策与回退操作写入不可篡改审计日志。CEO 可只读查看决策记录，不具备决策权限。

**范围边界**：

1. 包含：MKT Leader 对当前活跃规则版本的 Go/No-Go 标注；观察期（14 天）拦截机制；Go 后可选星标更新；No-Go 二次确认与自动回退至星标版本；决策依据必填；决策审计日志；决策历史记录只读查看
2. 不包含：
   - 规则版本内容的编辑或发布（职责归属 T-01/E-01 F-05 与 T-01/E-05 F-01）
   - KPI 数据的计算或展示（由 F-01 负责）
   - 版本效果并排对比（由 F-02 负责）
   - 通知渠道配置与通知模板管理

## 业务规则 (Business Rules)

- **[BR-01]观察期约束**：参见 Epic [BR-05]。新规则版本生效后须满 14 个自然日（观察期）方可对该版本执行 Go/No-Go 决策；观察期内决策入口不可操作，须提示剩余天数。
- **[BR-02]决策权限**：参见 Epic [BR-06]。仅 MKT Leader 角色可执行 Go/No-Go 决策与回退操作；CEO 角色为只读查看。
- **[BR-03]决策值约束**：每次标注须从"Go（继续采用）"与"No-Go（回退至星标版本）"二值中选择其一，不可为空。
- **[BR-04]决策依据必填**：提交时"决策依据"说明文字为必填项，不可为空白。`[推导-待确认]` 建议最短不低于 10 个字。
- **[BR-05]No-Go 二次确认**：参见 Epic [BR-08]。No-Go 回退操作须经二次确认，明确展示"将回退至星标版本 X"，防止误操作。
- **[BR-06]回退自动执行**：参见 Epic [BR-07]。No-Go 确认后系统须自动向上游（T-01/E-01 F-05 或 T-01/E-05 F-01）发起版本切换请求，将规则切回至当前星标版本；被回退的版本保留在版本历史中，不删除。
- **[BR-07]Go 后星标更新**：Go 决策提交后，系统须提示 MKT Leader 是否将当前版本标记为新星标；此操作为可选，不强制。
- **[BR-08]决策审计留存**：参见 Epic [BR-09]。所有 Go/No-Go 决策、回退操作须实时写入不可篡改审计日志（GC-04），审计条目包含：操作人、操作时间、规则类型、决策值、涉及版本号、决策依据全文。
- **[BR-09]决策记录不可篡改**：决策一经提交即进入已生效状态，不可撤回或修改。如需更新判断，须对后续版本新建决策记录。
- **[BR-10]合规留存**：参见 Epic [BR-10]。决策记录保留 ≥ 36 个月，不可删除。
- **[BR-11]星标前置**：执行 No-Go 回退前，该规则类型须已存在星标版本；若无星标版本则 No-Go 不可操作，须提示先在 F-02 中标记星标。

## 业务对象定义 (Business Object Definition)

### Go/No-Go 决策记录 (Go/No-Go Decision Record)

Go/No-Go 决策记录是 MKT Leader 针对当前活跃规则版本所作出的正式效果验收判断，是本特性的核心业务对象。每次提交产生一条独立的不可变记录，完整保留在审计日志中。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 决策记录编号 | 系统自动生成的唯一标识，用于审计追溯 |
| 规则类型 | 本次决策所针对的规则分类：线索评分规则 / 培育触发规则 |
| 决策对象版本号 | 被评估的当前活跃规则版本标识 |
| 对比基准版本号 | 决策时的星标版本标识（No-Go 回退目标） |
| 决策结论 | Go（继续采用）或 No-Go（回退至星标版本），二值必填 |
| 决策依据 | MKT Leader 填写的文字说明，解释本次决策的业务理由，必填 |
| 提交人 | 执行提交操作的 MKT Leader 账号，系统自动记录 |
| 提交时间 | 系统记录的提交操作精确时间戳 |
| 回退执行状态 | 仅 No-Go 决策适用：回退成功 / 回退失败 `[推导-待确认]`；Go 决策此字段为空 |
| 星标更新标记 | 仅 Go 决策适用：是否在 Go 后将当前版本更新为星标 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | MKT Leader 已登录；当前活跃规则版本已满 14 天观察期；该规则类型已有星标版本 | MKT Leader 选择 Go，填写决策依据，点击提交 | 决策记录成功写入审计日志，包含规则类型、版本号、决策结论、依据全文、提交人、时间；页面展示提交成功反馈 | US-01 |
| AC-02 | MKT Leader 已登录；当前活跃规则版本生效不满 14 天 | MKT Leader 尝试执行 Go/No-Go 决策 | 决策入口不可操作，系统提示"观察期未满，剩余 X 天"（参见 BR-01） | US-01 |
| AC-03 | MKT Leader 已选择决策结论 | MKT Leader 将决策依据留空，点击提交 | 系统阻止提交，提示"决策依据为必填项"（参见 BR-04） | US-01 |
| AC-04 | MKT Leader 已登录；当前活跃规则版本已满观察期；该规则类型已有星标版本 | MKT Leader 选择 No-Go，填写依据，点击提交 | 系统弹出二次确认，明确展示"将回退至星标版本 X"（参见 BR-05） | US-02 |
| AC-05 | MKT Leader 在二次确认窗口中确认回退 | 确认回退 | 系统自动向上游发起版本切换请求，规则版本切换为星标版本；决策记录（含回退执行状态）写入审计日志；被回退的版本保留在版本历史中 | US-02 |
| AC-06 | MKT Leader 刚完成 Go 决策提交 | 系统提示是否将当前版本标记为新星标 | MKT Leader 选择"是"→当前版本成为星标，旧星标自动取消，星标变更写入审计日志；选择"否"→星标不变 | US-03 |
| AC-07 | MKT Leader 已登录；当前规则类型已存在若干历史决策记录 | MKT Leader 查看决策历史列表 | 列表完整展示该规则类型所有已提交决策，每条包含：版本号、决策结论、依据、提交时间；所有记录只读 | US-04 |
| AC-08 | CEO 以只读角色登录 | CEO 查看 Go/No-Go 决策记录 | CEO 可查看全部决策记录与审计日志，但不可执行任何决策操作；页面无决策入口 | US-05 |
| AC-09 | MKT Leader 已登录；该规则类型尚未设定星标版本 | MKT Leader 尝试执行 No-Go 决策 | 系统提示"请先在版本对比中标记星标版本"，No-Go 不可操作（参见 BR-11） | US-02 |
