# Feature Definition - F-03 历史规则效果趋势分析

## 特性概述 (Feature Overview)

**所属 Epic**：T-03/E-04 规则效果验收看板

**业务定义**：

历史规则效果趋势分析特性为 MKT Leader 提供对任意已完结历史时段内核心 KPI（线索质量分布、跟进成功率、MQL→SQL 转化率）走势的回溯能力。MKT Leader 可指定查询时段与时间粒度，系统沿时间轴呈现 KPI 走势，并在时间轴上标注各规则版本切换的时间节点，展示各版本存续区间及其对应的 KPI 表现。通过将 KPI 变化趋势与规则版本切换动作对照，MKT Leader 可进行事后归因分析，判断效果差异是持续趋势还是短期波动，为 Go/No-Go 决策提供历史参照。CEO 可只读查看趋势分析结果。本特性依托 T-03/E-01 与 T-03/E-02 的历史数据，以及 T-01/E-01 F-05 与 T-01/E-05 F-01 的版本时间轴，不独立计算任何 KPI 数值。

**范围边界**：

1. 包含：按指定历史时段（36 个月数据保留范围内）呈现 KPI 走势；支持周 / 月 / 季三档时间粒度切换；在趋势时间轴上标注规则版本切换时间节点（线索评分规则与培育触发规则）；展示各规则版本存续区间及其 KPI 表现；MKT Leader 可操作，CEO 只读
2. 不包含：
   - 当期进行中周期的 KPI 实时监控（属 F-01 职责范围）
   - 历史 KPI 数据的自主计算或另立数据源（须引用上游历史记录，参见 Epic [BR-02]）
   - 规则版本并排对比（属 F-02 职责范围）
   - 历史数据的导出或归档功能 `[推导-待确认]`

## 业务规则 (Business Rules)

- **[BR-01]历史时段约束**：可查询的历史时段须为当前日期往前不超过 36 个月的已完结周期（参见 Epic [BR-10]）；进行中的统计周期不得纳入本特性范围。`[推导-待确认]` 进入回溯视图时默认展示最近一个已完结自然月的周粒度走势。
- **[BR-02]数据引用约束**：参见 Epic [BR-02]。历史 KPI 数据须且仅引用 T-03/E-01 与 T-03/E-02 的历史输出记录；本特性不得另立数据源或对历史数据执行重新计算。
- **[BR-03]版本时间轴来源约束**：参见 Epic [BR-01]。趋势时间轴上的规则版本切换节点须且仅引用 T-01/E-01 F-05（线索评分规则版本库）与 T-01/E-05 F-01（培育触发规则版本库）的版本发布记录；本特性不得自行维护版本时间信息。
- **[BR-04]时间粒度定义**：周粒度以自然周（周一至周日）为一期，月粒度以自然月为一期，季粒度以自然季度（Q1–Q4）为一期；默认展示周粒度。`[推导-待确认]`
- **[BR-05]权限区分**：参见 Epic [BR-06]。MKT Leader 可操作趋势分析（指定时段、切换粒度）；CEO 仅可查看，无操作权限。
- **[BR-06]数据留存边界**：参见 Epic [BR-10]。当所选时段部分超出 36 个月留存边界时，边界内数据正常呈现，超出部分须以明确的不可用提示标注。

## 业务对象定义 (Business Object Definition)

### 历史趋势分析视图 (Historical Trend Analysis View)

历史趋势分析视图是 MKT Leader 在单次回溯查询中，系统依据所选时段与时间粒度呈现的 KPI 走势与规则版本切换节点的逻辑集合。它并非持久化对象，仅在当次查询会话中存在。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 规则类型 | 本次趋势分析所针对的规则分类：线索评分规则 / 培育触发规则 |
| 查询时段起止 | MKT Leader 指定的回溯起始日期与终止日期，终止日期须早于系统当日 |
| 时间粒度 | 本次查询选定的走势展示颗粒度：周 / 月 / 季 |
| KPI 数据点序列 | 在所选时段与粒度下，各核心 KPI 按时间单元排列的数值序列；来源于 T-03/E-01 和 T-03/E-02 的历史记录 |
| 版本切换节点列表 | 在所选时段内，来自 T-01/E-01 F-05 和 T-01/E-05 F-01 的版本切换记录，包含切换日期、切换前后版本号 |
| 版本存续区间 | 每个规则版本在所选时段内的实际生效起止日期及其对应 KPI 表现 |

### 规则版本切换节点 (Rule Version Switch Node)

规则版本切换节点是趋势时间轴上对一次规则版本切换事件的只读标注，来源于 T-01/E-01 F-05 或 T-01/E-05 F-01 的版本发布记录。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 切换日期 | 规则版本切换正式生效的日期，对应趋势时间轴上的标注位置 |
| 规则类型 | 线索评分规则（来源 T-01/E-01 F-05） / 培育触发规则（来源 T-01/E-05 F-01） |
| 切换前版本号 | 本次切换前生效的规则版本标识 |
| 切换后版本号 | 本次切换后新生效的规则版本标识 |
| 版本存续区间 | 切换后版本的生效起止日期；若该版本在查询时段终止前仍为活跃版本，终止日期对应查询时段终止日期 |
| 变更摘要 | `[推导-待确认]` 由 T-01 版本发布方填写的本次规则调整说明 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | MKT Leader 已登录；T-03/E-01 和 T-03/E-02 已提供所选时段内的历史数据；F-01 已上线；所选时段为 36 个月内的已完结周期 | MKT Leader 指定一个历史月份（如 2026-01）并以周粒度提交查询 | 系统呈现该月内各周 KPI 走势，数据点与上游历史记录一致；加载 ≤ 3 秒（参见 Epic NFR-01） | US-01 |
| AC-02 | 同 AC-01；所选时段内上游对某周历史数据无记录 | MKT Leader 查询包含数据空缺周的时段 | 有效周的数据正常展示；空缺周以"数据不可用"标注呈现，不以零值替代；其他数据点不受影响 | US-01 |
| AC-03 | MKT Leader 已登录；所选时段内存在至少一次规则版本切换记录 | MKT Leader 完成趋势查询后查看时间轴 | 时间轴上在切换日期位置显示节点标注，包含切换日期、规则类型、切换前后版本号；节点数量与上游版本记录一致 | US-02 |
| AC-04 | 同 AC-03；所选时段内包含一个完整的版本存续区间 | MKT Leader 查看某版本存续区间 | 系统展示该版本的生效起止日期及区间内各核心 KPI 表现，数值来源于同一趋势查询结果 | US-02、US-03 |
| AC-05 | MKT Leader 已登录；所选时段内无任何版本切换记录 | MKT Leader 查看时间轴 | 时间轴上不显示任何版本切换节点；`[推导-待确认]` 可提示"所选时段内无规则版本切换记录"；KPI 走势正常展示 | US-03 |
| AC-06 | MKT Leader 已完成一次月粒度趋势查询 | MKT Leader 切换为周粒度 | KPI 数据点重新按周统计单元组织；版本切换节点标注位置保持不变；刷新 ≤ 3 秒 | US-01 |
| AC-07 | CEO 以只读角色登录 | CEO 进入历史趋势分析视图 | CEO 可查看全部趋势数据与版本切换节点，但不可操作时段指定、粒度切换等交互功能；页面无分析操作入口 `[推导-待确认]` | US-04 |
