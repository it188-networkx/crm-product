# Feature Definition - F-03 历史回溯与趋势分析

## 特性概述 (Feature Overview)

**所属 Epic**：T-03/E-04 Go/No-Go 验收看板

**业务定义**：
历史回溯与趋势分析特性为管理层（以 CEO 为核心用户）提供对任意已完结历史时段 KPI 达成率走势的回溯能力。管理层可自行指定查询时段与时间粒度，系统沿所选时间轴呈现 KPI 达成率的演变曲线，并在时间轴上标注各策略调整发生的时间节点；同时展示各规则版本存续区间及其对应的 KPI 表现，使管理层能够将 KPI 走势变化与具体策略调整动作相对照，支持事后归因分析与周期规律总结。本特性依托 T-03/E-01（漏斗 KPI 历史数据）与 T-03/E-02（渠道 ROI 历史数据）提供的历史记录，以及 T-01/E-01 与 T-01/E-05 提供的策略版本时间轴，不独立重新计算任何 KPI 数值。本特性聚焦历史已完结周期的趋势洞察，与 F-01 所覆盖的当期实时监控在业务职责上严格区分，须以 F-01 正式上线为集成前提。

**范围边界**：
1. 包含：按管理层指定的历史时段（在 36 个月数据保留范围内）呈现 KPI 达成率走势；支持日 / 周 / 月 / 季四档时间粒度的自由切换；在趋势时间轴上标注策略调整时间节点（机会管理策略与市场渠道策略版本的切换时间点）；呈现各规则版本存续区间及区间内对应的 KPI 表现；依据角色权限对历史数据的可见范围进行过滤
2. 不包含：当期进行中周期的 KPI 实时监控（属 F-01 职责范围）；历史 KPI 数据的自主计算或另立数据源（须引用上游历史记录，参见 Epic 层级规则 [BR-02]）；规则版本内容的详细查阅与版本间并排对比（属 F-02 职责范围）；历史 KPI 目标值的新增或修改；[推导-待确认] 历史趋势数据的导出或归档功能

## 业务规则 (Business Rules)

- **[BR-F03-01]历史时段选择约束**：可查询的历史时段须为当前日期往前不超过 36 个月的已完结周期；用户所选时段的起止日期不得包含当前仍在进行中的统计周期；[推导-待确认] 进入回溯视图时默认展示最近一个已完结自然季度的月粒度走势

- **[BR-F03-02]数据引用约束**：参见 Epic 层级规则 [BR-02]。历史漏斗 KPI 数据须且仅引用 T-03/E-01（[DEP-01]）的历史输出记录；历史渠道维度 KPI 数据须且仅引用 T-03/E-02（[DEP-02]）的历史输出记录；本特性不得另立数据源或对历史数据执行任何重新计算操作

- **[BR-F03-03]仅面向已完结历史时段**：历史回溯视图严格限定于已完结统计周期（即时段终止日期早于系统当日）；当期正在进行中的统计周期的数据不得纳入本特性的呈现范围，当期监控职责由 F-01 承担

- **[BR-F03-04]策略版本时间轴来源约束**：趋势时间轴上的策略调整时间节点须且仅引用 T-01/E-01（[DEP-03]）发布的机会管理策略版本时间轴与 T-01/E-05（[DEP-04]）发布的市场渠道策略版本时间轴；本特性不得在 E-04 内部自行维护或调整任何策略版本的时间信息

- **[BR-F03-05]时间粒度与走势数据点对齐**：趋势时间轴上每个数据点所代表的统计期次须与所选时间粒度严格对齐：日粒度对应自然日，周粒度对应自然周（周一至周日），月粒度对应自然月，季粒度对应自然季度（Q1–Q4）；[推导-待确认] 当所选时段跨度与所选粒度组合导致数据点数量超过合理上限时，系统须提示用户调整粒度或缩短时段

- **[BR-F03-06]权限隔离**：参见 Epic 层级规则 [BR-05]。历史回溯视图中可见的市场、渠道、漏斗阶段维度数据范围，须严格依据当前登录角色的权限配置（GC-06）进行过滤；受权限限制而不可见的维度须有明确提示，不得以空白或数据缺失状态呈现

- **[BR-F03-07]历史数据留存边界**：参见 Epic 层级规则 [BR-06]。KPI 历史数据保留周期不少于 36 个月且不可删除；当用户所选时段的部分范围超出留存边界时，边界内的数据正常呈现，超出部分须以明确的不可用提示标注，不得影响边界内已有数据的正常展示

## 业务对象定义 (Business Object Definition)

### 历史趋势分析报告 (Historical Trend Analysis Report)

历史趋势分析报告是管理层在单次回溯操作范围内，系统依据所选时段与时间粒度生成的 KPI 达成率走势的逻辑集合。它并非持久化归档对象，而是在当次查询会话中，对上游 T-03/E-01 和 T-03/E-02 历史记录的只读聚合呈现。每份报告唯一对应一个"查询时段 × 时间粒度 × 权限可见范围"的组合，其核心价值在于将历史 KPI 走势与策略调整时间节点在同一时间轴上对齐，供管理层进行事后归因分析。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 查询时段起止 | 管理层指定的历史回溯起始日期与终止日期，终止日期须早于系统当日，且两者之差不超过 36 个月数据留存边界 |
| 时间粒度 | 本次查询所选定的走势展示颗粒度，取值为：日 / 周 / 月 / 季 |
| 覆盖 KPI 指标范围 | 本次报告纳入走势展示的 KPI 指标集合，与当前登录角色的权限可见范围一致 |
| KPI 达成率数据点序列 | 在所选时段与粒度下，各 KPI 指标按时间单元排列的达成率数值序列；数据点来源于 [DEP-01] 和 [DEP-02] 的历史记录 |
| 关联策略调整节点数量 | 在所选时段内，来自 T-01/E-01 和 T-01/E-05 版本时间轴的策略调整时间节点总数，[推导-待确认] 若数量为零则在时间轴上不显示任何标注 |
| 数据留存边界说明 | [推导-待确认] 若所选时段部分超出 36 个月留存边界，本属性记录边界截断位置及不可用范围的起始日期，供管理层明确知晓数据完整性状态 |
| 权限可见范围 | 本次报告依据角色权限（GC-06）过滤后实际可见的维度范围，说明哪些市场、渠道或漏斗阶段数据被纳入展示 |

### 策略调整时间节点 (Strategy Adjustment Time Node)

策略调整时间节点是趋势时间轴上对一次策略版本切换事件的只读业务标注，其内容来源于 T-01/E-01 或 T-01/E-05 的版本发布记录，E-04 不对其进行任何写操作。每个节点代表一个策略版本从旧版本切换至新版本的确切时间，以及新版本在历史时段内持续生效的区间范围。通过将多个节点按时间顺序排列在趋势时间轴上，管理层可直观观察每一段策略存续区间内 KPI 走势的表现，从而支持因果归因推断。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 节点日期 | 策略版本切换正式生效的日期，对应趋势时间轴上的标注位置；来源于上游版本库的版本生效日期字段 |
| 策略类型 | 该调整所属的策略体系，取值为：机会管理策略（来源 T-01/E-01）/ 市场渠道策略（来源 T-01/E-05）|
| 切换前版本号 | 本次切换前生效的策略版本标识，来源于对应上游版本库 |
| 切换后版本号 | 本次切换后新生效的策略版本标识，即本节点所代表的版本 |
| 版本存续区间起止 | 切换后版本的实际生效起始日期，以及该版本被下一个版本取代前最后一个生效日期；若该版本在所选查询时段终止前仍为生效版本，则终止日期对应查询时段终止日期 [推导-待确认] |
| 变更摘要 | [推导-待确认] 由 T-01 版本发布方填写的本次策略调整的业务描述，在趋势时间轴上作为节点悬浮说明呈现，供管理层快速了解调整背景 |

| AC-04 | 同 AC-03；所选时段内包含一个完整的策略版本存续区间（即某版本的生效日期与被取代日期均在时段内） | CEO 选取时间轴上任意一个策略调整时间节点 | 系统呈现该节点对应版本的完整存续区间（起止日期）；同时关联展示该存续区间内对应 KPI 指标的达成率走势数据，数值来源于同一份历史趋势分析报告，不单独发起新的数据获取 | US-02、US-03 |
| AC-05 | CEO 已登录；所选历史时段内 T-01/E-01 与 T-01/E-05 均无任何版本切换记录 | CEO 完成历史趋势查询后查看趋势时间轴 | 时间轴上不显示任何策略调整时间节点标注；[推导-待确认] 时间轴下方显示"所选时段内无策略调整记录"的提示信息；KPI 达成率走势本身正常展示，不受影响 | US-03 |
| AC-06 | CEO 已完成一次历史趋势查询，当前以月粒度展示所选时段的走势 | CEO 将时间粒度从月切换至周 | 系统以周为统计单元重新组织同一时段的历史 KPI 达成率数据，走势数据点数量从月粒度扩展至对应的周粒度数量；策略调整时间节点标注位置保持不变，仍精确对应版本切换日期在新粒度下的时间轴坐标；刷新完成时长不超过 3 秒（参见 Epic 层级规则 [NFR-01]） | US-04 |
| AC-07 | CEO 已登录；所选历史时段的起始日期早于系统当日往前 36 个月的留存边界（即部分时段超出可用数据范围） | CEO 提交包含超出留存边界日期的历史时段查询 | 位于 36 个月留存边界之内的时段数据正常呈现；超出边界的时段在时间轴上以明确的"数据已超出保留期限"标注显示，不展示任何数值；整体查询不报错，可用数据段不受影响（参见 Epic 层级规则 [BR-06]） | US-01 |
