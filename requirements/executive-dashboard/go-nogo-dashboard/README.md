> 关联 Issue：[#42](https://github.com/it188-networkx/crm-product/issues/42)

---

## 1. 业务背景

### As-Is（当前状态）

- CEO/管理层对各市场/渠道/阶段的 KPI 达成情况缺乏统一视图，只能靠人工汇总 Excel 判断策略是否奏效
- 策略调整无历史版本追踪，无法回溯某次规则修改对实际业绩的影响，"Go 还是 No-Go"决策缺乏数据支撑
- Q/N 决策通常在季度末才触发，错过最佳调整时机

### To-Be（目标状态）

- 提供按市场/渠道/阶段维度的 KPI 达成率实时看板，辅助 CEO 在周/月频率做 Go/No-Go 决策
- 支持规则版本切换与并排对比，量化评估不同策略参数对 KPI 的影响
- 内置历史回溯功能，可复盘任意时段策略执行效果，为下期规划提供依据

> [缺少需求输入-待补充]：KPI 阈值（如"渠道 ROI > X 则 Go"）需与业务战略目标对齐，须与 VP 销售/CFO 访谈确认

## 2. 核心 Feature 拆分

| Feature ID | Feature 名称 | 优先级 | 简述 |
|---|---|---|---|
| F-01 | KPI 达成率多维追踪看板 | P1 | 按市场/渠道/阶段展示 KPI 实际值 vs 目标值，支持周/月/季切换，红黄绿色温标注达成状态 |
| F-02 | 规则版本切换与对比分析 | P1 | 支持选择不同策略规则版本（来自 T-01 E-01/E-05）并排对比 KPI 影响，量化规则变更收益 |
| F-03 | 历史回溯与趋势分析 | P1 | 支持任意历史时段回溯，展示 KPI 达成率走势，标注重大策略调整时间节点 |

## 3. 业务规则约束（Business Rules）

| 规则 ID | 描述 |
|---|---|
| BR-01 | KPI 目标值由管理员在设置页预设，须与 T-03/E-01 漏斗指标口径保持一致，不允许各维度自定义 |
| BR-02 | 规则版本切换须来自 T-01/E-01 或 T-01/E-05 产生的版本号，E-04 不得创建新规则版本，只读引用 |
| BR-03 | 渠道维度 KPI 数据依赖 E-02（渠道 ROI）已计算的结果，E-04 不重复计算 |
| BR-04 | 所有 CEO 的 Go/No-Go 决策操作（标注/提交）须写入审计日志（GC-04），纳入操作追踪 |
| BR-05 | 历史回溯权限受 GC-02 约束，不同角色可查看的历史数据范围由权限配置决定 |
| BR-06 | GC-06（合规留存）：KPI 历史数据及决策记录保留 ≥ 36 个月，不可删除 |

- **GC-06** 合规留存：KPI 历史数据与决策记录保留 ≥ 36 个月
