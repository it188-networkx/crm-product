# Feature Definition - F-04 ROI 公式配置管理 (ROI Formula Configuration Management)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-02] 渠道 ROI 对比分析 (Channel ROI)](../README.md)

**业务定义**：

该特性用于为管理员提供 ROI 计算口径的统一配置管理能力，包括归因模型、收益口径、成本口径、归因时间窗与异常值处理规则的创建、审批与版本管理。它关注的是"ROI 计算口径由谁定义、如何生效、历史口径如何追溯对比"，确保 F-01、F-02、F-03 展示的 ROI 数据始终基于统一且可审计的计算基准，不涉及渠道数据展示或内容归因分析本身。

**范围边界**：

1. 包含：
   - ROI 计算口径的创建，涵盖归因模型（最后触点 / 多触点 / 时间衰减）、收益口径（商机预估价值 / 实际成交金额）、成本口径（渠道总投入 / 单线索成本）、归因时间窗与异常值处理规则
   - 口径配置的审批流程，新建或修改的口径须经审批后方可生效
   - 口径配置的版本管理，每次变更产生新版本，旧版本保留供历史对比
   - 历史版本对比，管理员可并排查看不同版本口径的差异
   - 配置权限控制，仅限 CEO/管理层、市场负责人与 CRM 产品设计负责人操作
2. 不包含：
   - 渠道 ROI 数据的展示与渠道间对比（属于 F-01 渠道 ROI 总览与对比）
   - 内容主题维度的归因分析（属于 F-02 内容主题归因分析）
   - 渠道线索明细与商机明细查阅（属于 F-03 渠道投入产出下钻）
   - ROI 数据本身的计算执行逻辑（计算引擎读取本特性的生效口径后执行，不在本特性范围内定义）

## 业务规则 (Business Rules)

- **[BR-01]口径统配**：参见 Epic [BR-02]——渠道 ROI 的归因模型、收益口径与成本口径须通过本特性由管理员统一配置，不得由数据分析人员在前置处理中自行修改。
- **[BR-02]权限分级**：参见 Epic [BR-04]——口径配置的创建与修改权限仅限 CEO/管理层、市场负责人与 CRM 产品设计负责人；销售人员不具备查看或修改口径配置的权限。
- **[BR-03]口径变更协同**：参见 Epic [BR-03]——本特性的口径一经确认，不得在 E-04 Go/No-Go 看板上线后单独调整，须协同变更。
- **[BR-04]版本不可篡改**：口径配置的历史版本一经生成，不得被修改或删除；修改操作须产生新版本，旧版本保留完整记录供审计与对比使用（参见 Epic [NFR-02]）。
- **[BR-05]审批生效**：新建或修改的口径配置须经审批流程后方可成为生效口径；审批未通过的配置草稿不得被 F-01、F-02、F-03 读取使用。[推导-待确认]
- **[BR-06]单一生效口径**：同一时间点仅允许一套口径配置处于生效状态；新口径审批通过后自动替代旧口径生效，旧口径转为历史版本。[推导-待确认]

## 业务对象定义 (Business Object Definition)

### ROI 口径配置

ROI 口径配置是一套完整的 ROI 计算基准参数集合，定义了 F-01、F-02、F-03 展示数据所依赖的计算规则，是本特性管理的核心业务对象。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 口径版本号 | 标识该配置的唯一版本编号，自动生成，不可修改 |
| 归因模型配置 | 定义允许使用的归因模型集合（最后触点 / 多触点 / 时间衰减）及各模型的权重或衰减参数 |
| 收益口径定义 | 定义商机总价值的统计口径（如使用商机预估价值或实际成交金额） |
| 成本口径定义 | 定义渠道线索成本的统计口径（如使用渠道总投入或单线索成本） |
| 归因时间窗 | 定义归因计算的时间回溯窗口（如线索触达后多长时间内的成交可归因至该渠道） |
| 异常值处理规则 | 定义异常数据（如极端高值商机、疑似重复线索）的处理策略 |
| 创建人 | 记录该配置的创建者 |
| 创建时间 | 记录该配置的创建时间 |
| 审批状态 | 标识该配置当前的审批状态（草稿 / 待审批 / 已审批生效 / 已驳回） |
| 审批人 | 记录审批操作的执行者 |
| 审批时间 | 记录审批完成的时间 |
| 生效状态 | 标识该配置是否为当前生效口径（生效 / 历史版本） |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 当前用户为 CEO 或 CRM 产品设计负责人 | 创建一套新的 ROI 口径配置（填写归因模型、收益口径、成本口径、归因时间窗、异常值处理规则） | 配置保存为草稿状态，可提交审批；配置内容完整性校验通过 | US-01, US-05 |
| AC-02 | 已存在一套草稿状态的口径配置 | 提交审批 | 配置状态变更为"待审批"；审批通过后状态变更为"已审批生效"，原生效口径自动转为历史版本（参见 BR-06）；审批未通过的配置保留为"已驳回"草稿 | US-02 |
| AC-03 | 系统中存在至少 2 个历史版本的口径配置 | 查看口径配置版本列表 | 展示所有版本的口径配置，按创建时间倒序排列，每条记录展示版本号、创建人、审批状态与生效状态 | US-03 |
| AC-04 | 口径配置版本列表已打开，选择 2 个版本 | 发起版本对比 | 并排展示两个版本的归因模型、收益口径、成本口径、归因时间窗与异常值处理规则的差异；有变更的字段被显著标注 | US-03 |
| AC-05 | 当前用户为市场负责人 | 查看当前生效口径配置 | 展示当前生效口径的完整配置详情（只读）；不提供创建或修改入口 | US-04 |
| AC-06 | 当前用户为销售人员 | 尝试访问口径配置管理页面 | 页面不可访问，无入口（参见 BR-02） | US-06 |
| AC-07 | 已有一套新口径配置审批通过并生效 | 打开 F-01 渠道 ROI 总览视图 | F-01 展示的 ROI 数据基于新生效口径计算；口径版本号与新配置的版本号一致 | US-07 |
