# Feature Definition - F-02 内容主题归因分析 (Content Topic Attribution Analysis)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-02] 渠道 ROI 对比分析 (Channel ROI)](../README.md)

**业务定义**：

该特性用于追踪各内容主题从内容曝光到最终成交的全链路转化路径，将内容主题的业务贡献量化为可按渠道、负责人与时间窗口对比的数据，并以漏斗形式可视化各环节流失率。它关注的是"哪类内容主题对成交贡献最高、高贡献主题与低效主题如何区分、内容与渠道的交叉归因结论是否一致"，不涉及单条线索或单个商机的明细查阅，也不定义贡献率口径本身的配置管理。

**范围边界**：

1. 包含：
   - 各内容主题按"曝光→点击→留资→线索→商机→成交"六阶段漏斗链路的转化率与贡献量展示
   - 按渠道、负责人、时间窗口筛选内容主题归因数据
   - 从 F-01 渠道 ROI 总览视图的某一渠道切入，查看该渠道下各内容主题的归因漏斗（承接 Epic 场景一步骤 3 的跨特性联动路径）
   - 高贡献率主题标注（贡献率显著高于均值）与长尾低效主题识别
   - 内容主题贡献率的计算，遵循 Epic [BR-06] 口径（主题线索贡献率 = 主题线索数 / 全部线索数；主题成交贡献率 = 主题成交金额 / 全部成交金额）
2. 不包含：
   - 单条线索或商机的明细数据查阅（属于 F-03 渠道投入产出下钻）
   - 渠道维度的 ROI 总览对比（属于 F-01 渠道 ROI 总览与对比）
   - 内容贡献率口径的配置管理（属于 F-04 ROI 公式配置管理，P1 范围外）
   - 参见 Epic [DEP-02]：成交端数据须与 E-01 销售漏斗的漏斗成单状态保持同源，不得独立统计成交量
   - 参见 Epic [ASM-02]：若特定渠道的曝光量与点击量数据不可从外部媒介平台获取，对应渠道的转化链路仅从留资环节起算，并在视图中标注数据起点

## 业务规则 (Business Rules)

- **[BR-01]链路一致**：参见 Epic [BR-01]——本特性使用的转化链路（曝光→点击→留资→线索→商机→成交）中，线索、商机、成交阶段的定义须与 E-01 销售漏斗的阶段定义保持一致；同一商机在本特性与 E-01 中的转化状态数字必须一致，不得出现两套转化链路。
- **[BR-02]口径变更协同**：参见 Epic [BR-03]——内容主题归因使用的渠道 ROI 数据被 E-04 Go/No-Go 看板引用；本特性指标口径变更须与 E-04 协同，不得单独调整。
- **[BR-03]权限分级**：参见 Epic [BR-04]——CEO 与管理层可查看全渠道全主题的归因分析视图；市场负责人可查看其职责范围渠道的主题归因数据；销售人员不具备访问权限。
- **[BR-04]贡献率口径**：参见 Epic [BR-06]——内容主题线索贡献率 = 主题线索数 / 全部线索数；内容主题成交贡献率 = 主题成交金额 / 全部成交金额；两项口径须与 F-01 渠道统计数据保持同源，不得独立定义统计范围。
- **[BR-05]漏斗同源约束**：本特性成交端数据（商机→成交阶段）须从 E-01 销售漏斗的成单状态读取，不得在本特性内独立维护成交数量或成交金额（参见 Epic [DEP-02]）。
- **[BR-06]高贡献标注阈值**：内容主题的贡献率须高于当前筛选范围内所有主题平均贡献率的一定倍数方可被标注为"高贡献主题"；具体倍数阈值待定（TBD，责任归属：市场负责人确认），在阈值确认前默认使用 1.5 倍均值作为分流基准，并在视图中标注"阈值待确认"。[推导-待确认]
- **[BR-07]曝光数据缺失降级**：若某渠道的曝光量或点击量数据不可获取，该渠道对应主题的漏斗链路仅从留资环节起算；视图须标注该渠道的数据起点为"留资"，不得将该渠道纳入曝光→点击环节的转化率统计，但仍可参与留资及后续环节的贡献率计算。
- **[BR-08]主题归属唯一性**：同一线索只能归属于一个内容主题；若一条线索关联多个内容触点，须依据当前生效的归因模型口径（由 F-04 配置）确定最终归属主题，不得在本特性中自行拆分分摊。[推导-待确认]
- **[BR-09]低效主题标注阈值**：内容主题的贡献率低于当前筛选范围内所有主题平均贡献率的一定比例时，须被标注为“低效主题”；具体比例阈值待定（TBD，责任归属：市场负责人确认），在阈值确认前默认使用 0.5 倍均值作为分流基准，并在视图中标注“阈值待确认”。[推导-待确认]

## 业务对象定义 (Business Object Definition)

### 内容主题归因记录

内容主题归因记录是某一内容主题在指定筛选条件（渠道、负责人、时间窗口）下的全链路转化数据汇总，是本特性漏斗可视化的最小数据单元。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 内容主题标识 | 标识该记录所属的内容主题分类（如技术白皮书、行业案例、产品演示等） |
| 归属渠道 | 标识该主题在哪个渠道下的贡献数据，用于支持渠道维度筛选 |
| 时间窗口 | 标识该记录所覆盖的统计周期 |
| 曝光量 | 该主题在时间窗口内获得的内容曝光次数；数据来源于外部媒介平台，缺失时标注数据起点 |
| 点击量 | 该主题在时间窗口内获得的内容点击次数；数据来源于外部媒介平台，缺失时标注数据起点 |
| 留资量 | 通过该主题触达后完成留资行为的用户数 |
| 线索量 | 该主题归因的有效线索总数，须与 F-01 同源统计 |
| 商机量 | 该主题线索进入商机阶段的数量，须与 E-01 漏斗阶段保持一致 |
| 成交量 | 该主题关联的最终成交数，须从 E-01 成单状态读取 |
| 成交金额贡献 | 该主题归因的成交金额之和，用于计算主题成交贡献率 |
| 线索贡献率 | 主题线索数 / 全部线索数，按 BR-04 口径计算 |
| 成交贡献率 | 主题成交金额 / 全部成交金额，按 BR-04 口径计算 |
| 高贡献标注 | 标识该主题是否达到高贡献率阈值，依据 BR-06 判定 |
| 低效标注 | 标识该主题是否低于低效阈值，依据 BR-09 判定 |
