# Feature Definition - F-03 渠道投入产出下钻 (Channel Investment-Return Drill-Down)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-02] 渠道 ROI 对比分析 (Channel ROI)](../README.md)

**业务定义**：

该特性用于为 CEO 提供从渠道汇总数据到单渠道明细数据的下钻路径，将渠道 ROI 总览视图中的关键指标关联至具体的线索明细、商机关联记录与成交记录，使 CEO 能够在决策前自验证汇总数据的真实性与可信度。它关注的是"总览视图的某项指标是由哪些具体业务事实支撑的、单条线索或商机是否确实归属该渠道、成交记录是否真实可查"，不涉及渠道间的横向对比汇总，也不涉及内容主题维度的归因分析。

**范围边界**：

1. 包含：
   - 从 F-01 渠道 ROI 总览视图的某一渠道入口进入单渠道详情页
   - 单渠道下的线索明细列表（含渠道来源标记、进入时间、当前状态）
   - 线索关联的商机信息（商机状态、预估价值）
   - 成交记录展示（成交时间、成交金额）
   - 明细列表按时间段排序
   - 关键汇总指标（线索量、商机量、成交金额）的下钻路径，供 CEO 在总览数字与明细数字之间核对一致性
2. 不包含：
   - 渠道间横向对比数据（属于 F-01 渠道 ROI 总览与对比）
   - 内容主题维度的归因分析（属于 F-02 内容主题归因分析）
   - 线索或商机的编辑、转派、状态变更（属于销售执行模块，范围外）
   - 渠道来源标记的修改（任何角色均不得在本特性中修改，参见 Epic [BR-05]）
   - 参见 Epic [DEP-01]：渠道成本数据由外部系统或手动录入提供；本特性的成本展示依赖外部数据，缺失时对应渠道仅展示量化指标，不展示成本维度数据

## 业务规则 (Business Rules)

- **[BR-01]链路一致**：参见 Epic [BR-01]——本特性展示的线索到商机转化记录须与 E-01 销售漏斗的阶段状态定义保持一致，明细数据中的商机阶段标识须与漏斗视图中的阶段定义相同。
- **[BR-02]口径统配**：参见 Epic [BR-02]——本特性明细列表中展示的线索成本计算须基于 F-04 配置的当前生效口径，不得在明细层自行换算或调整。
- **[BR-03]权限分级**：参见 Epic [BR-04]——CEO 与管理层可下钻查看全渠道所有线索与商机明细；市场负责人可查看其职责范围渠道的明细数据；销售人员不具备访问权限。
- **[BR-04]来源锁定**：参见 Epic [BR-05]——线索的渠道来源标记须在线索进入系统时自动关联且不可修改；本特性在展示线索渠道来源标记时须处于只读状态，不提供任何修改入口。
- **[BR-05]下钻一致性**：当用户从 F-01 总览视图的某一指标下钻进入本特性时，明细页面展示的数据范围须与总览视图中该指标的统计口径、时间窗口及归因模型保持一致；汇总数字与明细数字须可相互核对，不得出现总览与明细统计口径不同导致的数字差异。[推导-待确认]
- **[BR-06]成本数据缺失**：当某渠道的成本数据缺失时，明细页面中成本相关字段显示为"待录入"；不得用 0 或估算值填充，以免误导数据核对。
- **[BR-07]明细排序规则**：线索明细列表默认按线索进入时间倒序排列；支持按时间段筛选，筛选条件须与 F-01 当前选择的时间窗口联动，保持数据一致性。[推导-待确认]

## 业务对象定义 (Business Object Definition)

### 渠道线索明细记录

渠道线索明细记录是指在某一渠道下，经渠道来源标记关联的单条线索的业务档案，包含线索进入信息、当前处理状态及后续商机关联情况，是 CEO 验证渠道汇总数据真实性的最小业务单元。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 线索标识 | 标识该条线索的唯一业务编号，可追溯至线索创建记录 |
| 渠道来源标记 | 标识该线索进入系统时自动关联的渠道来源，只读，不可修改 |
| 线索进入时间 | 标识该线索首次进入系统的时间，作为时间窗口匹配与排序基准 |
| 线索当前状态 | 标识该线索当前所处的业务阶段（如待跟进、已接触、已转商机等），须与 E-01 漏斗阶段一致 |
| 关联商机标识 | 标识该线索转化后对应的商机编号；未转化时为空 |
| 商机预估价值 | 关联商机的预估成交金额，供汇总核对使用 |
| 商机当前状态 | 关联商机所处的阶段（如跟进中、报价、谈判、成交、丢单），须与 E-01 漏斗阶段一致 |
| 成交金额 | 若商机已成交，记录实际成交金额；未成交时为空 |
| 成交时间 | 若商机已成交，记录成交发生时间；未成交时为空 |

### 渠道明细汇总

渠道明细汇总是单渠道详情页顶部展示的关键指标汇总，用于将 F-01 总览视图中的渠道汇总数字与本页明细数据建立核对关联。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 渠道标识 | 标识当前详情页所属的渠道来源 |
| 统计时间窗口 | 标识当前明细数据所覆盖的时间周期，与 F-01 总览的时间窗口联动 |
| 归因模型标识 | 标识当前明细数据所基于的归因模型口径，与 F-01 总览联动 |
| 线索总数 | 当前时间窗口内该渠道下的线索明细总条数，须与 F-01 总览中该渠道的线索量一致 |
| 商机总数 | 线索明细中已转化为商机的数量 |
| 成交总金额 | 明细记录中已成交商机的金额之和，须与 F-01 总览中该渠道的商机总价值可对应 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 当前用户为 CEO；F-01 总览视图已展示某渠道数据 | 点击该渠道的下钻入口进入详情页 | 进入该渠道的单渠道详情页，展示该渠道在当前时间窗口与归因模型下的线索明细列表；详情页的汇总数字（线索总数）与 F-01 总览中该渠道的线索量一致（参见 BR-05） | US-01 |
| AC-02 | 当前用户为市场负责人，尝试访问非其职责范围渠道的详情页 | 访问该渠道详情页 | 该渠道详情页不可访问，无下钻入口（参见 BR-03） | US-06 |
| AC-03 | 单渠道详情页已打开，存在已转化为商机的线索 | 查看线索明细列表中某条已关联商机的线索 | 展示该线索对应的商机状态与预估价值；商机阶段标识与 E-01 漏斗中的阶段定义一致（参见 BR-01） | US-02 |
| AC-04 | 单渠道详情页已打开，存在已成交的商机 | 查看成交记录 | 展示成交时间与成交金额；成交记录与 E-01 漏斗成单状态同源，明细成交金额之和与汇总显示的成交总金额一致 | US-03 |
| AC-05 | 单渠道详情页已打开 | 将排序方式切换为"线索进入时间升序" | 线索明细列表按线索进入时间从早到晚重新排列；时间范围与 F-01 总览当前选择的时间窗口保持一致（参见 BR-07） | US-04 |
| AC-06 | 单渠道详情页已打开，查看线索明细列表中某条线索的渠道来源标记 | 尝试修改渠道来源标记 | 渠道来源标记处于只读状态，不存在修改入口（参见 BR-04） | US-05 |
| AC-07 | 某渠道的成本数据缺失 | 进入该渠道详情页查看成本相关字段 | 成本字段显示“待录入”，不显示 0 或估算值（参见 BR-06）；其他字段（线索量、商机量、成交金额）正常展示 | US-07 |
