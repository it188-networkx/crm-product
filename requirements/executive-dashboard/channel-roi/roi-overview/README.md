# Feature Definition - F-01 渠道 ROI 总览与对比 (Channel ROI Overview & Comparison)

## 特性概述 (Feature Overview)

**所属 Epic**：[[E-02] 渠道 ROI 对比分析 (Channel ROI)](../README.md)

**业务定义**：

该特性用于向 CEO 提供跨渠道投入产出的统一量化视图，将各渠道的线索量、线索成本、转化率与商机总价值汇聚为可横向对比的业务数据，并支持多种归因模型的并排对比。它关注的是"哪个渠道带来了多少业务价值、各渠道相互之间如何排序、当前口径下的 ROI 结论是否可信"，不涉及单渠道明细数据的逐条查阅，也不定义归因口径本身的配置管理。

**范围边界**：

1. 包含：
   - 各渠道线索量、线索成本、线索到商机转化率、商机总价值的横向对比展示
   - 多渠道按 ROI 排序视图，并对投入产出比显著低于均值的渠道进行标注
   - 多归因模型（最后触点 / 多触点 / 时间衰减）的横向并排对比视图
   - 时间窗口切换（7 天 / 30 天 / 90 天 / 自然月），作为本 Epic 全局时间控制源头
2. 不包含：
   - 单渠道明细数据查阅（属于 F-03 渠道投入产出下钻）
   - 内容主题维度的归因分析（属于 F-02 内容主题归因分析）
   - ROI 计算口径的创建与版本管理（属于 F-04 ROI 公式配置管理，P1 范围外）
   - 参见 Epic [DEP-01]：渠道成本数据由外部系统或手动录入提供；当成本数据缺失时，本特性仅展示线索量分布，无法完成 ROI 计算，并标注数据覆盖状态
   - 参见 Epic [DEP-03]：ROI 计算口径须在本特性与 E-04 Go/No-Go 看板设计阶段同步对齐，指标口径一经确认不得单独调整

## 业务规则 (Business Rules)

- **[BR-01]链路一致**：参见 Epic [BR-01]——渠道 ROI 中使用的转化率指标（线索到商机转化率）须与 E-01 销售漏斗的阶段状态定义保持一致，不得在本特性中独立定义转化节点。
- **[BR-02]口径统配**：参见 Epic [BR-02]——渠道 ROI 的归因模型、收益口径与成本口径须通过 F-04 由管理员统一配置；本特性展示的数据必须严格基于已生效的口径配置，不得在视图层自行换算。
- **[BR-03]口径变更协同**：参见 Epic [BR-03]——渠道 ROI 数据被 E-04 Go/No-Go 看板引用为 KPI 达成率基线；本特性口径变更须与 E-04 协同，不得单独调整。
- **[BR-04]权限分级**：参见 Epic [BR-04]——本特性仅向 CEO 与管理层开放全渠道对比视图；市场负责人可查看其职责范围渠道的数据；销售人员不具备访问权限。
- **[BR-05]来源锁定**：参见 Epic [BR-05]——本特性展示的渠道归属数据基于线索进入时自动打标的渠道来源，不允许用户在本视图中修改渠道归属。
- **[BR-06]贡献率口径**：参见 Epic [BR-06]——渠道线索占比使用"渠道线索数 / 全部线索数"口径，须与 F-02 内容主题归因分析的统计口径同源。
- **[BR-07]低于均值标注**：当某渠道的 ROI 值低于当前时间窗口内所有渠道 ROI 均值时，该渠道须在视图中被显著标注；若有效渠道数量少于 2 个，不执行均值对比标注。[推导-待确认]
- **[BR-08]成本缺失降级**：当某渠道的成本数据缺失时，该渠道不参与 ROI 数值计算，仅展示线索量与转化率，并在视图中标注"成本数据待补充"；不得将成本缺失渠道纳入 ROI 排序，避免误导决策。
- **[BR-09]归因模型并排**：多归因模型对比视图须在同一时间窗口与渠道维度下展示；切换时间窗口时，三种模型数据须同步刷新，不得出现模型间数据时间窗口不一致的情况。[推导-待确认]

## 业务对象定义 (Business Object Definition)

### 渠道 ROI 快照

渠道 ROI 快照是某一渠道在指定时间窗口与归因模型配置下的投入产出汇总记录，是本特性横向对比视图的最小数据单元。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 渠道标识 | 标识该快照所属的渠道来源（如展会、信息流广告、内容营销等） |
| 时间窗口 | 标识该快照所覆盖的统计周期（7 天 / 30 天 / 90 天 / 自然月） |
| 归因模型 | 标识该快照采用的 ROI 计算归因模型（最后触点 / 多触点 / 时间衰减） |
| 线索量 | 该渠道在时间窗口内归入的有效线索总数 |
| 线索成本 | 该渠道在时间窗口内每获取一条线索的平均成本；成本数据来源于外部系统或手动录入 |
| 线索到商机转化率 | 该渠道线索进入商机阶段的比率，须与 E-01 漏斗阶段定义一致 |
| 商机总价值 | 该渠道在时间窗口内关联商机的预估价值之和 |
| ROI 值 | 基于当前生效口径配置计算的渠道投入产出比；成本缺失时为空值 |
| 数据覆盖标注 | 标识该渠道的成本或来源标记数据是否存在缺失，供决策参考 |

### 归因模型配置引用

归因模型配置引用是从 F-04 ROI 公式配置管理中读取的当前生效计算口径快照，用于确保本特性展示的 ROI 数据具备可追溯的计算基准。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 口径版本号 | 标识当前使用的 ROI 计算口径版本，供审计与历史对比使用 |
| 归因模型列表 | 当前生效配置中允许展示的归因模型集合 |
| 收益口径定义 | 商机总价值的统计口径说明 |
| 成本口径定义 | 渠道线索成本的统计口径说明 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 当前用户为 CEO 或管理层；至少 2 个渠道有完整的线索数据 | 打开渠道 ROI 总览视图 | 展示各渠道的线索量、线索成本、线索到商机转化率与商机总价值，各数据均基于当前生效口径配置计算（参见 BR-02） | US-01 |
| AC-02 | 当前用户为市场负责人 | 打开渠道 ROI 总览视图 | 仅展示该市场负责人职责范围内渠道的数据，全渠道汇总入口不可见（参见 BR-04） | US-06 |
| AC-03 | 至少 2 个渠道有有效 ROI 计算值 | 查看渠道列表 | 渠道按 ROI 值从高到低排序；ROI 值低于当前时间窗口内所有有效渠道均值的渠道被显著标注（参见 BR-07） | US-02 |
| AC-04 | 渠道 ROI 总览视图已打开 | 将时间窗口切换为 30 天 | 所有渠道的线索量、转化率、商机价值及 ROI 数据均基于新时间窗口重新计算并刷新；多归因模型对比视图同步刷新（参见 BR-09） | US-03 |
| AC-05 | 渠道 ROI 总览视图已打开，ROI 口径配置中包含至少 2 种归因模型 | 切换至多归因模型对比视图 | 在相同时间窗口与渠道维度下，并排展示最后触点、多触点与时间衰减三种归因模型的渠道 ROI 排序结果 | US-04 |
| AC-06 | 多归因模型对比视图已打开 | 切换时间窗口 | 三种归因模型的数据同步刷新，不存在模型间数据时间窗口不一致的情况（参见 BR-09） | US-04 |
| AC-07 | 某渠道的成本数据缺失 | 查看该渠道在总览视图中的行 | 该渠道仅展示线索量与转化率，ROI 值显示为"成本数据待补充"，不参与 ROI 排序（参见 BR-08）；视图整体正常加载，其他渠道数据不受影响 | US-05 |
