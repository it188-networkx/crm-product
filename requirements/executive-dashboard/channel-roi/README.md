# Epic - [E-02] 渠道 ROI 对比分析 (Channel ROI)

> 需求史诗定义（A0303）｜所属主题：T-03 CEO 决策驾驶舱
> **Epic 编号**：E-02 ｜ **优先级**：P0 ｜ **状态**：规划中

---

## 概述 (Overview)

**业务背景：**

公司在多个渠道（展会、信息流广告、老客转介绍、内容营销等）上持续投入，但各渠道的线索量、转化率与最终成交价值从未被统一量化。CEO 在决定是否加大某渠道投入时无可信数据支撑，只能依赖销售人员的主观反馈。内容主题的贡献度（哪类文章或方案带来了实际成交）同样缺乏追踪，导致内容团队资源分配无依据。

**核心目标：**

本 Epic 旨在构建各渠道线索量、转化率与商机价值的量化对比视图，并含内容主题归因分析，让 CEO 基于可信数据做出渠道加码或收缩与内容选题聚焦的资源分配决策。

**价值预估：**

- **渠道决策自主**：CEO 通过渠道 ROI 数据做渠道投入决策的事项占比从 0% 提升至 ≥ 60%（M6）。（分贡献指标，支撑 Theme T-03 核心 KPI：决策自主占比 ≥ 60%）
- **渠道优化**：提供跨渠道 ROI 对比数据，使 CEO 可识别高投入低产出渠道，优化渠道资源配置。[推导-待确认]
- **内容选题有据**：内容主题归因分析使内容团队可基于贡献率数据聚焦高转化选题，减少低效内容投入。[推导-待确认]

## 业务场景 (Business Scenarios)

### 场景一：季度渠道投入复盘与资源调整决策

**用户**：CEO（王总）

**情境**：季度末，CEO 需要基于各渠道过去一个季度的实际投入产出数据，决定下一季度是否调整渠道资源分配。

**业务流程**：

1. CEO 查看各渠道的线索量、线索成本、转化率与商机总价值的对比视图。
2. 系统按渠道 ROI 排序并高亮投入产出比显著低于均值的渠道。
3. CEO 选择某渠道查看其内容主题归因分析，了解该渠道下哪类内容贡献了实际成交。
4. 系统展示该渠道内容主题按"曝光→点击→留资→线索→商机→成交"链路的各环节转化率与贡献量。
5. CEO 基于数据决定渠道加码或收缩，记录决策依据并通知相关团队。

**跨特性协同**：步骤 1-2 依赖 F-01（渠道 ROI 总览与对比），步骤 3-4 依赖 F-02（内容主题归因分析），步骤 5 依赖 F-03（渠道投入产出下钻）。

### 场景二：内容选题效果追踪

**用户**：市场负责人（MKT Leader）

**情境**：每月内容策略评审，市场负责人需要了解过去一个月各内容主题对最终成交的贡献，以决定下月内容选题方向。

**业务流程**：

1. 市场负责人查看内容主题归因分析，按选题维度对比各主题的全链路转化率与商机贡献量。
2. 系统展示各主题的曝光量、留资转化率与最终成交金额贡献。
3. 市场负责人筛选时间窗口（7/30/90 天）与渠道维度，识别高贡献主题与长尾低效主题。
4. 系统对贡献率显著高于均值的主题进行标注，辅助选题优先级决策。
5. 市场负责人基于分析结果完成下月内容选题规划，确认聚焦方向。

**跨特性协同**：步骤 1-4 依赖 F-02（内容主题归因分析），步骤 3 的时间窗口切换依赖 F-01（渠道 ROI 总览）的统一时间范围控制。

## 业务规则 (Business Rules)

| 编号 | 规则描述 | 适用 Feature |
| :--- | :--- | :--- |
| [BR-01]链路一致 | F-02 内容主题归因的转化链路定义（曝光→点击→留资→线索→商机→成交）必须与 E-01 销售漏斗中的阶段定义保持一致，不得出现两套转化链路，同一商机在两个 Epic 中的转化状态数字必须一致。 | F-01, F-02, F-03 |
| [BR-02]口径统配 | 渠道 ROI 的计算口径（归因模型、收益口径、成本口径）须通过 F-04 由管理员在系统中统一配置，不得由数据分析人员在前置处理中自行修改，以保证 GC-02 数据主权约束。 | F-01, F-03, F-04 |
| [BR-03]协同变更 | E-04 Go/No-Go 验收看板引用本 Epic 的渠道 ROI 数据作为 KPI 达成率计算基线，E-02 的指标口径一经确认，不得在 E-04 上线后单独调整，须协同变更。 | F-01, F-02 |
| [BR-04]权限分级 | 渠道数据的访问权限须遵循 GC-06 权限分级约束：CEO 与管理层可查看全渠道对比视图，市场负责人可查看本职责范围渠道详情，销售人员不具备渠道 ROI 数据访问权限。 | F-01, F-02, F-03, F-04 |
| [BR-05]来源锁定 | 渠道来源标记须在线索进入系统时自动关联，不得允许事后手动修改渠道归属，避免数据污染影响 ROI 计算准确性。[推导-待确认] | F-01, F-03 |
| [BR-06]贡献率口径 | 内容主题贡献率的计算口径统一为：主题贡献率（线索）= 主题线索数 / 全部线索数，主题贡献率（成交）= 主题成交金额 / 全部成交金额；该口径须与 F-01 的渠道统计数据保持同源。 | F-01, F-02 |

## 约束与依赖 (Constraints & Dependencies)

| 编号 | 类别 | 描述 |
| :--- | :--- | :--- |
| [DEP-01]外部数据 | 外部依赖 | 渠道线索来源标记（各渠道进入时打标）与成本数据（广告费、人力折算）须由外部系统或手动录入提供；缺失时 F-01 渠道 ROI 计算降级为仅展示线索量分布，无法完成 ROI 计算。 |
| [DEP-02]漏斗同源 | 跨 Epic 数据约束 | E-01 销售漏斗全链路视图提供商机各阶段状态记录，F-02 内容主题归因的成交端数据须与 E-01 的漏斗成单状态保持同源，不得独立统计。 |
| [DEP-03]口径对齐 | 跨 Epic 数据约束 | E-04 Go/No-Go 验收看板依赖本 Epic 的渠道 ROI 数据作为 KPI 达成率的计算基线；指标口径须在 E-02 与 E-04 设计阶段同步对齐。 |
| [ASM-01]来源打标 | 前提假设 | 假设各渠道线索数据进入 CRM 系统时已携带渠道来源标记，且标记规范统一；若来源标记缺失或不规范，渠道归因分析降级为仅覆盖有标记数据的子集，并标注数据覆盖率。[推导-待确认] |
| [ASM-02]曝光采集 | 前提假设 | 假设内容主题的曝光量与点击量可通过外部媒介平台或官网分析工具采集；若特定渠道的曝光数据不可获取，F-02 对应渠道的转化链路仅从留资环节起算，并标注数据起点。[推导-待确认] |
