# Feature Definition - F-03 检索反馈与热度排序 (Retrieval Feedback & Hotness Ranking)

> 需求特性定义（A0304）｜所属主题：T-04 营销知识库
> **Feature 编号**：F-03 ｜ **优先级**：P0 ｜ **状态**：规划中
> 关联 Epic：[T-04/E-02 知识检索与场景推送](../README.md)

## 特性概述 (Feature Overview)

**所属 Epic**：[T-04/E-02 知识检索与场景推送](../README.md)

**业务定义**：
本特性允许售前对在检索（F-01）和推送（F-02）场景中使用的知识单元提交结构化反馈（有用 / 不相关 / 已过时），通过系统性收集反馈数据驱动知识单元热度分更新，使后续检索和推送的排序结果持续贴近售前实际使用需求。同时，将汇总的反馈数据呈现在内容运营看板，为内容运营员识别需修订内容提供数据依据，形成「使用 → 反馈 → 热度更新 → 排序优化 → 内容修订」的正向改进闭环。本特性依赖 F-01 与 F-02 的上线，不得早于两者独立集成。

**范围边界**：

1. 包含：售前对已展示知识单元提交反馈标签（有用 / 不相关 / 已过时）、反馈权重影响知识单元热度分计算、内容运营看板展示反馈汇总数据与热度趋势。
2. 不包含：基于反馈直接触发知识单元下线（参见 Epic BR-05）、知识单元的修订编辑流程（属 E-01）、反馈标签分类体系的定义（由 Theme 级统一维护，本特性仅消费）、检索或推送的核心排序算法（属 F-01、F-02 和 E-03 分别负责）。

## 业务规则 (Business Rules)

- 参见 Epic [BR-05]（反馈治理）：反馈数据仅用于热度排序优化与运营看板展示，不得直接触发内容下线；下线须由内容运营员在 E-01 中手动审核；反馈标签统一采用 Theme 级分类法（有用 / 不相关 / 已过时）。
- **[BR-01]反馈对象范围**：售前仅可对在本次会话中通过 F-01 检索结果或 F-02 推送列表中已展示过的知识单元提交反馈，不得对从未展示过的知识单元提交反馈。[推导-待确认]
- **[BR-02]反馈唯一性**：同一售前对同一知识单元在同一使用场景（检索 / 推送）中只允许存在一条有效反馈；重新提交时，新反馈覆盖原反馈记录，原反馈不再计入热度分计算。[推导-待确认]
- **[BR-03]热度分权重**：「有用」反馈对热度分正向贡献，「不相关」与「已过时」反馈对热度分负向贡献；具体权重系数待运营团队确认（TBD，默认分流：有用 +1，不相关 −0.5，已过时 −1）；新反馈覆盖旧反馈时，热度分同步撤销旧权重并计入新权重。[推导-待确认]
- **[BR-04]运营看板数据范围**：内容运营看板展示可配置时间窗口内的反馈数据聚合结果（有用反馈数、负面反馈数、热度趋势）；时间窗口默认值待运营团队确认（TBD）；看板不得展示包含个人身份信息的原始反馈明细。[推导-待确认]

## 业务对象定义 (Business Object Definition)

### 知识单元反馈记录

知识单元反馈记录是售前针对特定知识单元提交的单次反馈，承载反馈标签与来源场景信息，是热度分计算和运营看板聚合分析的原子数据单位。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 知识单元标识 | 标识本次反馈针对的是哪个知识单元 |
| 反馈提交人 | 标识提交反馈的售前顾问，用于唯一性校验（参见 BR-02） |
| 反馈标签 | 有用 / 不相关 / 已过时，采用 Theme 级分类法（参见 Epic BR-05） |
| 来源场景 | 标识反馈发生在检索（F-01）还是推送（F-02）场景，用于唯一性判断与分析 |
| 提交时间 | 反馈提交的时间戳，用于时间窗口聚合与时效性判断 |

### 知识单元热度汇总

知识单元热度汇总是系统对某一知识单元所有有效反馈数据的聚合结果，直接决定该知识单元在检索结果（F-01）和推送列表（F-02）中的排序权重。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 知识单元标识 | 标识热度汇总属于哪个知识单元 |
| 热度分 | 基于所有有效反馈权重累积计算的综合评分（参见 BR-03） |
| 有用反馈数 | 该知识单元收到的「有用」反馈总量，用于运营看板展示 |
| 负面反馈数 | 该知识单元收到的「不相关」与「已过时」反馈总量，用于运营看板展示 |
| 最近反馈时间 | 最新一条有效反馈的提交时间，辅助运营员判断数据时效性 [推导-待确认] |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 售前在当前会话的检索结果中看到某知识单元；该售前对该知识单元在检索场景下尚无有效反馈 | 提交「有用」反馈 | 反馈记录登记成功，该知识单元热度分正向更新（参见 BR-03）；该知识单元展示已提交反馈状态 | US-01, US-03 |
| AC-02 | 售前在当前会话的推送列表中看到某知识单元；该售前对该知识单元在推送场景下尚无有效反馈 | 提交「已过时」反馈 | 反馈记录登记成功，该知识单元热度分负向更新（参见 BR-03）；该知识单元展示已提交反馈状态 | US-01, US-03 |
| AC-03 | 售前已对某知识单元在检索场景下提交过「有用」反馈 | 在同一场景下再次查看该知识单元 | 该知识单元展示当前已提交的「有用」反馈状态，不允许重复提交相同场景下的反馈（参见 BR-02） | US-03 |
| AC-04 | 售前已对某知识单元提交过「不相关」反馈；同一售前在同一场景下重新提交「有用」反馈 | 提交新的「有用」反馈 | 新反馈覆盖原「不相关」反馈；热度分撤销原负向权重并计入新正向权重（参见 BR-02, BR-03） | US-01 |
| AC-05 | 内容运营员打开内容运营看板；指定时间窗口内存在多条反馈记录 | 查看反馈汇总数据 | 按知识单元展示有用反馈数、负面反馈数及热度趋势；展示数据为聚合结果，不包含个人身份信息（参见 BR-04） | US-02 |
| AC-06 | 某知识单元在指定时间窗口内收到大量「已过时」反馈 | 内容运营员查看运营看板 | 该知识单元在负面反馈数排行中可见；系统不自动触发该知识单元下线，下线须由内容运营员在 E-01 中手动操作（参见 Epic BR-05） | US-02 |
