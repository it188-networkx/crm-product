# T-04/E-02 知识检索与场景推送

> 需求史诗定义（A0303）｜所属主题：T-04 营销知识库
> **Epic 编号**：E-02 ｜ **优先级**：P0 ｜ **状态**：规划中
> 关联 Issue：[#46](https://github.com/it188-networkx/crm-product/issues/46)

---

## 1. 业务背景

### As-Is（当前状态）

- 售前在跟进商机时需在多个系统手动检索案例/话术，平均花费 15~30 分钟无法落地有效内容
- 知识分散无统一入口，无法按客户行业/阶段快速筛选匹配度高的内容
- 知识内容与商机详情页割裂，售前需来回切换应用获取支持材料

### To-Be（目标状态）

- 提供统一知识检索界面，支持按客户类型/行业/销售阶段多维筛选，召回 E-01 已发布知识单元
- 在商机详情页嵌入「推荐知识」面板，根据商机属性（行业、阶段、客户规模）智能推送相关案例/话术/解决方案
- 收集检索和推送反馈（点击/使用/忽略），驱动热度排序优化

> [缺少需求输入-待补充]：智能推送的相关性算法优先级（规则 vs 向量相似度）需与 E-03 协调确认

## 2. 核心 Feature 拆分

| Feature ID | Feature 名称 | 优先级 | 简述 |
|---|---|---|---|
| F-01 | 多维度知识检索 | P0 | 提供关键词 + 多维筛选器（客户类型/行业/竞品/销售阶段）的知识检索界面，结果按热度+匹配度排序，仅展示"已发布"知识单元 |
| F-02 | 商机场景智能推送 | P0 | 在 T-01 E-03 商机详情页嵌入「推荐知识」侧边面板，基于商机属性自动推送 Top-N 相关知识，支持展开预览和一键复制 |
| F-03 | 检索反馈与热度排序 | P0 | 售前可对检索/推送的知识单元提交"有用/不相关/过时"反馈，反馈权重影响热度分，内容运营看板展示热度趋势 |

---

## 3. 业务规则约束（Business Rules）

| 规则 ID | 描述 |
|---|---|
| BR-01 | 检索结果和推送内容只能来自 E-01 状态为"已发布"的知识单元，归档/草稿/审核中状态禁止出现 |
| BR-02 | 客户特定案例与行业通用知识须按 GC-02 数据权限隔离：客户专属内容仅对相关商机负责人可见 |
| BR-03 | 商机详情页「推荐知识」面板展示上限 N=5 条（可配置），最多占用右侧栏 40% 空间，不影响核心商机信息展示 |
| BR-04 | 检索/推送结果须包含知识单元的版本号与发布时间，避免售前使用过时版本 |
| BR-05 | 反馈数据仅用于热度排序优化，不得直接触发内容下线，下线须由内容运营员手动审核 |

## 4. 非功能需求（NFR）

| 编号 | 类型 | 要求 |
|---|---|---|
| NFR-01 | 性能 | 知识检索响应 ≤ 1s（P95，基于关键词+3维筛选器组合查询） |
| NFR-02 | 推送延迟 | E-01 发布新知识单元后，推荐候选池更新延迟 ≤ 10 分钟 |
| NFR-03 | 权限 | 检索和推送结果严格按 GC-02 数据权限实时过滤，API 层不缓存越权数据 |
