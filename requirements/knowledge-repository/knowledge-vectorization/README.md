## 1. 业务背景

### As-Is（当前状态）

- T-02 AI 建议系统（T-02 E-02 AI 跟进辅助）在生成跟进策略时缺乏行业贴合的知识支撑，建议内容泛化、缺乏说服力
- AI 建议无引用来源标注，售前无法判断建议依据，降低采纳意愿
- 知识资产以结构化文本存储，AI 无法高效做语义相似度匹配

### To-Be（目标状态）

- 将 E-01 已审核发布的知识单元向量化后存入向量数据库，供 T-02 AI 系统在推理时做语义检索增强（RAG 模式）
- AI 生成建议时若引用了向量检索到的知识，须明确标注来源知识单元 ID 和版本（GC-01 AI 边界约束）
- 建立引用质量反馈闭环：售前对 AI 引用内容提交反馈，反馈汇入 E-01 内容运营看板驱动知识质量迭代

> [缺少需求输入-待补充]：向量模型选型（内部自托管 vs 外部 API）需与技术架构确认，影响数据出境合规边界

## 2. 核心 Feature 拆分

| Feature ID | Feature 名称 | 优先级 | 简述 |
|---|---|---|---|
| F-01 | 知识单元向量化入库 | P1 | E-01 发布知识单元时触发向量化流水线：文本分块 → 嵌入模型编码 → 写入向量数据库；E-01 归档事件触发向量下线，保障向量库与知识库状态同步 |
| F-02 | AI 建议知识引用与来源标注 | P1 | T-02 E-02 AI 跟进助手在推理时调用向量检索接口（RAG），将 Top-K 语义相关知识段注入 Prompt；AI 输出须标注引用来源（知识单元 ID + 版本），供售前溯源 |
| F-03 | 引用质量反馈闭环 | P1 | 售前可对 AI 引用的知识片段标注"有帮助/不准确/已过时"；反馈汇总写入 E-01 内容运营看板；反馈量达到阈值时触发内容运营员复查通知 |

## 3. 业务规则约束（Business Rules）

| 规则 ID | 描述 |
|---|---|
| BR-01 | 只有 E-01 状态为"已发布"的知识单元才能进入向量化流水线，草稿/审核中/归档状态禁止向量化入库 |
| BR-02 | AI 建议若引用了向量知识，必须在输出文本中标注来源（GC-01 AI 边界约束），禁止将引用内容以 AI 原创形式展示 |
| BR-03 | 向量模型编码的知识文本须在系统内部处理，不得将未脱敏的客户信息发送给外部 API（GC-03 数据合规） |
| BR-04 | E-01 归档事件触发向量数据库中对应条目标记下线（软删除），已生成但引用了下线知识的历史建议须标注"引用内容已更新" |
| BR-05 | 向量化任务失败须告警并重试（最多 3 次），持续失败须阻断对应知识单元的 E-02 检索推送，并通知内容运营员 |

## 4. 非功能需求（NFR）

| 编号 | 类型 | 要求 |
|---|---|---|
| NFR-01 | 向量化延迟 | E-01 发布事件触发后，向量化入库完成延迟 ≤ 30 分钟（含编码 + 写库） |
| NFR-02 | 检索性能 | RAG 向量检索（Top-K=5）延迟 ≤ 500ms（P95），不阻塞 AI 建议生成主路径 |
| NFR-03 | 数据隔离 | 向量数据库中的知识向量按 GC-02 数据权限标记，RAG 检索须过滤无权限知识 |

## 8. 全局约束

- **GC-01** AI 边界约束：AI 输出中引用的知识必须标注来源，禁止以原创形式呈现
- **GC-02** 数据权限隔离：向量检索结果按权限过滤，不越权返回无权知识
- **GC-03** 信息脱敏：向量化输入须为 E-01 已脱敏发布的知识，禁止处理原始草稿
- **GC-04** 操作审计：向量化任务执行日志（含触发事件、入库状态、失败原因）保留 ≥ 12 个月
