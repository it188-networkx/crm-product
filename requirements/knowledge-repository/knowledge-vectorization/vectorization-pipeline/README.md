# Feature Definition - [T-04/E-03/F-01] 知识单元向量化入库 (Knowledge Unit Vectorization Pipeline)

## 特性概述 (Feature Overview)

**所属 Epic**：[[T-04/E-03] AI 知识向量化引用 (Knowledge Vectorization)](../README.md)

**业务定义**：

本特性负责在 E-01 知识单元发生发布或归档状态变更时，维护向量存储与知识库状态的实时同步：知识单元发布时触发向量化处理并写入向量存储，使其处于语义检索就绪状态；知识单元归档时触发对应向量条目软删除，使其退出检索范围；向量化任务失败时告警并自动重试，保障向量覆盖率。

它关注的是"向量存储中的可检索条目是否与知识库中的已发布条目保持状态一致"，以及"任务异常能否被及时发现与恢复"。本特性不定义如何执行语义检索、如何呈现检索结果或如何处理引用质量反馈。

**范围边界**：

1. 包含：E-01 发布事件触发向量化入库、E-01 归档事件触发向量条目下线（软删除）、向量化任务失败告警与自动重试、任务执行状态与日志记录。
2. 不包含：语义检索执行与 Top-K 结果返回（F-02）、检索结果来源标注（F-02）、引用质量反馈与复查通知（F-03）、知识单元内容的录入、审核与发布管理（属 E-01 范畴）。
3. 上游依赖：参见 Epic [DEP-01]（本特性仅处理 E-01 发布/归档事件，须订阅 E-01 状态变更通知，E-01 未稳定上线前无法集成）；参见 Epic [ASM-01]（假设 E-01 在本特性集成前已积累不少于 100 个已发布知识单元）；参见 Epic [ASM-02]（嵌入模型选型须在集成前确定，选型结果影响向量化处理的输入规格，待技术架构确认后方可锁定实现约束）。

## 业务规则 (Business Rules)

**继承自 Epic（适用本 Feature，不在此重复定义）：**

- 参见 Epic [BR-01] 发布门禁：仅 E-01 状态为"已发布"的知识单元可进入向量化处理，草稿、审核中、已归档状态的知识单元禁止向量化入库。
- 参见 Epic [BR-02] 归档联动下线：E-01 归档事件必须触发向量存储中对应条目标记为下线（软删除），后续语义检索不得返回下线条目。
- 参见 Epic [BR-03] 失败阻断：向量化任务失败须告警并重试（最多 3 次），持续失败须阻断对应知识单元的检索可用性并通知内容运营员介入。
- 参见 Epic [BR-08] 知识脱敏：向量化处理的输入必须为 E-01 已完成脱敏审核并发布的知识内容，不得处理含客户可识别信息的原始草稿。

**本 Feature 局部规则：**

- **[BR-01] 任务幂等性**：同一知识单元 ID + 版本号的向量化任务重复触发时，不得创建重复的向量条目；系统须判断当前版本是否已存在"检索就绪"状态的向量条目，若已存在则跳过本次处理并记录幂等跳过日志。`[推导-待确认]`
- **[BR-02] 版本迭代处理**：知识单元被重新发布（版本升级）时，前一版本对应的向量条目须在新版本向量化完成后标记为下线；同一知识单元的不同版本向量条目不得同时处于"检索就绪"状态。`[推导-待确认]`
- **[BR-03] 软删除完整性**：向量条目下线须采用软删除方式（保留记录、更新状态与下线时间），不得物理删除，以满足 Epic [NFR-03] 规定的不少于 12 个月审计追溯要求。`[推导-待确认]`

## 业务对象定义 (Business Object Definition)

### 向量化任务

向量化任务是对知识单元状态变更事件（发布/归档）的执行记录，用于追踪向量化或下线操作的触发来源、执行状态与结果，是实现任务告警与重试机制的业务凭证。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联知识单元 | 标识本次任务所处理的知识单元 ID 及版本号 |
| 触发事件类型 | 标识触发本次任务的事件类型（发布/归档） |
| 任务状态 | 标识当前任务的执行进度（待处理/处理中/已完成/失败） |
| 重试次数 | 记录当前任务已执行的失败重试次数，用于判断是否已达最大重试上限（参见 Epic BR-03） |
| 失败原因 | 任务失败时记录的失败原因描述，供内容运营员介入参考 |
| 触发时间 | 任务被触发的时间，作为向量化延迟计算的业务起点（参见 Epic NFR-01） |
| 完成时间 | 任务最终完成（成功或放弃重试）的时间 |

### 向量条目

向量条目是知识单元在向量存储中的可检索副本，其状态（检索就绪/已下线）与知识库中对应知识单元的发布状态保持同步，承载使 AI 建议系统能够语义检索该知识内容所需的元信息。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 关联知识单元 | 标识本条目对应的知识单元 ID 及版本号；来源元数据不可丢失（参见 Epic BR-04） |
| 条目状态 | 标识当前条目是否可被语义检索命中（检索就绪/已下线） |
| 入库时间 | 向量条目首次写入向量存储的时间 |
| 下线时间 | 条目被标记为下线时的时间，用于审计追溯（参见 Epic NFR-03） |
| 下线原因 | 标识条目下线的业务原因（知识单元归档/向量化任务持续失败） |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | E-01 中知识单元状态为"已发布"且已完成脱敏审核；当前不存在该知识单元此版本的"检索就绪"向量条目 | E-01 发出发布事件 | 向量化任务创建并处理完成；新向量条目写入向量存储且标记为"检索就绪"；从触发事件到入库完成的时间不超过 30 分钟（参见 Epic NFR-01） | US-01 |
| AC-02 | E-01 中知识单元从"已发布"变更为"已归档"；对应向量条目处于"检索就绪"状态 | E-01 发出归档事件 | 对应向量条目标记为"已下线"（软删除，记录保留）；下线后语义检索不再返回该条目（参见 Epic BR-02；参见 BR-03） | US-02 |
| AC-03 | 向量化任务已创建且处于处理中 | 向量化处理失败 | 系统自动重试，最多重试 3 次；每次失败均生成告警记录；3 次均失败后，对应知识单元的检索可用性被阻断，同时向内容运营员发出持续失败通知（参见 Epic BR-03） | US-03 |
| AC-04 | 同一知识单元 ID + 版本号已存在"检索就绪"状态的向量条目 | 再次收到该知识单元相同版本的发布事件 | 系统不创建重复向量条目；已有条目状态保持不变；系统记录幂等跳过日志（参见 BR-01） | US-01 |
| AC-05 | E-01 中知识单元状态为草稿、审核中或已归档 | 系统收到针对该知识单元的向量化请求 | 系统拒绝向量化处理，不写入任何向量条目（参见 Epic BR-01） | US-01 |
