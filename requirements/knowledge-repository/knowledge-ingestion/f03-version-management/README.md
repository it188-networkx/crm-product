# Feature Definition - T-04/E-01/F-03 知识单元版本管理 (Knowledge Unit Version Management)

> 需求特性定义（A0304）｜所属史诗：T-04/E-01 知识资产结构化录入与审核
> **Feature 编号**：F-03 ｜ **优先级**：P0 ｜ **交付阶段**：Phase 2 ｜ **状态**：规划中

---

## 特性概述 (Feature Overview)

**所属 Epic**：[T-04/E-01 知识资产结构化录入与审核](../README.md)

**业务定义**：
该特性用于对已发布的知识单元实施受控的内容迭代，确保每次内容更新均经过完整的审核与脱敏流程，并在新版本发布后自动将旧版本归档，同时保留历史快照供审计查阅。它关注的是"已发布知识单元如何被修订、修订内容如何重新进入审核通道、版本之间的状态切换规则，以及历史版本如何被访问"，不定义修订内容的录入细节（沿用 F-01 录入流程）、审核脱敏操作细节（由 F-02 负责），也不定义下游 E-02/E-03 的索引更新逻辑。

**范围边界**：

1. 包含：已发布知识单元的修订发起；修订内容进入 F-02 审核与脱敏发布流程；新版本发布后旧版本自动归档；历史版本快照查阅（供审计使用）；版本号管理。
2. 不包含：新知识单元的首次录入（F-01）；审核与脱敏操作（F-02）；已归档版本的恢复发布；知识单元的完全删除（历史快照不可删除）；版本之间的内容差异对比视图（可在后续迭代考虑）[推导-待确认]。
3. 参见 Epic [DEP-01]：新版本发布后，E-02 须订阅"已发布"事件更新检索索引；旧版归档后 E-02 须同步移除对应内容。
4. 参见 Epic [DEP-02]：新版本发布后，E-03 须订阅"已发布/已归档"事件同步向量数据库。

## 业务规则 (Business Rules)

- **[BR-01]修订前置条件**：版本修订仅可针对状态为"已发布"的知识单元发起；草稿状态或已归档状态的知识单元不得再次发起修订。
- **[BR-02]修订状态隔离**：一个知识单元同一时间只允许存在一个处于修订流程中（草稿或待审状态）的新版本；已有进行中的修订版本时，系统不允许再次发起新的修订[推导-待确认]。
- **[BR-03]旧版维持发布**：新版本修订草稿提交至 F-02 审核期间，旧版本保持"已发布"状态，继续对 E-02 和 E-03 可见，不中断下游服务。
- **[BR-04]自动归档**：新版本通过 F-02 审核并发布后，旧版本状态自动变更为"已归档"；已归档版本不得出现在 E-02 检索结果或 E-03 向量数据库中（参见 Epic [BR-05]）。
- **[BR-05]历史快照保留**：所有已归档的历史版本快照不可删除，须保留供审计查阅；历史版本内容对审计人员可查，对普通用户不可见（参见 Epic [BR-03]、[BR-05]）。
- **[BR-06]版本号规则**：每次新版本发布后，系统自动递增版本号；版本号须在知识单元历史版本列表中可追溯[推导-待确认]。
- **[BR-07]操作留痕**：修订发起、版本发布、旧版归档等操作须写入审计日志，包含操作者、时间戳、操作类型及版本号变更信息（参见 Epic [BR-03]）。
- **[BR-08]事业部隔离**：版本修订发起权限限于知识单元所属事业部/产品线范围内的知识运营员；跨事业部访问须经显式授权（参见 Epic [BR-06]）。
- **[BR-09]归档后 E-02/E-03 同步**：旧版本自动归档后，系统须触发订阅事件通知 E-02 移除归档内容、通知 E-03 更新向量数据；事件触发须在归档操作成功后发出（参见 Epic [DEP-01]、[DEP-02]）[推导-待确认]。

## 业务对象定义 (Business Object Definition)

### 知识单元版本

知识单元版本是一个已发布知识单元在特定时间点的完整内容快照，承载该版本的正文内容、元数据、版本号及生命周期状态，是版本追溯和审计的基本业务单元。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属知识单元 | 标识该版本归属于哪个知识单元记录 |
| 版本号 | 版本的顺序标识，随每次新版本发布自动递增 |
| 版本状态 | 当前版本所处的生命周期状态：草稿 / 待审 / 已发布 / 已归档 |
| 正文内容 | 本版本的知识内容主体（经脱敏处理后的内容） |
| 分类标签 | 本版本的行业/客户类型/竞品/销售阶段分类标签 |
| 修订发起人 | 发起本次修订的用户身份 |
| 修订发起时间 | 本版本草稿创建的时间戳 |
| 发布时间 | 本版本状态变更为"已发布"的时间戳，草稿/归档版本为空 |
| 归档时间 | 本版本状态变更为"已归档"的时间戳，非归档版本为空 |

### 版本操作记录

版本操作记录是知识单元版本生命周期中每次状态变更的审计凭证，确保版本迭代全流程可追溯、不可篡改。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属知识单元版本 | 标识操作针对的具体版本 |
| 操作类型 | 操作分类，如修订发起、版本发布、版本归档 |
| 操作者 | 执行该操作的用户身份 |
| 操作时间 | 操作发生的时间戳 |
| 操作摘要 | 操作的简短描述，包含版本号变更信息 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 知识运营员访问一个状态为"已发布"的知识单元，且该知识单元当前无进行中的修订版本 | 知识运营员发起内容修订 | 系统创建该知识单元的新版本草稿，草稿状态为"草稿"，修订发起操作写入审计日志（参见 BR-01、BR-07） | US-01 |
| AC-02 | 知识单元处于"草稿"或"已归档"状态，或已有一个进行中的修订版本 | 知识运营员尝试发起修订 | 系统拒绝发起修订，提示具体原因（参见 BR-01、BR-02） | US-01 |
| AC-03 | 新版本草稿已创建，旧版本处于"已发布"状态 | 知识运营员查看知识单元状态 | 旧版本仍显示为"已发布"并可被 E-02 和 E-03 正常消费；新版本草稿进入 F-02 审核流（参见 BR-03） | US-02 |
| AC-04 | 新版本草稿已进入 F-02 审核流，审核员完成脱敏审核并提交发布 | F-02 审核发布流程执行完成 | 新版本状态变更为"已发布"；审计日志记录版本发布操作；E-02 和 E-03 收到"已发布"事件通知（参见 BR-07、BR-09） | US-03 |
| AC-05 | 新版本状态已变更为"已发布" | 系统自动执行旧版本归档 | 旧版本状态自动变更为"已归档"；旧版本从 E-02 检索结果和 E-03 向量数据库中移除；归档操作写入审计日志；E-02 和 E-03 收到"已归档"事件通知（参见 BR-04、BR-07、BR-09） | US-04 |
| AC-06 | 审计人员查询某知识单元的历史版本列表 | 审计人员打开历史版本快照 | 系统展示该知识单元所有历史版本（含已归档版本）的快照，包含版本号、发布时间、归档时间及版本内容；历史快照对普通用户不可见（参见 BR-05） | US-05 |
