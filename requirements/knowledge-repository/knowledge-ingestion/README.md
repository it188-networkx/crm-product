# T-04/E-01 知识资产结构化录入与审核

> 需求史诗定义（A0303）｜所属主题：T-04 营销知识库
> **Epic 编号**：E-01 ｜ **优先级**：P0 ｜ **状态**：规划中
> 关联 Issue：[#44](https://github.com/it188-networkx/crm-product/issues/44)

---

## 1. 业务背景

### As-Is（当前状态）

- 行业解决方案、成功案例、竞品话术以非结构化方式分散在 PPT/Word/邮件中，售前无法快速定位
- 知识内容未经审核即流通使用，存在敏感客户信息泄露风险（违反 GC-03 信息脱敏要求）
- 没有版本管理机制，内容过时率高，质量反馈无闭环

### To-Be（目标状态）

- 建立统一的结构化知识录入通道，支持分类标签（行业/客户类型/竞品/阶段）与元数据填写
- 所有内容经人工审核和脱敏处理后方可发布，形成可信知识单元（E-02 检索和 E-03 向量化的共同输入）
- 内容版本化管理，支持修订历史追踪与质量反馈闭环

> [缺少需求输入-待补充]：审核流程 SLA（提交到发布的最长时限）需与知识运营团队确认

## 2. 核心 Feature 拆分

| Feature ID | Feature 名称 | 优先级 | 简述 |
|---|---|---|---|
| F-01 | 结构化内容录入 | P0 | 提供知识单元录入表单，支持分类标签（行业/客户类型/竞品/销售阶段）、文本/附件上传、元数据填写，保存为草稿状态等待审核 |
| F-02 | 审核与脱敏发布 | P0 | 审核员工作台：列表查看待审内容，执行脱敏标记（含客户名称/金额/人员）后提交发布；拒绝时须填写退回原因；发布后知识单元状态变为"已发布"触发 E-02/E-03 消费 |
| F-03 | 知识单元版本与质量反馈 | P0 | 已发布知识单元支持版本迭代（修订后重新进入审核流），记录版本历史；销售使用后可提交"有用/过时/错误"反馈，反馈汇总到内容运营看板 |

## 3. 业务规则约束（Business Rules）

| 规则 ID | 描述 |
|---|---|
| BR-01 | 未经审核的知识单元不得出现在 E-02 检索结果或 E-03 向量数据库中（严格状态门控） |
| BR-02 | 脱敏处理（GC-03）强制要求：所有客户名称、人员姓名、具体金额数字在发布前须经审核员标记并替换为泛化描述 |
| BR-03 | 每条录入操作、审核决定（通过/拒绝）、脱敏操作均须写入审计日志（GC-04），日志不可删除 |
| BR-04 | 知识单元分类标签由管理员预设可选值，录入人不得自由填写标签，防止标签碎片化影响 E-02 检索精度 |
| BR-05 | 版本迭代时，旧版本知识单元在新版本发布后自动归档（不可检索/引用），但历史快照保留供审计查阅 |

---

## 4. 非功能需求（NFR）

| 编号 | 类型 | 要求 |
|---|---|---|
| NFR-01 | 审核 SLA | 知识单元从提交到审核完成 ≤ 72 小时（含脱敏处理，需人工 SLA 配合） |
| NFR-02 | 安全 | 脱敏处理结果在保存时加密存储（GC-03），原始草稿访问权限限审核员及以上 |
| NFR-03 | 可追溯 | 审计日志包含操作者、时间戳、操作类型及变更内容摘要，保留 ≥ 12 个月 |

## 5. 跨 Epic 依赖

| 依赖标识 | 说明 |

## 8. 全局约束

- **GC-03** 信息脱敏：发布前强制脱敏，原始内容访问受严格权限控制
- **GC-02** 数据权限隔离：录入的知识单元按所属事业部/产品线隔离可见性
- **GC-04** 操作审计：录入/审核/脱敏操作全程留存不可篡改日志
