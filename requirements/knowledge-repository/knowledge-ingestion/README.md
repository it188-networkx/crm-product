# T-04/E-01 知识资产结构化录入与审核

> 需求史诗定义（A0303）｜所属主题：T-04 营销知识库

## 概述 (Overview)

**业务背景：**

行业解决方案、成功案例、竞品话术以非结构化方式分散在 PPT/Word/邮件中，售前无法快速定位可用内容；知识内容未经审核即在团队内流通，存在客户敏感信息泄露风险（违反 GC-03 信息脱敏要求）；内容无版本管理，过时率高，质量反馈无闭环。

**核心目标：**

构建一套受控的知识资产录入与发布通道，确保所有进入知识库的内容均经结构化分类、人工脱敏审核和版本化管理，成为 E-02 检索与 E-03 向量化引用的唯一可信来源。

**价值预估：**

- 年化人力节省等效 ≥ ¥50 万，减少售前手动检索与整理时间（M12 验收；支撑 T-04 目标：提升知识复用率）
- 知识单元发布审核通过率 ≥ 80%（M6 GA；支撑 T-04 目标：建立可信知识资产）
- 脱敏合规率达 100%，首次上线即达标（M3；支撑 GC-03 合规要求）
- 销售质量反馈"有用"占比 ≥ 70%（M9；支撑 T-04 目标：知识内容质量持续改进）

> **待确认（缺少需求输入）**：审核流程 SLA（提交到发布的最长时限）需与知识运营团队确认后固化

## 业务场景 (Business Scenarios)

### 场景一：售前录入竞品话术并发布上线

**用户**：售前顾问、审核员

**情境**：售前顾问在支持商机后，将本次使用的竞品应对话术结构化录入知识库，供团队复用。

**业务流程**：

1. 售前顾问发起知识录入，选择分类标签（行业/客户类型/竞品/销售阶段），填写正文并携带关联商机上下文预填元数据。
2. 系统将内容保存为草稿状态，等待审核。
3. 审核员对待审内容进行脱敏标记，将客户名称、人员姓名、具体金额替换为泛化描述。
4. 审核员提交发布决定；系统将知识单元状态变更为"已发布"，并通知 E-02 和 E-03 消费。

### 场景二：知识单元内容修订与历史版本管理

**用户**：知识运营员、审核员

**情境**：已发布的解决方案因产品升级导致内容过时，需要发起版本修订并归档旧版本。

**业务流程**：

1. 知识运营员对已发布的知识单元发起修订，提交新版本内容。
2. 系统将修订内容状态设为草稿，进入 F-02 审核与脱敏发布流程。
3. 新版本审核通过并发布后，系统自动将旧版本归档：不可被检索或引用，但历史快照保留供审计。
4. 系统通知 E-02 和 E-03 更新索引与向量数据。

### 场景三：销售反馈内容质量驱动运营改进

**用户**：销售、知识运营员

**情境**：销售在跟进商机时使用了一份知识单元，发现内容已过时，提交反馈以触发内容修订。

**业务流程**：

1. 销售对已使用的知识单元提交"过时"反馈并附简要说明。
2. 系统将反馈汇总至内容运营看板。
3. 知识运营员查看看板，识别高频负反馈的知识单元，对目标知识单元发起修订，进入场景二的版本管理流程。

## 业务规则 (Business Rules)

### 发布状态门控

- **[BR-01]发布门控**：未审核通过的知识单元不得出现在 E-02 检索结果或 E-03 向量数据库中；只有状态为"已发布"的知识单元方可被下游消费。

### 脱敏合规

- **[BR-02]强制脱敏**：所有客户名称、人员姓名、具体金额数字在发布前须经审核员显式标记并替换为泛化描述；审核员须明确确认"脱敏完成"后方可提交发布（GC-03）。

### 操作审计

- **[BR-03]操作留痕**：每条录入操作、审核决定（通过/拒绝）、脱敏操作均须写入审计日志，包含操作者、时间戳、操作类型及变更内容摘要；审计日志不可删除、不可篡改（GC-04）。

### 分类标签治理

- **[BR-04]标签受控**：知识单元分类标签由管理员预设可选值，录入人不得自由填写标签；防止标签碎片化影响 E-02 检索精度。

### 知识单元版本

- **[BR-05]旧版归档**：新版本发布后，旧版本自动归档：不可被检索或引用，但历史快照不可删除，保留供审计查阅（配合 BR-03）。

### 可见性隔离

- **[BR-06]事业部隔离**：知识单元按录入者所属事业部/产品线隔离可见性；跨事业部访问须经显式授权（GC-02）。

## 非功能需求 (Non-Functional Requirements)

- **[NFR-01]审核SLA**：知识单元从提交到审核完成 ≤ 72 小时（含脱敏处理；依赖人工 SLA 配合，需与知识运营团队确认）。
- **[NFR-02]录入响应**：录入内容提交后，系统返回草稿保存确认的业务感知延迟 ≤ 3 秒。
- **[NFR-03]数据安全**：原始草稿（含未脱敏内容）访问权限限审核员及以上角色；脱敏后内容加密存储（GC-03）。
- **[NFR-04]审计保留**：审计日志保留期限 ≥ 12 个月，支持按操作者、时间范围、操作类型检索，不可删除。

---

## 约束与依赖 (Constraints & Dependencies)

- **[DEP-01]E-02检索**：E-01 发布的知识单元是 E-02 检索库的唯一合法来源；E-02 须订阅"已发布"事件触发索引更新。
- **[DEP-02]E-03向量化**：E-01 发布的知识单元是 E-03 向量化输入；E-03 须订阅"已发布/已归档"事件同步向量数据库。
- **[DEP-03]商机上下文**：F-01 支持从商机详情页（T-01/E-03）携带上下文预填录入表单；依赖 T-01/E-03 提供上下文传递能力，E-01 先行上线时此功能降级为手动填写。
- **[ASM-01]审核员就绪**：假设知识运营团队在 Phase 2 GA 前完成培训，具备支撑 72h SLA 人工审核流的能力；若未就绪，审核将积压，不影响系统上线。
- **[ASM-02]历史内容迁移**：假设存量历史内容（PPT/Word）的批量导入由独立专项项目处理；本 Epic 仅覆盖 Phase 2 后的增量录入。
