# Feature Definition - T-04/E-01/F-02 审核与脱敏发布 (Review and Desensitization Publishing)

> 需求特性定义（A0304）｜所属史诗：T-04/E-01 知识资产结构化录入与审核
> **Feature 编号**：F-02 ｜ **优先级**：P0 ｜ **交付阶段**：Phase 2 ｜ **状态**：规划中

---

## 特性概述 (Feature Overview)

**所属 Epic**：[T-04/E-01 知识资产结构化录入与审核](../README.md)

**业务定义**：
该特性用于对草稿状态的知识单元实施合规审查与发布决策，是知识资产进入可信发布状态的唯一通道。它关注的是"审核员如何识别并消除草稿中的敏感信息、如何对草稿作出发布或退回决定，以及发布后如何通知下游消费方"。不定义草稿的录入过程（由 F-01 负责），也不定义已发布内容的版本迭代流程（由 F-03 负责）。

**范围边界**：

1. 包含：审核员查看待审草稿；对正文中客户名称、人员姓名、具体金额等敏感信息进行显式标记与泛化替换；审核员确认脱敏完成后提交发布决定；审核不通过时填写退回原因并退回给录入人；发布后触发 E-02 检索和 E-03 向量化订阅事件。
2. 不包含：草稿录入（F-01）；版本修订内容的录入入口（F-03 入口，修订内容重新进入本 Feature 审核流）；审核员的权限配置与分配（管理员职能）；E-02 检索与 E-03 向量化的具体处理逻辑（分属 E-02、E-03 范围）。
3. 参见 Epic [DEP-01]：E-02 须订阅 E-01 发布的"已发布"事件以触发索引更新。
4. 参见 Epic [DEP-02]：E-03 须订阅 E-01 的"已发布/已归档"事件以同步向量数据库。

## 业务规则 (Business Rules)

- **[BR-01]审核员权限**：仅具备审核权限的用户可访问待审草稿列表及草稿原文（含未脱敏内容）；录入人及其他角色不得查看草稿原文中未脱敏的敏感信息（参见 Epic [NFR-03]）。
- **[BR-02]强制脱敏确认**：审核员须在明确标记并替换所有敏感信息（客户名称、人员姓名、具体金额）后，显式勾选"脱敏完成"确认项，方可提交发布决定；未勾选时系统不允许提交（参见 Epic [BR-02]）。
- **[BR-03]发布门控**：仅状态为"草稿"且经审核员提交发布决定的知识单元，方可变更为"已发布"状态；"已发布"状态的知识单元方可被 E-02 和 E-03 消费（参见 Epic [BR-01]）。
- **[BR-04]退回必填原因**：审核员选择退回草稿时，退回原因为必填项；未填写原因时系统不允许提交退回（参见 US-04）。
- **[BR-05]状态变更留痕**：审核决定（发布或退回）、脱敏操作须写入审计日志，包含操作者、时间戳、操作类型（审核发布/审核退回/脱敏标记）及变更内容摘要；审计日志不可删除、不可篡改（参见 Epic [BR-03]）。
- **[BR-06]发布事件触发**：知识单元发布后，系统须触发"已发布"订阅事件通知 E-02 检索和 E-03 向量化；事件触发须在发布操作成功后发出，事件失败不影响发布状态，但须记录失败情况供运营监控（参见 Epic [DEP-01]、[DEP-02]）[推导-待确认]。
- **[BR-07]审核 SLA**：知识单元从草稿提交到审核完成的业务 SLA 目标为 ≤ 72 小时（参见 Epic [NFR-01]）；系统须支持按超时时长标记待审草稿的等待状态，便于运营监控 SLA 达成情况[推导-待确认]。
- **[BR-08]事业部可见性**：审核员仅可查看和处理其被授权的事业部/产品线范围内的待审草稿（参见 Epic [BR-06]）。

## 业务对象定义 (Business Object Definition)

### 脱敏标记

脱敏标记是审核员在草稿正文中识别的敏感信息片段及其对应的泛化替换描述，是证明审核员履行脱敏职责的业务凭证，也是审计追溯的重要组成部分。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属知识单元草稿 | 标识该标记归属于哪个待审草稿 |
| 敏感信息类型 | 敏感信息的分类，如客户名称、人员姓名、金额数字 |
| 原始文本片段 | 被标记的敏感原文内容，仅审核员可见 |
| 泛化替换描述 | 替换敏感信息的泛化表达，如"某制造业客户"、"某头部客户" |
| 标记操作者 | 执行脱敏标记的审核员身份 |
| 标记时间 | 脱敏标记创建的时间戳 |

### 审核决定记录

审核决定记录是审核员对草稿作出发布或退回判断的正式业务凭证，用于追溯知识单元进入"已发布"状态的责任归属，以及退回流转的历史轨迹。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 所属知识单元 | 标识该决定针对的知识单元 |
| 审核结论 | 发布或退回两种结论之一 |
| 脱敏完成确认 | 审核员是否显式确认完成了脱敏操作 |
| 退回原因 | 退回结论时审核员填写的原因说明，发布时为空 |
| 审核员 | 作出审核决定的用户身份 |
| 决定时间 | 审核决定提交的时间戳 |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 审核员已登录，系统中存在一个或多个处于"草稿"状态的待审知识单元，且属于该审核员被授权的事业部范围 | 审核员访问待审草稿列表 | 列表展示该审核员有权限处理的全部待审草稿，超出权限范围的草稿不显示（参见 BR-01、BR-08） | US-01 |
| AC-02 | 审核员打开一个待审草稿，浏览正文内容，识别出客户名称、人员姓名或金额等敏感信息 | 在正文中标记敏感信息片段并输入泛化替换描述 | 系统记录脱敏标记（敏感类型、原文片段、替换描述），标记操作写入审计日志（参见 BR-02、BR-05） | US-02 |
| AC-03 | 草稿正文中不含任何需要脱敏的敏感信息 | 审核员浏览正文，确认无需标记 | 审核员可跳过标记步骤，直接进入发布确认环节；脱敏完成确认项可在无标记情况下勾选 | US-02 |
| AC-04 | 审核员已完成正文脱敏标记（或确认无需标记），勾选"脱敏完成"并提交发布 | 提交发布决定 | 知识单元状态变更为"已发布"；审核决定写入审计日志；系统触发"已发布"订阅事件通知 E-02 和 E-03（参见 BR-03、BR-05、BR-06） | US-03 |
| AC-05 | 审核员未勾选"脱敏完成"确认项，尝试提交发布 | 提交发布决定 | 系统拒绝发布，提示必须确认脱敏完成（参见 BR-02） | US-03 |
| AC-06 | 审核员判定草稿内容不达标，填写退回原因并提交退回 | 提交退回决定 | 知识单元状态变更回"已退回"；退回原因与审核决定记录写入审计日志；录入人可查看退回状态与原因（参见 BR-04、BR-05） | US-04 |
| AC-07 | 草稿已被退回，录入人查看其提交的知识单元列表 | 录入人查看退回的草稿详情 | 录入人可看到草稿状态为"已退回"并查看审核员填写的退回原因（参见 BR-04） | US-05 |
