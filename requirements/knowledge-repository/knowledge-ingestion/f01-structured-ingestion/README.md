# Feature Definition - T-04/E-01/F-01 结构化内容录入 (Structured Content Ingestion)

> 需求特性定义（A0304）｜所属史诗：T-04/E-01 知识资产结构化录入与审核
> **Feature 编号**：F-01 ｜ **优先级**：P0 ｜ **交付阶段**：Phase 2 ｜ **状态**：规划中

---

## 特性概述 (Feature Overview)

**所属 Epic**：[T-04/E-01 知识资产结构化录入与审核](../README.md)

**业务定义**：
该特性用于将售前顾问、知识运营员等知识贡献者的隐性知识资产转化为结构化的可审核草稿，建立知识进入受控发布通道的第一道入口。它关注的是"知识单元如何被完整、规范地录入系统并进入等待审核的状态"，不定义录入后的审核流程（由 F-02 负责），也不定义已发布内容的版本迭代（由 F-03 负责）。

**范围边界**：

1. 包含：知识单元新建录入（分类标签选择、正文与附件提交、元数据填写）；携带商机上下文预填元数据；录入后状态为"草稿"等待审核。
2. 不包含：存量历史内容的批量迁移；知识单元审核与发布流程；版本修订录入（版本迭代入口由 F-03 提供，进入本 Feature 的录入流程）；分类标签的管理与维护（管理员职能）。
3. 参见 Epic [DEP-03]：F-01 支持从商机详情页（T-01/E-03）携带上下文预填元数据；依赖 T-01/E-03 提供上下文传递能力，E-01 先行上线时此功能降级为手动填写。

## 业务规则 (Business Rules)

- **[BR-01]草稿状态**：知识单元录入提交后，系统须将其保存为"草稿"状态；草稿状态的知识单元不得出现在 E-02 检索结果或 E-03 向量数据库中（参见 Epic [BR-01]）。
- **[BR-02]标签受控**：录入人在选择分类标签时，仅可从管理员预设的可选值中选择，不得自由填写；每个分类维度（行业/客户类型/竞品/销售阶段）必须至少选择一个标签（参见 Epic [BR-04]）。
- **[BR-03]操作留痕**：录入操作须写入审计日志，包含操作者、时间戳、操作类型（新建草稿）及内容摘要（参见 Epic [BR-03]）。
- **[BR-04]事业部归属**：录入的知识单元自动归属于录入者所属事业部/产品线，初始可见性范围由此确定（参见 Epic [BR-06]）。
- **[BR-05]正文必填**：知识单元正文为必填项；正文为空时，系统不允许提交录入。
- **[BR-06]附件限制**：附件为可选项；单次录入支持上传的附件数量上限及单文件大小上限须由运营团队配置后确定（阈值待定，责任归属：知识运营团队；默认分流策略：超出限制时系统拒绝上传并提示具体限制数值）[推导-待确认]。
- **[BR-07]商机上下文预填**：当录入由商机上下文携带触发时，系统应以商机数据预填行业、客户类型、销售阶段标签；预填值仅作为默认选项，录入人可修改；若 T-01/E-03 尚未就绪，则降级为手动填写（参见 Epic [DEP-03]）。
- **[BR-08]修订入口隔离**：版本修订录入须关联已发布的知识单元；全新录入须创建新的知识单元记录，不得关联现有已发布单元（此边界由 F-03 版本管理入口控制）[推导-待确认]。

## 业务对象定义 (Business Object Definition)

### 知识单元草稿

知识单元草稿是录入人提交的未经审核的知识资产原始版本，承载知识内容本体及用于检索和归类的元数据，是进入 F-02 审核流的唯一输入。

| 业务属性 | 业务说明 |
| :--- | :--- |
| 标题 | 知识单元的简短业务名称，用于列表展示和快速识别 |
| 正文 | 知识内容主体，包含话术、解决方案或案例描述 |
| 分类标签—行业 | 知识内容适用的行业分类，取自管理员预设可选值 |
| 分类标签—客户类型 | 知识内容适用的客户类型，取自管理员预设可选值 |
| 分类标签—竞品 | 知识内容针对的竞品对象，取自管理员预设可选值 |
| 分类标签—销售阶段 | 知识内容适用的销售阶段，取自管理员预设可选值 |
| 附件 | 与知识内容相关的原始材料文件，可选 |
| 关联商机 | 录入时关联的商机记录，用于追溯知识来源，可选 |
| 录入者 | 发起录入操作的用户身份，用于审计与可见性归属 |
| 所属事业部 | 录入者归属的事业部/产品线，决定初始可见性范围 |
| 录入时间 | 草稿首次提交的时间戳，作为审计基准 |
| 状态 | 当前生命周期状态，F-01 完成后为"草稿" |

## 验收标准 (Acceptance Criteria)

| AC | 前置条件 | 触发动作 | 预期结果 | US Ref |
| :--- | :--- | :--- | :--- | :--- |
| AC-01 | 录入人已完成标题、正文填写，并从预设可选值中选择了所有分类维度的标签 | 提交录入 | 系统保存知识单元草稿，草稿状态为"草稿"，录入操作写入审计日志（参见 BR-01、BR-03）；录入人看到草稿保存确认提示 | US-01, US-04 |
| AC-02 | 录入人提交录入时附带一个或多个附件文件 | 提交含附件的录入 | 附件与知识单元草稿关联保存；附件数量或大小超出限制时，系统拒绝上传并提示具体限制值（参见 BR-06） | US-02 |
| AC-03 | 录入人从携带商机上下文的入口（T-01/E-03）发起录入，且 T-01/E-03 上下文传递能力已就绪 | 进入录入界面 | 系统以商机的行业、客户类型、销售阶段数据预填对应分类标签；录入人可修改预填值（参见 BR-07） | US-03 |
| AC-04 | 录入人未填写正文，或未选择任意分类维度的标签 | 提交录入 | 系统拒绝提交，提示具体缺失的必填项（参见 BR-02、BR-05） | US-01 |
| AC-05 | 录入提交成功 | 系统完成草稿保存 | 业务感知响应延迟 ≤ 3 秒（参见 Epic [NFR-02]）；录入人可在草稿列表中查看已提交的草稿 | US-04 |
| AC-06 | 知识运营员通过 F-03 版本修订入口发起录入，关联已发布的知识单元 | 提交修订版正文内容 | 系统保存新版本草稿并关联原知识单元，草稿状态为"草稿"，进入 F-02 审核流（参见 BR-08） | US-05 |
